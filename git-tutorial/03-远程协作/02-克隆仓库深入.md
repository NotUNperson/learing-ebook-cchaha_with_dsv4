# 02 克隆仓库深入

## 本节你会学到什么

- 理解 `git clone` 到底下载了什么——不只是文件
- 用"复制整个相册包括修改记录"的类比理解 clone
- 知道 clone 和普通下载（Download ZIP）的本质区别
- 学会使用 `git clone` 命令
- 进入克隆下来的仓库，探索它的版本历史

## 正文

### 先讲一个故事

小明想学习一个开源项目，他上了 GitHub，点了一下那个绿色的 "Code" 按钮，然后下载了一个 ZIP 文件。解压后，他看到了一堆代码文件，觉得"嗯，拿到了"。

但当他用 `git log` 的时候，终端冷冰冰地告诉他：**fatal: not a git repository**。

小明懵了：我明明把代码下载下来了啊，为什么 Git 不认？

### 下载 ZIP 能拿到什么

用 "Download ZIP" 按钮下载一个项目，你拿到的是：

- 项目**当前版本**的所有文件
- 就是此时此刻的代码快照

你拿不到的是：

- 这个项目从创建到现在的任何提交记录
- 谁在什么时候改了什么
- 各个分支的代码
- 任何版本历史

用类比来说：下载 ZIP 就像**从云相册里下载了最新一张照片的打印版**。你拿到了照片没错，但你不知道这张照片拍了多少次才拍好，也不知道之前修过哪里。

### `git clone` 又是什么

`git clone` 做的事情完全不同。它会：

1. 把远程仓库的**整个 `.git` 目录**下载下来，包含所有提交记录
2. 把**所有分支**的信息都拉下来
3. 把**所有标签**（tag，后面会讲）也拉下来
4. 自动把工作目录切换到默认分支的最新版本

用类比来说：`git clone` 就像**把别人的云相册整个复制一份到你的手机上**——不只是最新的那张照片，而是从第一张到最新一张的所有照片，包括每张照片的拍摄时间、修改记录、以及别人删了又恢复的过程，全部都有。

### 到底哪些东西被下载了——一张表格说清楚

| | Download ZIP | git clone |
|---|---|---|
| 当前版本的文件 | 有 | 有 |
| 完整的提交历史 (`git log`) | 没有 | 有 |
| 所有分支 (`git branch -a`) | 没有 | 有 |
| 所有标签 (`git tag`) | 没有 | 有 |
| 可以继续用 Git 做版本管理 | 不能（不是仓库） | 可以 |
| 可以推回远程（如果有权限） | 不能 | 可以 |

### 类比深入：复制整个相册

假设你朋友有一个云相册，里面有他从大学到现在十年的照片，共 2000 张。他每次修图、加滤镜、删掉重拍的痕迹都保存在云端。

- 你点"下载当前照片"，你只拿到最近一张他发朋友圈的照片。这就是 Download ZIP。
- 你点"迁移整个相册到我的账号"，你拿到了从十年前到现在每一张照片、每一次修改。这就是 `git clone`。

对 Git 来说，"照片"就是代码文件，"修改痕迹"就是提交记录。

### `git clone` 命令详解

命令最基本的样子：

```bash
git clone <仓库地址>
```

比如克隆 GitHub 上一个叫 `hello-world` 的项目：

```bash
# 用 HTTPS 方式克隆
git clone https://github.com/octocat/hello-world.git

# 用 SSH 方式克隆（需要先配置 SSH 密钥）
git clone git@github.com:octocat/hello-world.git
```

HTTPS 和 SSH 的区别：HTTPS 每次推送需要输入用户名和密码（或 token），SSH 配置好密钥后不用每次验证。新手推荐先用 HTTPS，简单直接。

执行 clone 后，Git 做了什么事：

1. 在你当前目录下创建一个新文件夹，名字默认是项目名（比如 `hello-world`）
2. 在这个文件夹里初始化一个 Git 仓库（`git init`）
3. 添加远程仓库，别名叫做 `origin`，指向你克隆的那个地址
4. 下载远端仓库的所有数据（对象、引用等）
5. 把默认分支的最新版本检出到工作目录

所以 clone 之后，你不需要再手动执行 `git init` 和 `git remote add`——Git 都帮你做好了。

### 指定克隆下来的文件夹名

如果你想自己指定文件夹名，在 URL 后面加上：

```bash
# 克隆下来后文件夹叫 "my-copy"，而不是默认的 "hello-world"
git clone https://github.com/octocat/hello-world.git my-copy
```

### 探索克隆下来的仓库

克隆完成后，进去看看：

```bash
cd hello-world
ls -la           # 看到项目文件和隐藏的 .git 目录
git log --oneline  # 看到完整的提交历史——注意，这些不是你做的，是项目作者做的
git remote -v     # 看到 origin 指向你克隆的地址
git branch -a     # 看到所有分支，包括远程分支
```

你会发现，明明你什么都没做，但一个完整的项目就在眼前了——这就是 clone 的威力。

### 一个容易混淆的点

> "我 clone 了一个仓库，为什么我不能直接 push？"

因为你只是**复制**了别人的仓库。你有读的权限（可以拉取），但没有写的权限（不能推送）。就像你下载了别人的云相册到手机，你可以看，但不能往别人相册里加照片。

如果你想贡献代码，你需要"fork"（派生）——后面会详细讲。现在只需要记住：clone 是下载，不代表你获得了上传权限。

## 动手试试

1. 打开浏览器，访问 [github.com/octocat/hello-world](https://github.com/octocat/hello-world)（这是 GitHub 官方的示例项目）。
2. 在终端里找一个你用来练习的目录。
3. 执行 `git clone https://github.com/octocat/hello-world.git`
4. 进入克隆下来的目录：`cd hello-world`
5. 逐个执行这些命令，观察输出：
   - `git log --oneline` — 看看你能看到多少个提交？
   - `git remote -v` — origin 指向哪里？
   - `git branch -a` — 有哪些分支？
6. 对比一下：如果你用 Download ZIP 下载并解压，执行 `git log` 能成功吗？去试试看！

## 本节小结

`git clone` 不是简单的下载文件，而是把整个远程仓库——包括所有提交历史、所有分支、所有版本记录——完整复制到本地，就像把别人的云相册整个搬到你手机里一样。
