# 05 GitHub 基础操作

## 本节你会学到什么

- 理解 GitHub 不只是"存代码的网盘"，而是开发者社交平台
- 学会创建 GitHub 仓库
- 学会在 GitHub 网页上浏览代码、提交历史、分支
- 认识 Issues——项目的问题跟踪和讨论区
- 了解 Star、Fork、Watch 这些社交功能

## 正文

### GitHub 不只是"代码网盘"

很多新手听到 GitHub 的第一反应是："哦，就是网上存代码的地方。"

这个理解没有错，但就像说"微信就是发短信的软件"一样，虽然对，但远远不够。

GitHub 是一个**开发者社交平台**。你可以理解为：

> GitHub = 代码托管 + 社交网络 + 协作工具 + 项目展示

用一个更具体的类比：

- 如果把代码比作**照片**，GitHub 就是**朋友圈 + 相册**。你发照片（上传代码），朋友点赞（Star），大家在评论区讨论（Issues/Discussions），有人帮你把照片修得更好看（Pull Request）。
- 如果把代码比作**小说**，GitHub 就是**起点中文网**。作者上传章节（commit），读者追更（Watch），还有热心读者帮忙改错别字（贡献代码）。

### 注册 GitHub 账号

如果你还没有账号，现在就去注册一个：

1. 打开 [github.com](https://github.com)
2. 点击右上角的 "Sign up"
3. 输入邮箱、密码、用户名
4. 验证邮箱
5. 完成！

用户名建议：用英文或拼音，最好短一点、好记。你以后的项目链接里都会带着这个名字，就像你的开发者名片。

### 在 GitHub 上创建仓库

你已经在 01 节创建过一次了，这里我们再过一遍并了解每个选项的含义。

1. 登录后，点击右上角的 "+" 号，选择 "New repository"
2. 填写信息：

   | 字段 | 含义 | 怎么填 |
   |------|------|--------|
   | Repository name | 仓库名称 | 英文，短横线分隔，如 `my-first-project` |
   | Description | 项目简介 | 一句话说清楚这个项目是干嘛的 |
   | Public / Private | 公开还是私密 | 公开：全世界都能看到。私密：只有你能看到 |
   | Add a README file | 是否添加 README | 建议勾选，这样创建完就有第一个文件 |
   | Add .gitignore | 忽略文件模板 | 根据你用的编程语言选择 |
   | Choose a license | 选择开源协议 | 先选 MIT（最宽松，后面可以改） |

3. 点击 "Create repository"，完成！

### GitHub 页面上有什么——界面导览

创建仓库后，你会看到仓库主页。我们来认识一下每个区域：

**顶部导航栏**（Code 标签页下）：
- **Code**（代码浏览）：当前所在位置，展示所有文件和文件夹
- **Issues**：问题跟踪，相当于项目的"待办事项 + 论坛"
- **Pull requests**：合并请求，别人提交的代码改动
- **Actions**：自动化流水线（暂时不用管）
- **Projects**：项目看板（暂时不用管）
- **Wiki**：项目文档
- **Security**：安全检查

**代码区域**：
- 文件列表：和电脑上的文件夹一样，可以点击进去
- 每个文件点击可以查看内容
- 右上角可以看到 "X commits"（提交次数）、"Y branches"（分支数）
- 点击 "commits" 可以看到完整的提交历史（就和 `git log` 一样！）

**右侧边栏**：
- **About**：项目简介
- **Releases**：发布的版本
- **Languages**：使用的编程语言统计

### 在网页上浏览提交历史

点击代码列表上方的 "X commits" 链接（在时钟图标旁边），你会看到这个项目的所有提交记录，和 `git log --oneline` 的输出非常像。

点击某一个提交，可以看到：
- 这个提交修改了哪些文件
- 每个文件增加了什么内容（绿色，前面有 `+`）
- 删除了什么内容（红色，前面有 `-`）

这种"加绿减红"的对比视图在 Git 里叫做 **diff**（差异对比）。GitHub 把它做成了非常直观的网页版。

### Issues——项目的讨论区

Issues 是 GitHub 上最重要的协作功能之一。你可以把它理解为项目的"论坛 + 待办清单"。

一个 Issue 可以是：

- **Bug 报告**："登录页面的按钮点不动！"
- **功能建议**："能不能加一个暗色模式？"
- **问题求助**："我在安装时遇到这个错误，怎么解决？"
- **讨论话题**："下个版本要不要重构数据库？"

如何创建 Issue：

1. 进入仓库，点击顶部 "Issues" 标签
2. 点击绿色的 "New issue" 按钮
3. 填写标题和描述
4. 点击 "Submit new issue"

Issues 支持 Markdown 格式，你可以插入代码块、图片、链接。也可以在下面留言讨论。

### 三个重要的社交按钮：Star、Watch、Fork

在仓库主页的右上角，有三个按钮：

| 按钮 | 图标 | 含义 | 类比 |
|------|------|------|------|
| **Star** | 星星 | 收藏 / 点赞 | 朋友圈的"赞"。点的越多，这个项目越受欢迎 |
| **Watch** | 眼睛 | 关注 / 订阅通知 | 关注了某人朋友圈，有新动态就通知你 |
| **Fork** | 叉子 | 派生 / 复制到自己的账号下 | 把别人的朋友圈相册完整复制一份到自己名下 |

- **Star** 是最轻量的操作。你喜欢一个项目就给它 Star，表示支持，也方便你在自己的 Star 列表里找到它。
- **Watch** 意味着你会收到这个项目的动态通知（新的 Issue、新的 PR、新的提交等）。
- **Fork** 是最重要的协作动作。它会在你的 GitHub 账号下创建一个完全的副本。之后你就可以在自己的副本上修改，然后通过 Pull Request 把修改贡献回原项目。具体操作在下一节讲。

### 个人主页和 Profile

点击 GitHub 右上角你的头像，选择 "Your profile"，这就是你的个人主页。

你的 Profile 包括：
- 你的头像和简介
- 你 Star 过的仓库
- 你的贡献记录（绿色方块图）
- 你创建的仓库列表

很多招聘方会看你的 GitHub Profile 来了解你的编程能力。所以整理好它，像整理简历一样。

### 查找和探索开源项目

GitHub 的搜索栏非常强大。你可以搜索：

- 项目名称
- 编程语言（比如 `language:python`）
- 特定话题（比如 `topic:machine-learning`）
- 文件内容

试着搜索 `language:python stars:>1000`，看看有哪些热门的 Python 项目。点进去看看它们的 Issues、提交历史、贡献者列表——这能让你学到很多真实的工程实践。

## 动手试试

以下操作全部在网页端完成，不需要命令行：

1. 如果没有账号，去 [github.com](https://github.com) 注册一个。
2. 创建一个新仓库，勾选 "Add a README file"，用 Public 公开。
3. 进入仓库，浏览文件列表、点击 README.md 查看内容。
4. 点击 "commits" 链接，查看提交历史。
5. 创建一个 Issue（标题随意，比如 "测试 Issue 功能"），写一段描述，然后提交。再在 Issue 下面留一条评论。
6. 回到仓库主页，点一下 Star 按钮。
7. 搜索 `octocat/hello-world`，进入这个仓库，给它也点个 Star。
8. 去你的 Profile 页面，看看你的 Star 列表和贡献图。

## 本节小结

GitHub 不只是存代码的地方，而是一个集代码托管、问题追踪、代码审查、社交互动于一体的开发者平台，Issues 是项目讨论区，Star/Watch/Fork 构成了它的社交体系。
