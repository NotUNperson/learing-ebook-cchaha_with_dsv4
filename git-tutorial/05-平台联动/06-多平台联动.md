# 06 多平台联动

## 本节你会学到什么

- 理解为什么一个项目可能需要放在多个托管平台上
- 给本地仓库添加多个远程地址（多个 remote）
- 同时或分别推送到 GitHub、Gitee、GitLab
- 掌握 `git remote set-url` 的使用场景

## 一个生活类比：多平台发短视频

你是一个短视频博主。你拍好一个视频后，不会只发到一个平台——你会同时发到抖音、B站、小红书。视频内容是一样的（同一个 Git 仓库），但你给每个平台都发送了一份（push 到多个 remote）。

为什么要发多个平台？
- 抖音用户多，但视频压缩厉害 -> 类似 GitHub 生态好，但国内访问可能慢
- B站画质好，适合长视频 -> 类似 Gitee 速度快，适合国内协作
- 小红书适合图文 -> 类似 GitLab 的 CI/CD 功能很强

不同的平台有不同的长处。你的项目也可以"一份代码，多处存放"。

## 为什么要同时推送到多个平台

### 原因一：国内访问速度

GitHub 在国外，国内访问有时很慢甚至打不开。如果你在国内有一个团队，每次 `git clone` 等半天，很影响效率。Gitee（码云）是国内平台，访问飞快——你可以把 GitHub 当"主仓库"，Gitee 当"国内镜像"。

### 原因二：备份

不要把鸡蛋放在一个篮子里。虽然 GitHub 很稳定，但万一哪天账号出问题、或者平台做策略调整，至少你还有其他平台的副本。

### 原因三：不同用途

- GitHub：开源社区，让更多人看到你的项目
- Gitee：国内用户下载速度快
- GitLab：有些公司自建的 GitLab 服务器作为内部开发平台

### 原因四：公司要求

有些公司规定代码必须存放在公司的 GitLab 服务器上，但你也想在 GitHub 上维护一个开源版本。多 remote 就能满足这种需求。

## 核心概念：一个仓库可以有多个 Remote

不要再把 `origin` 想成"唯一的远程地址"。`origin` 只是一个**别名**，你可以有多个别名指向多个不同的远程地址。

```
本地仓库
   ├── remote "origin"   → git@github.com:用户名/项目.git
   ├── remote "gitee"    → git@gitee.com:用户名/项目.git
   └── remote "gitlab"   → git@gitlab.com:用户名/项目.git
```

你可以选择推送到其中一个，也可以一次性全部推送。

## 一步步操作：添加多个远程平台

### 准备工作

假设你已经有了一个 GitHub 仓库，并且本地已经配好了 `origin` 指向它。

```bash
# 当前状态
git remote -v
# origin  git@github.com:用户名/项目.git (fetch)
# origin  git@github.com:用户名/项目.git (push)
```

### 第一步：在 Gitee 上创建同名仓库

1. 登录 [Gitee](https://gitee.com)
2. 点击右上角 **"+"** → **新建仓库**
3. 仓库名和 GitHub 上保持一致（方便管理）
4. **同样不要勾选** README、.gitignore 等（保持空仓库）
5. 创建完成后，复制 SSH 地址，形如：`git@gitee.com:用户名/项目.git`

### 第二步：在 GitLab 上创建同名仓库（可选）

1. 登录 [GitLab](https://gitlab.com)
2. 点击 **New project** → **Create blank project**
3. 仓库名保持一致
4. 创建完成后，复制 SSH 地址

### 第三步：添加第二个 remote

```bash
# 给 Gitee 仓库添加一个 remote，叫 "gitee"
git remote add gitee git@gitee.com:用户名/项目.git
```

### 第四步：添加第三个 remote（如果需要）

```bash
# 给 GitLab 仓库添加一个 remote，叫 "gitlab"
git remote add gitlab git@gitlab.com:用户名/项目.git
```

### 第五步：验证

```bash
git remote -v
```

输出应该类似：

```
gitee   git@gitee.com:用户名/项目.git (fetch)
gitee   git@gitee.com:用户名/项目.git (push)
gitlab  git@gitlab.com:用户名/项目.git (fetch)
gitlab  git@gitlab.com:用户名/项目.git (push)
origin  git@github.com:用户名/项目.git (fetch)
origin  git@github.com:用户名/项目.git (push)
```

## 分别推送

```bash
# 只推送到 GitHub
git push origin main

# 只推送到 Gitee
git push gitee main

# 只推送到 GitLab
git push gitlab main
```

一次性推到所有平台：

```bash
# 方法：逐个执行
git push origin main && git push gitee main && git push gitlab main
```

你可以把这行命令写成脚本，或者配一个 Git 别名来简化。

## 同时推送到所有 Remote（进阶）

Git 原生支持在一个 push 命令中推送所有 remote：

```bash
# 推送到所有已配置的远程仓库
git remote | xargs -I {} git push {} main
```

但更优雅的方式是设置一个特殊的 remote 地址，让它包含多个 URL：

```bash
# 给 origin 添加第二个 URL（git push 时会同时推到两个地址）
git remote set-url --add origin git@gitee.com:用户名/项目.git
```

这样当你 `git push origin main` 时，Git 会同时推送到 GitHub 和 Gitee。检查效果：

```bash
# 查看 origin 的所有 URL
git remote -v
# origin  git@github.com:用户名/项目.git (fetch)
# origin  git@github.com:用户名/项目.git (push)
# origin  git@gitee.com:用户名/项目.git (push)
```

注意：`fetch` 只有一个地址（GitHub），`push` 有两个地址。这意味着 `git pull` 只会从 GitHub 拉取，但 `git push origin` 会同时推到两个平台。

### 如果你不想用这种"叠加"方式

如果你希望保持干净、明确的 remote 关系，就还是用最直接的方式——给每个平台一个独立的 remote 名字（`origin`、`gitee`、`gitlab`），分别推送。这对于初学者来说更清晰、不容易搞混。

## 日常使用建议

对于初学者和大多数个人项目，推荐以下策略：

| 策略 | 做法 |
|------|------|
| **主仓库** | GitHub（`origin`）——功能最全，生态最好 |
| **国内镜像** | Gitee（`gitee` remote）——方便国内用户 clone |
| **定期同步** | 每次 push 时顺便推一下 Gitee |

日常开发流程：

```bash
# 写完代码后
git add .
git commit -m "新的改动"

# 推到 GitHub（主仓库）
git push origin main

# 顺便推到 Gitee（如果国内有需要）
git push gitee main
```

## Pull 的时候从哪拉？

添加了多个 remote 后，`git pull` 默认还是从 `origin` 拉取。如果你想从 Gitee 拉取（比如国内网络快），可以：

```bash
# 临时从 Gitee 拉取一次
git pull gitee main

# 或者把 origin 的 fetch 地址也改成 Gitee
git remote set-url origin git@gitee.com:用户名/项目.git
```

> 注意：用 `git remote set-url` 会覆盖原有的地址。如果你只是想临时切换，建议保持 origin 指向 GitHub（主仓库），需要时手动 `git pull gitee main`。

## 多个 Remote 的常见操作

### 查看所有 remote 的详细信息

```bash
git remote -v                # 只看地址
git remote show origin       # 看某个 remote 的详细信息
git remote show gitee
```

### 重命名 remote

```bash
git remote rename gitee gitee-mirror
```

### 删除一个 remote

```bash
git remote remove gitee
```

### 修改某个 remote 的地址

```bash
# 比如你的 GitHub 用户名改了，仓库地址也变了
git remote set-url origin git@github.com:新用户名/项目.git
```

## 动手试试

1. 登录 Gitee（没有账号的话花1分钟注册一个，免费）
2. 在 Gitee 上创建一个和 GitHub 同名的空仓库
3. 把上一节 SSH 配置里生成的那个公钥也添加到 Gitee 的 SSH 设置中
4. 在本地你的项目里运行 `git remote add gitee git@gitee.com:用户名/项目.git`
5. 运行 `git remote -v` 确认现在有两个 remote
6. 分别推送到 GitHub 和 Gitee：`git push origin main && git push gitee main`
7. 去 Gitee 网页刷新，确认代码出现了
8. 试着改一个文件，再分别推一次

## 本节小结

一个 Git 仓库可以同时和多个远程托管平台"交朋友"——用 `git remote add` 给每个平台一个别名，推送时 `git push <别名> <分支>` 即可。一份代码，多处存放，备份和协作两不误。
