## 本节你会学到什么

- 把前面 7 节的所有知识串联起来，完成一个完整的 Git 工作流
- 模拟真实场景：创建笔记项目、多次修改、查看历史、撤销错误
- 体验"用 Git 管理一个项目"的完整流程
- 建立日常使用 Git 的自信心

## 正文

### 这是总复习，也是首次"实战"

恭喜你学到了这里。在前 7 节中，我们分别学习了：

1. 创建仓库 —— `git init`
2. 三个区域 —— 办公桌 / 文件夹 / 档案柜
3. 追踪改动 —— `git add`
4. 正式存档 —— `git commit`
5. 查看历史 —— `git log`
6. 查看状态 —— `git status` 和 `git diff`
7. 后悔药 —— `git restore`、`git reset`、`git revert`

现在我们要把这些知识连成一条线，模拟一个真实的使用场景：**用 Git 管理你的个人学习笔记**。

### 场景设定

假设你正在学习编程，每天都会写一些学习笔记。你想用 Git 管理这些笔记，这样：

- 每天写了什么，有历史可查
- 如果某天写错了内容，可以找回之前的版本
- 可以清楚看到自己的学习进展

这不是一个虚构的场景——很多程序员真的用 Git 来管理自己的笔记、博客、甚至写书。Git 不只是给代码用的，任何文本文件的版本管理都可以用它。

### 操作流程概览

我们将按以下步骤操作：

1. **初始化项目**：在电脑上创建一个"学习笔记"文件夹，初始化为 Git 仓库
2. **第一次提交**：创建第一篇笔记，提交到仓库
3. **日常笔记更新**：连续几天添加笔记，每次用清晰的提交信息
4. **修改已有笔记**：发现之前的笔记有误，修改并提交
5. **查看学习历史**：用 git log 回顾自己的学习历程
6. **处理一次意外**：不小心删了一段重要笔记，用 git restore 恢复
7. **整理提交历史**：发现有一次提交信息写得不好，用 git reset 修正
8. **查看改动细节**：用 git diff 审视某次修改

### 详细的参考步骤

#### 第一阶段：初始化

```bash
# 创建项目目录并进入
mkdir my-study-notes
cd my-study-notes

# 初始化为 Git 仓库
git init
```

#### 第二阶段：第一次提交

创建第一篇笔记 `2026-04-27-git-basics.md`：

```markdown
# Git 基础学习笔记

## git init
- 把普通文件夹变成 Git 仓库
- 会在文件夹中创建 .git 隐藏目录
- .git 目录存放所有版本数据

## 三个区域
- 工作区 = 办公桌（写代码的地方）
- 暂存区 = 文件夹（准备存档的地方）
- 仓库 = 档案柜（永久保存的地方）
```

然后提交：

```bash
git add .
git commit -m "docs: 添加 Git 基础学习笔记"
```

#### 第三阶段：日常更新

第二天，你学了 `git add` 和 `git commit`，更新笔记：

```bash
# 修改已有文件，追加新内容
echo "" >> 2026-04-27-git-basics.md
echo "## git add" >> 2026-04-27-git-basics.md
echo "- 把工作区的修改放入暂存区" >> 2026-04-27-git-basics.md
echo "- 支持选择性暂存（git add -p）" >> 2026-04-27-git-basics.md

git add .
git commit -m "docs: 补充 git add 相关内容"
```

第三天，学了 `git commit`：

```bash
echo "" >> 2026-04-27-git-basics.md
echo "## git commit" >> 2026-04-27-git-basics.md
echo "- 把暂存区内容永久存入仓库" >> 2026-04-27-git-basics.md
echo "- 提交信息要清晰描述'做了什么'" >> 2026-04-27-git-basics.md

git add .
git commit -m "docs: 补充 git commit 相关内容"
```

创建新笔记文件 `2026-04-28-terminal-basics.md`：

```markdown
# 终端基础学习笔记

## 常用命令
- ls：列出当前目录的文件
- cd：切换目录
- pwd：显示当前路径
- mkdir：创建目录
```

```bash
git add .
git commit -m "docs: 添加终端基础学习笔记"
```

#### 第四阶段：修正错误

你发现之前写的 Git 笔记中有一个错误——把"暂存区"写成了"缓存区"：

```bash
# 修改错误内容...
```

```bash
git add 2026-04-27-git-basics.md
git commit -m "fix: 修正笔记中的术语错误"
```

#### 第五阶段：回顾历史

```bash
# 看看这几天的学习记录
git log --oneline

# 看看具体的修改内容
git log --stat

# 只看笔记文件的修改历史
git log --oneline -- 2026-04-27-git-basics.md
```

#### 第六阶段：意外恢复

假设你不小心删掉了笔记中一段重要的内容：

```bash
# 查看当前状态，发现文件被修改了
git status

# 看看具体改了什么
git diff 2026-04-27-git-basics.md

# 后悔了，恢复文件
git restore 2026-04-27-git-basics.md

# 确认恢复成功
git status
```

#### 第七阶段：修正提交信息

你发现最后一次提交的信息有错别字。因为还没推送给别人看，可以放心用 reset：

```bash
git reset --soft HEAD~1
git commit -m "fix: 修正笔记中的术语错误（已更正）"
```

### 这个场景教会你什么

到这一步，你已经完整地走过了一个 Git 用户最常走的路径：

- **开始工作前**：`git status` 看看现状
- **工作过程中**：修改文件，定期 `git add` 和 `git commit`
- **遇到问题时**：用 `git status` 和 `git diff` 诊断，用 `git restore` 或 `git reset` 修复
- **回顾时**：用 `git log` 翻看历史

这就是 Git 的日常。不是火箭科学，只是一个帮你管好文件版本的工具。你现在已经完全有能力在自己的项目中使用它了。

## 动手试试

打开 Git Bash，从头到尾完成上面的完整流程。不要复制粘贴——自己敲每一条命令：

1. 创建一个名为 `my-study-notes` 的文件夹并初始化仓库
2. 创建第一篇笔记，提交（练习 `git init` + `git add` + `git commit`）
3. 修改笔记内容 2-3 次，每次提交（练习"修改 → add → commit"循环）
4. 创建第二篇笔记，提交
5. 用 `git log --oneline` 查看你的学习历史
6. 故意改错一个地方，用 `git restore` 恢复（练习后悔药）
7. 做一次提交，然后用 `git reset --soft HEAD~1` 撤销并重新提交（练习修正提交信息）
8. 最后用 `git log --oneline --graph` 看一眼完整的提交历史——这就是你的学习足迹

## 本节小结

Git 本地操作的核心循环就是：修改 → 暂存 → 提交 → 查看历史，遇到错误时用合适的后悔药。掌握了这个循环，你就已经是一个能独立使用 Git 管理项目的开发者了。

## 本章总结

"本地版本管理"是 Git 的基石。回顾本章学到的内容：

- `git init` —— 创建仓库（给文件夹装上时光机）
- 三个区域 —— 办公桌（工作区）→ 文件夹（暂存区）→ 档案柜（仓库）
- `git add` —— 挑选要存档的内容放进文件夹
- `git commit` —— 正式归档，写好标签
- `git log` —— 翻看档案柜的历史记录
- `git status` / `git diff` —— 随时了解自己在哪里、改了什么
- `git restore` / `git reset` / `git revert` —— 犯错了有后悔药

你已经掌握了 Git 本地操作的全部核心技能。下一步，就是学习如何通过 GitHub 等平台，把自己的仓库分享出去，和他人协作——这就是下一章"远程协作"的内容了。
