# 01 创建第一个仓库

## 本节你会学到什么

- 理解"仓库"是什么，以及它为什么像一个时间胶囊
- 用 `git init` 把一个普通文件夹变成 Git 仓库
- 探索 `.git` 目录，看看 Git 把版本信息藏在哪里
- 区分"把文件夹变成仓库"和"把仓库克隆下来"这两种场景

## 正文

### 什么是仓库？

想象你正在写一本小说。你每天写一点，改一点。有一天你突然想找回三个月前删掉的一段对话——如果你没有特意保存过历史版本，这段对话就永远消失了。

Git 的仓库（Repository，简称 repo）就是帮你解决这个问题的。你可以把它理解成一个**带有时间记录功能的文件夹**。在这个文件夹里，你不仅能存放文件，还能给文件的每一次修改"拍一张照片"，以后随时可以翻出任何一张"老照片"来看。

另一个类比：仓库就像一个**日记本**。你每次写日记都会写上日期和内容，以后翻看日记就能知道某一天发生了什么。Git 仓库也是这样，只不过它记录的不是你的生活，而是你的代码（或任何文本文件）的变化历史。

### 两种方式得到一个仓库

在"认识 Git"模块里，你已经用 `git clone` 从 GitHub 上克隆过一个仓库。那是**第一种方式**：别人已经建好了一个仓库，你把它复制一份到自己的电脑上。

现在我们要学的是**第二种方式**：你自己创建一个全新的仓库。也就是说，你不需要任何远程服务器，不需要 GitHub，就能在你的电脑上创建一个受 Git 管理的文件夹。

### 用 git init 创建仓库

打开 Git Bash，来到一个空文件夹（或者你想管理的项目文件夹），输入：

```bash
git init
```

就这一行命令。执行后，Git 会告诉你它创建了一个空的仓库。你的文件夹现在变成了一个 Git 仓库。

这个操作**不会**改变你文件夹里的任何现有文件。它只是在文件夹里悄悄创建了一个隐藏目录，叫做 `.git`。

### .git 目录里面有什么？

`.git` 目录是 Git 仓库的"大脑"和"心脏"。所有版本历史、配置、暂存信息都在这里面。

用 `ls -la` 可以看到这个隐藏目录：

```bash
ls -la .git
```

你会看到类似这样的结构：

- `HEAD` —— 指向当前分支的指针（告诉自己"我现在在哪条线上工作"）
- `config` —— 这个仓库的配置信息（比如你的用户名和邮箱）
- `objects/` —— 存放所有版本数据的"库房"，你的每次提交都以某种形式存在这里
- `refs/` —— 存放分支和标签的引用（"指针们"）
- `index` —— 暂存区的信息（后面章节会详细讲）

不用担心记不住这些。现在你只需要知道：`.git` 目录就是 Git 的"档案室"，它默默地把一切版本信息都存放在里面。只要你保留这个 `.git` 目录，你的所有版本历史就都安全。

> **重要提醒**：永远不要手动修改或删除 `.git` 目录里的内容，除非你知道自己在做什么。删掉 `.git` 就等于把整个版本历史都扔了——你文件夹里的文件还在，但所有历史记录都消失了。

### 仓库 vs 普通文件夹

| 普通文件夹 | Git 仓库 |
|-----------|----------|
| 只有文件 | 有文件 + `.git` 目录 |
| 没有版本历史 | 每次提交都被记录 |
| 改了就改了，没法后悔 | 可以回到任意历史版本 |
| 不知道改过什么 | 能精确知道每次改了什么 |

### 什么时候不该使用 git init？

- 如果你想参与一个已存在的项目，应该用 `git clone` 而不是 `git init`
- 不要在已有仓库的文件夹里再执行 `git init`（嵌套仓库会造成混乱）
- 不要把整个"我的文档"目录变成一个巨大的 Git 仓库——仓库应该对应一个项目

## 动手试试

1. 打开 Git Bash，进入你的桌面上某个空文件夹（如果没有，先创建一个）
2. 执行 `git init`
3. 用 `ls -la` 看看 `.git` 目录是否出现了
4. 用 `ls -la .git` 看看里面有哪些文件和文件夹
5. 尝试再执行一次 `git init` —— Git 会告诉你这个文件夹已经是一个仓库了

## 本节小结

`git init` 把一个普通文件夹变成 Git 仓库，相当于给这个文件夹装上了"时光机"。所有版本历史都藏在 `.git` 隐藏目录里。
