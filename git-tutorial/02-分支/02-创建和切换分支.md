# 02 创建和切换分支

## 本节你会学到什么

- 学会用 `git branch` 查看和创建分支
- 学会用 `git switch` 和 `git checkout` 在分支之间切换
- 理解 `HEAD` 指针是什么，它是如何工作的
- 学会创建分支并立刻切换过去的快捷方式

---

## 正文

### 分支的三兄弟命令

在 Git 中，和分支直接相关的有三个命令，你可以把它们想象成三个好兄弟，各有分工：

| 命令 | 分工 | 类比 |
|------|------|------|
| `git branch` | 查看/创建/删除分支 | 像"通讯录"：看看有哪些分支，新建联系人，删除联系人 |
| `git switch` | 切换分支（Git 2.23+ 新命令） | 像"切换电视频道"：从1台换到2台 |
| `git checkout` | 切换分支（老命令，功能更多） | 像"万能遥控器"：不止能换台，还能做很多别的事 |

> **新手建议**：如果你是新手，切换分支时优先使用 `git switch`，语义更清晰，不会和 `git checkout` 的其他功能混淆。`git checkout` 既可以切换分支，又可以恢复文件，功能太多反而不容易记住。不过你在很多老教程中会看到 `git checkout`，所以两个都该认识。

### 查看分支：git branch

打开终端，进入任意一个 Git 仓库，输入：

```bash
git branch
```

输出类似：

```
* main
  feature
```

- `*` 号标记的是你**当前所在的分支**（就像电视遥控器上的频道号高亮）
- 没有 `*` 的是仓库中的其他分支

加上 `-v` 参数可以看到每个分支最后提交的简要信息：

```bash
git branch -v
```

### 创建分支：git branch <分支名>

创建一个新分支，就像在笔记本上贴一张新的标签贴纸：

```bash
git branch feature-login
```

这条命令做了什么？它在当前的位置上创建了一个新指针，名字叫 `feature-login`，指向你当前所在的那次提交。

**重要概念**：创建新分支时，Git 并不会"复制"任何文件。它只是创建了一个**新的指针**（一个新的书签），指向你当前的位置。因此创建分支非常快，几乎瞬间完成。

用图表示：

```
创建前：
A --- B --- C  (main, HEAD -> main)
           ^
           你在这里

运行 git branch feature-login 后：
A --- B --- C  (main, HEAD -> main, feature-login)
           ^
           三个标签都指向同一个位置 C
```

注意，创建分支后，你**仍然在原来的分支上**。`git branch` 只是创建了分支，不会自动切换过去。

### 切换分支：git switch / git checkout

创建了分支之后，你需要"走"过去才能在那个分支上工作。

**使用 git switch**（推荐新手使用）：

```bash
git switch feature-login
```

**使用 git checkout**（老命令）：

```bash
git checkout feature-login
```

两条命令的效果完全一样：把你"传送"到 `feature-login` 分支。

切换分支后，运行 `git branch` 确认一下：

```
  main
* feature-login
```

星号出现在 `feature-login` 前面了，说明你已经切换成功。

### 创建分支并立刻切换（快捷方式）

如果你要创建新分支并立刻切换过去，不需要打两条命令：

```bash
# 方式一：使用 git switch -c（推荐）
git switch -c feature-login

# 方式二：使用 git checkout -b
git checkout -b feature-login
```

`-c` 表示 create（创建），`-b` 表示 branch（分支）。两条命令都等于"创建 + 切换"。

### HEAD：你脑袋上的箭头

在前面的图里你看到了 `HEAD -> main` 这样的标记。`HEAD` 是什么？

**HEAD 就是一个指针，指向你当前所在的分支。** 可以通俗理解成"你脑袋上的箭头"——你的头朝向哪个分支，你就在哪个分支上工作。

当你切换分支时，HEAD 也跟着移动：

```
# 你在 main 分支上
HEAD -> main

# 你切换到 feature-login 分支
HEAD -> feature-login

# 你在 feature-login 上做了一次提交
HEAD -> feature-login（随分支自动向前移动）
```

每次你提交代码时，HEAD 指向的那个分支会自动向前移动（指向新的提交）。

### 切换分支时，工作区发生了什么

这是很多新手困惑的地方。当你从 `main` 切换到 `feature-login` 时，你看到编辑器里的文件内容**可能会变化**！

为什么？因为不同分支上可能有不同的文件内容。Git 会把你工作区的文件**替换成目标分支的文件版本**。这就像你在看电视剧，从第 5 集切换到第 6 集——画面变了。

**重要规则**：切换分支前，请确保当前分支没有未提交的修改。如果有，Git 通常会阻止切换（或者提示你先提交或暂存）。你可以用 `git status` 检查。

### 删除分支

当一个分支的任务完成了（比如功能已合并回主分支），你可以删掉它：

```bash
git branch -d feature-login
```

`-d` 表示 delete。如果分支上的内容还没有被合并，Git 会警告你，防止误删。如果你确定要强制删除，用 `-D`（大写）：

```bash
git branch -D feature-login
```

> **类比**：`-d` 像删除文件到回收站（有保护），`-D` 像 Shift+Delete 永久删除（没保护）。

### 重命名分支

```bash
git branch -m 旧名称 新名称
```

`-m` 表示 move（移动/重命名）。

## 动手试试

在任意一个 Git 仓库中完成以下操作（5 分钟内）：

1. 运行 `git branch` 记录当前有哪些分支
2. 创建一个名为 `test-branch` 的新分支
3. 用 `git switch test-branch` 切换过去
4. 在 `test-branch` 上创建一个新文件并提交
5. 用 `git log --oneline --graph --all` 查看分支图
6. 切换回原来的分支，观察文件是否"消失"了
7. 用 `git branch -d test-branch` 删除这个练习分支

## 本节小结

分支就是指针，创建分支就是贴标签，切换分支就是移动 HEAD 箭头——记住这三句话，分支操作就掌握了。
