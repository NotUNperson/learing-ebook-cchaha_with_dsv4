## 正文

### 任务背景

恭喜你学完了 Git 分支的基础知识。在最后一节，我们来一次"实战演习"。

**场景设定**：你是一名前端开发者，正在维护一个简单的网页项目。目前 `main` 分支上有一个基础的 HTML 页面。你的任务是：通过分支工作流，给页面添加一个导航栏功能。

整个流程如下：

```
1. 从 main 创建功能分支
2. 在功能分支上开发导航栏（多次提交）
3. 与此同时，main 分支上有人改动了同一个文件（模拟协作）
4. 把 main 的最新改动合并/变基到你的功能分支
5. 解决冲突
6. 把功能分支合并回 main
7. 清理现场（删除功能分支）
```

### 完整操作流程

#### 第一步：准备工作

```bash
# 创建项目目录
mkdir nav-project
cd nav-project
git init

# 创建初始 HTML 页面
cat > index.html << 'EOF'
<!DOCTYPE html>
<html>
<head>
    <title>我的网站</title>
</head>
<body>
    <h1>欢迎来到我的网站</h1>
    <p>这是网站的主要内容区域。</p>
</body>
</html>
EOF

git add index.html
git commit -m "初始提交：基础 HTML 页面"
```

#### 第二步：创建功能分支并开发

```bash
# 创建并切换到功能分支
git switch -c feature/navbar

# 在 index.html 中添加导航栏
# 编辑 index.html，在 <body> 后面加入导航栏代码
```

修改后的 `index.html`：

```html
<!DOCTYPE html>
<html>
<head>
    <title>我的网站</title>
</head>
<body>
    <nav>
        <a href="/">首页</a>
        <a href="/about">关于</a>
        <a href="/contact">联系</a>
    </nav>
    <h1>欢迎来到我的网站</h1>
    <p>这是网站的主要内容区域。</p>
</body>
</html>
```

```bash
git add index.html
git commit -m "添加导航栏骨架"

# 再加一些样式（模拟多次提交）
# 在 <nav> 标签中加入 style 属性，或者添加 <style> 块
```

再次修改 `index.html`（在 `<head>` 中添加样式）：

```html
<!DOCTYPE html>
<html>
<head>
    <title>我的网站</title>
    <style>
        nav { background: #333; padding: 10px; }
        nav a { color: white; margin-right: 15px; text-decoration: none; }
    </style>
</head>
<body>
    <nav>
        <a href="/">首页</a>
        <a href="/about">关于</a>
        <a href="/contact">联系</a>
    </nav>
    <h1>欢迎来到我的网站</h1>
    <p>这是网站的主要内容区域。</p>
</body>
</html>
```

```bash
git add index.html
git commit -m "为导航栏添加样式"
```

#### 第三步：模拟 main 分支也有改动

打开另一个终端窗口，或者切回 main 分支：

```bash
git switch main

# 模拟"同事在 main 上修改了同一个文件的同一区域"
# 同事把 <h1> 标签改成了不同的内容
```

修改 `index.html`（在 `<body>` 区域添加了些不同的东西）：

```html
<!DOCTYPE html>
<html>
<head>
    <title>我的网站</title>
</head>
<body>
    <header>
        <h1>欢迎来到我的网站（全新改版！）</h1>
    </header>
    <p>这是网站的主要内容区域。</p>
</body>
</html>
```

```bash
git add index.html
git commit -m "首页改版：用 header 包裹标题，更新标题文案"
```

#### 第四步：同步 main 的最新改动到功能分支

```bash
# 切回功能分支
git switch feature/navbar

# 方式一：用 merge
git merge main

# Git 会报告冲突！因为两个分支都修改了 <body> 区域的开始部分
```

#### 第五步：解决冲突

打开 `index.html`，你会看到冲突标记：

```html
<!DOCTYPE html>
<html>
<head>
    <title>我的网站</title>
    <style>
        nav { background: #333; padding: 10px; }
        nav a { color: white; margin-right: 15px; text-decoration: none; }
    </style>
</head>
<body>
<<<<<<< HEAD
    <nav>
        <a href="/">首页</a>
        <a href="/about">关于</a>
        <a href="/contact">联系</a>
    </nav>
    <h1>欢迎来到我的网站</h1>
=======
    <header>
        <h1>欢迎来到我的网站（全新改版！）</h1>
    </header>
>>>>>>> main
    <p>这是网站的主要内容区域。</p>
</body>
</html>
```

**分析冲突**：
- HEAD（你的 feature/navbar）：有导航栏，和原来的标题
- main（同事的版本）：有 header 包裹的新标题，没有导航栏

**解决方式**：两者都要——保留导航栏，同时采用新的 header 结构。

手动编辑，删除所有冲突标记，整合为：

```html
<!DOCTYPE html>
<html>
<head>
    <title>我的网站</title>
    <style>
        nav { background: #333; padding: 10px; }
        nav a { color: white; margin-right: 15px; text-decoration: none; }
    </style>
</head>
<body>
    <nav>
        <a href="/">首页</a>
        <a href="/about">关于</a>
        <a href="/contact">联系</a>
    </nav>
    <header>
        <h1>欢迎来到我的网站（全新改版！）</h1>
    </header>
    <p>这是网站的主要内容区域。</p>
</body>
</html>
```

```bash
# 标记冲突已解决
git add index.html

# 完成合并
git commit -m "合并 main：整合导航栏和首页改版，解决冲突"
```

#### 第六步：合并回 main

```bash
# 切到 main
git switch main

# 合并功能分支（这次应该是三路合并，但不会有冲突，因为之前已经解决了）
git merge feature/navbar

# 查看最终的分支历史
git log --oneline --graph --all
```

#### 第七步：清理

```bash
# 删除功能分支
git branch -d feature/navbar

# 验证
git branch
```

最终，你的 `main` 分支上是一个集成了导航栏和首页改版的完整页面。

### 回顾完整流程

```
操作                          Git 命令
----                          --------
查看当前状态                   git status, git branch, git log
创建功能分支                   git switch -c feature/navbar
在功能分支上开发（多次提交）     git add, git commit
切换回主分支                   git switch main
模拟他人的改动                 git add, git commit
将主分支改动合并到功能分支      git merge main
解决冲突                      编辑文件, git add, git commit
将功能分支合并回主分支          git switch main, git merge feature/navbar
清理分支                      git branch -d feature/navbar
查看最终历史                  git log --oneline --graph --all
```

## 动手试试

这个练习本身就是动手实践的内容。请从头到尾完整地执行一遍上述流程（约 10-15 分钟）。

**进阶挑战**（如果你还有余力）：
1. 这次不用 `git merge`，而用 `git rebase` 把功能分支变基到 main 的最新提交上，观察冲突解决过程的差异
2. 再创建一个 `feature/footer` 分支，添加页脚，然后合并回 main，体验多个功能分支的管理
3. 在合并时故意不解决冲突就提交——看看 Git 会不会阻止你（答案：会）

## 本节小结

分支工作流的核心就四步：分出去（switch -c）、写代码（add + commit）、合进来（merge）、删分支（branch -d）——重复千遍也别乱。
