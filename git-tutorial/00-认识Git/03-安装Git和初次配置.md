# 03-安装 Git 和初次配置

## 本节你会学到什么

- 在你的 Windows 电脑上成功安装 Git
- 配置你的名字和邮箱（这是 Git 身份系统的基础）
- 验证安装是否成功
- 理解为什么要做这些配置

---

## 安装前的心理建设

这一节可能是整个教程里看起来最"技术"的一节，但别紧张。安装 Git 跟你安装一个游戏或者办公软件差不多——下载、双击、一路点"下一步"，就完成了。

**你不需要理解安装程序的每一个选项是什么意思。** 这个教程只用默认设置就够了。如果以后有人跟你说要改某个选项，那时再改也不迟。

## 第一步：下载 Git

1. 打开浏览器，访问 Git 的官方网站：https://git-scm.com
2. 页面右侧会有一个显眼的下载按钮，上面写着你的系统版本（比如 Windows）
3. 点击下载，你会得到一个 `.exe` 安装文件

如果官网下载比较慢，国内用户也可以访问 Git 的镜像站：https://npm.taobao.org/mirrors/git-for-windows/ —— 选择一个最新版本下载即可。

## 第二步：运行安装程序

双击你下载的安装文件，一路按照下面的指引操作（别的页面保持默认设置，直接点"Next"就行）：

| 安装步骤 | 你要做什么 | 为什么 |
|---------|-----------|-------|
| 选择组件 | 保持默认，直接 Next | 默认选项对初学者最友好 |
| 选择默认编辑器 | **建议选 Notepad++ 或 VS Code**（如果你装了），否则默认的 Vim 对新手不太友好 | 之后 Git 会让你写"提交说明"，需要用一个编辑器来写字 |
| 调整分支名称 | 保持默认，Next | 这是高级话题，现在不用管 |
| PATH 环境 | **选 "Git from the command line and also from 3rd-party software"**（第二个选项） | 这个选项让你能在任何地方打开命令行使用 Git |
| HTTPS 传输 | 保持默认，Next | 使用 Git 自带的加密传输工具 |
| 换行符转换 | **选 "Checkout Windows-style, commit Unix-style line endings"**（第一个选项） | 保证你和用 Mac/Linux 的同事协作时不会因为换行符出问题 |
| 终端模拟器 | **选 "Use Windows' default console window"** | 对你来说最熟悉，后面的教程也在普通命令行里操作 |
| 其他选项 | 全部保持默认，一路 Next 到底 | 默认即最佳 |

点击 Install，等进度条跑完就可以了。

## 第三步：打开命令行

安装完成后，我们要确认一下 Git 是否成功装上了。这里需要打开**命令行**。

如果你从来没有用过命令行，别怕——把它想象成一个"用文字控制电脑的聊天框"就好。你输入一句话，电脑给你一个回答。

在 Windows 上打开命令行最简单的方式：

1. 按键盘上的 `Win` 键（四个小方块的 Windows 徽标键）
2. 输入 `cmd`
3. 按回车（Enter）

你会看到一个黑底白字的窗口——这就是你的"命令行终端"。

## 第四步：验证安装

在命令行窗口里输入下面这行（注意，`--version` 前面有两个短横线），然后按回车：

```bash
git --version
# 查看你安装的 Git 的版本号
```

如果安装成功，你会看到类似这样的输出：

```
git version 2.44.0.windows.1
```

具体的数字可能不同，这不重要。重要的是——如果出现了 `git version x.x.x` 这几个字，说明 Git 已经安好了，恭喜！

如果你看到的是 `'git' 不是内部或外部命令`，说明安装中可能没选对环境变量。回到安装程序重新跑一遍，在"PATH 环境"那一步确认选了第二个选项。或者重启命令行窗口再试一次。

## 第五步：配置你的身份

现在你要告诉 Git："我是谁"。为什么？因为 Git 是协作工具——你做的每一次"存档"，都会签上你的名字和邮箱。这样别人才能知道这个修改是谁做的、怎么联系你。

在命令行里依次输入以下两条（把名字和邮箱换成你自己的）：

```bash
git config --global user.name "Zhang San"
# --global 意思是"全局设置，对所有 Git 仓库生效"
# user.name 后面跟你的名字，可以用中文或英文，建议用拼音

git config --global user.email "zhangsan@example.com"
# 邮箱建议用真实的（虽然不会验证），将来和代码托管平台配合时用得上
```

**参数解释**：
- `git config`：Git 的"设置"命令
- `--global`：表示这是全局设置，对你电脑上所有 Git 仓库都生效。没有这个参数的话，配置只对当前文件夹有效
- `user.name` / `user.email`：你要设置的具体项目

## 第六步：确认配置成功

输入以下命令查看你的配置是否生效：

```bash
git config --global --list
# --list 表示"列出所有设置"
```

你会看到类似这样的输出：

```
user.name=Zhang San
user.email=zhangsan@example.com
```

如果这两行出现了，而且值是你刚才输入的——大功告成！

## 关于"命令行"的额外说明

这一节你第一次接触了命令行。后面我们会用到更多的命令。现阶段你只需要知道：

- 你输入一行文字，按回车，电脑执行
- 执行完会显示结果，然后光标回到原位等你输入下一条
- 这就跟你用微信发消息一样——只不过对方是电脑而不是人

不用担心记不住命令。用多了自然记住。每次给出命令时，教程都会解释它的中文含义。

## 动手试试

1. 打开命令行，输入 `git --version`，确认看到版本号
2. 输入 `git config --global --list`，确认看到你的名字和邮箱
3. 在命令行里输入 `mkdir git-practice` 然后 `cd git-practice`——我们创建了一个空文件夹，后面会用到它
   ```bash
   mkdir git-practice
   # 创建一个叫 git-practice 的文件夹（make directory 的缩写）
   
   cd git-practice
   # 进入这个文件夹（change directory 的缩写）
   ```
4. 把命令行窗口留着不要关，下一节继续用

## 本节小结

Git 安装和配置就像给一把新工具开箱并刻上你的名字——做完这一步，你的电脑已经准备好进入真正的 Git 世界了。
