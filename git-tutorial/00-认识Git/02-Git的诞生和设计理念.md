- 明白为什么 Git 即使在没有网络的情况下也能正常工作

## 一个传奇的诞生故事

2005 年，世界上最著名的开源项目——Linux 操作系统内核——遇到了一个麻烦。

当时 Linux 内核的开发者们使用一个叫 BitKeeper 的商业工具来管理代码。这个工具是付费的，但给开源社区免费使用。然而，因为一些摩擦，BitKeeper 的公司撤回了免费使用权。Linux 社区突然失去了版本控制工具。

换作普通团队，可能会开会讨论、评估市面上已有的替代品、排期迁移。但 Linux 项目的创始人林纳斯-托瓦兹（Linus Torvalds）不是普通人——就是那个写出了 Linux 内核和后来轰动全球的操作系统的家伙。

**他没有去"找"一个工具。他用了两周时间，自己写了一个。**

这个工具就是 Git。林纳斯给它定的设计目标是：

- **速度快**——Linux 内核有几十万个文件，不能慢
- **支持分布式协作**——全世界的开发者都在改 Linux，不能依赖某一台中心服务器
- **数据完整**——任何人的篡改或损坏都能被发现
- **支持非线性开发**——不同的人可以同时在不同的方向上改代码，互不干扰

这些目标，今天读起来可能觉得抽象。别急，我们一步步拆解。

## 集中式 vs 分布式：用共享相册来理解

在 Git 出现之前，主流的版本控制工具（比如 SVN）大多是**集中式**的。Git 带来了**分布式**的理念。

这两个词听起来很学术，我们用一个生活中的类比来区分。

### 集中式版本控制 —— 图书馆的借书登记簿

想象你们办公室有一本共用的相册，存在公司服务器上：
- 谁想看相册，必须连到服务器
- 谁想加照片，必须先把相册从服务器上"借"出来
- 加完照片再"还"回去
- 关键问题：**如果服务器挂了，或者你出差在飞机上没有网，你连看照片都做不到**

这就是集中式：**历史记录和一整套操作能力都在中央服务器上。你的电脑只是一个"工作现场"。**

### 分布式版本控制 —— 每人家里都有一整套相册

现在想象你们把整本相册复印了，每个人家里放一套完整的：
- 你可以在自己家里翻看任何一页，不需要联网
- 你可以在自己家里添加新照片、重新排序，不影响别人
- 当你准备好跟别人分享你的修改时，把你的那套和别人那套"同步"一下就行
- 关键优势：**你的电脑上有完整的历史记录。你拥有一切，不依赖任何一台服务器**

这就是分布式：**每个人手里的都是完整仓库。中央服务器只是大家约定好的一个"交换中心"，并非必需品。**

| 维度 | 集中式（SVN 等） | 分布式（Git） |
|------|-----------------|---------------|
| 历史记录存在哪 | 只存在服务器上 | 每个人电脑上都有一份完整的 |
| 离线能工作吗 | 基本不能 | 完全可以 |
| 服务器挂了怎么办 | 所有人停工，历史可能丢失 | 没影响，随便谁的电脑上都有完整备份 |
| 类比 | 图书馆唯一的一本登记簿 | 每人家里一本完整的相册 |

这解释了为什么你常听到"克隆仓库"这个词——因为你真的会得到**完整的一份**，不只是最新的文件，而是全部历史。

## Git 的"快照"思维——另一个关键设计

大部分版本控制工具记录变化的方式是：**记下文件 A 的第 3 行被改成了什么**（这叫"增量记录"或"差异记录"）。

Git 不是这么想的。Git 的思维方式更像是在**拍照片**。

每当你说"保存这一版"，Git 会给所有文件拍一张"快照"（snapshot）：
- 文件改过了？拍一张新的。
- 文件没改？不重复存储，而是指回上一张快照里的那个文件（节省空间）。

这就像你给房间拍照：
- 传统方式：只记下"沙发往左移了 20 厘米，桌上多了一杯咖啡"
- Git 的方式：直接拍一张新照片。没变的家具在照片里还在，但不占额外存储（因为 Git 知道"那把椅子上一张照片里就有，不用再存一次"）

**快照思维的好处**：
- 查看任何一个历史版本时，Git 不用从头计算"改过什么"，而是直接拿出那一张快照
- 操作速度快得多
- 数据更不容易出错

## Git 的设计哲学

总结一下 Git 的核心设计理念，它们会影响你后面的每一步操作：

1. **每个参与者拥有完整仓库**——你的电脑上的不是"副本"，是"原件之一"
2. **基于快照而非差异**——版本就像一叠即时成像的照片
3. **几乎所有操作都是本地的**——快，且不需要网络
4. **数据完整性内建**——每一个版本的内容都会生成一个唯一的指纹，任何改动都逃不掉
5. **分支极其廉价**——开一个新方向做尝试，几乎零成本（这个我们后面会详细学）

## 动手试试

我们现在还没装 Git，但可以做一个小思考实验：

1. 打开你的手机相册，想象你没有办法把照片存到本地——每次想翻看某张照片，都必须连到一个中央服务器，等它一张一张传给你
2. 再想象，如果你手机存着全部照片，随时可以翻看、编辑、发给朋友，只在想"和云上同步"的时候才联网
3. 哪种体验更好？第二种就是 Git 给你的感觉

## 本节小结

Git 诞生于 Linux 内核社区的需求，核心哲学是"每人一整套、快照而非差异、本地操作优先"——这套哲学会让你的开发体验从"出门必须带钥匙串"变成"口袋里啥都有"。
