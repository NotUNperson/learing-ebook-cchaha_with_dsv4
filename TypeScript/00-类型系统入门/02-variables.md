# 02 变量声明：let、const 与 var

## 本节你会学到什么

- 用 `let` 声明可变变量，理解它和 C++ 普通变量的对应关系
- 用 `const` 声明不可变绑定，弄清楚它和 C++ 的 `const` 哪里像、哪里不一样
- 理解 `var` 的历史问题，知道为什么现代代码里都不用 `var` 了
- 掌握块作用域（block scope）的概念，告别变量"越狱"的困惑

## 正文

上一节我们看了 TypeScript 和 C++ 的宏观差异。从这一节开始，我们真正动手写代码。先学最基础的东西：变量声明。

### let：TypeScript 里的"int"

在 C++ 里，声明一个整数变量：

```cpp
int age = 25;
age = 26;  // 可以修改
```

在 TypeScript 里，最接近的等价写法是：

```typescript
let age: number = 25;
age = 26;  // 也可以修改
```

注意类型标注的位置：C++ 是 `类型 变量名 = 值`（`int age = 25`），TypeScript 是 `let 变量名: 类型 = 值`（`let age: number = 25`）。类型跑到了冒号后面。第一次看到这个语法可能会觉得别扭，但习惯之后你会发现它读起来很像英文："let age, of type number, be 25"（让变量 age，类型为 number，值为 25）。

不过，TypeScript 比 C++ 更聪明的一点在于：你通常**不需要**写那个类型。直接用 `let age = 25`，TypeScript 就自动推断出 `age` 是 `number` 类型。C++ 里 `auto age = 25` 也能推导，但很多 C++ 教程不推荐过度使用 `auto`，而 TypeScript 社区里，能不写类型就不写是标准做法。

### const：TypeScript 里的"const int"

C++ 的 `const`：

```cpp
const double PI = 3.14159;
// PI = 3.14;  // 编译错误，不能修改
```

TypeScript 的 `const`：

```typescript
const PI = 3.14159;
// PI = 3.14;  // 编译错误，不能修改
```

看起来很像是吧？但有一个**关键区别**：

C++ 的 `const` 加在类型前面，表示"这个变量的值在整个生命周期内不能改变"。

TypeScript 的 `const` 表示"这个**绑定**不能重新指向别的东西"。什么意思呢？打个比方：C++ 的 `const` 像是把盒子用胶水封死了，里面的东西不能换。TypeScript 的 `const` 像是把盒子上贴的标签用502胶水粘住了——你不能把标签（变量名）撕下来贴到另一个盒子上。但是，如果盒子里面装的是可以修改的东西（比如一个数组），你**可以**修改盒子里的内容。

```typescript
const numbers = [1, 2, 3];
numbers.push(4);       // 没问题！可以往数组里加东西
// numbers = [5, 6];   // 错误！不能把标签贴到另一个数组上
```

这个区别非常重要。很多人刚学 TypeScript 的时候以为 `const` 让所有东西都变成不可变的，结果被坑了。

### 生活类比：标签贴纸 vs 固定内容

把变量名想象成一张贴在储物箱上的标签贴纸。

- **let**：标签贴在箱子上，你可以把标签撕下来贴到另一个箱子上（重新赋值），也可以更换箱子里的东西。
- **const**：标签贴死了，不能撕下来换地方。但箱子里装的是什么东西，你能不能换，取决于这个箱子本身是什么类型。如果是个"数组箱子"，你可以往里塞新东西；如果是个"数字箱子"，你就不能换里面的数字。
- C++ 的 **const**：箱子本身被封死了，不管里面是数字还是数组，都换不了。

用快递行业类比：
- `let` = 你的快递柜号码可以随时换一个柜子放
- `const` = 你的快递柜号码固定了，但柜子里可以放不同的包裹

### var：一段该知道但不要用的历史

如果你看网上的旧教程，可能会看到 `var` 这个关键字。

```typescript
var name = "Tom";
```

`var` 是 JavaScript（以及 TypeScript）最早的变量声明方式，但它有一个著名的问题：**函数作用域，而不是块作用域**。请看这段代码：

```typescript
if (true) {
    var x = 10;   // 用 var 声明
}
console.log(x);   // 10 —— x 逃出了 if 块！
```

如果用 `let` 就不会：

```typescript
if (true) {
    let y = 10;   // 用 let 声明
}
// console.log(y);  // 编译错误！y 只在 if 块内可见
```

这就像你在一间屋子里说的话（var），结果整栋楼都听见了；而 let 和 const 只让这间屋子里的人听见。

**所以，一句话总结**：永远用 `let` 和 `const`，不要用 `var`。`var` 只存在于历史里，你只需要知道它为什么不好就够了。

### 完整对比表

| 特性 | C++ | TypeScript |
|------|-----|------------|
| 可变变量 | `int x = 1;` | `let x: number = 1;` |
| 不可变变量 | `const int x = 1;` | `const x = 1;` |
| 类型推导 | `auto x = 1;` | `let x = 1;`（自动推导） |
| 作用域 | 块作用域 `{}` | `let`/`const` 是块作用域，`var` 是函数作用域 |
| 重新赋值 | 基本类型不能改 const | `const` 只锁定绑定，不锁定对象内容 |

## 动手试试

1. 声明一个 `let` 变量 `score` 初始值为 0，然后改成 100，打印出来。
2. 声明一个 `const` 变量 `playerName`，尝试修改它（看编译器报什么错）。
3. 声明一个 `const` 数组 `colors = ['red', 'blue']`，用 `push` 加一个 'green'，看是否可以。用赋值的方式 `colors = ['yellow']` 看是否可以。
4. 在一个 `if` 块里用 `var` 声明一个变量，在 `if` 外面访问它，体会"变量越狱"。
5. 对比用 `let` 做同样的事，看区别。

## 本节小结

用 `let` 声明可变变量，用 `const` 声明不可变绑定——注意 `const` 只锁绑定不锁内容，这和 C++ 的 `const` 有本质区别；永远不要用 `var`。
