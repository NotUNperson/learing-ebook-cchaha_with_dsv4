# 04-04 数据持久化：为什么需要卷？

## 本节你会学到什么

- 理解容器存储的"临时性"本质
- 亲眼看到容器删除后数据丢失的惨状
- 理解 Docker Volume 的设计动机
- 区分容器存储层和卷的概念

---

用电脑的时候，你肯定知道两样东西的区别：**内存**和**U盘**。

电脑内存里存的东西，一关机就没了。你写了一半的文档如果只放在内存里，突然断电——哭都来不及。U盘就不一样了，你把文件拷进去、拔掉、插到另一台电脑上，文件还在。

Docker 容器的文件系统，就像**内存**：容器一删，里面的数据就没了。Docker Volume 就是那个 **U盘**：你可以把它"插"到容器上，容器读写它；容器删了，U盘里的数据还好好地在那里；你还能把同一个 U盘"拔下来"插到另一个容器上。

---

## 血的教训：容器删了，数据没了

先来见证一下这个残酷的现实：

```bash
# 启动一个容器，在里面创建一个文件
$ docker run -it --name test alpine:latest sh
/ # echo "important data" > /data.txt
/ # cat /data.txt
important data
/ # exit

# 删掉容器
$ docker rm test

# 重新创建一个同名容器
$ docker run -it --name test alpine:latest sh
/ # cat /data.txt
cat: can't open '/data.txt': No such file or directory
```

文件没了！虽然容器名字一样，但 Docker 创建的是一个全新的容器，新的文件系统，旧的数据不会自动迁移过来。

这也是为什么 Docker 官方文档反复强调：**容器的文件系统是临时的（ephemeral）**。把重要数据直接放在容器文件系统里，等于把论文存在内存里不点保存——早晚要哭。

---

## 数据库在容器里的困境

想象你用 Docker 跑 MySQL：

```bash
$ docker run -d --name mydb -e MYSQL_ROOT_PASSWORD=secret mysql:8
```

数据库跑得好好的，存了几百万条用户数据。有一天你不小心执行了：

```bash
$ docker rm -f mydb    # 删了！
```

全完了。整个数据库——没了。你可能会说"我怎么会不小心删容器呢"，但在真实运维中，升级镜像、迁移服务、清理环境都会导致容器被删除。如果你没有把数据持久化到卷里，删容器就等于格式化硬盘。

---

## 容器为什么是"失忆体质"？

Docker 容器用的是"分层文件系统"。从镜像启动容器时，Docker 在镜像的只读层之上加了一个**可写层**（thin writable layer）。这个可写层只在容器存活期间存在。

- 镜像层 = 只读模板（你下载的 ubuntu 镜像）
- 可写层 = 你在容器里做的所有修改（创建文件、修改配置）

容器一删除，Docker 把这层"撕掉"——你的修改也跟着进了垃圾桶。

```
+------------------+
|  可写层 (你的修改)  |  <-- 容器一删，这层就没了
+------------------+
|  镜像层 3          |
+------------------+
|  镜像层 2          |
+------------------+
|  镜像层 1 (ubuntu) |
+------------------+
```

---

## Volume 的原理：外挂存储

Volume 则完全不同。它绕过容器的文件系统，直接"挂载"到宿主机文件系统上一个由 Docker 管理的目录。

```
容器                   宿主机
+-------+             +------------------+
| /data | ----(挂载)---> | /var/lib/docker  |
|       |             |   /volumes/        |
|       |             |   myvol/_data/     |
+-------+             +------------------+
```

容器看到的是 `/data` 目录，但实际数据存在宿主机上的 `/var/lib/docker/volumes/...` 里。容器删了，宿主机上的数据目录还在——这就是"U盘"不丢的原理。

---

## 三种挂载方式预览

Docker 提供了三种方式来持久化数据：

| 方式         | 管理方     | 类比                         |
| ------------ | -------- | ---------------------------- |
| Named Volume | Docker   | 插在电脑上的外接硬盘，系统帮你管 |
| Bind Mount   | 你自己    | 自己从抽屉里拿出来的 U 盘       |
| tmpfs        | 内存     | 便签纸，写完就撕              |

接下来三节我们逐一深入，但现在你先记住：**凡是需要持久化的数据，一定要用 Volume 或 Bind Mount**。

2. 退出容器，用 `docker rm` 删掉它
3. 重新创建同名容器，检查文件是否还在

---

## 本节小结

容器文件系统是临时的，删了就没了，就像电脑内存断电就清空；Volume 就像 U盘，独立于容器生命周期，插拔数据不丢。
