## 公寓楼 vs 独栋别墅

如果你刚来一座城市，需要一个地方住。你有两种选择：

**方案 A：租一栋独栋别墅。** 这栋别墅有自己完整的地基，独立的水电系统，独立的供暖锅炉，独立的花园和车库。跟隔壁邻居完全隔离，互不影响。住着很舒服，但是贵，一栋别墅光地基就占一大片地。

**方案 B：租一间公寓。** 这栋公寓楼有很多户人家，大家共享整栋楼的地基，共享水电总管道，共享电梯和走廊。你家和邻居家之间只用一堵墙隔开。生活质量差不多，但成本低太多了，一栋楼可以住几十户人家。

独栋别墅就是**虚拟机（VM）**，公寓楼就是**容器（Container）**。

- 虚拟机：每个 VM 有自己的完整操作系统（Guest OS），就像每栋别墅有自己的地基。一台物理机上跑 10 个 VM，就是 10 套完整的操作系统内核，非常占用资源。
- 容器：所有容器共享宿主机的操作系统内核，就像公寓住户共享地基和水电总管道。一台机器上跑几十上百个容器轻轻松松。

## 一张图看懂区别

我们画不出真正的图，但你可以这样理解架构：

```
传统虚拟机架构：                    容器架构：
+-----------------------+          +-----------------------+
|       App A           |          |    App A  |  App B    |
+-----------------------+          +-----------+-----------+
|   Bins/Libs (依赖)     |          | Bins/Libs | Bins/Libs |
+-----------------------+          +-----------+-----------+
|   Guest OS (完整系统)   |          |  Docker Engine        |
+-----------------------+          +-----------------------+
|   Hypervisor (虚拟层)   |          |  Host OS (宿主机系统)    |
+-----------------------+          +-----------------------+
|   Host OS (宿主机)      |          |  Server Hardware       |
+-----------------------+          +-----------------------+
|   Server Hardware      |
+-----------------------+
```

虚拟机：每个应用下面都有一层完整的操作系统，然后是 Hypervisor 虚拟层，最后才是真正的物理宿主机系统。

容器：所有应用共享一个 Docker Engine，Docker Engine 直接跑在宿主机操作系统上，不需要额外的虚拟层和 Guest OS。

## 启动速度的差异

现在你应该能理解为什么容器启动这么快了。

- 启动一个虚拟机 = 重新盖一栋别墅。从打地基（启动 Guest OS）开始，通常需要几十秒到几分钟。
- 启动一个容器 = 打开公寓的门。进程启动即可，通常只要几百毫秒到几秒。

你试试在电脑上启动一个完整的 Ubuntu 虚拟机要多久？运气好也得 30 秒往上。而 `docker run ubuntu echo hello`，连一秒都不到。

## 什么时候用 VM，什么时候用容器？

**用虚拟机的情况：**
- 你需要完全隔离的安全环境（比如跑不受信任的代码）
- 你需要运行不同内核的操作系统（比如在 Linux 主机上跑 Windows 应用）
- 你需要完整的系统级功能（比如自定义内核模块）

**用容器的情况：**
- 你要部署应用，追求快速启动和低资源占用
- 你要做微服务架构，一台机器上跑很多服务
- 你需要一致的开发、测试、生产环境
- 你要做 CI/CD 持续集成和部署

对于大多数应用开发和部署场景来说，容器是更好的选择。这也是为什么 Docker 这么流行。

## 一个常见的误解

新手常常以为"容器就是轻量级虚拟机"。这个理解不算全错，但容易误导。

更准确的理解是：**容器是一个被隔离的进程组**。它不是虚拟机，它没有自己的内核，它跟你宿主机上的其他进程共享同一个内核。只不过 Docker 用 Linux 的 namespace 和 cgroup 技术，让这个进程组以为自己在一个独立的环境里运行——有自己的文件系统、自己的网络、自己的进程空间。

就像你去住酒店：你住的房间给了你"这是我的私人空间"的感觉，但整栋酒店大楼的结构、供水、供电都是共享的。你不是住在独立的别墅里，你只是被隔开了而已。

## 动手试试

在你的机器上运行以下两个命令，对比启动时间：

```bash
# 容器启动一个 Ubuntu 并运行命令（毫秒级）
time docker run --rm ubuntu echo "Hello from container"

# 如果你有 VirtualBox 之类的虚拟机软件，试试启动一个完整 VM
# 直观感受一下启动时间的差距
```

如果还没装 Docker 或者还没拉取 ubuntu 镜像，别急，后续章节会讲到。先记住这个对比的概念就好。

## 本节小结

虚拟机是独栋别墅，每户自带完整地基（操作系统），隔离性强但笨重；容器是公寓楼，共享地基（内核），轻量快速但共享宿主机系统。
