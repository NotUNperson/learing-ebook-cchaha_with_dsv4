## 正文
### 表单是什么 —— 你填过的所有"申请表"

你在生活中一定填过无数表格：银行开户表、体检登记表、快递面单、考试报名表。这些表格有一个共同的结构：
- 有固定的**字段**（姓名、电话、地址...）
- 有些字段是**必填的**（不填就不能提交）
- 填错了会**提示你修改**（"请输入正确的手机号"）
- 填好后**提交**，数据被送到后台处理

Django 表单（Form）就是帮你在网页上实现这套"填表-验证-提交-处理"流程的工具。

### GET vs POST —— 看菜单 vs 下单

HTTP 协议中有多种"请求方法"，最常用的两个是 GET 和 POST。

用去餐厅吃饭来类比：

- **GET 请求** = 你向服务员**要菜单**。"我看看有什么菜"——你只是在读取信息，不会改变任何东西。浏览器访问网页、点链接，都是在发 GET 请求。

- **POST 请求** = 你填好点菜单**交给服务员**。"我要这个、这个、还有这个"——你在提交数据，改变了某些东西（你的订单进入了系统）。

在 Web 开发中：
- GET 用于**读取**数据（查看文章、搜索、浏览页面）。
- POST 用于**提交**数据（登录、注册、发文章、留言）。

同一个 URL（比如 `/blog/create/`）可以同时响应 GET 和 POST：
- GET 请求 → 显示空白表单
- POST 请求 → 处理用户提交的数据

### Django 表单的两种形式

Django 提供两种表单：

**1. `forms.Form`** —— 手动定义每个字段，不关联数据库。

适合：联系表单、搜索框、登录表单（不直接对应某个 Model）。

```python
from django import forms

class ContactForm(forms.Form):
    name = forms.CharField(max_length=100, label="你的名字")
    email = forms.EmailField(label="邮箱地址")
    message = forms.CharField(label="留言内容", widget=forms.Textarea)
```

**2. `forms.ModelForm`** —— 基于 Model 自动生成表单字段。

适合：创建文章、用户注册（直接对应某个 Model）。

```python
from django import forms
from .models import Post

class PostForm(forms.ModelForm):
    class Meta:
        model = Post
        fields = ["title", "content", "is_published"]
        labels = {
            "title": "文章标题",
            "content": "文章内容",
        }
```

`ModelForm` 是 Django 的"快捷方式"——你不用手动定义 title、content 等字段，Django 会从 Model 中自动推断每个字段的类型和约束。

### 在视图中处理表单

表单视图的经典模式是"GET 显示表单，POST 处理提交"：

```python
from django.shortcuts import render, redirect
from .forms import PostForm

def create_post(request):
    if request.method == "POST":
        # 用户提交了表单
        form = PostForm(request.POST)  # 用提交的数据创建表单
        if form.is_valid():            # 验证数据是否合法
            post = form.save()         # ModelForm 可以直接保存到数据库！
            # 保存成功后，跳转到新创建的文章页面
            return redirect("detail", post_id=post.id)
    else:
        # GET 请求，显示空白表单
        form = PostForm()

    return render(request, "blog/create_post.html", {"form": form})
```

关键点解析：
- `request.method` 判断请求类型（`"GET"` 或 `"POST"`）。
- `PostForm(request.POST)` 用用户提交的数据填充表单。Django 会自动从 POST 数据中提取对应字段的值。
- `form.is_valid()` 验证数据：必填字段是否填了？邮箱格式对不对？长度有没有超？全部通过才返回 `True`。
- `form.save()` 是 `ModelForm` 特有的方法——直接创建并保存一条数据库记录。
- `redirect()` 跳转到新页面。这很重要——防止用户刷新页面时重复提交表单。

### cleaned_data —— 验证通过的"干净数据"

如果你用的是普通 `Form`（不是 `ModelForm`），保存前需要手动从表单中取数据：

```python
if form.is_valid():
    name = form.cleaned_data["name"]
    email = form.cleaned_data["email"]
    message = form.cleaned_data["message"]
    # 手动处理这些数据（发邮件、存数据库等）
```

`cleaned_data` 是一个字典，包含所有验证通过的字段值。为什么叫"cleaned"（干净的）？因为 Django 已经帮你做了数据清洗和格式转换——比如 `IntegerField` 的值已经从字符串变成了真正的整数。

### 在模板中渲染表单

模板中渲染表单非常简单：

```html
<form method="post">
    {% csrf_token %}
    {{ form.as_p }}
    <button type="submit">提交</button>
</form>
```

- `{% csrf_token %}` —— 必须有的安全令牌（马上解释）。
- `{{ form.as_p }}` —— 以段落方式渲染所有字段（每个字段包在 `<p>` 标签里）。
- 还可以用 `{{ form.as_table }}`（表格方式）、`{{ form.as_ul }}`（列表方式），或者逐个字段手动渲染。

### CSRF 防护 —— 为什么要写 {% csrf_token %}

CSRF（跨站请求伪造）是 Web 安全里最常见的攻击之一。用场景来理解：

1. 你登录了银行网站（浏览器里有了你的银行登录 cookie）。
2. 你打开了另一个恶意网站。
3. 恶意网站偷偷向银行网站发送了一个"转账"请求。
4. 因为你浏览器还保存着银行的登录状态，银行以为是你本人在操作，钱就被转走了。

Django 的防护方式：在每个表单里嵌入一个**随机生成的令牌**（`{% csrf_token %}`）。服务器收到 POST 请求时会检查令牌是否正确——恶意网站无法获取这个令牌，所以它的伪造请求会被拒绝。

类比：CSRF 令牌就像银行给你的"动态口令 U 盾"。就算有人知道你的账号密码，没有这个物理设备也转不走钱。

你只需要记住一件事：**所有 POST 表单里都必须写 `{% csrf_token %}`**，否则 Django 会拒绝请求并报 403 错误。

## 本节小结
GET = 看菜单（读数据），POST = 下单（提交数据）。ModelForm 让你几行代码就完成"填表-验证-保存"全流程。`{% csrf_token %}` 是必带的安全令牌。
