## 正文
### URL 路由 —— 餐厅菜单上的"点餐指引"

走进一家餐厅，你会先拿到菜单。菜单上写着：
- "宫保鸡丁 —— 28 元 —— 由张师傅负责"
- "麻婆豆腐 —— 22 元 —— 由李师傅负责"

你指着"宫保鸡丁"对服务员说"我要这个"，服务员就去告诉张师傅，张师傅开始做菜。

在 Django 里：
- **URL（网址）** = 菜单上的菜名。比如 `/blog/`、`/about/`、`/post/5/`。
- **视图函数** = 负责这道菜的厨师。每个 URL 都有一个对应的视图函数来处理。
- **URL 路由配置** = 那张菜单。它记录了哪个 URL 对应哪个视图函数。

专业术语叫"URL 路由"（URL Routing），其实就是"网址导航"的意思。用户输入了某个 URL，Django 就根据路由表找到对应的视图函数去处理。

### 两层路由 —— 总台接听 + 转接到店铺

Django 的路由分两层：

**第一层：项目级路由（`myblog/urls.py`）**

这是"商场总服务台"。有人打来电话，总台判断"这个人找的是哪家店"，然后转接过去。

```python
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path("admin/", admin.site.urls),
    path("blog/", include("blog.urls")),  # 以 /blog/ 开头的请求 → 转给 blog 应用
]
```

`include("blog.urls")` 的意思是："所有以 `/blog/` 开头的网址，交给 blog 应用自己处理，我就不管了。"

**第二层：应用级路由（`blog/urls.py`）**

这是你"店铺内部的导航"。`blog/urls.py` 需要你自己创建：

```python
from django.urls import path
from . import views  # 导入同目录下的 views.py

urlpatterns = [
    path("", views.index, name="index"),        # /blog/ → index 视图
    path("about/", views.about, name="about"),   # /blog/about/ → about 视图
    path("post/<int:post_id>/", views.detail, name="detail"),  # 带参数
]
```

### 视图函数 —— 厨师的工作台

视图函数（View）就是 Django 里的"厨师"。它接收顾客的"点单"（request），做出一盘"菜"（response）。

最简单的视图函数：

```python
# blog/views.py
from django.http import HttpResponse

def index(request):
    # 任何视图函数的第一个参数都是 request
    # request 包含了这次请求的所有信息
    return HttpResponse("<h1>欢迎来到我的博客！</h1>")
```

几点说明：
- `request` 是 Django 自动传给你的，包含了用户请求的所有信息：访问了什么 URL、用了什么方法（GET/POST）、提交了什么数据、浏览器类型等等。
- `HttpResponse` 是最简单的返回方式，直接返回一段字符串（通常是 HTML）。
- 视图函数**必须返回一个响应对象**（HttpResponse 或其子类）。不返回的话 Django 会报错。

### 路径参数 —— 动态的 URL

你的博客肯定不止一篇文章，每篇文章需要一个独立的 URL。你不会为每篇文章都写一行 `path()`，而是用**路径参数**：

```python
# /blog/post/1/、/blog/post/2/、/blog/post/3/ 都匹配这一行
path("post/<int:post_id>/", views.detail, name="detail")
```

`<int:post_id>` 的含义：
- `<...>` 表示这是一个"变量部分"，不是固定的文字。
- `int` 是类型转换器，告诉 Django "这里只能匹配整数"。
- `post_id` 是参数名，Django 会把这个值传给视图函数。

对应的视图函数多了一个参数：

```python
def detail(request, post_id):
    return HttpResponse(f"<h1>文章 #{post_id}</h1>")
```

Django 内置的路径转换器：

| 写法 | 匹配内容 | 示例 |
|------|---------|------|
| `<int:id>` | 整数 | `/post/5/` → id=5 |
| `<str:name>` | 字符串（默认） | `/user/zhangsan/` → name="zhangsan" |
| `<slug:title>` | 字母、数字、横线、下划线 | `/post/hello-world/` → title="hello-world" |
| `<uuid:id>` | UUID 格式 | 略 |
| `<path:url>` | 完整路径（包含 /） | 略 |

### `name` 参数 —— 给 URL 起个名字

你注意到每个 `path()` 都有一个 `name` 参数：

```python
path("", views.index, name="index"),
```

这个 `name` 是给路由起的"小名"。有了名字之后，你就可以在代码里用名字来引用这个 URL，而不是硬编码 `"/blog/"`。这样做有一个巨大的好处：如果以后你改了 URL 规则（比如 `/blog/` 改成 `/articles/`），只要 `name` 不变，所有引用这个路由的地方会自动更新。

在视图中反向生成 URL：

```python
from django.shortcuts import redirect
# 跳转到 name="detail" 的页面，传入 post_id=5
return redirect("detail", post_id=5)
# 自动生成 /blog/post/5/ 这个 URL
```

## 本节小结
urls.py = 菜单（网址 → 视图函数），视图函数 = 厨师（收 request，返回 response），路径参数让 URL 动起来。
