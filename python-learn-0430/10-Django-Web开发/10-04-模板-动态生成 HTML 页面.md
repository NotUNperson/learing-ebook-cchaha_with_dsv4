# 10-04 模板：动态生成 HTML 页面

## 本节你会学到什么
- 理解模板的作用：把 HTML 和 Python 数据分离
- 学会使用模板变量 `{{ }}` 和模板标签 `{% %}`
- 学会用 `render()` 函数渲染模板
- 学会使用模板继承来避免重复代码

## 正文
### 为什么需要模板 —— 手写 HTML 太笨了

上一节我们是这样返回网页的：

```python
def index(request):
    return HttpResponse("<h1>欢迎来到我的博客</h1><p>今天天气真好</p>")
```

这有几个明显的问题：
1. HTML 混杂在 Python 代码里，看得眼花。
2. 如果要改页面布局，得去 Python 代码里翻找。
3. 如果一个页面有几百行 HTML（现实中的网页都这样），你的视图函数会长到可怕。
4. 最关键的——你没法把**动态数据**塞进去。比如"博客首页显示所有文章"——每篇文章在数据库里，你怎么把它们变成 HTML 里的一行行内容？

Django 的模板系统就是解决这些问题的。

### 模板的类比 —— 作文填空题

想象你是一个语文老师，让你班上的 40 个学生每人写一篇"我的周末"作文。如果每个学生都从白纸开始写，40 篇作文千奇百怪，有的忘了写日期，有的忘了写名字。

于是你印了一叠**填空作文模板**：

```
我的周末
姓名：________
日期：________

这个周末，我去了________，在那里我看到了________。
我觉得________。
```

骨架是固定的（标题、结构），但空白处每个人填的不同。

Django 模板就是这个思路：
- **骨架（HTML 结构）** 写在 `.html` 模板文件里，固定不变。
- **空白横线（模板变量）** 用 `{{ 变量名 }}` 表示，等渲染时再填。
- **指令语句（模板标签）** 用 `{% 语句 %}` 表示，比如"循环显示所有文章"、"如果已登录就显示用户名"。

### 模板变量 —— 填空的横线

模板变量用两个大括号包裹：`{{ variable }}`。

模板文件（`templates/blog/index.html`）：

```html
<h1>{{ title }}</h1>
<p>欢迎你，{{ username }}！</p>
<p>当前时间：{{ now }}</p>
```

视图函数中传数据：

```python
def index(request):
    context = {
        "title": "我的博客",
        "username": "小明",
        "now": "2024-03-15 14:30:00",
    }
    return render(request, "blog/index.html", context)
```

Django 渲染后的最终 HTML：

```html
<h1>我的博客</h1>
<p>欢迎你，小明！</p>
<p>当前时间：2024-03-15 14:30:00</p>
```

`render()` 函数做了三件事：
1. 找到模板文件 `blog/index.html`。
2. 把 `context` 字典里的数据"填入"模板的 `{{ }}` 位置。
3. 返回填好数据的完整 HTML（包装成 HttpResponse）。

### `render()` 详解

```python
render(request, template_name, context=None)
```

三个参数：
- `request`：Django 传给你的请求对象（每个视图必有的第一个参数）。
- `template_name`：模板文件的路径。Django 默认在 `templates/` 目录下查找。
- `context`：一个字典，键就是模板中可用变量名，值就是实际数据。

### 模板标签 —— 模板里的"编程语句"

模板不只是填填空——它还可以做循环、条件判断等逻辑操作。这些操作用 `{% %}` 包裹，叫"模板标签"。

**循环：**

```html
<ul>
    {% for post in posts %}
        <li>
            <a href="/blog/post/{{ post.id }}/">{{ post.title }}</a>
        </li>
    {% empty %}
        <li>暂无文章</li>
    {% endfor %}
</ul>
```

`{% for %}` 会遍历 `posts` 列表，每次迭代生成一个 `<li>` 标签。`{% empty %}` 是可选的——如果列表为空，就显示"暂无文章"。

**条件判断：**

```html
{% if user.is_authenticated %}
    <p>欢迎回来，{{ user.username }}！</p>
{% else %}
    <p>请先<a href="/login/">登录</a></p>
{% endif %}
```

Django 模板语言的逻辑标签和 Python 的关键字非常像（`for`、`if`、`else`），区别只是用 `{% %}` 包起来，而且必须有关闭标签（`{% endfor %}`、`{% endif %}`）。

### 模板继承 —— 一次写好，处处复用

一个网站的所有页面，通常有相同的头部（导航栏）、尾部（版权信息）、侧边栏。如果每个页面都复制粘贴这些 HTML，以后想改导航栏的一个链接，得改几十个文件——这简直是噩梦。

Django 的**模板继承**完美解决这个问题。

**父模板**（`templates/base.html`）—— 定义"骨架"：

```html
<!DOCTYPE html>
<html>
<head>
    <title>{% block title %}默认标题{% endblock %}</title>
</head>
<body>
    <nav>
        <a href="/">首页</a>
        <a href="/about/">关于</a>
    </nav>

    {% block content %}
    <!-- 这里是留给子模板填充内容的区域 -->
    {% endblock %}

    <footer>版权信息</footer>
</body>
</html>
```

**子模板**（`templates/blog/index.html`）—— 只写"肉"：

```html
{% extends "base.html" %}

{% block title %}博客首页{% endblock %}

{% block content %}
    <h1>欢迎来到我的博客</h1>
    {% for post in posts %}
        <p>{{ post.title }}</p>
    {% endfor %}
{% endblock %}
```

`{% block %}` 就像在父模板里挖了几个"坑"，子模板用 `{% extends %}` 继承后，只需要往这些坑里填内容。导航栏和页脚只在一处定义，所有页面自动共享——修改一处，全局生效。

### 模板文件的目录结构

Django 默认在 `settings.py` 的 `TEMPLATES` 配置中指定模板目录。推荐的做法是：

```
myblog/
├── templates/             # 在项目根目录创建（和 manage.py 同级）
│   ├── base.html          # 基础模板（骨架）
│   └── blog/              # 按应用分文件夹
│       ├── index.html     # 博客首页
│       ├── detail.html    # 文章详情页
│       └── create.html    # 创建文章页
```

在 `settings.py` 中确保 `DIRS` 包含这个路径：

```python
TEMPLATES = [
    {
        "BACKEND": "...",
        "DIRS": [BASE_DIR / "templates"],  # 添加这一行
        "APP_DIRS": True,
        # ...
    },
]
```

`APP_DIRS: True` 表示 Django 也会在每个应用的 `templates/` 子目录下查找模板。

6. 在 `context` 字典里传入一个文章列表（先手动写死几条数据），在模板中用 `{% for %}` 循环显示。

## 本节小结
模板 = 填空作文（骨架固定，内容动态），变量用 `{{ }}`，标签用 `{% %}`，`render()` 负责把数据填进去。模板继承让你写一次骨架，处处复用。
