# 06-03 激活与退出虚拟环境

## 本节你会学到什么
- 在不同操作系统下激活虚拟环境
- 判断自己当前是否在虚拟环境中
- 退出（停用）虚拟环境

## 正文

### 用"走进办公室"类比激活

上节我们已经"租到了办公室"（创建了虚拟环境）。但光有办公室还不够，你得**走进去**才能开始工作。

激活虚拟环境，就是"走进去"。激活之后，你在终端里运行的 `python` 和 `pip` 都是这个环境专属的，装的库也只影响这个环境。

退出虚拟环境，就是"走出办公室，下班回家"。

### Windows 下激活虚拟环境

打开 PowerShell 或命令提示符，进入到你的项目目录，然后：

```bash
venv\Scripts\activate
```

如果你的虚拟环境名字不是 `venv`，替换一下就行，比如 `myenv\Scripts\activate`。

激活成功的标志是：**终端提示符前面会出现环境名字**，像这样：

```
(venv) C:\Users\你的用户名\test_project>
```

那个 `(venv)` 就是视觉提示：你现在在虚拟环境里面了。

### Mac / Linux 下激活虚拟环境

```bash
source venv/bin/activate
```

同样，成功后终端提示符前面会出现 `(venv)`：

```
(venv) $ 
```

注意 Mac/Linux 用的是正斜杠 `/`，Windows 用的是反斜杠 `\`。另外 Mac/Linux 多了 `source` 命令（也可以用 `. venv/bin/activate`，效果一样）。

### 怎么确认自己已经在虚拟环境里？

三个方法：

1. **看提示符**：最直观，前面有括起来的环境名。
2. **用 `which` 或 `where` 命令**：查看当前用的是哪个 Python。

   Windows 上：
   ```bash
   where python
   ```
   会显示 `...\venv\Scripts\python.exe`（带 venv 路径），而不是 `C:\Python311\python.exe`。

   Mac/Linux 上：
   ```bash
   which python
   ```
   会显示 `.../venv/bin/python`。

3. **查看 `sys.prefix`**：
   ```bash
   python -c "import sys; print(sys.prefix)"
   ```
   如果输出的是你的虚拟环境路径，那就在环境里。

### 退出虚拟环境

在任何操作系统上，退出命令都一样：

```bash
deactivate
```

运行后，终端提示符前面的 `(venv)` 消失，你又回到了全局环境。

### 一个完整的操作流程

假设你在 Windows 上，项目叫 `my_project`：

```bash
# 1. 进入项目目录
cd D:\my_project

# 2. 创建虚拟环境（只需要做一次）
python -m venv venv

# 3. 激活虚拟环境
venv\Scripts\activate

# 4. 现在提示符变成 (venv)，可以开始安装包、写代码了
pip install requests

# 5. 工作完成，退出虚拟环境
deactivate
```

每次打开新的终端窗口，都需要**重新激活**。虚拟环境的激活只在当前终端窗口有效，关掉就没了（这是设计如此，不是 bug）。

### 一个小提醒

如果你用的是 IDE（比如 VS Code、PyCharm），IDE 通常会自动检测到项目里的虚拟环境，并在你打开终端时自动激活。但自己手动打开终端（非 IDE 内置的）时，还是得手动激活。

## 动手试试

1. 用上节创建的 `test_project` 文件夹。
2. 激活里面的虚拟环境（根据你的操作系统选命令）。
3. 运行 `where python`（Windows）或 `which python`（Mac/Linux），确认 Python 路径指向虚拟环境。
4. 运行 `deactivate` 退出，然后再运行一次 `where python` / `which python`，对比两次的结果。
5. 重复激活和退出几遍，熟悉这个"进门出门"的流程。

## 本节小结

激活虚拟环境 = 进入工作状态，`deactivate` = 下班退出；不同操作系统激活命令稍有不同，但退出命令一样。
