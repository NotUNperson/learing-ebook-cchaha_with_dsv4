# 06-02 创建虚拟环境

## 本节你会学到什么
- 使用 `venv` 命令创建虚拟环境
- 理解虚拟环境目录的结构
- 知道在哪里创建虚拟环境比较合适

## 正文

### 用"租办公室"类比创建虚拟环境

上一节我们说了，虚拟环境是给每个项目一个独立厨房。那么创建虚拟环境，就像是**去物业那里登记，拿到这个厨房的钥匙**。

在 Python 里，"物业"就是 `venv` 模块（Python 3.3 开始内置，不需要额外安装）。这个模块专门负责创建和管理虚拟环境。

### 创建虚拟环境的命令

打开终端（Windows 用 PowerShell 或命令提示符，Mac/Linux 用终端），输入：

```bash
python -m venv myenv
```

这行命令是什么意思？拆开看：

- `python`：调用 Python 解释器
- `-m venv`：以模块方式运行 `venv`（"venv" 是 virtual environment 的缩写）
- `myenv`：虚拟环境的名字（可以随便取，常见的有 `venv`、`env`、`.venv`）

运行之后，你会在当前目录下看到一个叫 `myenv` 的文件夹。注意，这个文件夹是**在你当前所在的位置生成**的，所以创建前最好先 `cd` 到你的项目目录。

### 虚拟环境文件夹里有什么？

创建完成后，进入 `myenv` 目录看看。在 Windows 上大概长这样：

```
myenv/
    Include/          # C 语言头文件（基本用不到，不用管）
    Lib/              # 库文件夹
        site-packages/  # 第三方库就装在这里
    Scripts/          # 激活脚本和其他工具
        activate       # 激活脚本（Windows）
        pip.exe        # 这个环境专属的 pip
        python.exe     # 这个环境专属的 Python
    pyvenv.cfg        # 配置文件
```

在 Mac/Linux 上，`Scripts/` 会变成 `bin/`，但功能一模一样。

最重要的几个部分：

| 文件/目录 | 作用 |
|----------|------|
| `Lib/site-packages/` | 你装的第三方库全部存在这里 |
| `Scripts/python.exe` | 这个环境专属的 Python 解释器 |
| `Scripts/pip.exe` | 这个环境专属的 pip（包管理器） |
| `Scripts/activate` | 激活脚本，下节讲 |
| `pyvenv.cfg` | 记录这个环境关联的是哪个 Python |

### 虚拟环境应该放在哪里？

两种常见习惯：

1. **放在项目目录里面**（推荐新手）：比如你的项目在 `D:/my_project/`，就在这个目录下创建 `venv`。好处是项目和它的虚拟环境绑在一起，一看就知道。
2. **统一放在一个地方**（适合管理多个项目）：比如把所有虚拟环境放到 `~/.virtualenvs/`。但需要额外工具（如 `virtualenvwrapper`），对我们现阶段来说不必要。

推荐的做法是：在每个项目的根目录下创建虚拟环境，名字就叫 `venv` 或 `.venv`。

```bash
cd my_project
python -m venv venv
```

### 命名建议

虚拟环境的名字只是文件夹名，所以你喜欢叫什么都可以。但业界惯例是用这几个名字：

- `venv`（最常用）
- `.venv`（前面加个点，Linux/Mac 下是隐藏文件夹）
- `env`

用 `venv` 就好，简单明了。

## 动手试试

1. 在你的桌面或任意位置，新建一个文件夹叫 `test_project`。
2. 打开终端，`cd` 到这个文件夹。
3. 运行 `python -m venv venv` 创建虚拟环境。
4. 查看 `venv` 文件夹的内容，看看里面有哪些目录和文件。

不需要写 Python 代码，这一步全在终端里完成。完成这一步，你就"租到了自己的办公室"！

## 本节小结

`python -m venv 环境名` 创建虚拟环境，本质上就是生成一个包含独立 Python 和 pip 的文件夹。
