## 正文
### pygame 是什么 —— 做游戏的工具箱

想象一下，你想自己动手做一个风筝。你需要竹条做骨架、纸张做蒙面、胶水粘合、绳子牵引。如果每个东西都自己从零做——自己砍竹子、自己造纸、自己搓绳子——那你可能花一个礼拜都做不完。

做游戏也一样。如果你想用最原始的 Python 做一个游戏，你得自己处理怎么在屏幕上画东西、怎么检测键盘按了什么键、怎么让画面流畅地刷新。这些事情每一项都极其复杂。

pygame 就是那个"做风筝的工具箱"。它帮你把最底层的、最麻烦的事情（画窗口、检测键盘鼠标、播放声音、管理画面刷新）都处理好了，你只需要关心一件事——我的游戏怎么玩。

具体来说，pygame 能帮你做这些事：

- **创建窗口**：不需要理解操作系统的窗口系统，两行代码就能创建一个游戏窗口。
- **绘制图形和图片**：可以画圆、画方、画线，也可以加载 PNG/JPG 图片显示在窗口里。
- **处理输入**：键盘按键、鼠标点击和移动，都能轻松获取。
- **播放声音和音乐**：背景音乐、音效，都能搞定。
- **碰撞检测**：判断两个物体是不是撞在一起了。
- **精灵系统**：批量管理游戏中所有会动的角色。

### 安装 pygame —— 把工具箱搬回家

pygame 不是 Python 自带的，需要用 pip 安装。打开你的终端（命令行），输入：

```bash
pip install pygame
```

或者如果你用的是 Python 3：

```bash
pip3 install pygame
```

等它跑完，看到 `Successfully installed pygame-xxx` 就说明装好了。

验证是否装好，可以在 Python 交互环境里试一下：

```python
import pygame
print(pygame.version.ver)
```

如果没有报错，并且打印出了版本号（比如 `2.5.2`），恭喜你，工具箱已经就位了。

### 第一个 pygame 程序 —— 打开工具箱看看里面有什么

新手学任何库，第一件事不是写出多厉害的游戏，而是确认"这东西能跑"。我们写一个最简短的程序来验证一切正常：

```python
import pygame

# 1. 初始化 —— 打开工具箱
pygame.init()
print("pygame 初始化成功！")

# 2. 创建一个 400x300 的窗口
screen = pygame.display.set_mode((400, 300))
pygame.display.set_caption("Hello pygame!")

# 3. 让窗口保持打开 3 秒
pygame.time.wait(3000)

# 4. 退出 —— 收拾工具
pygame.quit()
```

运行这段代码，你会看到一个黑色的窗口弹出来，标题是"Hello pygame!"，3 秒后自动关闭。这就是 pygame 的"Hello World"。

每一行的含义：

- `pygame.init()` —— 初始化 pygame 的所有模块。相当于把工具箱打开，把工具都摆好。
- `pygame.display.set_mode((400, 300))` —— 创建一个 400 像素宽、300 像素高的窗口。`display` 模块负责"显示器"相关功能；`set_mode` 是"设置模式"。
- `pygame.display.set_caption(...)` —— 设置窗口的标题文字。
- `pygame.time.wait(3000)` —— 让程序暂停 3000 毫秒（即 3 秒）。
- `pygame.quit()` —— 关闭 pygame，释放资源。相当于用完工具后把它们都收好。

运行 `examples/09-01-pygame-intro.py`，你会看到更详细的版本信息，包括 pygame 有哪些功能模块可用。

## 本节小结
pygame 是 Python 做 2D 游戏最常用的库，用 pip 一行命令就能安装，核心思路是"它帮你处理底层脏活，你专注游戏逻辑"。
