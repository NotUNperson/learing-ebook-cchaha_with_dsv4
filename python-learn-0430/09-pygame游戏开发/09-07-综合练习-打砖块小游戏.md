## 正文
### 打砖块是什么样的游戏

打砖块（Breakout）是电子游戏史上最经典的作品之一，1976 年由雅达利公司推出。玩法极其简单：

- 屏幕上方排列着几排彩色砖块。
- 屏幕下方有一块可以左右移动的挡板。
- 一个小球在屏幕上弹来弹去。
- 你移动挡板接住球，让球弹回去撞碎砖块。
- 消除所有砖块就赢了。
- 球掉出屏幕底部就失去一条命，命用完了就输了。

就是这么简单的规则，却让人玩得上瘾。我们的目标就是用 pygame 把它做出来。

### 游戏的整体结构

把我们的 280 行代码拆开来，可以分为这几个部分：

**1. 常量定义区**
所有游戏参数的"集中营"：窗口大小、颜色、挡板大小、球的速度、砖块的行列数……把这些数字集中定义在文件顶部，想调参数的时候不用满文件翻找。

```python
WIDTH, HEIGHT = 800, 600
PADDLE_WIDTH = 120
BALL_SPEED_X = 5
BRICK_ROWS = 7
BRICK_COLS = 10
# ...
```

**2. 游戏状态变量**
描述游戏"此刻是什么样子"的所有变量：
- `paddle_x`：挡板的位置
- `ball_x`, `ball_y`：球的位置
- `ball_dx`, `ball_dy`：球的速度（方向和快慢）
- `ball_moving`：球是否正在飞行
- `bricks`：所有砖块的状态列表
- `lives`：剩余生命
- `score`：得分
- `game_over`：游戏是否结束
- `win`：是否胜利

**3. 主循环的四步**
和之前学的一样，每帧重复：事件处理 → 更新状态 → 绘制画面 → 控制帧率。

### 球碰到砖块的反弹逻辑

这段代码值得特别讲解，因为它涉及到怎么判断球从哪个方向撞到砖块：

```python
# 球和砖块的中心点
ball_center_x = ball_x + BALL_SIZE / 2
ball_center_y = ball_y + BALL_SIZE / 2
brick_center_x = rect.x + BRICK_WIDTH / 2
brick_center_y = rect.y + BRICK_HEIGHT / 2

# 球中心到砖块中心的偏移量
dx = ball_center_x - brick_center_x
dy = ball_center_y - brick_center_y

# 判断水平还是垂直碰撞
if abs(dx) * (BRICK_HEIGHT / 2) > abs(dy) * (BRICK_WIDTH / 2):
    ball_dx = -ball_dx  # 水平反弹
else:
    ball_dy = -ball_dy  # 垂直反弹
```

这个判断逻辑的直觉是这样的：比较球中心相对于砖块中心的角度。如果球在砖块的"左/右"方向更远，就是水平碰撞；如果在"上/下"方向更远，就是垂直碰撞。

乘以宽高比是为了把矩形"校正"成正方形来判断方向，避免因矩形太扁或太窄导致误判。

### 球碰到挡板的"角度控制"

如果球碰到挡板时总是原路弹回，游戏会非常无聊。我们让球碰到挡板的不同位置时，弹射的角度也不同：

```python
# 偏移量：球的中心相对于挡板中心的位置
offset = (ball_x + BALL_SIZE/2) - (paddle_x + PADDLE_WIDTH/2)
# 偏移量越大，水平速度越大
ball_dx = offset / 30 * abs(BALL_SPEED_X)
```

球碰到挡板左边 → 球往左弹；碰到右边 → 球往右弹。这给了玩家一定的"控球"能力，让游戏更有策略性。

### 游戏的状态管理

这个游戏有几种截然不同的"状态"：

| 状态 | `ball_moving` | `game_over` | 表现 |
|------|---------------|-------------|------|
| 等待发球 | `False` | `False` | 球停在挡板上方，等待玩家按空格 |
| 游戏进行中 | `True` | `False` | 球在飞行，正常处理碰撞 |
| 胜利 | `False` | `True` + `win=True` | 显示"恭喜通关" |
| 失败 | `False` | `True` + `win=False` | 显示"游戏结束" |

把状态管理清楚，是写出不乱的游戏代码的关键。
