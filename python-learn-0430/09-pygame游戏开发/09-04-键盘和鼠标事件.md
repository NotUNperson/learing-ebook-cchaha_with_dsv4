## 正文
### 事件是什么 —— 游戏手柄的每一个"信号"

玩过游戏机的都知道，你按手柄上的 A 键，游戏里的角色就会跳一下。你推摇杆，角色就会走。你的每一个操作，都会产生一个"信号"传给游戏。

pygame 里管这些"信号"叫**事件（Event）**。每个事件都包含类型信息（是什么类型的事件）和附加数据（比如按了哪个键、鼠标在什么位置）。

常见的事件类型：

| 事件类型 | 什么时候触发 | 附带信息 |
|---------|------------|---------|
| `QUIT` | 玩家点窗口右上角的 X | 无 |
| `KEYDOWN` | 键盘有键被按下 | `event.key`（哪个键） |
| `KEYUP` | 键盘有键被松开 | `event.key`（哪个键） |
| `MOUSEBUTTONDOWN` | 鼠标按键被按下 | `event.pos`（坐标）、`event.button`（左/中/右键） |
| `MOUSEBUTTONUP` | 鼠标按键被松开 | 同上 |
| `MOUSEMOTION` | 鼠标移动了 | `event.pos`、`event.rel`（相对移动量） |

### 事件处理的三层结构

每一帧，我们都要处理事件的"收件箱"：

```python
for event in pygame.event.get():      # 第1层：遍历所有"新信"
    if event.type == pygame.QUIT:     # 第2层：判断"信的类型"
        running = False
    elif event.type == pygame.KEYDOWN: # 第2层：另一种信的类型的处理
        if event.key == pygame.K_UP:   # 第3层：读取信的具体内容
            player_y -= 5
```

三层结构就像这样：
- 第一层：看看收件箱里有没有新信（遍历事件列表）。
- 第二层：判断每封信是什么类型（催缴单？广告？私人信件？）。
- 第三层：如果这封信我关心，就读它的具体内容（催缴单上写了多少钱？）。

### KEYDOWN 事件 —— 玩家按了哪个键

```python
if event.type == pygame.KEYDOWN:
    if event.key == pygame.K_UP:
        y -= 5       # 向上键 → 向上移动
    elif event.key == pygame.K_DOWN:
        y += 5       # 向下键 → 向下移动
```

常用的按键常量：
- 方向键：`K_UP`、`K_DOWN`、`K_LEFT`、`K_RIGHT`
- 字母键：`K_a` ~ `K_z`
- 数字键：`K_0` ~ `K_9`、`K_1` ~ `K_9`
- 功能键：`K_SPACE`（空格）、`K_RETURN`（回车）、`K_ESCAPE`（ESC）

### 持续按键 vs 单次按键

`KEYDOWN` 事件只在按键**被按下的那一瞬间**触发一次。如果你想实现"按住方向键持续移动"的效果，需要用另一种方式：

```python
keys = pygame.key.get_pressed()  # 获取当前所有按键的状态
if keys[pygame.K_LEFT]:
    x -= speed  # 只要左键是"按住"状态，就持续移动
```

`pygame.key.get_pressed()` 返回一个"按键状态字典"——每个键对应一个布尔值（`True` 表示正在按下，`False` 表示没按下）。在 09-05 的示例中会用到这种方式。

### 鼠标事件

鼠标事件有三个关键属性：
- `event.pos`：鼠标的 `(x, y)` 坐标。
- `event.button`：1 = 左键，2 = 中键，3 = 右键。
- `event.buttons`：在 MOUSEMOTION 事件中，表示当前有哪些键被按住（`event.buttons[0]` 表示左键状态）。

```python
if event.type == pygame.MOUSEBUTTONDOWN:
    if event.button == 1:  # 左键点击
        player_x, player_y = event.pos  # 把方块移到鼠标点击的位置
```

## 本节小结
pygame 的事件系统像一个收件箱，每帧遍历新事件，根据事件类型和内容做出响应。键盘用 KEYDOWN，鼠标用 MOUSEBUTTONDOWN/MOTION，退出用 QUIT。
