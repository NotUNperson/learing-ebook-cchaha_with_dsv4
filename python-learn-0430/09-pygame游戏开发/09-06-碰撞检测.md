## 正文
### 碰撞检测是什么 —— 碰碰车的感应器

你去游乐场玩过碰碰车吗？当你的车和别人的车撞在一起的时候，"砰"的一下，你就知道撞上了。游戏里的碰撞检测就是那个"砰"——判断两个游戏角色是不是碰到了一起。

为什么碰撞检测这么重要？
- 子弹碰到敌人 → 敌人受伤
- 主角碰到金币 → 得分
- 角色碰到墙壁 → 不能穿过去
- 球碰到球拍 → 反弹回去

几乎所有的游戏互动，底层都是碰撞检测。

### Rect 对象 —— 每个角色的"碰撞感应范围"

我们知道，`pygame.Rect` 是一个矩形对象。它不光能描述一个矩形的位置和大小，还自带了一套碰撞检测的方法。你可以把它理解成每个角色身上带着的"感应器腰带"。

创建一个 Rect：

```python
# 方式1：直接创建
player_rect = pygame.Rect(100, 200, 50, 50)
#                         x,   y,   宽, 高

# 方式2：从 Surface 的 get_rect() 获取
player_rect = player_image.get_rect()
player_rect.x = 100
player_rect.y = 200
```

Rect 有很多方便的属性：

```python
rect = pygame.Rect(100, 50, 200, 150)

rect.x           # 100    左上角 x
rect.y           # 50     左上角 y
rect.width       # 200    宽度
rect.height      # 150    高度
rect.left        # 100    左边界（= x）
rect.right       # 300    右边界（= x + width）
rect.top         # 50     上边界（= y）
rect.bottom      # 200    下边界（= y + height）
rect.center      # (200, 125)  中心点坐标
rect.centerx     # 200    中心点 x
rect.centery     # 125    中心点 y
```

### colliderect() —— 判断两个 Rect 是否重叠

这是碰撞检测最核心的方法：

```python
if player_rect.colliderect(enemy_rect):
    print("撞到了！")
```

`colliderect()` 返回 `True` 或 `False`。它检查两个矩形是否有重叠区域——哪怕只重叠了 1 像素，也会返回 `True`。

### 碰撞后要做什么

碰撞检测只是"发现问题"，发现问题之后要"解决问题"。常见的处理方式：

**1. 让物体消失（"吃掉"效果）**
比如主角碰到金币，金币消失，得分增加：

```python
for coin in coins:
    if coin["alive"] and player_rect.colliderect(coin["rect"]):
        coin["alive"] = False
        score += 10
```

**2. 反弹（墙壁、球拍效果）**
比如球碰到墙壁或球拍时反弹：

```python
if ball_rect.colliderect(wall_rect):
    ball_speed_x = -ball_speed_x  # 水平方向反弹
```

**3. 阻止移动（"不可穿越"效果）**
比如角色不能穿过墙壁：

```python
# 向下移动之前先检查
proposed_rect = player_rect.copy()
proposed_rect.y += speed
if not proposed_rect.colliderect(wall_rect):
    player_rect.y += speed  # 只有不碰撞才允许移动
```

### 碰撞检测的几种常用方法一览

| 方法 | 作用 |
|------|------|
| `rect.colliderect(other_rect)` | 两个矩形是否碰撞 |
| `rect.collidepoint(x, y)` | 点 (x, y) 是否在矩形内（常用于鼠标点击检测） |
| `rect.collidelist(list_of_rects)` | 返回列表中第一个碰撞的索引，不碰撞返回 -1 |
| `rect.contains(other_rect)` | other_rect 是否完全在 rect 内部 |
| `rect.clip(other_rect)` | 返回两个矩形重叠部分的 Rect |

碰撞检测 = 判断两个 Rect 是否重叠。`colliderect()` 是最常用的方法。碰撞检测 + 响应逻辑 = 游戏互动的核心。
