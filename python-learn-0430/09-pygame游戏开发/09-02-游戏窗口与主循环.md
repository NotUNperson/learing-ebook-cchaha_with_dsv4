# 09-02 游戏窗口与主循环

## 本节你会学到什么
- 理解游戏窗口和主循环的概念
- 学会创建窗口并设置标题
- 掌握主循环的四个步骤：事件处理、更新、绘制、控制帧率
- 学会正常退出游戏

## 正文
### 窗口就是舞台，主循环就是持续的演出

想象你在看一场话剧。演员们在一个舞台上表演，舞台的大小是固定的，背景幕布可以更换。演出从开始到结束，演员在台上不断地说话、走动、互动——这是一个持续的过程，不会演一分钟就停下来。

pygame 里的窗口，就相当于话剧的**舞台**。`set_mode()` 定义了这个舞台有多大（800x600 像素？还是 400x300？）。

而主循环（也叫游戏循环），就是那场**持续的演出**。只要演出没有结束，就一直在循环：处理观众的掌声和嘘声（事件处理），更新演员的位置和状态（更新逻辑），在舞台上表演给观众看（绘制画面）。

如果只有窗口没有主循环，那窗口会一闪而过——程序创建了窗口就结束了，就像搭好舞台却不上演节目。

### 主循环的四个步骤 —— 像心跳一样规律

每个游戏的主循环都在重复四个步骤，每一秒重复几十次：

**第一步：事件处理**

观众（玩家）会做出各种动作——按键盘、点鼠标、关窗口。这些动作在 pygame 里叫作"事件"（Event）。每帧都要检查有没有新的"观众动作"要处理。

```python
for event in pygame.event.get():
    if event.type == pygame.QUIT:   # 观众要离场了
        running = False
```

`pygame.event.get()` 返回从上一次循环到这次循环之间发生的所有事件。就像一个收件箱，你每一帧去查看里面的新信件。

**第二步：更新游戏状态**

根据事件来改变游戏的状态——角色移动、分数增加、敌人刷新等等。本节还没有实际的游戏逻辑，所以这一步暂时是空的。

**第三步：绘制画面**

把更新后的游戏世界"画"到窗口上：背景、角色、文字等等。画完之后调用 `pygame.display.flip()` 把画好的内容显示出来——就像话剧里的"换幕"。

```python
screen.fill((30, 30, 50))   # 用深蓝色填充背景
pygame.display.flip()       # 把画好的画面显示出来
```

**第四步：控制帧率**

如果机器很快，主循环可能一秒钟跑几千次，画面会闪得看不清。我们需要控制节奏，让它稳定在每秒 60 帧（FPS = Frames Per Second）。

```python
clock.tick(60)  # 确保每秒不超过 60 帧
```

60 FPS 意味着每帧大约 16 毫秒，这是游戏画面的"流畅标准"——低于 30 会觉得卡，60 是丝般顺滑。

### 正常的退出 —— 收拾完再走

你会注意到，我们没有直接写 `break` 退出循环，而是用 `running = False`。这样做的好处是：主循环会完整地跑完最后一圈，然后执行循环后面的清理代码：

```python
pygame.quit()
sys.exit()
```

`pygame.quit()` 负责释放 pygame 占用的资源，相当于谢幕后工作人员收拾舞台。如果你直接关窗口而不调用 quit，有些资源可能不会被释放（虽然在现代系统上影响不大，但这是好习惯）。

## 本节小结
游戏窗口 = 舞台，主循环 = 持续演出，每帧做四件事（事件、更新、绘制、控帧），退出前要清理资源。
