## 正文

### 刻在石头上的字 vs 写在白板上的字

想象你有两种写字的方式：

- **写在白板上**：写错了可以擦掉重写，想改就改。这就像**列表**——可以随时修改。
- **刻在石头上**：刻上去就改不了了，只能看，不能动。这就像**元组**——一旦创建就不能修改。

元组（tuple）和列表长得像"双胞胎"，但有一个关键区别：**元组是不可变的**。

### 创建元组

元组用**圆括号** `()` 创建（列表用方括号 `[]`）：

```python
# 列表用方括号
fruits_list = ["苹果", "香蕉", "橘子"]

# 元组用圆括号
fruits_tuple = ("苹果", "香蕉", "橘子")
```

### 元组能做的事

**访问（和列表一样，用索引）：**

```python
colors = ("红", "黄", "蓝")
print(colors[0])   # 红
print(colors[1])   # 黄
print(colors[-1])  # 蓝
```

**获取长度：**

```python
print(len(colors))   # 3
```

### 元组不能做的事

**不能修改！不能修改！不能修改！**

```python
colors = ("红", "黄", "蓝")
# colors[0] = "绿"     # 报错！元组不支持修改
# colors.append("绿")  # 报错！元组没有 append
# colors.remove("红")  # 报错！元组没有 remove
```

元组就像一份**已经盖章的合同**——你可以看内容，但改不了任何一个字。

### 为什么要有元组？

你可能会想：既然列表能做元组的所有事，还能修改，为什么要用元组？

几个常见理由：

1. **你不希望数据被改**：比如一年有 12 个月、一周有 7 天——这些是固定的，不该被任何人改。用元组可以"锁住"数据。

```python
MONTHS = ("一月", "二月", "三月", ..., "十二月")   # 月份永远不变
```

2. **性能更好**：因为元组不可变，Python 处理元组比列表更快。数据量大的时候差别很明显。

3. **可以做字典的键**（后面会学）：列表不能做字典的键，元组可以。

### 单元素元组的小陷阱

如果元组里只有一个元素，必须加一个逗号：

```python
not_a_tuple = ("hello")     # 这不是元组！这只是括号里的字符串
real_tuple = ("hello",)     # 这才是元组（注意末尾的逗号）

print(type(not_a_tuple))    # <class 'str'> — 竟然是字符串
print(type(real_tuple))     # <class 'tuple'> — 这才是元组
```

为什么？因为 Python 分不清 `("hello")` 是"我强调一下 hello"还是"这是一个元组"。加个逗号就是明确告诉 Python："这是元组，只有一个元素。"

### 列表 vs 元组：一张对比表

| 特性 | 列表 (list) | 元组 (tuple) |
|------|-----------|------------|
| 创建符号 | `[]` 方括号 | `()` 圆括号 |
| 能否修改 | 能 | 不能 |
| 常用方法 | append、remove、sort 等 | 只有 count、index |
| 使用场景 | 数据经常变动 | 数据固定不变 |
| 类比 | 白板（可擦写） | 石刻（不可改） |

## 本节小结

元组和列表很像，但元组创建后不能修改——就像刻在石头上的字。当你需要保护数据不被改动时，用元组；当数据需要灵活变动时，用列表。
