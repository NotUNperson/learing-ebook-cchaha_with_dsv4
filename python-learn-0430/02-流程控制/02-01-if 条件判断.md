# 02-01 if 条件判断

## 本节你会学到什么

- 理解缩进的重要性

## 正文

### 出门前的"如果"

每天早上出门前，你都会在心里做一个判断：

> **如果**下雨了，我就带伞。
> **如果**不下雨，我就不带。

这就是编程里的 `if` —— "如果……就……"。

在 Python 里写出来是这样的：

```python
if weather == "下雨":
    print("带伞出门")
```

拆开来看：
- `if` —— "如果"
- `weather == "下雨"` —— "外面下雨了吗？"（还记得 `==` 是比较吗？）
- `:` —— "那么，做下面的事"
- `print("带伞出门")` —— "带伞出门"

### 缩进：代码的"缩进"就是"从属关系"

注意第二行前面有**4 个空格**（或按一次 Tab 键）。这个缩进非常重要——它告诉 Python "这行代码属于上面的 if"。

你可以理解为：在待办清单上，主任务写在左边，子任务往右缩进一格：

```
出门准备：
    - 如果下雨：
        带伞          <-- 缩进表示这是"如果下雨"下的子任务
    - 拿钥匙
    - 穿鞋
```

只有缩进在 `if` 下面的代码，才会在条件满足时执行。不缩进的代码，不管条件是否满足都会执行：

```python
weather = "下雨"

if weather == "下雨":
    print("带伞")           # 缩进了——下雨才执行
    print("穿雨鞋")         # 缩进了——下雨才执行
print("出门")               # 没缩进——不管下不下雨都执行
```

输出：

```
带伞
穿雨鞋
出门
```

如果把 `weather` 改成 `"晴天"`，输出就只有：

```
出门
```

前面两行不执行——因为条件不满足。

### if 的结构解剖

```python
if 条件:
    条件满足时执行这里的代码
    这里也执行
这里一定会执行（不在 if 里面）
```

关键点：
- `if` 后面一定要有**冒号** `:` —— 就像在说"注意了，我要说条件了"
- 条件下面必须**缩进** —— 一般是 4 个空格或 1 个 Tab
- 同一个 `if` 下面可以有**多行**缩进的代码
- **不缩进**就表示"if 结束，这是正常代码了"

### 条件可以是各种比较

```python
score = 85

if score >= 60:
    print("及格了！")

if score < 60:
    print("没及格")

if score == 100:
    print("满分！太厉害了！")
```

任何返回 True 或 False 的表达式都能放在 `if` 后面。还记得上一章学的比较运算和逻辑运算吗？它们就是 `if` 的最好搭档。

### 条件也可以直接是布尔值

```python
is_raining = True

if is_raining:
    print("带伞")    # is_raining 是 True，所以会执行

is_raining = False

if is_raining:
    print("带伞")    # 这句不会执行，因为 is_raining 是 False
```

### 常见的缩进错误

```python
# 错误示范 1：忘了缩进
# if 5 > 3:
# print("5大于3")     # 报错！必须缩进

# 错误示范 2：忘了冒号
# if 5 > 3            # 报错！必须加冒号
#     print("5大于3")

# 错误示范 3：缩进不一致
# if 5 > 3:
#     print("第一行")
#       print("第二行")   # 报错！缩进要对齐
```

## 动手试试

1. 写一个程序：定义一个变量 `temperature = 30`，如果温度大于 28，就打印"好热，开空调"
2. 定义一个变量 `hour = 7`，如果 hour 小于 12，打印"早上好"
3. 试一试：把一个打印语句缩进在 if 下面，另一个不缩进。分别把条件设为 True 和 False，观察执行结果

## 本节小结

`if` 让程序有了"选择能力"——条件满足才执行，就像你出门前看天气决定带不带伞。缩进是关键，它告诉 Python 哪些代码属于 `if`。
