## 正文
### 为什么需要函数 -- 菜谱 vs 做菜

想一想：你妈妈教你做番茄炒蛋，她会先把步骤写下来：

1. 打鸡蛋，搅拌
2. 切番茄
3. 热锅倒油
4. 先炒蛋，再放番茄
5. 加盐出锅

这写下来的步骤就是**菜谱** -- 在编程里，这叫**定义函数**（defining a function）。

每次你按照这份菜谱真正下厨做菜，那就是**调用函数**（calling a function）。菜谱写一次，可以做无数次菜。函数也是：定义一次，可以调用无数次。

**不写函数的代码长这样**（同样的逻辑复制粘贴了三遍）：

```python
# 计算 3 + 5 的和并打印
result = 3 + 5
print(f"3 + 5 = {result}")

# 计算 10 + 20 的和并打印
result = 10 + 20
print(f"10 + 20 = {result}")

# 计算 100 + 200 的和并打印
result = 100 + 200
print(f"100 + 200 = {result}")
```

**写成函数长这样** -- 清爽多了：

```python
def add_and_print(a, b):
    result = a + b
    print(f"{a} + {b} = {result}")

add_and_print(3, 5)      # 3 + 5 = 8
add_and_print(10, 20)    # 10 + 20 = 30
add_and_print(100, 200)  # 100 + 200 = 300
```

### def -- 定义你的第一个函数

```python
def greet():
    print("你好！")
    print("欢迎学习 Python！")

# 调用函数
greet()
greet()  # 可以调用很多次
```

`def` 是 define（定义）的缩写。`greet` 是你给函数起的名字。括号 `()` 暂时是空的 -- 表示这个函数不需要任何"输入"。冒号后面的缩进部分是**函数体**，也就是函数具体要做的事情。

函数定义好之后不会自动执行 -- 你必须"叫"它（调用它），它才会干活。

### 函数让代码更有条理

```python
# 没有函数 -- 像一锅粥
print("=== 欢迎 ===")
print("正在验证用户...")
print("验证通过！")
print("正在加载数据...")
print("加载完成！")

# 有函数 -- 像分类好的工具箱
def show_welcome():
    print("=== 欢迎 ===")

def verify_user():
    print("正在验证用户...")
    print("验证通过！")

def load_data():
    print("正在加载数据...")
    print("加载完成！")

# 现在逻辑一目了然
show_welcome()
verify_user()
load_data()
```

## 本节小结
函数就像菜谱：定义一次，可反复调用，让代码不再重复、更有条理。
