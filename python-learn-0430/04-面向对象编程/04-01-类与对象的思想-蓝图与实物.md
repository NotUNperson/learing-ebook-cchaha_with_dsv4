# 04-01 类与对象的思想：蓝图与实物

- 初步建立面向对象的思维方式

## 正文
### 类与对象 -- 饼干模具和饼干

假设你要做饼干。你先得有一个**模具** -- 模具决定了饼干的形状（是星星形状还是小熊形状）。有了模具之后，你用面粉一压，就做出了一块具体的饼干。

- **模具** = **类（class）**：一个设计图、一个模板，定义了"这个东西有什么特征、能做什么"
- **饼干** = **对象（object）**：按照模具做出来的具体实物，每一块都是独立的

一个模具可以做出无数块饼干。一个类也可以创建出无数个对象。每块饼干可能有不同颜色（巧克力味、草莓味），但形状都来自同一个模具。

### 面向过程 vs 面向对象 -- 自己做菜 vs 去餐厅点菜

你以前写的代码，大体上是**面向过程**的：第一步干什么、第二步干什么、第三步干什么。这就像你自己在家做菜 -- 买菜、洗菜、切菜、炒菜，每一步都要你自己操心。

**面向对象**的思维则不同。它就像去餐厅吃饭：
- 你不用关心后厨怎么炒菜（你不用管每个细节）
- 你只需要告诉服务员"我要一份宫保鸡丁"（你调用对象的方法）
- 服务员（对象）自己知道怎么做，把菜端给你

在代码里：

```python
# 面向过程：一步步操作数据
name = "张三"
score = 90
print(f"{name}的成绩是{score}")  # 数据和处理数据的代码是分开的

# 面向对象：把数据和操作数据的代码打包在一起
class Student:
    def __init__(self, name, score):
        self.name = name
        self.score = score

    def show_score(self):
        print(f"{self.name}的成绩是{self.score}")

# 创建一个学生对象，让它自己汇报成绩
s = Student("张三", 90)
s.show_score()  # 你只要告诉它"汇报成绩"，它自己知道怎么做
```

面向对象的核心思想是：把**数据**和**操作数据的方法**打包在一起，变成一个"活的东西"（对象）。这更符合我们理解现实世界的方式 -- 学生自己知道自己的姓名和成绩，也能自己汇报。

### 为什么要学面向对象

随着程序越来越大，面向过程的代码会变得难以管理 -- 到处都是函数和数据，它们之间的关系不清晰。

面向对象让你可以：
- 把相关的数据和行为捆在一起（学生有姓名和成绩，也能汇报自己的成绩）
- 代码更容易理解和修改（改学生的行为，就去 Student 类里改）
- 方便复用和扩展（比如从"学生"派生出"大学生"，后面会讲）

## 动手试试
不用写代码，用生活中的例子思考：
1. 找一个你身边的东西（比如手机、台灯、宠物），说说它的"属性"（特征）和"方法"（能做什么）
2. 比如"手机"：属性 = 品牌、电量、屏幕尺寸；方法 = 打电话、拍照、充电
3. 写下你的分析，然后看看旁边的东西，是不是也能这样描述？

## 本节小结
类是模具（蓝图），对象是饼干（实物）；面向对象就是把数据和操作打包在一起，更像真实世界。
