# 2.14 显示模式

## 本节你会学到什么

- `display` 属性的四种核心值：`block`、`inline`、`inline-block`、`none`
- 块级元素和行内元素的本质区别
- `inline-block` 如何兼具两者的优点
- `display: none` 和 `visibility: hidden` 的关键差异
- 如何用 `display` 改变元素的默认行为

## 正文

### 元素的"性格"——display 属性

在前面的学习中，你已经接触过 `div` 和 `span`。你可能注意到了：`div` 总是一个人占一整行，而 `span` 乖乖地和周围的文字排在一起。这不是巧合，这是它们的"天性"决定的。

在 CSS 里，每个元素都有一个 `display` 属性，它决定了元素在页面上**怎么"站"**。你可以把页面想象成一个广场：

- 有些人喜欢**独占一行**，站在那谁也不让过——这就是 `block`
- 有些人喜欢**挤在一排**，像排队一样挨着站——这就是 `inline`
- 还有人既能**排在一起**，又能**控制自己的体型**——这就是 `inline-block`
- 最神奇的一种：**凭空消失**，连位置都不留——这就是 `none`

### 一、block —— 独占一行的"大胖子"

`block`（块级）元素就像一堵墙。你把一堵墙放在广场上，它从左到右把路全占了，后面的人只能绕到下一行去。

**block 元素的特点：**
- 独占一行（即使它的实际宽度只有 100px，右边空着也不让别人上）
- 宽度默认是父容器的 100%
- **可以设置** `width` 和 `height`（宽高可控）
- **可以设置** `margin` 和 `padding` 四个方向
- 像一块积木，一块一块从上往下摞

**常见的 block 元素**：`div`、`p`、`h1~h6`、`ul`、`ol`、`li`、`header`、`footer`、`section`

### 二、inline —— 随波逐流的"小字"

`inline`（行内）元素就像一句话里的一个字。文字从左往右流，这个字就跟着大部队一起走，前面是字，后面还是字，大家排在一行里。

**inline 元素的特点：**
- 不换行，和相邻的行内元素排在同一行
- 宽度由内容自动决定（有多少字就多宽）
- **不能设置** `width` 和 `height`（设了也没用，就像你不能规定一个字必须占 5 厘米宽）
- `margin` 和 `padding` **只有左右方向生效**，上下方向不影响布局
- 像流水里的叶子，随波逐流

**常见的 inline 元素**：`span`、`a`、`strong`、`em`、`img`（img 的特殊之处后面讲）

### 三、inline-block —— 排队的"固定大小卡片"

`inline-block` 是 block 和 inline 的"混血儿"，结合了两者的优点：

- **inline 的一面**：可以和别的元素排在同一行，不换行
- **block 的一面**：可以设置 `width`、`height`、四个方向的 `margin` 和 `padding`

这个比喻最贴切：**排队买票的人**。大家排成一排（inline 的特性），但每个人都有自己的体型——有人高有人矮有人胖有人瘦（block 的特性）。每个人都拿着一个固定大小的号码牌，不会因为个子小就挤没了。

`inline-block` 在制作导航栏、卡片列表、按钮组时非常常用。

### 四、none —— 隐身术

`display: none` 让元素**彻底从页面上消失**——不仅是看不见，而且**不占任何空间**。就像这个元素从来没在 HTML 里出现过一样。

这个特性非常有用：
- 制作交互菜单（点击按钮才出现子菜单）
- 响应式布局中隐藏某些元素（手机上不显示侧边栏）
- 标签页切换（只显示当前激活的标签内容）

**`display: none` vs `visibility: hidden`**：

| 属性 | 看得见吗 | 占空间吗 | 类比 |
|------|----------|----------|------|
| `display: none` | 看不见 | 不占 | 人离开队列，后面的人补上 |
| `visibility: hidden` | 看不见 | 占着 | 人还在队列里，只是隐身了，位置空着 |

## 动手试试

1. 打开示例文件，观察四个区块的表现。重点关注：block 怎么占满行、inline 怎么拒绝设置宽高、inline-block 怎么排成一行、none 怎么完全消失。
2. 把示例中某个 `div` 的 `display` 改成 `inline`，观察一个"块级元素"变成"行内元素"后会发生什么。
3. 把三个 `.inline-block-box` 的 `height` 改成不同的值（比如一个 60px、一个 100px、一个 140px），看看它们在同一行里怎么对齐的。
4. 把 `display: none` 改成 `visibility: hidden`，刷新页面，观察那个位置的空白还在不在。

## 本节小结

`display` 决定了元素的"居住方式"——block 独占一行像墙，inline 随波逐流像文字，inline-block 排成一行但体型可控，none 像隐身术一样彻底消失。
