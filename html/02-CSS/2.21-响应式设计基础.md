# 2.21 响应式设计基础

## 本节你会学到什么

- 什么是响应式设计
- 视口（viewport）的概念和 `<meta name="viewport">` 的重要性
- `rem` 和 `em` 单位的区别与推荐用法
- 百分比宽度和 `min-width`/`max-width` 的运用
- 流体布局（用相对单位代替固定 px）

## 正文

### 一件衣服，三个尺码

想象你开了一家服装店，但店里只卖均码衣服。高个子穿着短一截，矮个子穿着拖地——这就是一个**不响应式**的网页：在 27 寸大屏幕上看着刚好，换到手机上就得用两只手指放大缩小，字像蚂蚁一样小。

**响应式设计（Responsive Design）**就是给同一个网页准备不同的"尺码"——在宽屏幕上是 M 号，在平板上是 S 号，在手机上是 XS 号。同一个网站，不同设备上都好看。

### 一、视口（viewport）—— 手机端的"致命疏忽"

很多新手写完网页，在电脑上看着漂漂亮亮，用手机打开一看——字像蚂蚁，布局全毁。九成的原因是没有设置 viewport。

在 HTML 的 `<head>` 里，你需要这一行：

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

这行代码是做什么的？

手机浏览器为了能在小屏幕上显示为电脑设计的网页，会默认把页面缩小——就像一个 375px 宽的手机屏幕假装自己是 980px 宽。你的文字当然就被缩成蚂蚁了。

`width=device-width` 告诉手机："用你真实的屏幕宽度来显示，别假装自己是电脑屏幕。"`initial-scale=1.0` 是初始缩放比例为 1（不缩放）。

**一句话**：没有这行 meta 标签，移动端响应式免谈。从现在起，你写的每一个网页的 `<head>` 里都要有它。

### 二、固定 px 的"死板"

看这个例子：

```css
.container {
  width: 1200px;   /* 固定宽度 */
}
```

在 1920px 的屏幕上，1200px 是合适的。但在 375px 的手机上——1200px 远超过屏幕宽度，用户只能左右滑动。这就是固定 px 的致命缺陷。

### 三、rem 和 em —— 什么是"相对单位"

固定 px 不够灵活，我们需要**相对单位**。

**em**：相对于当前元素的**父元素**字号。如果父元素 `font-size: 16px`，那么 `2em = 32px`。

**rem（root em）**：相对于**根元素 `<html>`** 的字号。整个页面统一参照根字号。

```css
html {
  font-size: 16px;    /* 设置根字号，1rem = 16px */
}
h1 {
  font-size: 2rem;     /* 32px = 2 * 16px */
}
p {
  font-size: 1rem;     /* 16px */
  margin-bottom: 1.5rem;  /* 24px */
}
```

**为什么推荐 rem？**

em 会层层叠加——如果父元素字号是 2em，子元素再写 1.5em，那就是 2 * 1.5 * 根字号 = 3 倍根字号。你很难心算出一个元素到底多大。

rem 只参照根元素，全局一致，不会出现"祖传染色体"的连锁反应。写 `1.5rem`，它就是 1.5 倍根字号，不看爹妈。

**实用技巧**：通过修改根字号来整体缩放页面

```css
html {
  font-size: 16px;     /* 桌面端 */
}
@media (max-width: 600px) {
  html {
    font-size: 14px;   /* 手机上整体缩小一点 */
  }
}
```

改这一行，所有用 rem 的元素都会等比例缩放——这就是响应式的优雅之处。

### 四、百分比宽度 —— 随容器而变化

```css
.container {
  width: 90%;          /* 占父元素宽度的 90% */
  max-width: 800px;    /* 但最多不超过 800px */
  margin: 0 auto;      /* 居中 */
}
```

这个写法非常经典：宽度随屏幕变化（90%），但不会无限放大（max-width 限定了上限）。窄屏上它占 90%，宽屏上它最多 800px，居中显示。

### 五、min-width 和 max-width —— 给元素画"底线"

- `max-width`：我不超过这个值。适合用来防止内容在超宽屏幕上拉到不可读的程度。
- `min-width`：我不小于这个值。适合用来防止在窄屏上被挤扁。

```css
img {
  max-width: 100%;     /* 图片不会超过父容器宽度，再宽就缩小 */
  height: auto;        /* 保持比例 */
}
```

这个组合让图片永远不会撑破布局——在手机上自动缩小，在桌面上不超过原有尺寸。

### 六、流体布局 —— 用相对单位代替固定 px

**旧做法（固定布局）**：全用 px，在不同屏幕上"写死"。

```css
.sidebar { width: 300px; float: left; }
.content { width: 700px; float: left; }
```

**新做法（流体布局）**：用百分比和 rem，弹性适应。

```css
.page {
  display: flex;
  max-width: 1200px;    /* 桌面端上限 */
  margin: 0 auto;       /* 居中 */
  padding: 0 1rem;
}
.sidebar {
  flex: 0 0 200px;       /* 侧边栏固定 200px */
}
.content {
  flex: 1;               /* 内容区占满剩余 */
}
```

流体布局不一定全部用百分比——"弹性项 + max-width"是个很好的组合，让布局在某个范围内"自适应"。

### 七、视口相关单位

CSS 还提供了相对于视口尺寸的单位：

| 单位 | 含义 |
|------|------|
| `vw` | 视口宽度的 1% |
| `vh` | 视口高度的 1% |
| `vmin` | vw 和 vh 中较小的那个 |
| `vmax` | vw 和 vh 中较大的那个 |

```css
.hero {
  height: 80vh;           /* 占屏幕高度的 80% */
  font-size: calc(1rem + 2vw);  /* 根字号 + 视口宽度的 2%，随屏幕变大而变大 */
}
```

## 本节小结

响应式设计就是同一个网页在不同屏幕上自动调整——viewport meta 标签是基础，rem 做统一缩放，百分比和 max-width 让布局有弹性，相对单位替代固定 px 是核心思路。
