响应式设计基础 —— 同一网页，不同屏幕都好看

一、rem 单位 —— 统一参照根字号

这个标题用 1.5rem

这段正文用 1rem。所有 rem 都参照 html 元素的字号（浏览器默认 16px）。

这段小字用 0.875rem（约 14px），用来做注释文字。

这段大字用 1.25rem（约 20px），用来强调。

试着修改浏览器的最小字号设置，或者未来用媒体查询改 html 的 font-size——所有 rem 会自动跟随缩放！

二、em vs rem —— 为什么推荐 rem

父元素：font-size: 20px（1em = 20px）

我是子段落，font-size: 0.8em = 16px

我是孙 div，font-size: 1.5em = 30px（1.5 * 父20px）
我是曾孙 div，font-size: 1.5em = 45px（1.5 * 父30px）
看到没？em 层层叠加，越套越离谱！

这就是 em 的"祖传染色体"效应。rem 只看根元素，不存在这个问题，所以推荐用 rem。

三、百分比宽度 + max-width —— 弹性但有上限

width: 90%; max-width: 700px; min-width: 280px;

试着拖动浏览器窗口从很窄到很宽：

  • 窄屏（比如 320px 手机宽度）：我占 90%，约 288px，大于 min-width 280px
  • 中屏（比如 768px 平板）：我占 90%，约 691px
  • 宽屏（比如 1440px 桌面）：90% = 1296px，但 max-width 限制我为 700px

四、响应式图片 —— 永不撑破布局

max-width: 100%; height: auto;
（这是一个模拟图片的色块）
不管容器多宽多窄，我都不会撑破布局

五、固定布局 vs 流体布局

固定宽度（老做法）—— 窄屏上会爆

左列：380px 固定

在小于 800px 的屏幕上，这个布局会溢出。

右列：380px 固定

手机上根本看不全。

流体布局（Flexbox）—— 弹性适应

左列：flex: 1（自动伸缩）

不管屏幕多宽多窄，我和右列始终平分空间。

右列：flex: 1（自动伸缩）

拖一拖浏览器窗口试试！

六、vw/vh 视口单位

这个盒子的宽度是 80vw（视口宽度的 80%），max-width: 700px。

这个色条高度 = 15vh（视口高度的 15%）

总结

技术 作用 写法
viewport meta 手机端正确缩放 <meta name="viewport" content="width=device-width, initial-scale=1.0">
rem 统一参照根字号 font-size: 1.5rem;
百分比宽度 随父容器弹性 width: 90%;
max-width 宽度上限 max-width: 800px;
min-width 宽度下限 min-width: 280px;
max-width: 100% 图片不溢出 img { max-width: 100%; height: auto; }
