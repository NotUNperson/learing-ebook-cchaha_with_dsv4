盒模型（下）—— box-sizing 与 margin 合并

一、content-box（标准） vs border-box（替代）

两个盒子都有 width: 300px; padding: 25px; border: 5px，但 box-sizing 不同：

content-box（默认）—— width 只管内容区

content-box
设 width=300px, padding=25, border=5
实际宽度 = 300 + 50 + 10 = 360px
300px content 360px 实际

border-box（推荐）—— width 包含到边框

border-box
设 width=300px, padding=25, border=5
实际宽度 = 300px（content 自动缩到 240px）
300px 就是你看到的
提示：按 F12 打开开发者工具，分别点选上面两个盒子，在 Styles 面板底部的盒模型图中看实际尺寸——直观感受两者的区别。

二、为什么要全局设置 border-box

本页面的 CSS 开头写了：

*, *::before, *::after {
  box-sizing: border-box;
}

这意味着你设置的 width 就是最终看到的宽度，不用心算。下面两列布局因为有了 border-box，padding 和 border 不会让列撑破容器：

第一列：padding 和 border 包含在宽度内，不会溢出。
第二列：同样舒服地待在容器里。
第三列：所有列总宽正好等于容器。

三、margin 合并（垂直方向，取最大值）

下面两个盒子：第一个 margin-bottom: 30px，第二个 margin-top: 20px。

直觉上间距 = 30 + 20 = 50px。实际是 max(30, 20) = 30px！

盒子 A：margin-bottom: 30px
实际间距只有 30px（取最大值），不是 50px！
盒子 B：margin-top: 20px

四、父子 margin 合并（子元素的 margin "穿透"父元素）

发生合并的情况：

我的 margin-top: 40px
但父 div 没有 border/padding/overflow 来阻断
结果我的 margin-top 把父元素也带下去了！

注意上面这个区块：父 div 的顶部没有留白，子元素的 margin-top 直接"渗透"到父元素外面了。

阻止合并：给父元素加 border-top

我的 margin-top: 40px
父 div 有 border-top: 1px solid transparent
合并被阻断，margin 正常表现在父元素内部！

注意区别：父元素顶部留白了（有边界阻断），子元素 margin-top 正常表现为内部间距。

margin 合并关键点总结

  • 只发生在垂直方向（上下），水平方向不合并
  • 间距取最大值，不是相加
  • 可以通过 border、padding、overflow: hidden 阻断
  • Flexbox 和 Grid 布局中不存在 margin 合并
