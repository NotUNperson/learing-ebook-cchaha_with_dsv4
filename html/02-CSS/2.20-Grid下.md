# 2.20 Grid（下）—— 项目放置与布局选用

## 本节你会学到什么

- `grid-column` / `grid-row` 把项目放到指定格子
- `span` 关键字跨列/跨行
- `grid-template-areas` —— 用"画地图"的方式做布局
- Flexbox vs Grid 的选用原则
- 两者嵌套使用的技巧

## 正文

### 从"自动分配"到"精确落子"

上一节我们学了怎么画棋盘。但如果只是画好棋盘，让项目自动按顺序往里面填，那 Grid 只发挥了三成功力。

Grid 真正强大的是：你可以**精确指定每个项目的位置**——就像下象棋一样，把"炮"放在第 2 行第 3 列。

### 一、认识网格线

在 Grid 里，行列之间有一条条看不见的"线"。这些线从 1 开始编号：

```
       线1   线2   线3   线4
       |     |     |     |
线1 —— +-----+-----+-----+
       |     |     |     |
线2 —— +-----+-----+-----+
       |     |     |     |
线3 —— +-----+-----+-----+
       |     |     |     |
```

一个 3 列 x 2 行的网格就有 4 条列线和 3 条行线。

### 二、grid-column / grid-row —— 从第几条线到第几条线

用这两个属性指定项目跨越的线的范围：

```css
.item {
  grid-column: 1 / 3;   /* 从列线 1 到列线 3，跨两列 */
  grid-row: 1 / 2;      /* 从行线 1 到行线 2，第一行 */
}
```

这里的 `1 / 3` 意思是"从第 1 根线开始，到第 3 根线结束"——跨了两列。

也可以用 `span` 关键字，不说"到第几条线"，而是说"跨几个格子"：

```css
.item {
  grid-column: span 2;     /* 跨两列（从当前位置开始） */
  grid-row: span 3;        /* 跨三行 */
}
```

更简洁的简写：

```css
.item {
  grid-column: 2 / span 2;   /* 从第 2 条列线开始，跨 2 列 */
}
```

### 三、grid-template-areas —— "画地图"式布局

这是 Grid 里最直观、最受新手欢迎的布局方式。直接在 CSS 里用文字"画"出布局：

```css
.container {
  display: grid;
  grid-template-areas:
    "header  header  header"
    "sidebar content content"
    "footer  footer  footer";
  grid-template-columns: 200px 1fr 1fr;
  grid-template-rows: 60px 1fr 60px;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.content { grid-area: content; }
.footer  { grid-area: footer; }
```

**类比**：就像在餐桌上摆碗筷。"header" 占桌子最上面一整排，"sidebar" 占左边一个小碟子，"content" 占右边的大盘子，"footer" 占最底下的一整排。CSS 里的文字地图一眼就能看出布局长什么样。

这种写法的好处：
- 一眼看懂布局结构
- 改布局时直接改"地图"就行，不用改 HTML
- 配合媒体查询做响应式非常方便（直接换一张"地图"）

### 四、Flexbox vs Grid —— 什么时候用哪个

| 场景 | 推荐 | 原因 |
|------|------|------|
| 导航栏（横向一排链接） | Flexbox | 一维排列，Flexbox 最自然 |
| 卡片列表（等高、换行） | Flexbox | flex-wrap + gap 轻松搞定 |
| 整页布局（侧边栏+主内容+页头页脚） | Grid | 二维结构，行列同时控制 |
| 仪表盘（不对齐的格子） | Grid | grid-column/row 精确放置 |
| 居中一个元素 | Flexbox | 两行代码搞定 |
| 同时控制行和列间的对齐 | Grid | `align-items` + `justify-items` |
| 不确定项目数量（如搜索结果） | Flexbox | 自动换行，弹性伸缩 |
| 确定的行列结构（如照片墙） | Grid | 统一的网格结构 |

**核心原则**：一维用 Flexbox，二维用 Grid。两者可以嵌套——Grid 容器里放 Flexbox 组件，或者 Flexbox 容器里放 Grid 组件，完全没问题。

### 五、嵌套使用

最常见的是：**整页用 Grid 搭骨架，局部组件用 Flexbox**。

```
┌──────────────────────────────────┐
│            Header                │  ← Grid: header 区域
├──────────┬───────────────────────┤
│          │  [卡片1] [卡片2] [卡片3]│ ← 侧边栏 Grid 区域
│ Sidebar  │  ← 内容区是 Flexbox   │   主内容 Grid 区域
│          │  [卡片4] [卡片5] [卡片6]│   里面用 Flexbox 排列卡片
├──────────┴───────────────────────┤
│            Footer                │  ← Grid: footer 区域
└──────────────────────────────────┘
```

外层 Grid 负责"大格局"，内层 Flexbox 负责"小排队"。各司其职，相得益彰。

## 动手试试

1. 打开 `examples/2.20-grid-2.html`，观察三个演示区块。
2. 找到 `.grid-lines .item1`，把 `grid-column` 从 `span 2` 改成 `span 1`，看看它怎么变窄的。
3. 找到 `.grid-areas`，把 `grid-template-areas` 里的地图修改一下——比如让 "banner" 只占中间两列，看看布局怎么变。
4. 在脑海里过一遍：你最近见过的网页（B站首页、GitHub、知乎），哪些位置是 Flexbox，哪些位置是 Grid？

## 本节小结

Grid 可以用 `grid-column`/`grid-row` 精确放置项目到棋盘上，也可以用 `grid-template-areas` 用"画地图"的方式布局——后者最直观。核心选择原则：一维 Flexbox，二维 Grid，两者可以嵌套。
