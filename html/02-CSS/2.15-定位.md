# 2.15 定位

## 本节你会学到什么

- 五种定位方式：`static`、`relative`、`absolute`、`fixed`、`sticky`
- `top`/`right`/`bottom`/`left` 偏移属性的用法
- "定位祖先"是什么，`absolute` 怎么找参考点
- `z-index` 控制层叠顺序
- 每种定位的典型使用场景

## 正文

### 给元素发"入住许可证"

上一节学了 `display`，它决定了元素在标准流程里怎么排队。但如果你不想排队呢？你想让某个元素飞到页面右上角，或者钉在屏幕底部无论如何滚动都不走？

这时候就需要**定位（position）**。CSS 提供了五种定位方式，你可以把它们想象成物业发给住户的五种不同的"入住许可证"：

| 定位类型 | 类比 |
|----------|------|
| `static` | 普通人，老老实实排队 |
| `relative` | 挪了一下位子，但原来的坑还给你留着 |
| `absolute` | 离开队伍，找一个"有头有脸的长辈"做参照，站到指定位置 |
| `fixed` | 贴在屏幕玻璃上，你怎么移动窗口它都不走 |
| `sticky` | 平时跟队伍走，滚到某个位置就粘住不动了 |

### 一、static —— 默认的"普通人"

所有元素默认都是 `position: static`。这就是我们熟悉的**正常文档流**——block 元素从上往下摞，inline 元素从左往右排。

不需要特意写 `position: static`，因为本来就是。而且 static 元素不接受 `top`/`right`/`bottom`/`left` 偏移，也不参与 `z-index` 层叠——就像一个普通排队的人，你不能说"你往左挪 20 米站"，他会回答你："我是正常排队的，不搞特殊。"

### 二、relative —— "挪一下，但坑留着"

`position: relative` 让元素相对于**自己原本的位置**偏移。

最重要的特点：**原位置保留**。就像在电影院里，你站起来挪到了旁边一个座位，但你原来的座位上放了一个包占着——别人不能坐你的位置。页面上的其他元素不会填补你原来的空间。

```css
.relative-box {
  position: relative;
  top: 20px;    /* 从原本位置向下偏移 20px */
  left: 30px;   /* 从原本位置向右偏移 30px */
}
```

**典型用法**：微调元素位置、作为 absolute 子元素的"定位参考点"。

### 三、absolute —— "离开队伍，找长辈做参照"

`position: absolute` 让元素**脱离文档流**——它不再占据原来的位置，后面的元素会顶上来。然后它寻找最近的**有定位的祖先元素**（即 position 不是 static 的祖先）作为参考坐标系，用 `top`/`right`/`bottom`/`left` 确定自己的位置。

如果往上找了一圈，没有任何祖先有定位（全是 static），那就以整个页面（body）为参考。

**类比**：你不想排队了，离开队伍，然后在广场上找一个"有身份的长辈"——比如一个有 relative 定位的父 div——站在它的某个角上。如果找不到长辈，就以整个广场为参照。

**典型用法**：弹出菜单、提示框、图标角标（比如购物车上的数字小红点）。

### 四、fixed —— "粘在屏幕上的贴纸"

`position: fixed` 让元素相对于**浏览器窗口**定位，滚动页面也不会移动。就像在电脑屏幕上贴了一张便利贴——不管窗口里的内容怎么滚，便利贴始终在那个位置。

它也脱离文档流，不占空间。

**典型用法**："回到顶部"按钮、固定导航栏、客服浮窗、弹窗遮罩。

### 五、sticky —— "滚到这里就粘住了"

`position: sticky` 是 relative 和 fixed 的混合体：

- 在滚动到**阈值**之前，它表现得像 relative（正常位置）
- 一旦滚动超过阈值，它就像 fixed 一样固定在屏幕上

**类比**：就像那种可撕的便利贴，平时夹在笔记本的某一页里跟着翻，但当这一页翻开到某个角度时，便利贴就粘在桌面上不动了。

**典型用法**：表格的表头固定、分类标题在滚动时吸顶。

### 六、偏移属性 top / right / bottom / left

这四个属性只有在元素**有定位（非 static）**时才生效。它们表示元素相对于参考点的偏移：

- `top`：参考点顶部 → 向下偏移
- `right`：参考点右侧 → 向左偏移
- `bottom`：参考点底部 → 向上偏移
- `left`：参考点左侧 → 向右偏移

*注意：top 正数向下，这和我们数学里的坐标系 Y 轴方向相反。*

### 七、z-index —— 谁在上面

当定位元素重叠时，`z-index` 决定了谁盖在谁上面。值越大越靠上（越靠近用户的眼睛）。

**类比**：就像楼层——`z-index: 1` 在一楼，`z-index: 999` 在 999 楼，当然是高楼盖在低楼上面。

**重要规则**：
- 只有定位元素（非 static）才吃 z-index
- 相同 z-index 时，后来者居上（写在后面的 HTML 元素在上面）

## 动手试试

1. 打开 `examples/2.15-position.html`，依次观察五种定位的表现。
2. 找到 `.absolute-box` 的 CSS，把 `top: 0` 和 `right: 0` 改成 `bottom: 0` 和 `left: 0`，看看它跑到哪去了。
3. 给 `.fixed-banner` 换一个不同的 `z-index` 值，刷新后看看它和其他元素的重叠关系有没有变化。
4. 滚动页面试试 sticky 标题的效果——向下滚动时它应该粘在顶部。

## 本节小结

五种定位就是五种"入住许可证"——static 老实排队，relative 挪位留坑，absolute 找长辈做参照脱离队伍，fixed 粘在屏幕上，sticky 滚到就粘。偏移属性和 z-index 只在有定位时生效。
