# 2.7 颜色

## 本节你会学到什么

- 颜色关键字：`red`, `blue` 等
- 十六进制颜色：`#ff0000`
- RGB/RGBA：`rgb(255, 0, 0)`，A 是透明度
- HSL/HSLA：色相、饱和度、亮度——更直观的调色方式
- `currentColor` —— 继承当前文字颜色
- `opacity` 属性 vs rgba 透明度的区别

## 正文

### 给网页"上色"

你已经学过了怎么选中元素、怎么用伪类和伪元素，但到现在为止我们只用了几个基础颜色字。这一节我们专门来讲 CSS 里所有"表示颜色"的方式。

打个比方：颜色表示法就像"告诉别人你想要什么颜色"的不同方式——

- 关键字法："我要红色" —— 口语最直接，但不够精确
- 十六进制法：设计师用的色号 "#E74C3C" —— 精确但不太直观
- RGB 法："红255、绿0、蓝0" —— 像调配灯光
- HSL 法："色相0度、饱和度100%、亮度50%" —— 像在色盘上转动指针

### 方式一：颜色关键字

最简单直接的写法：直接写颜色的英文名。

```css
color: red;
background-color: lightblue;
border-color: tomato;
```

CSS 支持 140+ 种颜色关键字，常用的如下：

| 关键字 | 颜色 | 描述 |
|---|---|---|
| `black` / `white` | 黑白 | 基础色 |
| `gray` / `lightgray` / `darkgray` | 灰系列 | 不同深浅 |
| `red` / `blue` / `green` | 红蓝绿 | 三原色 |
| `tomato` | 番茄色 | 偏橙的红 |
| `dodgerblue` | 道奇蓝 | 明亮的蓝 |
| `gold` | 金色 | 偏黄 |
| `transparent` | 透明 | 完全透明 |

关键字优点是直观好记，缺点是粒度不够细——"我想要比 tomato 浅一点但又不能是 pink 的颜色"就没法表达了。这时就需要更精确的方式。

### 方式二：十六进制 —— 设计师的色号

就像你去涂料店说的色号"#E74C3C"。CSS 里最常用的颜色表示法。

格式：以 `#` 开头，后面跟 6 位（或 3 位）十六进制数字。

```
#RRGGBB
 ││││││
 │││││└─ 蓝色分量（00 ~ FF，十进制 0~255）
 ││││└── 蓝色分量高位
 │││└─── 绿色分量低位
 ││└──── 绿色分量高位
 │└───── 红色分量低位
 └────── 红色分量高位
```

```css
color: #ff0000;  /* 纯红：红全满，绿空，蓝空 */
color: #00ff00;  /* 纯绿 */
color: #0000ff;  /* 纯蓝 */
color: #000000;  /* 纯黑（全关） */
color: #ffffff;  /* 纯白（全开） */
color: #cccccc;  /* 浅灰（三色相同就是灰色） */
```

如果六位中每两位都相同，可以缩写成三位：

```css
#ff0000 → #f00  (红)
#00ff00 → #0f0  (绿) —— 其实是 #00ff00，别和 #0f0 搞混
#0000ff → #00f  (蓝)
#cccccc → #ccc  (灰)
#ffffff → #fff  (白)
```

### 方式三：RGB / RGBA —— 像调配灯光的旋钮

把颜色想象成三盏灯的旋钮：红(Red)、绿(Green)、蓝(Blue)，每盏灯亮度从 0~255。

```
rgb(255, 0, 0)     → 纯红（红开满，绿蓝全关）
rgb(0, 255, 0)     → 纯绿
rgb(0, 0, 255)     → 纯蓝
rgb(255, 255, 255) → 纯白（三灯全开）
rgb(0, 0, 0)       → 纯黑（三灯全关）
rgb(128, 128, 128) → 灰色（三灯一样亮）
```

**RGBA** 多加了一个参数 Alpha（透明度）——就像一块有色玻璃的"透光度"：

```css
/* 半透明红色：a=0.5 表示 50% 不透明，即半透明 */
background-color: rgba(255, 0, 0, 0.5);
/*                           红 绿 蓝  透明度(0=全透明, 1=不透明) */
```

### 方式四：HSL / HSLA —— 更符合直觉的调色方式

RGB 有个问题：你知道 `rgb(52, 152, 219)` 是什么颜色吗？光看数字完全想象不出来。HSL 就是来解决这个问题的——它用人类的思维来描述颜色：

```
hsl(色相, 饱和度%, 亮度%)
```

- **色相（Hue）**：0~360 度，在色环上的位置。0 是红、120 是绿、240 是蓝
- **饱和度（Saturation）**：0%~100%，颜色的"浓度"。0% 是灰色（没颜色），100% 是纯色
- **亮度（Lightness）**：0%~100%。0% 是黑，100% 是白，50% 是纯色

```css
color: hsl(0, 100%, 50%);     /* 纯红 */
color: hsl(120, 100%, 50%);   /* 纯绿 */
color: hsl(240, 100%, 50%);   /* 纯蓝 */
color: hsl(0, 0%, 50%);       /* 灰色（饱和度为 0） */
color: hsl(0, 100%, 90%);     /* 很浅的粉红（亮度很高） */
```

HSL 的优势很明显：想调"亮一点的蓝色"？把 L 值提高就行。想调"浅一点的红色"？把饱和度降低。想调互补色？色相加 180。

同样有 HSLA：

```css
color: hsla(240, 100%, 50%, 0.5);  /* 半透明的蓝色 */
```

### currentColor —— 跟当前文字颜色走

`currentColor` 是一个特殊的关键字，它等于当前元素的 `color` 属性值。有什么用呢？

```css
.icon {
  color: #3498db;                 /* 文字颜色 */
  border: 2px solid currentColor; /* 边框颜色自动跟文字颜色一致！ */
}

.icon:hover {
  color: #e74c3c;                 /* 悬停时改文字颜色 */
  /* 边框自动跟着变成 #e74c3c！不用再写一遍 border-color */
}
```

这就像指挥家说："乐团的着装颜色，跟首席小提琴手的领带颜色一致。"——改一处，其他自动跟随。

### opacity vs rgba 透明度的关键区别

你可能发现，CSS 里有两个地方能控制"透明"：

```css
/* 方式 A：用 opacity 属性 */
.element-a {
  opacity: 0.5;  /* 整个元素（包括里面的文字、子元素）都变成半透明 */
}

/* 方式 B：用 rgba/hsla 颜色 */
.element-b {
  background-color: rgba(0, 0, 0, 0.5);  /* 只有背景半透明，文字正常 */
}
```

区别一目了然：

- `opacity: 0.5` —— 整个元素像一块磨砂玻璃，**里面的所有东西（文字子元素）都会变透明**
- `rgba(0,0,0,0.5)` —— 只有这个颜色属性本身半透明，**文字和其他子元素不受影响**

用类比来说：
- `opacity` 是把整张照片的透明度调低（照片里的所有内容都会变淡）
- `rgba` 是只在某个图层上用半透明颜料（其他图层不受影响）

## 本节小结

CSS 颜色四种表示法：关键字（简单）、十六进制（精确）、RGB（灯光思维）、HSL（直觉调色）。`currentColor` 实现统一配色，`rgba` 控制单属性透明，`opacity` 控制整体透明。
