# 2.22 媒体查询

## 本节你会学到什么

- `@media` 语法和媒体查询的工作原理
- 断点（breakpoint）的概念和常用断点
- 移动优先（mobile-first）vs 桌面优先（desktop-first）
- 常见的媒体查询场景：宽度、打印、暗色模式
- 用媒体查询实现"手机单列 -> 平板两列 -> 桌面三列"

## 正文

### CSS 里的"条件判断"

上一节我们学了用 rem、百分比、max-width 让布局有一定的弹性。但有时候"弹性"不够——你想在手机上**换一种完全不同的布局**：手机上导航栏变成汉堡菜单，电脑上是完整横排；手机上卡片堆叠，电脑上是三列。

这时候就需要**媒体查询（Media Queries）**。`@media` 就像 CSS 里的 `if` 语句——"如果屏幕宽度小于 600px，就执行下面的样式"。

### 一、@media 基本语法

```css
@media (条件) {
  /* 条件满足时，这里的样式才生效 */
}
```

最常见的就是检查屏幕宽度：

```css
/* 当屏幕宽度 <= 768px 时 */
@media (max-width: 768px) {
  .container {
    flex-direction: column;   /* 从横排变成竖排 */
  }
}
```

`max-width: 768px` 的意思是"宽度最大不超过 768px"——也就是 768px 及以下生效。

`min-width: 769px` 的意思是"宽度最小不低于 769px"——也就是 769px 及以上生效。

### 二、断点（Breakpoint）—— 在哪个宽度切换布局

断点就是你设定布局切换的屏幕宽度。选择一个合适的断点很重要：

| 断点 | 大致对应 |
|------|----------|
| 480px | 大多数手机竖屏 |
| 600px | 手机横屏 / 小平板 |
| 768px | 平板竖屏 |
| 900px | 平板横屏 / 小笔记本 |
| 1024px | 笔记本 / 小桌面 |
| 1200px | 标准桌面 |
| 1440px+ | 大桌面 |

**注意**：这些是参考值，不要死记。你应该根据**内容**来决定断点——当内容看起来太挤或太散时，就是该切换的时候。

### 三、移动优先 vs 桌面优先

**移动优先（mobile-first）**：

先写手机端样式（最窄的），然后用 `min-width` 逐步加宽屏样式。

```css
/* 基础样式 = 手机端（默认） */
.column { width: 100%; }

/* 平板及以上 */
@media (min-width: 600px) {
  .column { width: 50%; }
}

/* 桌面及以上 */
@media (min-width: 900px) {
  .column { width: 33.33%; }
}
```

**桌面优先（desktop-first）**：

先写桌面端样式（最宽的），然后用 `max-width` 逐步收窄屏样式。

```css
/* 基础样式 = 桌面端（默认） */
.column { width: 33.33%; }

/* 平板及以下 */
@media (max-width: 899px) {
  .column { width: 50%; }
}

/* 手机 */
@media (max-width: 599px) {
  .column { width: 100%; }
}
```

**强烈推荐移动优先**，三个理由：
1. 手机上性能更紧张，先保证移动端的简洁
2. 移动优先的 CSS 往往更短更清晰
3. 移动端用户量已经超过桌面端

### 四、媒体查询不只是宽度

虽然 `max-width` / `min-width` 占了媒体查询用量的 90%，但媒体查询的能力远不止于此：

```css
/* 打印样式 —— 打印时隐藏导航栏和广告 */
@media print {
  nav, .advertisement { display: none; }
  body { font-size: 12pt; }
}

/* 暗色模式 —— 随系统设置自动切换 */
@media (prefers-color-scheme: dark) {
  body {
    background: #1a1a2e;
    color: #eee;
  }
}

/* 横屏 vs 竖屏 */
@media (orientation: landscape) {
  /* 横屏时的样式 */
}

/* 高清屏 */
@media (min-resolution: 2dppx) {
  /* 2 倍屏（Retina）时的样式，如换高清图 */
}
```

### 五、一个完整的响应式实例

手机单列 -> 平板两列 -> 桌面三列：

```css
.card-list {
  display: grid;
  gap: 20px;
  grid-template-columns: 1fr;          /* 手机：单列 */
}

@media (min-width: 600px) {
  .card-list {
    grid-template-columns: repeat(2, 1fr);  /* 平板：两列 */
  }
}

@media (min-width: 900px) {
  .card-list {
    grid-template-columns: repeat(3, 1fr);  /* 桌面：三列 */
  }
}
```

配合 Grid 的 `grid-template-areas`，你甚至可以在不同断点下换整张"地图"。

### 六、媒体查询不要滥用

一些新手的误区：给每个小小的尺寸变化写一个媒体查询，最后写出十几个断点。这会让 CSS 难以维护。

**原则**：能用流体布局（rem、百分比、flex）解决的，就不用媒体查询。媒体查询用于**结构性的布局变化**——比如从横排变竖排、从两列变单列。颜色、字号、间距的小变化尽量用相对单位自然处理。

## 动手试试

1. 打开 `examples/2.22-media-queries.html`，在浏览器里打开，然后把窗口从宽拖到窄。观察几个断点的布局变化。
2. 打开 Chrome DevTools 的 Device Toolbar（F12 然后点手机图标），切换到不同设备看看效果。
3. 找到 CSS 中的 `@media (min-width: 900px)` 这个断点，把里面的 `repeat(3, 1fr)` 改成 `repeat(4, 1fr)`，刷新看看桌面端变成了四列。
4. 试着用浏览器打印预览（Ctrl+P）看看这个页面——你会发现导航菜单在打印预览里消失了，因为有 `@media print`。

## 本节小结

`@media` 是 CSS 里的条件判断——在特定屏幕宽度（断点）切换布局。推荐移动优先（min-width 逐步增强），能用流体布局就别滥用媒体查询。
