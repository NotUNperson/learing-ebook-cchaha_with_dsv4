## 正文

### 万物皆盒子

CSS 里有一条最根本的认知，你把它刻在脑子里：

> **每一个 HTML 元素，都是一个矩形盒子。**

不管它看起来是什么——一段文字、一张图片、一个按钮——浏览器都在背后画了一个矩形。CSS 的布局、间距、大小，本质上就是控制这些盒子怎么摆、离多远、里面怎么装东西。

这就好比：在一个仓库里，所有货物都装在盒子里。有的盒子大，有的小；有的紧挨着，有的隔老远；有的盒子里塞了泡沫（内边距），有的盒子壁很厚。CSS 就是那个仓库管理员，指挥每个盒子怎么摆。

### 盒模型的四层结构

从**内到外**，每个盒子有四层。我们用"快递包裹"来类比：

```
┌──────────────────────────────────────┐
│              margin                  │  ← 4. 外边距：这个快递和别的快递之间的间距
│  ┌──────────────────────────────┐   │
│  │          border              │   │  ← 3. 边框：纸箱的厚度
│  │  ┌──────────────────────┐   │   │
│  │  │       padding        │   │   │  ← 2. 内边距：纸箱里的泡沫保护层
│  │  │  ┌──────────────┐   │   │   │
│  │  │  │   content    │   │   │   │  ← 1. 内容区：你真正要寄的东西
│  │  │  │   (文字/图片) │   │   │   │
│  │  │  └──────────────┘   │   │   │
│  │  └──────────────────────┘   │   │
│  └──────────────────────────────┘   │
└──────────────────────────────────────┘
```

用快递包类比：

| 层级 | CSS 属性 | 比喻 |
|---|---|---|
| 最里层 | **content** | 你要寄的东西——一本书 |
| 第二层 | **padding** | 包在书外面的泡沫/气泡膜，保护书不撞到箱子壁 |
| 第三层 | **border** | 纸箱的箱壁厚度，能看见的边界 |
| 最外层 | **margin** | 这个快递和旁边快递之间的距离（别贴太近，留点空间） |

### 逐层详解

#### 第一层：content（内容区）

内容区就是元素真正"装着"的东西——文字、图片、子元素。它是盒子的核心，其他三层都围着它转。

```css
div {
  width: 300px;   /* 内容区宽度（默认只设置 content 的宽度） */
  height: 200px;  /* 内容区高度 */
}
```

#### 第二层：padding（内边距）

内容到边框之间的距离。就像快递箱里的泡沫——让内容别直接贴在箱子壁上。

```css
div {
  padding: 20px;          /* 四边统一 20px */
  padding: 20px 40px;     /* 上下 20px，左右 40px */
  padding: 10px 20px 30px 40px;  /* 上 右 下 左（顺时针） */
  /* 也可单边设置 */
  padding-top: 10px;
  padding-right: 20px;
}
```

padding **是背景色的延伸**——如果你给元素设了背景色，padding 区域也会被填充。回到快递比喻：泡沫的材料颜色和箱子内部颜色是一样的。

#### 第三层：border（边框）

盒子的边界线。就像快递箱的厚度——它是一层可知可感的边界。

```css
div {
  border: 2px solid #333;  /* 简写：粗细 样式 颜色 */
  /* 也可单边设置 */
  border-left: 4px solid #e74c3c;
}
```

border 的颜色、样式、粗细我们下一节（2.13）会深入展开。

#### 第四层：margin（外边距）

这个盒子和相邻盒子之间的**外部**间距。就像仓库里两个快递之间要保持的距离。

```css
div {
  margin: 20px;          /* 四边统一 */
  margin: 20px auto;     /* 上下 20px，左右 auto（居中！） */
  margin: 10px 20px 30px 40px;  /* 上 右 下 左（顺时针，和 padding 一样） */
}
```

**margin: 0 auto 居中**是 CSS 里最经典的居中方式：左右都设为 auto，浏览器就会自动平均分配左右空间，让盒子水平居中。但仅对**有确定宽度**的块级元素有效。

> margin 和 padding 的值顺序规则完全一样：
> - 1 个值：四边相同
> - 2 个值：上下、左右
> - 3 个值：上、左右、下
> - 4 个值：上、右、下、左（顺时针）

### 用 Chrome DevTools 看盒模型

每个浏览器都有"开发者工具"（按 F12 打开）。这是你未来最亲密的战友。

1. 打开任意网页，按 **F12**
2. 点击左上角的"选择元素"图标（或按 Ctrl+Shift+C）
3. 鼠标在页面上点任意元素
4. 在右侧的 **Styles** 面板底部，你会看到这样一个图：

```
┌──────────┐
│  margin  │  橙色/浅棕色
│ ┌──────┐ │
│ │border│ │  黄色/黑色
│ │┌───┐ │ │
│ ││pad││ │  绿色
│ ││└─┘│ │ │
│ ││内容││ │  蓝色
│ │└───┘ │ │
│ └──────┘ │
└──────────┘
```

把鼠标悬停在这个图上，页面里对应元素就会高亮各层区域——比任何文字描述都直观。以后调试间距问题，第一反应就是打开 DevTools 看盒模型。

## 动手试试

1. 打开示例文件 `2.11-box-model.html`，观察每层颜色标注
2. 按 F12 打开开发者工具，选中任意演示元素，在 Styles 面板底部找到盒模型图
3. 修改某个元素的 `padding` 值，观察背底色是否跟着延伸
4. 试着修改 `margin` 值，观察元素间距的变化
5. 给一个元素设 `width: 200px`，再设 `padding: 50px` 和 `border: 5px`，在 DevTools 里看看实际宽度是多少（为下一节做铺垫）

## 本节小结

CSS 万物皆盒子，从内到外四层：content（内容本身）、padding（内边距=泡沫）、border（边框=箱壁）、margin（外边距=包裹间距）。DevTools 里的盒模型面板是你最好的老师。
