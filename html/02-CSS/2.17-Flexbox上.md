# 2.17 Flexbox（上）—— 弹性盒子容器属性

## 本节你会学到什么

- Flexbox 是什么，为什么它比 float 好一百倍
- 主轴（main axis）和交叉轴（cross axis）的概念
- 容器的五个核心属性：`flex-direction`、`flex-wrap`、`justify-content`、`align-items`、`align-content`
- `flex-flow` 简写
- 用排队的比喻理解每个属性的效果

## 正文

### 从"硬排队"到"智能排队"

回顾上一节的 float：为了做一排横向排列的盒子，我们要给每个盒子加 `float: left`，然后还要用 clearfix 修高度塌陷，还要手动算宽度防止溢出换行……简直像用算盘做微积分。

现在，欢迎来到现代 CSS 的明星——**Flexbox（弹性盒子）**。

Flexbox 是一维布局模式。"一维"的意思是说，它擅长处理**一个方向**上的排列——要么横着排，要么竖着排（不可能同时横着又竖着，那是 Grid 的活）。但在一维这个领域，它简直是完美的。

**核心概念**：只要给父元素写一句 `display: flex;`，所有子元素就自动变成了"弹性项目"，父元素变成了"弹性容器"。容器像一个智能管家，自动帮你分配空间、对齐、换行。

### 排队比喻

把 Flexbox 想象成一个排队系统：

- **弹性容器（flex container）**：排队区域的负责人
- **弹性项目（flex items）**：排队的人
- **主轴（main axis）**：队伍排的方向（从左往右，还是从上往下）
- **交叉轴（cross axis）**：和队伍垂直的方向（人们的肩膀方向）
- **justify-content**：人在队伍方向上怎么站（前面？中间？分散？）
- **align-items**：人在肩膀方向上怎么对（靠上？居中？靠下？）

### 一、开启 Flexbox

只需要一行代码：

```css
.container {
  display: flex;
}
```

就这么简单！子元素们立刻从 block 的"垂直堆叠"变成了一排横向排列。不需要 float，不需要 clearfix，不需要手动算宽度。

### 二、flex-direction —— 排队方向（主轴方向）

`flex-direction` 决定主轴的方向，也就是"队伍往哪个方向排"。

| 值 | 主轴方向 | 类比 |
|----|----------|------|
| `row`（默认） | 从左到右 | 横排，队伍从左往右延伸 |
| `row-reverse` | 从右到左 | 横排，但倒过来，最右边是第一个人 |
| `column` | 从上到下 | 竖排，队伍从上往下延伸 |
| `column-reverse` | 从下到上 | 竖排，但倒过来，最下面是第一个人 |

**理解交叉轴**：交叉轴总是和主轴垂直。如果主轴是横的（row），交叉轴就是竖的；如果主轴是竖的（column），交叉轴就是横的。

### 三、flex-wrap —— 是否换行

当一排装不下所有项目时，Flexbox 默认会把它们**挤瘦**而不是换行。但你可以用 `flex-wrap` 改变这个行为：

| 值 | 效果 | 类比 |
|----|------|------|
| `nowrap`（默认） | 不换行，挤一挤 | 水流进窄管道，压缩自己 |
| `wrap` | 装不下就换行 | 流水遇到墙，拐弯 |
| `wrap-reverse` | 换行但反向（往上换行） | 倒流 |

`flex-flow` 是 `flex-direction` 和 `flex-wrap` 的简写：

```css
.container {
  flex-flow: row wrap;   /* 横向排列 + 换行 */
}
```

### 四、justify-content —— 主轴对齐

`justify-content` 控制项目在**主轴方向**上怎么分布。这是最常用的 Flexbox 属性之一。

还是排队比喻——一群人站在一条直线上：

| 值 | 效果 | 类比 |
|----|------|------|
| `flex-start`（默认） | 靠主轴起点 | 队伍靠左（或靠上）站 |
| `flex-end` | 靠主轴终点 | 队伍靠右（或靠下）站 |
| `center` | 居中 | 队伍站在中间 |
| `space-between` | 两端对齐，中间均分 | 第一个人贴左墙，最后一个人贴右墙，剩下的人均匀散布 |
| `space-around` | 每个项目两边有相等的间距 | 每个人周围都有一样的"呼吸空间" |
| `space-evenly` | 所有间隙完全相等 | 项目之间、项目与墙之间距离都相同 |

### 五、align-items —— 交叉轴对齐

`align-items` 控制项目在**交叉轴（和主轴垂直）方向**上怎么对齐。

| 值 | 效果 | 类比 |
|----|------|------|
| `stretch`（默认） | 拉伸填满交叉轴 | 所有人被拉成和最高的那个人一样高 |
| `flex-start` | 靠交叉轴起点 | 所有人的肩膀对齐上面 |
| `flex-end` | 靠交叉轴终点 | 所有人的脚对齐下面 |
| `center` | 居中 | 大家的中间线对齐（最常用！） |
| `baseline` | 文字基线对齐 | 第一行文字对齐，不管盒子大小 |

### 六、align-content —— 多行时行与行之间的对齐

只有启用了 `flex-wrap: wrap` 且有多行时，`align-content` 才生效。它控制的是"行"与"行"之间的间距分布方式。值和 `justify-content` 类似（flex-start / flex-end / center / space-between / space-around / space-evenly / stretch）。

## 动手试试

1. 打开 `examples/2.17-flexbox-1.html`，观察默认 `display: flex` 下子元素的排列。
2. 找到 `.demo-flex`，把 `flex-direction` 从 `row` 改成 `column`，看看子元素怎么从横排变成竖排。
3. 修改 `justify-content` 的值为 `space-between`、`space-around`、`space-evenly`，分别观察区别（重点关注边框边距）。
4. 修改 `align-items` 的值为 `flex-start`、`flex-end`、`center`，观察盒子在交叉轴上的位置变化。
5. 注释掉 `flex-wrap: wrap` 那行（变回默认的 nowrap），观察盒子们是怎么被挤压的。

## 本节小结

Flexbox 给父元素写 `display: flex`，子元素就自动成了弹性项目。用 `flex-direction` 定排队方向，`justify-content` 管主轴对齐，`align-items` 管交叉轴对齐，`flex-wrap` 管是否换行。比 float 好用一百倍。
