# 2.18 Flexbox（下）—— 弹性项目属性

## 本节你会学到什么

- `flex-grow`：有多余空间时怎么分配（分蛋糕）
- `flex-shrink`：空间不够时怎么压缩（一起挨饿）
- `flex-basis`：项目初始大小
- `flex` 简写属性（强烈推荐使用）
- `align-self`：单个项目覆盖容器的 `align-items`
- `gap`：项目之间的间距
- 三个经典布局模式：水平垂直居中、圣杯布局、等分列

## 正文

### 每个队员的"个性"

上一节我们学了容器的属性——那是"整体规则"。但每个弹性项目也可以有自己的"个性"，就像一个团队里，领导定了基调（容器属性），但每个队员可以发挥自己的特点（项目属性）。

这些项目属性写在**子元素**上，而不是父容器上。

### 一、flex-grow —— 分蛋糕（放大比例）

`flex-grow` 决定当容器有**多余空间**时，这个项目能分到多少。

默认值是 `0`，意思是"有多的我也不要"。

```css
.item {
  flex-grow: 1;   /* 参加分蛋糕，蛋糕按比例分 */
}
```

**类比**：一家人吃蛋糕。爸爸说"我已经吃饱了"（flex-grow: 0），妈妈要了 1 份（flex-grow: 1），小孩要了 2 份（flex-grow: 2）。如果蛋糕还剩 300 克，妈妈分到 100 克，小孩分到 200 克。

如果所有项目都设 `flex-grow: 1`，它们就**均分**多余空间。

### 二、flex-shrink —— 一起挨饿（缩小比例）

`flex-shrink` 决定当容器**空间不够**时，这个项目要多"瘦身"。

默认值是 `1`，意思是"大家一起挨饿"。

如果设为 `0`，这个项目就**不缩小**——别人挤扁了它也保持原样。

**类比**：公交车上人太多，需要挤一挤。`flex-shrink: 0` 的人就像举着易碎蛋糕上车的——"别挤我！我不能缩！"

### 三、flex-basis —— 初始大小

`flex-basis` 设定了项目在**分配多余空间之前**的初始大小。可以理解成"基准宽度"（主轴为 row 时）或"基准高度"（主轴为 column 时）。

```css
.item {
  flex-basis: 200px;  /* 初始是 200px，然后再根据 grow/shrink 调整 */
}
```

`flex-basis` 和 `width` 的区别：
- `width` 是硬规定（严格多少 px）
- `flex-basis` 是"起点"——空间多余就增长，空间不够就压缩

### 四、flex 简写 —— 记住这个就好

日常开发中，你不用分开写三个属性，统一用 `flex` 简写：

```css
.item {
  flex: flex-grow flex-shrink flex-basis;
}
```

最常用的几个写法：

| 写法 | 含义 | 场景 |
|------|------|------|
| `flex: 1` | `flex: 1 1 0%` | 均分剩余空间，最常用！ |
| `flex: auto` | `flex: 1 1 auto` | 均分但考虑内容大小 |
| `flex: none` | `flex: 0 0 auto` | 固定大小，不参与分配 |
| `flex: initial` | `flex: 0 1 auto` | 默认值，只缩小不放大 |

**没有歧义的推荐**：想让几个项目平分一个容器？给每个项目写 `flex: 1` 就好了。

### 五、align-self —— "我偏不！"

`align-self` 允许**单个项目**覆盖容器的 `align-items` 设置。

```css
.item-special {
  align-self: flex-end;   /* 别人都居中了，就我偏要靠底站着 */
}
```

**类比**：队长说"全体居中对齐！"但有一个队员说"我偏不，我要靠上站。"——这就是 `align-self`。

### 六、gap —— 项目之间的"呼吸空间"

`gap` 是 CSS 的通用属性，不仅 Flexbox 能用，Grid 也能用。它设置的是项目之间的间距：

```css
.container {
  display: flex;
  gap: 20px;        /* 项目之间 20px 间距 */
  gap: 20px 10px;   /* 行间距 20px，列间距 10px */
}
```

在 `gap` 出现以前，人们用 `margin` 来制造间距，但会产生"第一个和最后一个元素外面多余的 margin"的问题。`gap` 完美解决了这个问题——它只在项目**之间**起作用。

### 七、三个经典布局模式

**模式一：水平垂直居中（最经典）**

```css
.container {
  display: flex;
  justify-content: center;  /* 主轴居中 */
  align-items: center;      /* 交叉轴居中 */
  height: 100vh;           /* 占满整个视口高度 */
}
```
这两行就能实现 CSS 里最让新手头疼的"水平垂直居中"——在 Flexbox 之前，这需要各种 hack 技巧。

**模式二：圣杯布局（页头 + 中间三列 + 页脚）**

```css
.page     { display: flex; flex-direction: column; min-height: 100vh; }
.middle   { display: flex; flex: 1; }
.sidebar  { width: 200px; }
.content  { flex: 1; }
```

**模式三：等分列（如三列卡片）**

```css
.card-list { display: flex; gap: 20px; }
.card      { flex: 1; }   /* 每张卡片均分宽度 */
```

## 本节小结

项目属性让每个弹性项目有自己的"个性"——flex（简写）管空间分配，align-self 管个别对齐，gap 管间距。`flex: 1` 均分、水平居中+垂直居中是日常开发中最常用的组合。
