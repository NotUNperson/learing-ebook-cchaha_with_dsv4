# 2.10 字体与图标

## 本节你会学到什么

- `@font-face` 自定义字体的用法
- Google Fonts 的引入方式（link 方式）
- 图标字体（Icon Font）的概念
- Font Awesome 等图标库的基本用法
- 现代开发中 SVG 图标的趋势

## 正文

### 不止于系统自带字体

上一节我们用了 web-safe 字体——那些操作系统自带的字体。但你要是见到所有网站都是 Arial 和 Times New Roman，那网页设计也太单调了。

自定义字体允许你使用**用户设备上可能没有的字体**——浏览器会按你给的地址下载字体文件，然后用它来渲染文字。就像你请客吃饭，不是去大家都有分店的连锁店，而是从外面请了一个私厨带着食材来。

### @font-face：自己"定义"一个字体

`@font-face` 是 CSS 里用来"声明一个自定义字体"的规则。你告诉浏览器：这个字体叫什么名字，它的文件在哪里。

```css
@font-face {
  font-family: "MyCustomFont";           /* 给字体起个名字，之后用这个名字引用 */
  src: url("fonts/MyCustomFont.woff2")   /* 字体文件路径 */
       format("woff2"),                   /* 告诉浏览器这是什么格式 */
       url("fonts/MyCustomFont.woff")     /* 备选格式 */
       format("woff");
  font-weight: normal;                    /* 这个文件对应哪个粗细 */
  font-style: normal;                     /* 正常还是斜体 */
}

/* 定义好后，像普通字体一样使用 */
body {
  font-family: "MyCustomFont", sans-serif;
}
```

字体文件有不同格式，兼容性各有不同：

| 格式 | 扩展名 | 说明 |
|---|---|---|
| WOFF2 | `.woff2` | 现代浏览器首选，压缩率高，文件小 |
| WOFF | `.woff` | 广泛兼容，次选 |
| TTF/OTF | `.ttf` `.otf` | 传统格式，文件较大 |
| EOT | `.eot` | 仅 IE 老版本需要 |

> 好消息：2026 年了，你只需提供 `.woff2` 基本就够了。如果需要兼容老旧浏览器，再加一个 `.woff`。

### Google Fonts：最省心的字体来源

Google Fonts 是一个免费、开源的字体库（fonts.google.com）。你不用自己下载字体文件、不用写 `@font-face`，Google 帮你全搞定了。

**使用步骤**：

1. 打开 [fonts.google.com](https://fonts.google.com)，挑一款字体
2. 选好想要的粗细
3. 复制它给的 `<link>` 标签，贴到你的 HTML `<head>` 里
4. 在 CSS 里像普通字体一样使用

示例（引入 Roboto + Noto Sans SC）：

```html
<head>
  <!-- Google Fonts 提供的 link 标签 -->
  <link rel="preconnect" href="https://fonts.googleapis.com">
  <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
  <link href="https://fonts.googleapis.com/css2?family=Roboto:wght@400;700&family=Noto+Sans+SC:wght@400;700&display=swap" rel="stylesheet">
</head>
```

```css
body {
  font-family: "Roboto", "Noto Sans SC", sans-serif;
  font-weight: 400;
}

h1 {
  font-family: "Roboto", "Noto Sans SC", sans-serif;
  font-weight: 700;  /* 粗体，只要你在 link 里包含了 700 粗细 */
}
```

> 注意：引入 Google Fonts 需要网络——如果用户离线，浏览器会回退到后备字体。`display=swap` 这个参数让浏览器先用后备字体显示文字，等字体加载完再"替换"，避免白屏。

### 图标字体：把图标当作"文字"

你可能会觉得奇怪——图标和字体有什么关系？

图标字体的核心思想：**每个图标对应一个 Unicode 字符**。就像字母 "A" 在字体里对应一个字形，图标字体把 "房子图标" 也作为一个字符存进去。

好处很明显：
- 像文字一样用 CSS 控制颜色、大小、阴影（你前面学的文字样式全部能套到图标上！）
- 矢量缩放，不会模糊
- 文件相对较小

### Font Awesome：最流行的图标字体库

使用步骤超级简单：

1. 引入 Font Awesome 的 CSS（CDN 方式最快）：

```html
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css">
```

2. 在 HTML 里用 `<i>` 标签加对应 class：

```html
<i class="fa-solid fa-house"></i>         <!-- 房子图标 -->
<i class="fa-solid fa-magnifying-glass"></i>  <!-- 搜索图标 -->
<i class="fa-brands fa-github"></i>        <!-- GitHub 品牌图标 -->
<i class="fa-solid fa-spinner fa-spin"></i> <!-- 旋转的加载图标 -->
```

3. 像调整文字一样调整图标：

```css
.fa-house {
  color: #3498db;
  font-size: 24px;
}
```

常用的还有 **Material Icons**（Google 出品，Google Fonts 旗下）、**Bootstrap Icons** 等。

### 现代趋势：SVG 图标

图标字体虽然方便，但有一些短板：
- 图标字体的本质是文字，有些浏览器会把图标翻译的文字读出来（无障碍体验差）
- 图标如果加载不出来，用户会看到奇奇怪怪的字符
- 多色图标困难

因此现代开发越来越倾向 **SVG 图标**（.svg 文件直接嵌入或用 SVG sprite）。但对于学习和快速原型开发，图标字体仍然是一个非常实用的工具。

> 你学 C 的时候用 `printf` 调试，做项目可能用 gdb——图标字体是 `printf`，SVG 是 gdb。两个都会当然最好，但先从简单的上手。

## 动手试试

1. 打开示例文件 `2.10-fonts.html`，看看 Google Fonts 字体的效果
2. 去 [fonts.google.com](https://fonts.google.com) 挑一款你喜欢的字体，替换示例中的字体
3. 去 [fontawesome.com/icons](https://fontawesome.com/icons) 搜几个图标，替换示例中的图标 class
4. 尝试给图标设置不同的 `font-size` 和 `color`，感受"图标就是文字"的概念
5. 把 Google Fonts 的 link 标签暂时注释掉，看页面是否回退到后备字体

## 本节小结

自定义字体让网页脱离"系统字体"的束缚——`@font-face` 自己定义，Google Fonts 省心引入。图标字体把图标当文字用，CSS 文字样式全兼容。SVG 是未来趋势，但图标字体是入门的趁手工具。
