# 2.2 层叠与优先级

## 本节你会学到什么

- "层叠"到底是什么——多重样式来源如何叠加
- 选择器权重的计算方法（标签=1, class=10, id=100, 内联=1000）
- 同权重下"后来居上"规则
- `!important` 是什么，为什么尽量不用

## 正文

### 什么是"层叠"

上一节我们知道了三种引入 CSS 的方式。但真正的"层叠"比那更复杂。假设你写了一段 HTML：

```html
<p class="intro" id="first-para">你好，世界！</p>
```

然后你从不同角度给这个段落设置了颜色：

```css
p              { color: black; }   /* 规则 A：所有段落 */
.intro         { color: blue; }    /* 规则 B：class 为 intro 的元素 */
#first-para    { color: red; }     /* 规则 C：id 为 first-para 的元素 */
```

现在问题来了：这个段落的文字到底是什么颜色？

答案是红色。为什么？因为浏览器在"层叠"这些规则时，会计算每条规则的**权重**，权重高的胜出。

### 权重计算：像扑克牌比大小

CSS 选择器的权重，可以抽象成一个四位数字 `(a, b, c, d)`，每一位代表不同层级：

| 选择器类型 | 权重值 | 类比 |
|---|---|---|
| 内联样式 (`style=""`) | a=1（即 1000 分） | 王牌 |
| id 选择器 (`#xxx`) | b=1（即 100 分） | K |
| class 选择器 (`.xxx`) / 属性选择器 / 伪类 | c=1（即 10 分） | 10 |
| 标签选择器 (`p`, `div`) / 伪元素 | d=1（即 1 分） | 1 |
| 通配符 `*` | 0 分 | 空气 |

计算规则也很简单：**把每条规则的各层级的数量分别填入对应位数，然后从左往右比大小。** 就像两个四位数比大小——先看千位，千位相同看百位，以此类推。

举例：

```css
/* 权重：(0,0,0,1) = 1 */
p { color: black; }

/* 权重：(0,0,1,0) = 10 — 比上面高，胜出 */
.intro { color: blue; }

/* 权重：(0,0,1,1) = 11 — 更高了，胜出 */
p.intro { color: green; }

/* 权重：(0,1,0,0) = 100 — 最高，胜出 */
#first-para { color: red; }

/* 权重：(1,0,0,0) = 1000 — 无敌，但这不算选择器，是内联样式 */
```

> 重要提醒：权重不是十进制！不是说你写 11 个标签选择器就能超过 1 个 class 选择器。每一位是独立的，10 个标签（0,0,0,10）仍然比不过 1 个 class（0,0,1,0）。就像你不能攒 10 张扑克牌 1 去赢 1 张扑克牌 10——规则不允许。

### "后来居上"规则

如果两条规则的权重完全相同呢？那就看顺序——**后面的覆盖前面的**。这个规则非常直观：同一份说明书，如果你在第 1 页写"门刷蓝色"，第 5 页又写"门刷红色"，工人当然按最新指示办事。

```css
p { color: blue; }   /* 先写的 */
p { color: red; }    /* 后写的 — 生效！ */
```

这也解释了为什么外部样式表（写在前面）、内部样式表（写在后面）、内联样式（写在最后面），越靠后的越优先——在相同选择器的情况下。

### !important：核武器，尽量别用

还有一个终极武器叫 `!important`：

```css
p {
  color: red !important;  /* 加了 !important，谁都别想覆盖我 */
}
```

加了 `!important` 后，这条规则会跳过正常的权重计算，直接成为最高优先级。但正因如此，它极难被覆盖——如果滥用，很快你的样式表就会变成一片"谁能喊得更响"的军备竞赛，谁也调试不动。

> 经验法则：`!important` 只在万不得已时使用。比如你使用的第三方 CSS 库（像 Bootstrap）里的某个样式你非覆盖不可，而且你无法修改 HTML 结构或提高自己选择器的权重。在日常学习中，把它当成"不存在"就好。

### 一张图总结优先级链条

```
!important > 内联样式(1000) > id(100) > class/属性/伪类(10) > 标签/伪元素(1) > 通配符*(0) > 继承样式
```

注意最右边多了个"继承样式"。这是什么？某些 CSS 属性会从父元素传递给子元素（比如字体、颜色），这些属性的优先级比 0 还低——任何选择器只要有明确的规则，就能覆盖掉继承来的样式。

## 本节小结

CSS 的"层叠"是三重规则的综合：来源优先级（内联>内部>外部）、选择器权重（id>class>标签）、出现顺序（后来居上）。理解这个，你就解决了"为什么我的样式不生效"这个新手的头号问题。
