# 2.23 过渡

## 本节你会学到什么

- `transition` 是什么——让属性值平滑变化
- 四个过渡子属性：`property`、`duration`、`timing-function`、`delay`
- 速度曲线（ease、linear、ease-in、ease-out 等）的区别
- 哪些属性可以过渡，哪些不能
- 用 `transition` 简写提高可读性

## 正文

### 从"啪"到"柔"

到现在为止，我们写的所有样式变化都是"啪"一下完成的——鼠标悬停，链接瞬间变色；点一下按钮，背景颜色立刻切换。

但在真实世界里，很少有事情是瞬间发生的。你按开关，灯是**慢慢亮起来**的（尽管很快）；你踩油门，车速是**逐渐加上去**的。

CSS 的 `transition`（过渡）就是让属性值的变化从"啪"变成"柔"——从一个状态平滑过渡到另一个状态。

**类比**：transition 就像动画片里的"帧间插值"。你只需要告诉 CSS "起始状态"和"结束状态"，中间的每一帧它自动帮你补上。

### 一、transition 四大要素

一个完整的过渡有四个要素：

| 属性 | 作用 | 类比 |
|------|------|------|
| `transition-property` | 哪些属性要过渡 | 你要对什么东西做渐变 |
| `transition-duration` | 过渡耗时 | 整个过程持续多长时间 |
| `transition-timing-function` | 速度曲线 | 是匀速变化还是先快后慢 |
| `transition-delay` | 延迟开始 | 等多久才开始 |

### 二、transition-property —— 过渡哪些属性

```css
.button {
  transition-property: background-color, transform;
  /* 让背景色和变形两个属性参与过渡 */
}
```

可以写 `all` 让所有可过渡的属性都平滑变化（但性能略差，不推荐在生产环境滥用）。

**哪些属性可以过渡？**
- 颜色相关：`color`、`background-color`、`border-color`
- 尺寸相关：`width`、`height`、`padding`、`margin`
- 位置相关：`top`、`left`、`right`、`bottom`
- 变形：`transform`（最推荐，性能最好）
- 透明度：`opacity`
- 阴影：`box-shadow`、`text-shadow`

**不能过渡的属性**：
- `display`（不能，中间态没有意义）
- `position`（不能）
- `font-family`（不能）

### 三、transition-duration —— 变化耗时

单位是 `s`（秒）或 `ms`（毫秒）：

```css
.button {
  transition-duration: 0.3s;    /* 0.3 秒 */
  transition-duration: 300ms;   /* 300 毫秒，同上 */
}
```

推荐值：**0.2s 到 0.5s**。太快感觉不到，太慢让人觉得迟钝。交互反馈（如 hover）用 0.2s-0.3s，大范围变化（如模态框弹出）用 0.3s-0.5s。

### 四、transition-timing-function —— 速度曲线

这是过渡中最"有灵魂"的参数。它决定了变化过程的速度：

| 值 | 速度特征 | 类比 |
|----|----------|------|
| `ease`（默认） | 慢 -> 快 -> 慢 | 汽车起步加速，停车减速 |
| `linear` | 全程匀速 | 高铁巡航，速度不变 |
| `ease-in` | 慢 -> 快 | 弹弓拉满后松手，越飞越快 |
| `ease-out` | 快 -> 慢 | 扔出去的石头落地前减速 |
| `ease-in-out` | 慢 -> 快 -> 慢 | 比 ease 更对称 |
| `cubic-bezier()` | 自定义曲线 | 自定义行为 |

绝大部分场景用 `ease` 就够了——它最自然。要做更精致的动画可以用 `cubic-bezier()` 调曲线。

### 五、transition-delay —— 延迟开始

```css
.button {
  transition-delay: 0.2s;   /* 鼠标放上后等 0.2 秒再开始变化 */
}
```

一般用不上，但在**菜单展开**等场景中有妙用——多个子项依次延迟出现，形成"涟漪"效果。

### 六、transition 简写

四个属性分开写很啰嗦，日常用简写：

```css
/* 格式：property duration timing-function delay */
.button {
  transition: background-color 0.3s ease;
  /* 背景色变化，0.3秒，缓动曲线 */
}

/* 给多个属性设置过渡 */
.card {
  transition: transform 0.3s ease, box-shadow 0.3s ease;
}
```

### 七、过渡写在谁身上

一个常见误区：把 `transition` 写在 `:hover` 状态上。

```css
/* 错误写法 */
.button:hover {
  transition: background-color 0.3s;  /* 移入有过渡，移出没有！ */
}

/* 正确写法 —— transition 写在元素本身上 */
.button {
  transition: background-color 0.3s;  /* 移入和移出都有过渡 */
}
.button:hover {
  background-color: red;
}
```

**原则**：`transition` 写在元素**默认状态**上，不要写在 `:hover` 里。这样才能保证来回都有过渡效果。

### 八、推荐用 transform 代替直接改尺寸

如果你想做一个"hover 时放大"的效果：

```css
/* 不推荐 —— 直接改 width/height 会触发布局重排，性能差 */
.button:hover {
  width: 120px;
  height: 50px;
}

/* 推荐 —— transform 是 GPU 加速的，丝般顺滑 */
.button:hover {
  transform: scale(1.1);   /* 放大到 110% */
}
```

`transform` 只触发合成，不触发布局重排，是性能最好的过渡方式。

## 动手试试

1. 打开 `examples/2.23-transition.html`，把鼠标悬停到各个按钮和卡片上，感受不同的过渡效果。
2. 找到第一个按钮的 CSS，把 `transition-duration` 从 `0.3s` 改成 `2s`，再 hover 试试——2 秒的过渡是不是过于拖沓？
3. 把 timing-function 从 `ease` 改成 `linear`，对比感受"缓动"和"匀速"的差异。
4. 找到 `.card` 的 `transition`，给它加一个 `opacity 0.3s`，然后在 `.card:hover` 里加 `opacity: 0.7;`，观察透明度过渡。
5. 试试给 `.transition-box` 加 `transition: width 0.5s`，然后在 hover 里改 `width`。再用 `transform: scale(1.2)` 做同样的效果，对比两者的流畅度。

## 本节小结

`transition` 让 CSS 属性变化从"啪"变"柔"。四个参数：property（变什么）、duration（变多久）、timing-function（怎么变）、delay（啥时候开始）。推荐简写，推荐用 transform 代替直接改尺寸。
