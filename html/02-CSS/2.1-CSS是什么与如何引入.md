# 2.1 CSS 是什么与如何引入

## 本节你会学到什么

- CSS 是什么，它和 HTML 的关系
- 三种引入 CSS 的方式：内联样式、内部样式表、外部样式表
- 三种方式各自的适用场景
- 基础优先级规则：内联 > 内部 > 外部
- 用 `link` 标签引入外部 CSS 文件

## 正文

### 从毛坯房到精装房

你已经学完了 HTML。HTML 是什么呢？它就像盖房子的钢筋水泥——搭出了房间的骨架：这里是标题，那里是段落，下面有张图片。但光有骨架的房子能住人吗？不能。墙是灰的，字是黑的，所有东西都干巴巴地堆在一起。

CSS 就是装修队。它负责给这个毛坯房刷墙、铺地板、挂窗帘、摆家具——让网页变得好看。

CSS 的全称是 **Cascading Style Sheets**，翻译过来叫"层叠样式表"。拆开看：

- **层叠（Cascading）**：多个样式可以叠加到同一个元素上，后面会详讲
- **样式（Style）**：字体、颜色、间距、背景……一切视觉相关的东西
- **表（Sheets）**：样式规则写在一起，像一张清单

打个比方：HTML 是你身体的骨骼和器官，CSS 就是你穿的衣服、戴的眼镜、梳的发型。同一个人（同一份 HTML），换一套 CSS 就能焕然一新——这也就是为什么有个著名网站叫 [CSS Zen Garden](http://www.csszengarden.com/)，同一份 HTML 被不同设计师配上不同的 CSS，呈现出来的页面天差地别。

### 三种引入方式

CSS 可以通过三种方式加到 HTML 里。我们用一个生活中的类比来理解：

想象你是学校管乐团的总指挥，需要给团员们发一份着装要求。

#### 方式一：内联样式（inline style）—— 一个一个口头通知

走到每个人面前，单独告诉他："你，穿白衬衫黑裤子。"

在 HTML 里，这就是直接写在元素的 `style` 属性里：

```html
<p style="color: red; font-size: 20px;">这段文字是红色的，字号 20px。</p>
```

- **优点**：最直接，说完立刻生效
- **缺点**：如果有 100 个人，你得说 100 遍，改一次也要改 100 处
- **适用场景**：临时测试、或者极少量的、不会重复的样式

#### 方式二：内部样式表（internal style sheet）—— 在群里发一条公告

在群里发一条消息："全体注意，从今天起都穿白衬衫黑裤子。"

在 HTML 里，这就是在 `<head>` 里写一个 `<style>` 标签：

```html
<head>
  <style>
    p {
      color: blue;
      font-size: 18px;
    }
  </style>
</head>
```

- **优点**：一个规则管所有同类元素，改一次全改
- **缺点**：只在这一个页面里有效，换一个页面又要重新发公告
- **适用场景**：单个页面的样式、演示 demo、小项目

#### 方式三：外部样式表（external style sheet）—— 印发一份着装手册

印一本小册子发给所有人："着装规范请参考《管乐团着装手册》第 3 页。"以后要改，只需要修订手册。

在 HTML 里，这就是创建一个 `.css` 文件，然后用 `<link>` 标签引入：

**style.css 文件**：

```css
p {
  color: green;
  font-size: 16px;
}
```

**HTML 文件**：

```html
<head>
  <link rel="stylesheet" href="style.css">
</head>
```

- **优点**：多个页面共用同一份 CSS，修改一处全站生效，像统一的制服规范
- **缺点**：多一次 HTTP 请求（浏览器要单独下载 css 文件）
- **适用场景**：几乎所有正式项目——这才是推荐的写法

### 基础优先级

如果三种方式同时存在，浏览器听谁的？记住一条**基础规则**：

> 内联 > 内部 > 外部

理由也很直观：口头单独通知（内联）比群公告（内部）更具体，群公告比通用手册（外部）更具体。越具体，优先级越高。

你在本节的示例代码里会亲眼看到这个规则的实际效果。

> 注意：这里说的是"基础规则"。真正的优先级计算比这个复杂（涉及选择器权重等），我们下一节《层叠与优先级》会深入讲。

## 本节小结

CSS 是网页的"衣服"，有三种穿法——内联（直接穿在元素身上）、内部（写在本页 head 里）、外部（独立 .css 文件用 link 引入）。越贴近元素的穿法越优先。
