# 2.16 浮动与清除

## 本节你会学到什么

- `float` 的本意：文字环绕图片
- `float: left` 和 `float: right` 的效果
- 浮动导致的**父元素高度塌陷**问题
- `clear` 属性清除浮动
- 经典的 clearfix 技巧（伪元素清除法）
- float 在现代布局中的定位：了解即可，布局交给 Flexbox/Grid

## 正文

### float 的本意 —— 报纸排版

想象你翻开一张报纸。报纸上经常有这样的排版：左边是一张图片，右边的文字绕着图片走。这就是 `float` 最初的设计目的——**让文字环绕图片**。

`float` 翻译过来叫"浮动"。你可以想象元素像一片叶子浮在水面上：它脱离了正常的水流（文档流），漂到了左边或右边，而水面上的其他东西（文字）会绕着它走。

### 一、float: left 和 float: right

```css
img {
  float: left;   /* 图片浮到左边，文字从右边环绕 */
  margin-right: 15px;
}
```

**float: left**：元素浮到父容器的左边，后面的内容从右边环绕。
**float: right**：元素浮到父容器的右边，后面的内容从左边环绕。

注意：浮动的元素会变成"块级化"——即使原本是 `span` 这样的 inline 元素，加了 `float` 后也能设置宽高了。

### 二、高度塌陷 —— 浮动的"副作用"

浮动有一个著名的副作用：**父元素高度塌陷**。

这是什么意思呢？正常情况下，父元素的高度由所有子元素撑起来——子元素有多高，父元素就有多高。但浮动元素脱离了文档流，父元素就"看不到"它们了，导致父元素的高度缩成 0（如果所有子元素都浮动了的话）。

**类比**：想象一个气球里装着几个小球。突然小球都浮到空中去了（float），气球的底部就瘪了——因为里面没有东西撑着。这就是高度塌陷。

这个效果在你想让父元素有一个背景色或者边框时尤为致命——父元素高度为 0，背景色根本看不见。

### 三、clear —— 清除浮动

`clear` 属性的作用是：让一个元素**不受前面浮动元素的影响**，强制换到浮动元素的下方。

```css
.clear-element {
  clear: both;   /* 清除左右两侧的浮动影响 */
}
```

`clear` 有三种值：
- `clear: left` —— 只清除左浮动
- `clear: right` —— 只清除右浮动
- `clear: both` —— 清除两侧浮动（最常用）

### 四、clearfix —— 经典修复技巧

既然每个浮动元素后面都手动加一个清除元素很麻烦，人们发明了 **clearfix** 技巧。它的原理是：在父元素的内容最后，插入一个看不见的"清洁工"，用它来撑起父元素的高度。

标准写法：

```css
.clearfix::after {
  content: "";          /* 必须！伪元素得有 content */
  display: block;       /* 伪元素默认是 inline，改成 block */
  clear: both;           /* 清除浮动 */
}
```

给浮动元素的父容器加上 `class="clearfix"`，就能自动修复高度塌陷问题。

**原理**：`::after` 在父元素内部的最后创建了一个透明的块级小助手，这个小助手用 `clear: both` 把自己放在了所有浮动元素的下方，从而撑起了父元素的高度。就像一个隐形的托底木板。

### 五、float 在现代 CSS 里还重要吗

坦率地说，**用来做整体页面布局的话，float 已经过时了**。现在我们有 Flexbox 和 Grid，这两把新工具比 float 好用一百倍。

但 float 仍然需要了解，原因是：
1. **文字环绕图片**这种事，还是 float 最方便、最自然
2. 你可能接手老项目，里面有很多 float 布局的历史代码
3. 面试偶尔会问
4. 理解了 float，你才能理解为什么后面会诞生 Flexbox 和 Grid

> 一句话总结：float 就像算盘——现在有了计算器（Flexbox/Grid），但你还是要认识算盘，知道它是干什么的。

## 本节小结

float 本意是文字环绕图片（像报纸排版），但会导致父元素高度塌陷。clear 和 clearfix 用来修复。现代布局优先用 Flexbox/Grid，float 主要用于文字环绕场景。
