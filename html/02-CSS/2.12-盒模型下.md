# 2.12 盒模型（下）

## 本节你会学到什么

- 标准盒模型（content-box）：`width` 只管 content
- 替代盒模型（border-box）：`width` 包含 content + padding + border
- 为什么推荐全局设置 `box-sizing: border-box`
- margin 合并现象及其规则
- 用"套内面积 vs 建筑面积"类比两种盒模型

## 正文

### 一个让人困惑的发现

上一节你知道了盒模型有四层。现在来做一个小计算：

```css
div {
  width: 200px;
  padding: 20px;
  border: 5px solid black;
}
```

这个 div 在屏幕上实际占多宽？

直觉告诉你：200px。但实际情况是：

```
实际宽度 = width + 左padding + 右padding + 左border + 右border
         = 200 + 20 + 20 + 5 + 5
         = 250px
```

这就是**标准盒模型**（content-box）——浏览器默认的行为。`width` 只设置内容区宽度，padding 和 border 是额外加上去的。

### 买房类比：套内面积 vs 建筑面积

如果你以后买过房就会秒懂：

| 盒模型 | 类比 | `width` 的含义 |
|---|---|---|
| **content-box**（标准盒模型） | **套内面积**——你家里实际能用的面积，墙体厚度不算在里面 | `width` = content 宽度 |
| **border-box**（替代盒模型） | **建筑面积**——算上了墙体和公摊，这是你真正"占"的面积 | `width` = content + padding + border |

用 content-box 设计一个宽 300px 的盒子时，你还要心算减去 padding 和 border 才能得到剩余的内容空间——就像你买房子时房地产商说"套内 80 平"，你要自己加墙厚才知道总面积。

border-box 反过来了：你说"我要一个总宽 300px 的盒子"——浏览器就把 content + padding + border 的总和算成 300，不会超。这才是大多数人心中的"宽度"。

### box-sizing 属性：切换盒模型

```css
/* 标准盒模型（浏览器默认）—— width = content */
div {
  box-sizing: content-box;
}

/* 替代盒模型（推荐）—— width = content + padding + border */
div {
  box-sizing: border-box;
}
```

一张图说明两种模型的区别：

```
      content-box (标准)              border-box (替代)
      ┌─────────────────┐            ┌─────────────────┐
      │    margin       │            │    margin       │
      │  ┌───────────┐  │            │  ┌───────────┐  │
      │  │  border   │  │            │  │  border   │  │
      │  │ ┌───────┐ │  │            │  │ ┌───────┐ │  │
      │  │ │ padding│ │  │            │  │ │ padding│ │  │
      │  │ │┌─────┐│ │  │            │  │ │┌─────┐│ │  │
      │  │ ││cont ││ │  │            │  │ ││cont ││ │  │
      │  │ │└─────┘│ │  │            │  │ │└─────┘│ │  │
      │  └───────────┘  │            │  └───────────┘  │
      └─────────────────┘            └─────────────────┘
      width = content                width = content+padding+border
      设 width=200, 实际 250        设 width=200, 实际就是 200
```

### 全局使用 border-box

几乎所有现代项目的 CSS 第一步就是：

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}
```

为什么？因为 border-box 让你不用心算。你说"这个盒子宽 300"，它就是 300。这是 CSS 社区公认的最佳实践。把它写在你的每个项目最开头，就像 C 程序第一行写 `#include <stdio.h>` 一样自然。

### margin 合并（margin collapsing）

margin 有一个诡异的特性，被叫做"margin 合并"。看这个场景：

```html
<div class="box1" style="margin-bottom: 30px;">上面</div>
<div class="box2" style="margin-top: 20px;">下面</div>
```

两个盒子之间的间距是多少？直觉告诉你：30 + 20 = 50px。但实际上只有 **30px**。

这就是 margin 合并：**垂直方向上，相邻元素的 margin 会重叠，取较大的那个值。** 就像两个人都想保持社交距离，但最终只需要保持最远的那个距离就能同时满足。

规则：
- **只发生在垂直方向**（水平方向不合并）
- **只发生在块级元素**之间
- **相邻**的 margin 会合并（包括父元素的第一个/最后一个子元素的 margin "穿透"到父元素外部）
- 最终间距取**最大值**（不是相加）

margin 合并不是什么"bug"，它是 CSS 规范有意设计的行为——目的是防止大段文本之间的间距过宽。但在实际布局中确实会让人困惑。

避免方法：
1. 改用 padding 替代 margin 做间距
2. 给容器加 `overflow: hidden`（或者 `border`、`padding`）形成"阻断"
3. 使用 Flexbox 或 Grid 布局（天然不存在 margin 合并问题）

### 父子 margin 合并

不只是兄弟元素，父元素的第一个/最后一个子元素的 margin 可能"穿透"父元素：

```html
<div style="background: #eee;">
  <p style="margin-top: 40px;">这个 p 的 margin-top 可能把整个父 div 也带下去</p>
</div>
```

你会以为 p 和 div 顶部有 40px 的距离。但实际上 div 的顶部也会被"带"下去 40px——就像孩子往上推，结果把爸爸也推出去了。

## 动手试试

1. 打开示例文件 `2.12-box-sizing.html`，对比 content-box 和 border-box 两个盒子的实际宽度
2. 按 F12 打开开发者工具，点选这两个演示盒子，在盒模型面板看它们的实际尺寸
3. 把全局 `box-sizing: border-box` 注释掉，再看布局是否会乱——理解它为什么重要
4. 观察 margin 合并演示区，修改两个盒子的 margin 值，验证"取最大值"的规则
5. 在父子 margin 合并演示区里，试试给父元素加 `border: 1px solid transparent` 看是否能阻断合并

## 本节小结

`box-sizing` 控制 `width` 的计算范围：content-box 只管内容（套内面积），border-box 管到边框（建筑面积）。永远在新项目开头写 `*, *::before, *::after { box-sizing: border-box; }`。垂直 margin 会合并取最大值，调试时别被它骗了。
