# 2.24 动画

## 本节你会学到什么

- CSS 动画和过渡的区别
- `@keyframes` 定义动画关键帧
- 动画属性：`name`、`duration`、`iteration-count`、`direction`、`timing-function`
- `animation` 简写
- 做一个呼吸灯和一个加载动画

## 正文

### 从"需要触发"到"自动运行"

上一节的 `transition` 很方便，但它有一个限制：**必须有人触发**。hover、focus、点击——总要等用户做了什么，过渡才开始。

但有时候你想让动画**自动运行**：页面一打开，加载图标就开始转圈；一个呼吸灯无限循环地亮暗交替；一段文字从左边滑入。这些不需要用户操作，自己就动。

这就是 CSS **动画（Animation）**的用武之地。

**类比**：transition 是自动门——你走近（hover），门打开。animation 是电风扇——插上电就自己转。

### 一、@keyframes —— 定义动画过程

`@keyframes` 是动画的核心。它定义了动画从开始到结束的整个过程。

```css
@keyframes 动画名 {
  0%   { /* 起始状态 */ }
  50%  { /* 中间状态 */ }
  100% { /* 结束状态 */ }
}
```

`0%` 是动画开始，`100%` 是动画结束。你可以用 `from` 和 `to` 代替 0% 和 100%：

```css
@keyframes fadeIn {
  from { opacity: 0; }       /* 从完全透明 */
  to   { opacity: 1; }       /* 到完全不透明 */
}
```

也可以在中间插任意多的关键帧（30%、75% 等），浏览器会自动计算帧与帧之间的过渡。

**类比**：`@keyframes` 就像一本翻页动画书。书的每一页画一个稍微不同的画面（关键帧），快速翻动时，画面就"动"起来了。你只需画关键的那几页，中间的过渡页浏览器自动帮你补。

### 二、把动画应用到元素上

定义好 `@keyframes` 之后，要用 `animation` 属性把它绑定到元素上：

```css
.element {
  animation-name: 动画名;            /* 用哪个 @keyframes */
  animation-duration: 2s;           /* 动画时长 */
  animation-timing-function: ease;  /* 速度曲线 */
  animation-delay: 0s;              /* 延迟开始 */
  animation-iteration-count: infinite; /* 播放次数 */
  animation-direction: normal;      /* 播放方向 */
  animation-fill-mode: forwards;    /* 结束后保持什么状态 */
}
```

### 三、animation-iteration-count —— 播放多少次

| 值 | 效果 |
|----|------|
| `1`（默认） | 播放一次 |
| `3` | 播放三次 |
| `infinite` | 无限循环 |

加载动画、呼吸灯就是用 `infinite`。

### 四、animation-direction —— 播放方向

| 值 | 效果 | 类比 |
|----|------|------|
| `normal`（默认） | 从头到尾 | 播完就回到起点 |
| `reverse` | 从尾到头 | 倒放 |
| `alternate` | 正-反-正-反交替 | 像钟摆，来回摇 |
| `alternate-reverse` | 反-正-反-正交替 | 倒着开始的钟摆 |

`alternate` 特别适合做"呼吸"效果——从小到大再回到小，丝般顺滑。

### 五、animation-fill-mode —— 动画结束后怎么样

动画结束后，元素默认会**弹回**最初的样子。如果你想让它保持在结束状态：

```css
.element {
  animation-fill-mode: forwards;  /* 停在最后一帧 */
}
```

常用值：
- `none`（默认）：动画结束后弹回原位
- `forwards`：停在最后一帧（保持最终状态）
- `backwards`：在延迟期间就应用第一帧的样式
- `both`：同时有 forwards 和 backwards

### 六、animation 简写

```css
.element {
  /* 格式：name duration timing-function delay iteration-count direction fill-mode */
  animation: pulse 2s ease-in-out infinite alternate;
}
```

日常开发中简写就够了。

### 七、实例：呼吸灯

```css
@keyframes pulse {
  0%   { opacity: 0.4; transform: scale(1); }
  50%  { opacity: 1;   transform: scale(1.05); }
  100% { opacity: 0.4; transform: scale(1); }
}

.dot {
  width: 30px;
  height: 30px;
  background: #e74c3c;
  border-radius: 50%;
  animation: pulse 2s ease-in-out infinite;
}
```

### 八、实例：加载旋转圈

```css
@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

.loader {
  width: 40px;
  height: 40px;
  border: 4px solid #ddd;
  border-top-color: #3498db;
  border-radius: 50%;
  animation: spin 0.8s linear infinite;
}
```

### 九、transition vs animation

| 维度 | transition | animation |
|------|------------|-----------|
| 触发方式 | 需要触发（hover、focus 等） | 自动运行 |
| 复杂性 | 简单，两个状态 | 任意多个关键帧 |
| 循环 | 不支持 | 支持（infinite、次数） |
| 适用场景 | 交互反馈（hover 效果） | 自动动画（加载、背景动效） |
| 往返 | 自动（移开就回去） | 需要手动设置 alternate |

## 动手试试

1. 打开 `examples/2.24-animation.html`，观察页面上的呼吸灯和加载动画。
2. 找到 `@keyframes pulse`，把 `opacity` 的最小值改成 `0.1`，刷新看看呼吸灯是不是"呼吸"得更深了。
3. 把加载圈的 `animation-duration` 从 `0.8s` 改成 `3s`，看看它转得有多慢。
4. 找到 `.slide-in`，把 `animation-iteration-count` 改成 `infinite`，看看它会不会不停滑入。
5. 自己写一个 @keyframes：让一个方块不断变换背景颜色（在红、蓝、绿之间循环）。

## 本节小结

CSS 动画用 `@keyframes` 定义关键帧过程，浏览器自动补间。transition 需要触发，animation 自动运行并支持无限循环。用 `alternate` 做呼吸效果，`infinite` 做持续动画。
