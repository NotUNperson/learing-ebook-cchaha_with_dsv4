# 1.3 head 中的元信息

## 本节你会学到什么

- `title` 标签的真正作用（不只是改个名字）
- `meta charset` 为什么必须放在最前面
- `viewport` 是什么--手机端适配的第一关
- `meta description` 怎样影响搜索结果
- 把这些元信息想象成"身份证和说明书"

---

## 正文

### 你去外国旅游，会带什么证件？

出国旅游时，你需要护照（证明你是谁）、签证（证明你能去）、机票（证明你坐哪班飞机）。没有这些，海关不放行。

一个 HTML 页面也一样。它需要把自己的"身份信息"交代清楚，搜索引擎和浏览器才会正确地处理和展示它。这些"身份信息"就写在 `<head>` 标签里面。

上一节我们说了 head 是"大脑"，这节我们打开大脑，看看里面具体装了哪些东西。

### title：浏览器标签页上的文字

```html
<title>我的第一个网页</title>
```

`<title>` 写在 head 里面，它的内容会显示在**浏览器标签页**上。

title 的作用不止于此：
- **收藏夹名称**：用户收藏你的页面时，title 就是默认的书签名。
- **搜索结果标题**：在 Google/Bing 搜索时，你的页面 title 就是搜索结果里那个蓝色大标题。
- **分享时的标题**：把链接发到微信/QQ 时，预览卡片上的标题通常就是 title。

所以 title 不能随便写。一个团购网站首页，title 不能只是"首页"，而应该是"XX 团购-美食电影酒店，超低价"。

### meta charset：不乱码的根本保证

```html
<meta charset="UTF-8">
```

这一行在 1.2 节见过。它的地位特殊：**必须写在 head 的最前面**（紧跟 `<head>` 之后），因为浏览器必须先知道用什么编码，才能正确读取后面的所有内容。

UTF-8 是万国码，支持中文、英文、日文、阿拉伯文……几乎所有文字。**永远用 UTF-8，永远第一时间声明。**

### viewport：手机端的第一道防线

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0">
```

这行代码是专门为手机浏览器准备的。没有它，你的页面在手机上会变得极小，用户得不断放大才能看清字。

拆开来解释：
- `width=device-width`：把页面宽度设为手机屏幕的实际宽度（而不是模拟成一个宽大的 PC 屏幕）。
- `initial-scale=1.0`：初始缩放比例是 1 倍（不放大也不缩小）。

有了它，页面在手机上就能正常显示了。没有它，你会发现一个 12px 的字在手机上像蚂蚁一样小。

### description：搜索引擎看到的"页面简介"

```html
<meta name="description" content="这是一个帮助零基础新手学习 HTML 的教程网站。">
```

这个标签不会显示在页面上，但搜索引擎会把它读到。在搜索结果中，title 下方那两行灰色小字通常就是 description。

好的 description 像一个电梯广告：
- 在大约 150 个字符以内。
- 准确概括页面内容。
- 包含用户可能会搜索的关键词。

### keywords：曾经显赫，如今过气

```html
<meta name="keywords" content="HTML, 教程, 前端, 编程">
```

早年 SEO（搜索引擎优化）盛行时，keywords 标签是兵家必争之地，很多网站往里面堆几百个关键词骗排名。后来搜索引擎学聪明了，**Google 和百度基本上不再看 keywords 标签**。现在你可以写，也可以不写--写了无害，但不加分。

### 其他常见 meta 标签

```html
<!-- 告诉 IE 浏览器用最高版本的渲染引擎 -->
<meta http-equiv="X-UA-Compatible" content="IE=edge">

<!-- 页面作者 -->
<meta name="author" content="小明">

<!-- Open Graph 协议：分享到社交平台时显示的标题、描述、图片 -->
<meta property="og:title" content="HTML 入门教程">
<meta property="og:description" content="从零开始学 HTML">
<meta property="og:image" content="https://example.com/cover.jpg">
```

Open Graph（og:）标签是 Facebook 发明的，但现在几乎所有社交平台（微信、QQ、微博、Twitter）都会读取它。分享链接时出现的那个"带图、带标题、带简介"的卡片，就是用 Open Graph 标签控制的。

## 动手试试

1. 打开你 1.2 节写的 `my-first.html`，在 head 里加上 description 和 viewport 两个 meta 标签。
2. 把 `<title>` 改成一段有意义的话（比如"小明的 HTML 学习笔记"），看看浏览器标签页的变化。
3. 用手机（或者 Chrome 开发者工具的手机模拟器）分别打开带 viewport 和不带 viewport 的页面，感受差异。

---

## 本节小结

head 里的元信息是给浏览器和搜索引擎看的"说明书"，不是你写给自己看的笔记--它们直接影响搜索排名、手机显示、社交分享效果。
