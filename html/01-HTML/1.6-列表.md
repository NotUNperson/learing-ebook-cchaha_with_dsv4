# 1.6 列表

## 本节你会学到什么

- 无序列表（ul）-- 像购物清单，顺序不重要
- 有序列表（ol）-- 像操作步骤，顺序很重要
- 描述列表（dl/dt/dd）-- 像词典的词条加释义
- 嵌套列表 -- 列表中套列表，像文章大纲

---

## 正文

### 你天天都在用列表

想想你日常生活中遇到的"列表"：

- 去超市买菜：鸡蛋、牛奶、面包、苹果（顺序不重要，放进购物车就行）
- 泡方便面：撕开包装 -> 放面饼 -> 倒开水 -> 等三分钟 -> 吃（顺序绝对不能错！）
- 查字典："算法"这个词的意思是"解决特定问题的有限步骤"（一个词条对应一个释义）

这三种情况恰好对应了 HTML 的三种列表。

### 无序列表 ul + li

```html
<ul>
  <li>鸡蛋</li>
  <li>牛奶</li>
  <li>面包</li>
  <li>苹果</li>
</ul>
```

- `ul` = unordered list（无序列表）。浏览器会给每项前面加一个小圆点（bullet point）。
- `li` = list item（列表项）。每一项都要包在 li 里。

ul 表示"这些都是并列的东西，顺序不重要"。就像你的购物清单。

### 有序列表 ol + li

```html
<ol>
  <li>撕开包装</li>
  <li>放入面饼和调料</li>
  <li>倒入沸水至刻度线</li>
  <li>盖上盖子等 3 分钟</li>
  <li>搅拌均匀即可食用</li>
</ol>
```

- `ol` = ordered list（有序列表）。浏览器会自动给每项前面加上数字序号（1, 2, 3...）。
- `li` 依然是每一项。

ol 表示"这些步骤有先后顺序，不能搞乱"。就像泡面步骤。

> 注意：ul 和 ol 的**直接子元素必须是 li**，不要往 ul/ol 里面直接塞文字或其他标签。li 里面倒是可以放任何东西。

### 描述列表 dl + dt + dd

```html
<dl>
  <dt>HTML</dt>
  <dd>超文本标记语言，用来描述网页结构。</dd>

  <dt>CSS</dt>
  <dd>层叠样式表，用来控制网页的外观和布局。</dd>

  <dt>JavaScript</dt>
  <dd>一种脚本语言，用来给网页添加交互行为。</dd>
</dl>
```

- `dl` = description list（描述列表）。
- `dt` = description term（被描述的词条）。
- `dd` = description definition（对该词条的解释）。

就像一个迷你词典。一个 dt 可以有多个 dd（一个词可能有多个释义）。

### 嵌套列表

列表里可以再放列表，制造层级结构，就像文章的大纲：

```html
<ul>
  <li>第一章：HTML 基础
    <ul>
      <li>1.1 HTML 是什么</li>
      <li>1.2 文档骨架</li>
      <li>1.3 head 元信息</li>
    </ul>
  </li>
  <li>第二章：CSS 入门
    <ul>
      <li>2.1 选择器</li>
      <li>2.2 颜色与字体</li>
    </ul>
  </li>
</ul>
```

注意嵌套时的缩进，能让结构一目了然。内层的 ul 是包在**外层 li 里面**的，不是在 ul 里面直接放 ul。

## 动手试试

用三种列表写一个页面：
1. 用 ul 列出你喜欢的三种食物。
2. 用 ol 列出你从起床到出门的步骤。
3. 用 dl 列出你学过的三个编程术语及其解释。
4. 用一个嵌套 ul，列出课程大纲（至少两层）。

---

## 本节小结

ul 无序（圆点）、ol 有序（数字）、dl 词条释义。选择哪种列表，取决于内容之间的关系--是并列？有序？还是定义？
