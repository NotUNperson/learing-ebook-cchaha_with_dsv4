## 正文

### 前面学了这么多，该上战场了

17 节下来，你已经学了很多 HTML 标签。但如果每个标签只是单独用过，你可能会觉得它们"各是各"的。一个真正的网页，是所有这些标签**组合在一起**工作的。

这节我们来做一个完整的新闻/博客文章页面。不写新概念，就是把前面学的串起来。

### 需求描述

做一个"科技新闻"页面，内容可以虚构，但结构要完整。要求用到以下元素：

**结构层面（1.15 节）：**
- header、nav、main、article、aside、footer

**内容层面：**
- 标题 h1-h3（1.4 节）
- 段落 p、br、hr（1.4 节）
- strong / em / mark / small（1.5 节）
- 无序列表 ul（1.6 节）
- 链接 a，含锚点（1.7 节）
- 图片 + figure/figcaption（1.8 节）
- 一个表格（1.9-1.10 节），例如"各大浏览器份额对比"或"排行榜"

## 动手试试

你的任务：**仿照上面的页面，做一个你自己的新闻/博客页面。** 主题自选，可以是：

- 你喜欢的游戏攻略
- 一部电影的评论
- 一个技术话题（如"C 语言和 JavaScript 的区别"）
- 你的学习周记

### 检查清单

做完后，对照以下清单逐项检查：

- [ ] 第一行有 `<!DOCTYPE html>`
- [ ] `<html lang="zh-CN">` 写了
- [ ] `<meta charset="UTF-8">` 在 head 里
- [ ] `<title>` 写了一句话（不是"无标题文档"）
- [ ] 用了至少一个语义化结构标签（header / nav / main / article / section / aside / footer）
- [ ] 有 h1 到 h3 至少三层标题
- [ ] 有 p 段落
- [ ] 至少用了 strong 或 em 或 mark 或 small 中的两个
- [ ] 有一个 ul 或 ol 列表
- [ ] 有一个 a 链接（可以先用 # 占位）
- [ ] 有一个 img，且写了 alt 属性
- [ ] 有一个表格
- [ ] 页面能正常在浏览器中打开，没有乱码

全部打勾，恭喜你--HTML 部分你已经通关了！
