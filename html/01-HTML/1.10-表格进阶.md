# 1.10 表格进阶

## 本节你会学到什么

- thead、tbody、tfoot：表格的逻辑分区，像表头/表体/表尾
- colspan：跨列合并，像横向推开相邻格子
- rowspan：跨行合并，像纵向推开上下格子
- 真实场景：发票、账单、复杂课表的制作

---

## 正文

### 基础表格够用吗？

上一节我们做的课程表只有 4 行 4 列，结构简单。但真实世界里的表格往往复杂得多：

- 账单表格有一个"合计"行在最后，需要和上面的数据区隔开。
- 课程表里，"午休"可能横跨所有列。
- 简历里的教育经历，时间段那一格可能占据好几行。

这些需求就要靠"分区"和"合并单元格"来解决。

### thead、tbody、tfoot：给表格分区

把表格理解成一个报告文件：

- **thead（表头）**：表格最上面的标题行，像文件抬头。"下面这张表是干什么的"。
- **tbody（表体）**：表格的主体数据区域，像文件正文。
- **tfoot（表尾）**：表格最下面的汇总行，如"合计"。像文件的结语。

```html
<table>
  <thead>
    <tr>
      <th>商品</th>
      <th>单价</th>
      <th>数量</th>
      <th>小计</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>键盘</td>
      <td>299</td>
      <td>1</td>
      <td>299</td>
    </tr>
    <tr>
      <td>鼠标</td>
      <td>149</td>
      <td>2</td>
      <td>298</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td colspan="3">合计</td>
      <td>597</td>
    </tr>
  </tfoot>
</table>
```

分区的好处：
1. **语义清晰**：看代码就知道哪是头、哪是身体、哪是尾巴。
2. **打印友好**：如果表格要打印多页，浏览器可以在每页顶部自动重复 thead。
3. **CSS 控制方便**：可以单独给 thead/tbody/tfoot 设置不同样式。
4. **无障碍**：屏幕阅读器能区分结构层次。

> 注意：HTML 里写的时候 thead 和 tfoot 在 tbody 前面，但浏览器显示时 tfoot 会自动放到表格最下面。

### colspan：横着合并

colspan 是 column span（跨列）的缩写。它让一个格子**横跨多列**，像推土机一样把右边的格子推开。

```html
<table>
  <tr>
    <td colspan="2">这个格子横跨两列</td>
    <td>普通格</td>
  </tr>
  <tr>
    <td>格1</td>
    <td>格2</td>
    <td>格3</td>
  </tr>
</table>
```

第一行本来该有 3 个 td，但第一个 td 的 `colspan="2"` 让它占了两个位置，所以这一行只需要写 2 个 td（一个跨两列的 + 一个普通的）。

课程表里的"午休"行：

```html
<tr>
  <td colspan="5">午休（12:00 - 14:00）</td>
</tr>
```

一行只有一个格子，横跨 5 列，表示"午休"贯穿整个上午到下午的间隔。

### rowspan：竖着合并

rowspan 是 row span（跨行）的缩写。它让一个格子**纵跨多行**，把下面的格子推开。

```html
<table>
  <tr>
    <td rowspan="2">这个格子<br>跨两行</td>
    <td>格2</td>
  </tr>
  <tr>
    <!-- 第一格被上面跨行的占了，这里只写一个 -->
    <td>格3</td>
  </tr>
</table>
```

第二行的第一个位置被上一行的 rowspan 给占了，所以第二行只写一个 td。

经典例子——课表里"数据结构"连上两节：

```html
<tr>
  <td rowspan="2">数据结构</td>
  <td>8:00-8:45</td>
  <td>理论课</td>
</tr>
<tr>
  <!-- 第一格被上一行占了，跳过 -->
  <td>8:55-9:40</td>
  <td>习题课</td>
</tr>
```

### colspan 和 rowspan 一起上

复杂表格里经常两种合并同时出现。比如一个账单：

```html
<table>
  <tr>
    <th>类别</th>
    <th>项目</th>
    <th>金额</th>
  </tr>
  <tr>
    <td rowspan="2">餐饮</td>
    <td>午餐</td>
    <td>25</td>
  </tr>
  <tr>
    <!-- "餐饮"格跨两行，这一行的第一格被占了 -->
    <td>晚餐</td>
    <td>40</td>
  </tr>
  <tr>
    <td colspan="2">餐饮小计</td>
    <td>65</td>
  </tr>
</table>
```

"餐饮"用 rowspan 跨了两行，"餐饮小计"用 colspan 跨了两列。

写复杂表格的一个建议：**先用纸画出表格的样子，再用 HTML 实现。** 尤其是合并单元格的时候，画出来就不容易算错每行该写几个 td。

## 本节小结

thead/tbody/tfoot 给表格分区，colspan 横着合并，rowspan 竖着合并。画表格前先打草稿。
