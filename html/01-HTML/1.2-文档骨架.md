# 1.2 文档骨架 -- 最小可运行的 HTML 页面

## 正文

### 盖房子需要框架

上一节我们把 HTML 比作"砖块"。这节我们来搭**房子的框架**。

任何一栋房子，不管装修成什么样，都得有地基、承重墙、屋顶。HTML 页面也一样，不管页面内容是什么，骨架结构是固定的。这个骨架长这样：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>页面标题</title>
</head>
<body>
  <!-- 这里写页面内容 -->
</body>
</html>
```

这就是"最小可运行的 HTML 页面"。虽然还没填内容，但它已经是一个合法的 HTML 文件了--浏览器看到它就知道："哦，这是一个网页。"

下面逐行拆解。

### DOCTYPE：第一行"暗号"

```html
<!DOCTYPE html>
```

`<!DOCTYPE html>` 必须写在文件的第一行（前面不能有空行）。

它的作用是**告诉浏览器用"标准模式"来渲染页面**。

为什么要专门说一声？因为在很久很久以前（IE6 时代），浏览器厂商各自为战，同样的 HTML 在不同浏览器里显示效果天差地别。后来大家坐下来定了一套统一标准，但为了不把老网站搞坏，浏览器保留了两种渲染模式：

- **怪异模式**：模拟老浏览器的行为，兼容老网站。
- **标准模式**：按最新的 W3C 规范渲染，所有现代浏览器效果一致。

`<!DOCTYPE html>` 就是告诉浏览器："用标准模式，别用怪异模式。"类似去餐馆点菜时先说一声"按菜单上的做，不要微辣"。

### html 标签：整个页面的"大老板"

```html
<html lang="zh-CN">
  ...
</html>
```

`<html>` 是所有元素的根元素，它包住整个页面的一切。里面只能有两个直属子元素：`<head>` 和 `<body>`，就像一个公司只有两个部门。

`lang="zh-CN"` 告诉浏览器和搜索引擎："这个页面的主要内容是简体中文。"这能帮助：
- 搜索引擎正确识别语言，给中国用户优先展示。
- 屏幕阅读器（盲人用的读屏软件）用正确的中文发音朗读。

### head 和 body：大脑和身体

HTML 页面就像一个"人"：

- **head = 大脑**：看不见，但很重要。它负责思考、规划，告诉浏览器关于这个页面的各种信息（标题、编码、关键词等）。
- **body = 身体**：看得见、摸得着。你在浏览器里看到的所有文字、图片、按钮，都在 body 里面。

```html
<head>
  <!-- 大脑中的信息：页面标题、字符编码、给搜索引擎看的描述…… -->
  <meta charset="UTF-8">
  <title>这是一个页面</title>
</head>

<body>
  <!-- 身体里的一切：用户能看到的内容 -->
  <h1>欢迎</h1>
  <p>你好！</p>
</body>
```

head 里面的东西**不会直接显示在页面上**（除了 title，它会出现在浏览器标签页上），而 body 里面的东西**全部会显示**。

### 字符编码 meta charset

```html
<meta charset="UTF-8">
```

这一行放在 head 的最前面。它告诉浏览器："这个文件是用 UTF-8 编码的。"

如果少了这一行，浏览器可能会用错误的编码去读你的文件，结果就是**中文全变成乱码**。UTF-8 是当今互联网的标准编码，几乎支持所有语言的字符。记住：**每个 HTML 文件，第一件事就是写 `<!DOCTYPE html>`，第二件事就是 `<meta charset="UTF-8">`。**

### 一个完整的"最小页面"

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>我的页面</title>
</head>
<body>
  <p>这是一个能跑的最小 HTML 页面。</p>
</body>
</html>
```

把它保存为 `.html` 文件，双击用浏览器打开，就能看到页面标题和那行文字。恭喜，你已经会"做网页"了--虽然只是万里长征第一步。

## 动手试试

1. 把上面的"最小页面"代码手打一遍（不要复制粘贴），保存为 `my-first.html`，用浏览器打开。
2. 尝试把 `lang="zh-CN"` 改成 `lang="en"`，看看页面显示有什么变化？（答案：视觉上没有变化，但浏览器和搜索引擎会认为页面是英文的。）
3. 故意删掉 `<meta charset="UTF-8">` 这一行，然后在 body 里写几个中文，保存，用浏览器打开--看看是不是乱码了。

---

## 本节小结

每个 HTML 文件的骨架都一样：DOCTYPE 声明 + html 包裹 head（大脑）和 body（身体）。这是所有网页的地基。
