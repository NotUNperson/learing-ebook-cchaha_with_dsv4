# 1.15 语义化结构标签

## 本节你会学到什么

- 语义化结构标签：header、nav、main、article、section、aside、footer
- div 和 span 是什么，以及为什么只用它们作"最后的选择"
- 语义化的三大好处：SEO、无障碍、代码可读性
- 用一篇博客文章演示整个页面布局

## 正文

### 你的页面在搜索引擎眼里是一团什么？

想象你是 Google 的搜索爬虫。你要看遍全世界的网页，把它们分类、排序、索引。你来到一个页面，看到的全是 `<div>`：

```html
<div>
  <div>
    <div>网站标题</div>
    <div>导航</div>
  </div>
  <div>
    <div>文章标题</div>
    <div>文章内容...</div>
    <div>广告</div>
  </div>
  <div>版权信息</div>
</div>
```

你完全不知道哪个是标题、哪个是导航、哪个是正文。全是 div，看不出来任何区别。

但如果作者用了语义标签：

```html
<header>网站标题</header>
<nav>导航</nav>
<main>
  <article>
    <h1>文章标题</h1>
    <p>文章内容...</p>
  </article>
  <aside>广告/侧边栏</aside>
</main>
<footer>版权信息</footer>
```

你一眼就知道：哦，这是头部，那是正文，那是侧边栏。搜索引擎也是这么"看"的。

### 语义化的三赢

**1. SEO（搜索引擎优化）**：搜索引擎能理解你的页面结构。在 `<main>` 里的内容权重更高，在 `<aside>` 里的内容权重较低。对百度/Google 友好，排名更容易靠前。

**2. 无障碍（Accessibility）**：盲人用屏幕阅读器"听"网页时，可以一键跳到 `<nav>`（导航区）、`<main>`（正文区）、`<footer>`（页脚区）。如果全是 div，只能一个字一个字地听。

**3. 代码可读性**：三个月后回头看你的代码，是看到一堆 `<div>` 更容易理解，还是看到 `<header>` `<nav>` `<article>` 更容易理解？

### 常用语义标签速览

```html
<!-- 页面头部：通常包含网站名、Logo、搜索框 -->
<header>
  <h1>我的博客</h1>
  <p>一个分享技术心得的地方</p>
</header>

<!-- 导航区：网站的主要链接，如"首页""关于""联系" -->
<nav>
  <a href="/">首页</a>
  <a href="/about">关于</a>
  <a href="/contact">联系</a>
</nav>

<!-- 主体内容区：一个页面只有一个 main -->
<main>
  <!-- 独立的内容单元：一篇文章、一篇博客 -->
  <article>
    <h2>文章标题</h2>
    <p>文章正文……</p>
  </article>

  <!-- 章节/区域：用于把内容分组 -->
  <section>
    <h3>评论区</h3>
    <p>这里放评论……</p>
  </section>
</main>

<!-- 侧边栏：相关但非核心的内容 -->
<aside>
  <h3>关于作者</h3>
  <p>作者简介……</p>
</aside>

<!-- 页脚：版权信息、联系方式、备案号等 -->
<footer>
  <p>&copy; 2026 我的博客. All rights reserved.</p>
</footer>
```

### 各标签的精确含义

| 标签 | 含义 | 注意 |
|------|------|------|
| `<header>` | 页面头部或文章头部 | 可以在页面级用，也可以在 article 里用 |
| `<nav>` | 主要导航链接集合 | 不是所有链接群都叫 nav，只放主导航 |
| `<main>` | 页面唯一的主体内容 | 一个页面**只能有一个** main |
| `<article>` | 独立、完整的内容单元 | 像一篇文章、一篇论坛帖子——可以独立分发的 |
| `<section>` | 内容分区 | 通常带一个标题。如果只是为了加样式，用 div |
| `<aside>` | 侧边栏/附加内容 | 和主体相关但不核心，去掉也不影响理解 |
| `<footer>` | 页面底部或文章底部 | 脚注、版权、联系方式、相关链接 |

### div 和 span：无语义的盒子

- `<div>`（division）是**块级**无语义容器。独占一行，用来分组。它是最后的选择——当你找不到合适的语义标签时才用。
- `<span>` 是**行内**无语义容器。不独占一行，用来包裹句子中的几个字。

```html
<div class="container">
  <p>这是一段文字，其中 <span class="highlight">这些字</span> 需要高亮。</p>
</div>
```

规则：**先想有没有语义标签，没有再选 div/span。**

## 动手试试

做一个简单的"个人博客首页"，用语义标签写结构：
- header：博客名称 + 一句话简介
- nav：首页 / 文章 / 关于我（写三个链接，# 占位）
- main + article：一篇文章（标题 + 段落）
- aside：作者简介（一张头像虚构的图 + 两句话）
- footer：版权信息
不用 CSS 美化，把结构搭对就行。做完后，把所有语义标签换成 div，比较一下，哪个更容易理解？

---

## 本节小结

语义标签让你的页面不再是"一堆 div"，而是有头有尾、有主有次的结构化文档。对搜索引擎、视障用户、未来的你自己，都是善举。
