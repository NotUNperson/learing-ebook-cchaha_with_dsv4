# 1.12 表单控件（上）—— 文本类输入

## 本节你会学到什么

- 文本类输入控件的多种 type：text / password / email / number / tel / url / search
- textarea：多行文本输入框
- placeholder：占位提示文字
- value / readonly / disabled 的区别
- 各类型在手机上的不同键盘适配

---

## 正文

### 填写表单像去办事——不同事项有不同格式要求

去银行办事时，不同栏目对填写格式有不同的要求：

- "姓名"栏：普通文字就行。
- "密码"栏：填写时显示为 `***`，防止旁边人偷看。
- "手机号"栏：只能填数字，在手机上会自动弹出数字键盘。
- "邮箱"栏：如果没填 `@`，银行系统会提醒你"格式不对"。

HTML 的 input 标签提供了不同的 type，来对应这些不同场景。选对 type，不仅能做基础校验，还能优化用户的输入体验。

### 文本类 input 类型全家福

```html
<!-- 普通文本 -->
<input type="text" name="username" placeholder="请输入用户名">

<!-- 密码（输入时显示为圆点） -->
<input type="password" name="pwd" placeholder="请输入密码">

<!-- 邮箱（手机端会弹出带 @ 的键盘） -->
<input type="email" name="email" placeholder="name@example.com">

<!-- 数字（手机端弹出数字键盘） -->
<input type="number" name="age" min="0" max="150" placeholder="请输入年龄">

<!-- 电话号码（手机端弹出电话键盘） -->
<input type="tel" name="phone" placeholder="请输入手机号">

<!-- 网址（手机端弹出带 .com 的键盘） -->
<input type="url" name="website" placeholder="https://example.com">

<!-- 搜索框（有些浏览器会在框内显示清除按钮） -->
<input type="search" name="q" placeholder="搜索...">
```

每一个 type 就像是告诉浏览器："嘿，这个输入框是专门填某某内容的，请在手机上弹对应的键盘，顺便帮我校验一下格式。"

### 手机上键盘的长相不一样

这是一个容易被忽略但非常重要的细节。在手机上：

- `type="text"`：弹出普通全键盘。
- `type="email"`：弹出带 `@` 和 `.com` 快捷键的键盘。
- `type="number"`：弹出纯数字键盘（有些手机还带小数点）。
- `type="tel"`：弹出电话拨号盘风格的键盘（纯数字，比 number 键盘更大）。
- `type="url"`：弹出带 `/` 和 `.com` 快捷键的键盘。
- `type="search"`：键盘右下角的"回车"变成"搜索"。

如果你不用正确的 type，用户填手机号时还得在全键盘里找数字，体验非常糟糕。选对 type，就是对用户的体贴。

### textarea：多行文本

```html
<textarea name="comment" rows="5" cols="40" placeholder="写下你的评论..."></textarea>
```

- `textarea` 不是空标签，它有开始标签和结束标签。内容写在中间（不是 value 属性里）。
- `rows` 和 `cols`：控制可见行数和列数（可用 CSS 覆盖，只是一个初始大小）。
- 适合评论、留言、简介等需要多行文字的场景。

### placeholder：占位提示

```html
<input type="text" placeholder="请输入您的姓名">
```

灰色提示文字，用户一开始输入就消失。适合给用户一个示例或提示。但**不能代替 label**：
- placeholder 消失后用户就看不到提示了。
- 屏幕阅读器可能忽略 placeholder。
- 年老用户可能以为输入框里已经有内容了。

### value、readonly、disabled

```html
<!-- value：设置默认值 -->
<input type="text" name="city" value="北京">

<!-- readonly：只读，能看到但改不了（数据会提交） -->
<input type="text" name="id" value="20240001" readonly>

<!-- disabled：禁用，灰色，不能改也不能提交 -->
<input type="text" name="code" value="已失效" disabled>
```

区别总结：

| 属性 | 能否编辑 | 能否选中复制 | 表单提交时是否传递 |
|------|---------|------------|-----------------|
| 普通 | 能 | 能 | 传递 |
| readonly | 否 | 能 | 传递 |
| disabled | 否 | 否 | 不传递 |

### name 属性再强调

如果 input 没有 name，表单提交时这个 input 的值会被忽略。**想让服务器收到这个值，必须写 name。**

## 动手试试

做一个"个人信息登记表"，包含：
- 姓名（text）
- 密码（password）
- 邮箱（email）
- 年龄（number）
- 手机（tel）
- 个人简介（textarea，5行）
- 所有字段都要有 label 和 placeholder
- 用一个 readonly 的 input 显示"用户编号"
- 在手机上打开这个页面（或用 Chrome 开发者工具的移动模拟器），点击各个输入框，观察不同 type 弹出的不同键盘

---

## 本节小结

text 类 input 有 7 种 type，不同的 type 不仅格式校验不同，手机键盘也不同。textarea 是多行文本。placeholder 不能替代 label。readonly 看得见改不了，disabled 灰色且不提交。

## 下一节预告

文本输入会了，可表单不只是打字。单选、多选、下拉框——让用户"选"而不是"敲"。
