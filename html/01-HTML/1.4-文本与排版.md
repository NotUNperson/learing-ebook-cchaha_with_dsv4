# 1.4 文本与排版

## 本节你会学到什么

- h1 到 h6：六级标题，像书本的章/节/小节
- p 标签：段落的基本单位
- br 标签：强制换行（像打字机的回车键）
- hr 标签：水平分割线
- 浏览器默认样式的概念

---

## 正文

### 写文章，要有章法

打开任何一本书，你会看到什么？书名（最大）、第一章标题（次大）、第一节标题（再次）、正文段落（正常大小）。这种"层级结构"让读者一眼就知道哪是重点、哪是细节。

网页也一样。你不能把所有字都写成一样大，那就像一篇没有标题、没有分段、密密麻麻的文章，没人愿意读。

HTML 提供了一套"文本排版"标签，帮助你把文字组织成有层次的内容。

### h1 到 h6：六级标题

```html
<h1>这是一级标题（最大）</h1>
<h2>这是二级标题</h2>
<h3>这是三级标题</h3>
<h4>这是四级标题</h4>
<h5>这是五级标题</h5>
<h6>这是六级标题（最小）</h6>
```

h 是 heading（标题）的缩写。数字越小，层级越高，字号越大。

对比一下书本的层级：
- h1 = 书名（一个页面最好只有一个 h1）
- h2 = 章标题
- h3 = 节标题
- h4 = 小节标题
- h5 / h6 = 更细碎的标题，用得较少

重点来了：**标题标签不是用来控制字大小的，而是用来表达文档结构的。** 你可以用 CSS 把 h2 变成 8px，把 p 变成 48px，但搜索引擎和屏幕阅读器仍然认为 h2 是"二级标题"。语义 > 外观。

### p 标签：段落

```html
<p>这是一个段落。段落和段落之间会有间距，浏览器自动帮你留白。</p>
<p>这是第二个段落。注意看，两个段落之间有明显的空白间隔。</p>
```

p 是 paragraph（段落）的缩写。一个 p 就是一个自然段。

在 HTML 代码里，你按 Enter 换行，浏览器会**忽略**你的换行。也就是说：

```html
<p>这是第一行。
这其实是第二行，
但浏览器会把它们
当成同一行显示。</p>
```

浏览器只认标签，不认你的源代码换行。

### br 标签：强制换行

如果你确实想在段落里换行（比如写一首诗），可以用 `<br>`：

```html
<p>
  床前明月光，<br>
  疑是地上霜。<br>
  举头望明月，<br>
  低头思故乡。
</p>
```

br 是 break（断行）的缩写。它是空标签，没有结束标签。把它理解为打字机上的**回车键**（换行，但不另起段落）。

注意：不要连续用多个 `<br><br><br>` 来制造空行。排版的事情留给 CSS 做，HTML 只管结构。

### hr 标签：水平分割线

```html
<p>上面是一个话题。</p>
<hr>
<p>下面切换到另一个话题了。</p>
```

hr 是 horizontal rule（水平分割线）的缩写。它在页面上画一条横线，表示话题的转换。也是空标签。

## 动手试试

用本节学到的标签，写一篇"我的自我介绍"短页面，要求包含：
- 一个 h1 标题（"关于我"）
- 两个 h2 小标题（比如"基本信息"和"爱好"）
- 至少三个 p 段落
- 至少一处 br 换行
- 一条 hr 分割线

不用追求好看，把结构搭对就行。

---

## 本节小结

标题用 h 系列、段落用 p、换行用 br、分割用 hr。这四个标签是所有网页文本排版的基本功。
