# 1.8 图片

## 本节你会学到什么

- img 标签：src（图片在哪）和 alt（替代文字）这两个核心属性
- alt 为什么必须写 -- 对视障用户和图片挂掉时至关重要
- width 和 height：控制图片尺寸
- 常用图片格式：JPEG / PNG / SVG / WebP 各自适合什么场景
- figure 和 figcaption：给图片配上说明文字

---

## 正文

### 一图胜千言，但一失足成千古恨

文字能让读者理解内容，但加了图才能真正抓住眼球。不过图片有个特点：**它最容易出问题。** 图挂了、加载慢、盲人看不到……如果处理不好，体验会非常糟糕。

HTML 提供了一套机制来处理这些问题。

### img 标签：图片的基本写法

```html
<img src="photo.jpg" alt="一张风景照片">
```

img 是空标签（没有结束标签）。两个核心属性：

- `src`（source）：图片文件的路径。可以用相对路径或绝对路径（规则和上一节链接的路径完全一样）。
- `alt`（alternative text）：替代文字。图片无法显示时，这段文字会代替它出现。视障用户用屏幕阅读器"听"网页时，读的就是 alt。

### alt 不是"随便写写"

很多新手觉得 alt 可有可无，写个 `alt="图片"` 敷衍了事。但 alt 非常重要：

- **视障用户**：屏幕阅读器会读出 alt 的内容，让他们知道这里有一张什么图。
- **图片加载失败**：网速慢、图片文件丢失、格式不支持时，alt 文字会显示在图片本该在的位置。
- **搜索引擎**：搜索引擎看不懂图片里有什么，但能读懂 alt 文字。写好了有助于图片被搜到。

写 alt 的原则：**如果闭上眼，别人把 alt 文字读给你听，你能不能知道这张图大概在表达什么？**

```html
<!-- 糟糕的 alt（等于没写） -->
<img src="chart.png" alt="图片">

<!-- 糟糕的 alt（冗余） -->
<img src="logo.png" alt="公司 Logo，这是一个 Logo">

<!-- 好的 alt（准确描述） -->
<img src="chart.png" alt="2025 年各季度销售额柱状图，Q4 最高达到 120 万">

<!-- 纯装饰性图片（比如背景花纹）可以空 alt -->
<img src="decoration.png" alt="">
```

### width 和 height：控制尺寸

```html
<img src="photo.jpg" alt="风景照片" width="400" height="300">
```

直接写数字就行，单位默认是像素（px）。不写的话，图片会按原始尺寸显示。

建议：在 HTML 里设置 width 和 height，可以让浏览器在图片还没加载完时就"预留好位置"，避免图片加载时页面疯狂跳动。

### 常用图片格式

| 格式 | 适合场景 | 特点 |
|------|---------|------|
| JPEG (.jpg) | 照片、写实图片 | 压缩率高，文件小，但不支持透明背景 |
| PNG (.png) | Logo、图标、需要透明背景的图 | 支持透明，无损压缩，但文件较大 |
| SVG (.svg) | 图标、简单插图 | 矢量格式，无限放大不失真，代码可以修改 |
| WebP (.webp) | 照片和图标都行 | Google 出品，压缩率比 JPEG 更高，同时支持透明，新一代推荐格式 |
| GIF (.gif) | 简单动图 | 仅 256 色，画质差，适合短动画 |

简单记忆：**照片用 JPEG，透明图用 PNG，图标用 SVG，追求体积小用 WebP。**

### figure 和 figcaption：图片 + 说明，天生一对

单独一张图意义有时不够明确，需要配一段说明文字：

```html
<figure>
  <img src="sales-chart.png" alt="2025 年四个季度销售额">
  <figcaption>图 1：2025 年各季度销售额对比，Q4 超预期增长 30%</figcaption>
</figure>
```

- `figure` 是"一个带说明的独立内容单元"。它和文章内容有关联，但去掉它也不影响文章主体的完整性。
- `figcaption` 是 figure 的说明文字，放在 figure 里面。

figure 是块级元素（独占一行），它不只是给图片用的，也可以给代码块、表格、引用等任何"带说明的独立内容"使用。

### 路径知识点回顾

图片的 src 和链接的 href 用的是同一套路径规则：

```html
<!-- 绝对路径 -->
<img src="https://example.com/images/photo.jpg" alt="照片">

<!-- 相对路径——同目录 -->
<img src="photo.jpg" alt="照片">

<!-- 相对路径——子目录 -->
<img src="images/photo.jpg" alt="照片">

<!-- 相对路径——上级目录 -->
<img src="../shared/logo.png" alt="Logo">
```

和链接一样，相对路径搬家时不用改，绝对路径搬家需要全改。

## 动手试试

1. 找一个你喜欢的图片，放在和 HTML 文件同一个目录下，用 img 标签显示它。
2. 故意把 src 写成一个不存在的文件名，看看浏览器怎么显示 alt 文字。
3. 用 figure + figcaption 包住图片，写一句说明。
4. 给图片设置 width="200"，观察效果。然后再试试不设 height（高度会自动按比例缩放）。

---

## 本节小结

img 是图片标签，src 说地址，alt 说替代文字。永远写好 alt，永远考虑"如果图挂了怎么办"。figure 帮你把图片和说明打包在一起。
