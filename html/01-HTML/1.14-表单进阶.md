# 1.14 表单进阶——HTML5 内置校验

## 本节你会学到什么

- required：必填字段，不填就过不去
- min / max：数字输入的范围限制
- minlength / maxlength：文本输入的长度限制
- pattern：正则表达式模式匹配（最强大的校验）
- 这些校验是"第一道防线"，纯 HTML 实现，不需要 JavaScript
- input type="submit" 触发表单提交流程

---

## 正文

### 表单校验就像门卫

你把表单交到服务器，就像把快递送到收货人手里。但如果在快递发出之前，门口就有一个门卫帮你检查一下--"单子填全了没？""手机号是不是少了一位？"--那就能减少很多麻烦。

浏览器内置了这套"门卫"系统，它就是 **HTML5 表单校验**。你只需要在标签上加几个属性，浏览器就会自动检查用户的输入。

前提：把提交按钮写成 `<input type="submit">` 或 `<button type="submit">`，并且按钮在 form 里面。点击提交按钮时，浏览器触发校验。

### required：这格必须填

```html
<input type="text" name="username" required>
```

加了 `required`，这个输入框就不能是空的。如果用户没填就点提交，浏览器会弹出一个提示气泡："请填写此字段。"（不同浏览器的提示文字略有不同）。

required 对几乎所有 input 类型都有效，包括 radio、checkbox、select 等。

### min / max：数字范围

```html
<input type="number" name="age" min="1" max="120" required>
```

`min` 和 `max` 设定数字的允许范围。用户填 0 或 150 会触发校验失败。这两个属性只对数字类输入有效（`type="number"`、`range`、`date` 等）。

### minlength / maxlength：文本长度

```html
<input type="text" name="username" minlength="3" maxlength="20" required>
```

- `minlength`：最少 N 个字符。
- `maxlength`：最多 N 个字符（浏览器甚至会阻止用户输入超过 maxlength 的内容）。

### pattern：正则表达式（大招）

这是最强大的校验属性。pattern 接受一个**正则表达式**作为值，用来匹配用户输入的内容。

如果你学过 C，可能对字符串处理有点概念。正则表达式就是一种"字符串的匹配模板"。

```html
<!-- 手机号：1 开头的 11 位数字 -->
<input type="tel" name="phone" pattern="1[3-9]\d{9}" placeholder="请输入手机号" required>
```

解释一下 `1[3-9]\d{9}` 这个正则：
- `1`：第一个字符必须是 1
- `[3-9]`：第二个字符是 3 到 9 之间的一个数字
- `\d{9}`：后面跟 9 个数字（\d 是 digit，数字的意思）
- 合起来：1 开头的 11 位数字，第二位是 3-9

几个常用正则：

```html
<!-- 6-20 位字母或数字 -->
<input type="text" name="username" pattern="[A-Za-z0-9]{6,20}" required>

<!-- QQ 号：5-11 位纯数字，不能 0 开头 -->
<input type="text" name="qq" pattern="[1-9]\d{4,10}">

<!-- 简单邮箱校验 -->
<input type="email" name="email" pattern="[^@]+@[^@]+\.[^@]+">
```

pattern 一般搭配 `title` 属性使用，用来给用户提示格式要求：

```html
<input type="text" name="username"
  pattern="[A-Za-z0-9]{6,20}"
  title="6-20 位，只能用字母和数字"
  required>
```

当校验失败时，title 的内容会出现在浏览器提示气泡里。

### 一个完整的带校验的注册表单

```html
<form action="/register" method="POST">
  <p>
    <label for="username">用户名：</label>
    <input type="text" id="username" name="username"
      required minlength="3" maxlength="20"
      pattern="[A-Za-z0-9]{3,20}"
      title="3-20 位字母或数字">
  </p>
  <p>
    <label for="password">密码：</label>
    <input type="password" id="password" name="password"
      required minlength="6"
      title="至少 6 位字符">
  </p>
  <p>
    <label for="age">年龄：</label>
    <input type="number" id="age" name="age"
      required min="1" max="120">
  </p>
  <p>
    <label for="email">邮箱：</label>
    <input type="email" id="email" name="email"
      required placeholder="name@example.com">
  </p>
  <p>
    <button type="submit">注册</button>
  </p>
</form>
```

### 重要提醒

HTML5 校验是**第一道防线**，但不是**最后一道防线**。为什么？

- 用户可以按 F12 打开开发者工具，直接删掉 HTML 里的 required 属性然后提交。
- 有些老浏览器不支持这些属性。
- 正则可以被故意绕过。

所以**服务器端也需要做校验**，永远不能只依赖前端。HTML5 校验是为了"让好人更方便，让普通人不容易犯错"，而不是防黑客。

## 动手试试

做一个"用户注册表单"，要求：
- 用户名：必填，3-15 位，只能用字母和数字
- 密码：必填，至少 6 位
- 确认密码：必填（用 pattern 做不了的校验--两次输入是否一致，这得用 JavaScript。先放一个普通 password 输入框占位，后面学了 JS 再回来补）
- 年龄：必填，18-100 之间
- 邮箱：必填，email 类型
- 一个 type="submit" 的提交按钮
- 试试故意不填、填错，观察浏览器弹出的提示

---

## 本节小结

required（必填）、min/max（数字范围）、minlength/maxlength（长度）、pattern（正则匹配）构成了浏览器的"门卫系统"，纯 HTML 就能做基础校验，但永远不能替代服务器端校验。
