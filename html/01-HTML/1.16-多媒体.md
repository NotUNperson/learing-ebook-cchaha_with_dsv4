## 正文

### 网页不只有文字和图片

前面 15 节，我们的页面一直是"安静的"：文字、图片、表格、表单。但真实的互联网充满了声音和动态画面——教程视频、背景音乐、播客、直播间。

HTML 提供了 audio 和 video 标签，让你直接在页面上播放媒体，不需要 Flash 插件（那个已经死透了）。

### audio：音频播放器

```html
<audio src="music.mp3" controls></audio>
```

- `src`：音频文件路径。
- `controls`：告诉浏览器显示播放控制条（播放/暂停按钮、进度条、音量）。如果不加 controls，用户什么都看不到、什么都操作不了——就像一台没有按钮的音响。

常用属性：

```html
<audio src="bgm.mp3" controls autoplay muted loop></audio>
```

- `autoplay`：页面加载后自动播放。注意：大多数浏览器会阻止自动播放（除非同时设置了 `muted`），这是为了防止用户打开页面突然被声音吓到。
- `muted`：静音播放。
- `loop`：循环播放，放完从头再来。

### video：视频播放器

```html
<video src="demo.mp4" controls width="640" height="360"></video>
```

和 audio 几乎一样的属性，多了 width 和 height 来控制视频区域的大小。

```html
<video controls autoplay muted loop width="640" height="360">
  <source src="demo.mp4" type="video/mp4">
  <source src="demo.webm" type="video/webm">
  你的浏览器不支持视频播放。
</video>
```

video 可以有结束标签，中间的"你的浏览器不支持视频播放"是降级文字——如果浏览器完全不支持 video 标签（极老旧的浏览器），就显示这句话。

### source 标签：格式备胎

不同的浏览器支持的媒体格式不一样：

- MP4：最通用，几乎所有浏览器都支持。
- WebM：Google 推出，压缩率高，但 Safari 支持较晚。
- MP3：最通用的音频格式。
- OGG：开源格式，部分浏览器支持。

为了解决兼容性问题，你提供多个格式，浏览器按顺序挑选第一个它认识的：

```html
<video controls width="640" height="360">
  <!-- 第一个 source：优先尝试 -->
  <source src="demo.webm" type="video/webm">
  <!-- 第二个 source：WebM 不兼容则用 MP4 -->
  <source src="demo.mp4" type="video/mp4">
  <!-- 都播不了，显示提示 -->
  你的浏览器不支持此视频格式。
</video>
```

浏览器从上往下读，选第一个它能播放的格式。这就像你请客人喝饮料：先问"可乐要吗"，不要再问"雪碧要吗"，都不要就喝水。

### iframe：页面里开一扇窗看邻居

iframe 是 inline frame 的缩写。它能在你的页面里嵌入**另一个完整的网页**。

```html
<iframe
  src="https://www.example.com"
  width="600"
  height="400"
  title="示例页面"
></iframe>
```

就像你在自己家墙上开了一扇窗户，透过窗户可以看到邻居家的电视。iframe 最常见的用途：

- 嵌入 YouTube / Bilibili 视频（视频网站提供的"分享 -> 嵌入"得到的代码就是一个 iframe）。
- 嵌入地图（如百度地图、Google Maps）。
- 嵌入第三方服务（如支付页面、在线文档）。

iframe 需要注意：

- `title` 属性必须写（给屏幕阅读器用的）。
- 有些网站不允许被 iframe 嵌入（如淘宝、银行等，出于安全考虑）。
- iframe 里的内容和你的页面是隔离的——你不能用你的 JavaScript 去操作 iframe 里的内容（跨域安全限制）。

### canvas：一块你可以画画的画布

```html
<canvas id="myCanvas" width="400" height="300"></canvas>
```

canvas 本身只是一个"画板"。要在这个画板上画线条、图形、动画，需要用 JavaScript。它的用途包括：图表绘制、游戏画面、图片滤镜、实时数据可视化。

## 动手试试

1. 找一个短的视频（或音频）文件，用 video/audio 标签嵌入页面，加上 controls 属性。
2. 用 iframe 嵌入 B 站或 YouTube 的一个视频（去视频页面找"分享"按钮，复制"嵌入代码"）。
3. 用 source 标签给 video 提供两个格式，理解"备胎机制"。
4. 试着不放 controls 属性，看看是不是什么都操作不了？

## 本节小结

audio 放声音，video 放画面，都用 controls 给用户控制权。source 提供多个格式备胎。iframe 是页面里的"另一扇窗"。canvas 是一块画布，等待 JavaScript 挥洒。
