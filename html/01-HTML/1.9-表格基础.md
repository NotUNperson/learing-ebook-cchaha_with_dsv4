## 正文

### 表格就是行和列的交叉

从小我们就接触表格：课程表、成绩单、账单。它们都有一个共同结构：**横的是行（row），竖的是列（column），交叉处是一个格子（cell）。**

HTML 的表格标签名很直白：

- `table` -- 整个表格
- `tr` -- table row（表格行）
- `td` -- table data（表格数据，也就是普通格子）
- `th` -- table header（表头格，特殊的数据格）

### 一个最简单的表格

```html
<table>
  <tr>
    <td>张三</td>
    <td>85</td>
    <td>92</td>
  </tr>
  <tr>
    <td>李四</td>
    <td>78</td>
    <td>88</td>
  </tr>
  <tr>
    <td>王五</td>
    <td>91</td>
    <td>79</td>
  </tr>
</table>
```

这是一个 3 行 3 列的表格。写表格的思考方式：**先分行，再分格。** 每一个 `<tr>` 就是一行，里面的 `<td>` 就是这一行的各个格子。

默认情况下，浏览器不会给表格画边框。你需要在 CSS 里加一句 `border: 1px solid #ccc`（后面 CSS 章节会教），表格格子才会出现线。

不过放心，我们在示例文件里加了一些"临时的 CSS"，让你现在就能看到清晰的表格线。

### th：表头格

表头格 `<th>` 和普通数据格 `<td>` 的区别：

- `<th>` 里的字默认**加粗**且**居中**。
- `<th>` 在语义上表示"这一格是表头，定义了这一行或这一列是什么"。

```html
<table>
  <tr>
    <th>姓名</th>
    <th>语文</th>
    <th>数学</th>
  </tr>
  <tr>
    <td>张三</td>
    <td>85</td>
    <td>92</td>
  </tr>
  <tr>
    <td>李四</td>
    <td>78</td>
    <td>88</td>
  </tr>
</table>
```

第一行的三个 `<th>` 就是表头，定义了"第一列是姓名""第二列是语文""第三列是数学"。

th 可以出现在行的最左边（定义这一行是什么），也可以出现在列的最上方（定义这一列是什么），或者两者都有。

### 一个表格的样子（加了边框）

在 HTML 标签上通过 CSS 加上边框后，表格终于"像表格"了。边框、间距、对齐这些我们会在 CSS 部分详细介绍。

### 表格不是用来排版的

早年（大约 2000 年代）很多网站用表格来做页面布局：左边放导航，右边放正文。表格标签把整个页面划分成一格一格的，然后把不同内容塞进不同的格子里。

这种做法已经被淘汰了。原因：
- 表格在语义上就是"展示数据"的，不是"摆放内容"的。
- 表格布局在不同屏幕尺寸下表现很差。
- 屏幕阅读器（盲人工具）会把表格内容按"第几行第几列"读出来，如果你的导航栏被包在 table 里，盲人用户会听到一连串的"行一列一、行二列一……"。

**规则：表格只用来展示数据，布局用 CSS 的 Flexbox 和 Grid（后面会学）。**

## 本节小结

表格由 table（整张表）、tr（行）、td（普通格）、th（表头格）组成。先写行，再把格填进行里。表格只用来展示数据，别拿它做页面布局。
