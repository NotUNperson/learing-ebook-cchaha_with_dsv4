# 0.4 搭建开发环境

## 本节你会学到什么

* 为什么需要一个好的代码编辑器，而不是用 Windows 自带的记事本
* 安装和配置 VS Code（全免费的行业标准选择）
* 安装 Live Server 插件，实现"保存即刷新"的体验
* 认识浏览器 DevTools 的四个核心面板（Elements、Console、Network、Sources）
* 一份本课程的标准工作流程

## 一、编辑器 VS 记事本：为什么要用专业工具

你可以用 Windows 记事本写 HTML。事实上，很多大学第一堂课就是让学生用记事本手写一个 HTML 文件。但很快你就会发现：

* 记事本没有**语法高亮**——所有文字一个颜色，标签和内容混在一起很难分辨。
* 记事本没有**自动补全**——你写 `<but` 它不会给你提示 `button`，每个标签名都要手打。
* 记事本没有**错误提示**——标签忘了闭合、缩进乱了，它不会告诉你。
* 记事本**默认编码可能是 ANSI**——写中文可能会乱码。

所以，选择一款能识别 HTML/CSS/JS 语法的代码编辑器，是效率和工作体验的巨大提升。市面上有很多选择（Sublime Text、Atom、WebStorm），但本课程推荐 **Visual Studio Code（VS Code）**——免费、开源、社区庞大、插件丰富。

> **类比 C 语言**：写 C 语言你会用 Dev-C++、VS、Code::Blocks、Clion 这样的 IDE，而不是记事本。VS Code 就像是前端开发的"IDE"，虽然它本质是编辑器，但通过插件可以做到 IDE 级别的体验。

## 二、安装 VS Code

**第一步：下载**

打开 https://code.visualstudio.com/ ，点击那个大大的"Download for Windows"按钮。下载完成后运行安装程序。

**第二步：安装时的重要选项**

安装过程中有几页选项，建议勾选下面这些：

* **"将 '通过 Code 打开' 操作添加到文件资源管理器的目录上下文菜单"**——这样你在任意文件夹上右键，就能选"通过 Code 打开"
* **"将 '通过 Code 打开' 操作添加到文件资源管理器的文件上下文菜单"**——同理，单个文件也能用 Code 打开
* **"将 Code 注册为受支持的文件类型的编辑器"**——让系统默认用 VS Code 打开 `.html`、`.css`、`.js` 这类文件
* **"添加到 PATH"**——这样你可以在命令行里输入 `code .` 来打开当前文件夹

**第三步：打开 VS Code，看一眼界面**

* 左侧是**活动栏**（Activity Bar）：最上面的 Explorer（文件浏览器）是你最常用的。
* 上方是**命令面板**（按 `Ctrl+Shift+P` 打开）：你能通过它执行几乎所有操作，比如输入 `theme` 换颜色主题。
* 中间是**编辑区域**：可以分屏编辑多个文件（右键标签选"Split"）。
* 底部是**状态栏**：显示当前文件的语言、编码、行号等信息。

**第四步：安装中文字体推荐**

如果你觉得英文界面不习惯，可以在 Extensions（Ctrl+Shift+X）里搜索 `Chinese (Simplified)`，安装微软官方的中文语言包，重启后界面就变成中文了。不过本课程会用英文术语（因为前端领域几乎所有文档和搜索关键词都是英文的），建议保持英文界面。

## 三、安装 Live Server 插件

这是本课程最重要的一个插件。它的功能很简单：**在你本地电脑上启动一个 HTTP 服务器，自动在浏览器中打开你的 HTML 文件，并且当你修改保存文件时自动刷新页面。**

没有它的话你的工作流是：改代码 -> 保存 -> 切到浏览器 -> 手动刷新。一天下来你至少要做几百次这个循环。Live Server 帮你省掉后两步。

**安装步骤**：

1. 打开 VS Code
2. 点击左侧的 **Extensions** 图标（或者按 `Ctrl+Shift+X`）
3. 在搜索框中输入 `Live Server`
4. 找到作者为 **Ritwick Dey** 的 Live Server 插件（通常排第一个，下载量超过四千万）
5. 点击 **Install** 按钮

**使用方法**：

1. 在 VS Code 中打开一个文件夹（File -> Open Folder），或者创建一个新文件夹作为你的项目目录
2. 在这个文件夹中创建一个 HTML 文件（比如 `index.html`）
3. 在编辑器中右键这个 HTML 文件，选择 **"Open with Live Server"**
4. 浏览器会自动打开并显示你的页面。地址栏里是 `http://127.0.0.1:5500/index.html`（而不是 `file:///C:/...`）
5. 现在修改你的 HTML 文件，保存（`Ctrl+S`），浏览器**自动刷新**，无需手动操作

还有一个更方便的方式：直接点击 VS Code 右下角状态栏的 **"Go Live"** 按钮，它会对当前编辑的文件启动 Live Server。

## 四、认识浏览器 DevTools

浏览器自带的开发者工具（Developer Tools，简称 DevTools）是前端开发最重要的工具，没有之一。按 **F12** 打开（或者按 `Ctrl+Shift+I`），你会看到一个面板。这个面板有多个标签页，我们重点认识四个：

### Elements 面板（元素）

显示页面的 DOM 树——你在 0.3 节学到的那个树形结构。你能看到每个 HTML 标签的嵌套关系，右侧还能看到应用到当前元素的 CSS 规则。

实用操作：
* 鼠标移到 DOM 树上的节点，页面中对应的元素会高亮。
* 双击文字可以直接修改，修改后页面立刻更新。
* 右键某个元素 -> "Edit as HTML"，可以任意修改 HTML 结构。
* 右侧 Styles 面板里可以临时修改 CSS，实时看到效果。

### Console 面板（控制台）

这是 JavaScript 的"命令行"。你可以在这里输入 JavaScript 代码并立刻看到结果。就像你在 C 语言里写 `printf()` 来调试一样，这里就是 `console.log()` 的输出位置。

实用操作：
* 输入 `document.title` 回车，看到当前页面的标题。
* 输入 `console.log("你好")` 回车，看到输出。
* 输入 `1 + 2` 回车，看到 `3`。
* 当你写的 JS 代码出错时，错误信息（红色的）也会出现在这里。

### Network 面板（网络）

显示页面加载过程中发生的**每一个** HTTP 请求。这在 0.1 节提到过——你会惊讶地发现一个网页实际上发了几十个请求。

实用操作：
* 打开 Network 面板，然后刷新页面（F5），观察瀑布流般的请求列表。
* 点击任意一条请求，能看到完整的 Request Headers 和 Response Headers——这就是你在 0.1 节学到的东西。
* 每行请求的右侧有状态码（200、404 等）和耗时（ms）。
* 顶部的"Disable cache"勾选框，勾上后每次都会从服务器重新获取（不会用缓存）。

### Sources 面板（源代码）

显示页面加载的所有源文件（HTML、CSS、JS），并且可以**断点调试** JavaScript。

实用操作：
* 左侧文件树列出所有源文件，点击可以查看内容。
* 在 JS 代码的行号上点一下，会打上一个蓝色的断点。刷新页面后，代码执行到这里会暂停。
* 暂停时你可以在右侧看到当前所有变量的值（就像 GDB 调试 C 语言一样）。

> **类比 C 语言**：Console 面板像你的 `printf` 输出窗口；Sources 面板的断点调试就像 GDB 里的 `break` + `print`。如果你学过 C 语言的调试，你会发现这套逻辑几乎一样。

## 五、本课程的标准工作流程

以下是你在本课程中每次写代码时的标准流程：

1. **打开 VS Code**，在 File -> Open Folder 中选择你的项目文件夹（本课程建议在 `D:/projects_learn/html-larn-20260426` 下组织所有代码）。
2. **创建或编辑 HTML/CSS/JS 文件**，用 VS Code 的编辑器写代码。
3. **右键 HTML 文件 -> "Open with Live Server"**，在浏览器中看到效果。
4. **按 F12 打开 DevTools**：
   * 用 Console 看 JS 的输出和错误
   * 用 Elements 查看和临时修改 DOM/CSS
   * 用 Network 观察 HTTP 请求
5. **修改代码 -> 保存（Ctrl+S）-> 浏览器自动刷新**，看新效果。如果有 JS 错误，在 Console 中排查。
6. **遇到不懂的，先在 Console 里实验**。比如不确定某个 JS 语法是否正确，先在 Console 里敲一遍看结果，再写到文件里。

## 动手试试

1. 如果你还没有安装 VS Code，现在就下载安装它（https://code.visualstudio.com/）。
2. 安装完成后，安装 Live Server 插件。
3. 创建一个测试文件夹，在里面新建 `test.html`，写入以下内容：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>环境测试</title>
</head>
<body>
    <h1>环境搭建成功!</h1>
    <p>如果你能看到这个页面，说明你的开发环境已经就绪。</p>
    <button onclick="console.log('按钮被点击了!')">点我，看 Console</button>
</body>
</html>
```

4. 右键 `test.html` -> "Open with Live Server"。
5. 浏览器打开后，按 F12 打开 DevTools，切到 Console 标签。
6. 点击页面上的按钮，观察 Console 里是否输出了"按钮被点击了!"。
7. 如果一切正常，恭喜你，开发环境搭建完成。

## 本节小结

VS Code 是你的主要工作台，Live Server 让你保存即刷新，浏览器 DevTools 让你透视网页内部的一切。这三者组合起来，就是前端开发的标准工作环境。
