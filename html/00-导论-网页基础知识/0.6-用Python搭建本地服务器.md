# 0.6 用 Python 搭建本地服务器

## 本节你会学到什么

* 用 Python 一行命令启动一个 HTTP 服务器
* 理解 `localhost` 和端口号是什么
* 用 Flask 写一个能返回 JSON 数据的迷你 API 服务
* 在浏览器中测试你亲手搭建的服务器

## 一、事前检查：你的电脑有 Python 吗

在开始之前，先确认你的电脑上有没有 Python。打开命令行（Win+R -> 输入 `cmd` 回车，或者 PowerShell），输入：

```bash
python --version
```

如果输出 `Python 3.x.x`（x 是具体版本号，比如 3.12.0），说明你已经有 Python 了，可以直接跳到第二部分。

如果没有（输出 `'python' 不是内部或外部命令`），你需要先安装 Python：

1. 打开 https://www.python.org/downloads/
2. 点击那个显眼的黄色"Download Python"按钮
3. 运行下载的安装程序
4. **重要**：安装页面的最底部，勾选 **"Add Python to PATH"**（把 Python 添加到系统环境变量）
5. 点击 Install Now，等待安装完成
6. 重新打开命令行，再输入 `python --version`，确认安装成功

## 二、极简方案：一行命令搞定

Python 自带了 HTTP 服务器模块。你只需要**一行命令**就能启动一个 HTTP 服务器，让它帮你托管当前目录下的所有文件。

**操作步骤**：

1. 打开命令行（PowerShell 或 CMD）
2. 用 `cd` 命令切换到你存放 HTML 文件的目录。比如你的项目在 `D:/projects_learn/html-larn-20260426`，就输入：

```bash
cd D:/projects_learn/html-larn-20260426
```

3. 输入以下命令然后回车：

```bash
python -m http.server 8000
```

你会看到输出类似：

```
Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...
```

这行输出的意思是：**一个 HTTP 服务器已经在你的电脑上启动，监听 8000 端口**。

4. 打开浏览器，在地址栏输入：

```
http://localhost:8000
```

你会看到当前目录下的所有文件列表。点击任意 `.html` 文件（比如 `test.html`），它就会在 `http://` 协议下打开！

**关闭服务器**：在命令行中按 `Ctrl+C`。

### 这行命令做了什么

拆解 `python -m http.server 8000`：

* `python -m`：告诉 Python"我要运行一个模块（module）"。
* `http.server`：Python 内置的一个模块，功能是启动一个简单的 HTTP 文件服务器。
* `8000`：端口号。你可以换成别的数字（比如 3000、8080），只要不和其他程序冲突。端口号像是"大楼里的房间号"——服务器的 IP 地址是大楼地址，端口号是具体的房间编号。

### `localhost` 是什么意思

`localhost` 是网络世界里一个特殊的名字，它永远指向**你自己的电脑**。在网络术语里，它的 IP 地址是 `127.0.0.1`（IPv4）或 `::1`（IPv6）。

你可以把 `localhost` 理解成"我自己的地址"。无论你是断开网线、关掉 Wi-Fi，还是不在任何网络中，`localhost` 永远都能访问——因为它在你的电脑内部完成通信，不需要任何外部网络。

所以 `http://localhost:8000` 的意思就是："访问我自己电脑上 8000 端口的那个 HTTP 服务器"。

## 三、进阶方案：用 Flask 写一个迷你 API 服务

上面 `python -m http.server` 方案虽然简单，但它只能"提供静态文件"——你把什么文件放在目录里，它就提供什么，不能做任何动态逻辑。如果想做一个"能接收请求并返回动态数据"的服务器，就需要一个**Web 框架**。

Flask 是 Python 里最流行的轻量级 Web 框架之一。写一个能返回 JSON 数据的 API 服务，只需要 10 行左右的代码。

### 第一步：安装 Flask

在命令行中输入：

```bash
pip install flask
```

如果 `pip` 不识别，先试试 `pip3 install flask`。安装过程中会下载几个文件，等待几十秒即可。成功后会显示 `Successfully installed flask-x.x.x`（以及几个依赖库）。

### 第二步：写服务代码

用 VS Code 新建一个文件，叫 `server.py`，写入以下代码：

```python
from flask import Flask, jsonify, request

app = Flask(__name__)

# 一个简单的"内存数据库"——实际上就是一个列表
# 学过数据结构你应该对这种列表操作很熟悉
tasks = [
    {"id": 1, "title": "学 HTML", "done": False},
    {"id": 2, "title": "学 CSS", "done": False},
    {"id": 3, "title": "学 JavaScript", "done": False},
]

@app.route('/')
def home():
    """根路径：返回一个欢迎信息"""
    return '<h1>欢迎使用任务管理 API</h1><p>访问 <a href="/api/tasks">/api/tasks</a> 查看任务列表</p>'

@app.route('/api/tasks')
def get_tasks():
    """返回全部任务的 JSON 数据"""
    return jsonify(tasks)

@app.route('/api/tasks/<int:task_id>')
def get_task(task_id):
    """返回单个任务的 JSON 数据"""
    task = next((t for t in tasks if t['id'] == task_id), None)
    if task is None:
        return jsonify({"error": "任务不存在"}), 404
    return jsonify(task)

@app.route('/api/tasks', methods=['POST'])
def add_task():
    """接收 POST 请求，添加一个新任务"""
    new_task = request.get_json()
    new_task['id'] = len(tasks) + 1
    tasks.append(new_task)
    return jsonify(new_task), 201

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=8000, debug=True)
```

### 第三步：运行服务器

在命令行中，进入 `server.py` 所在的目录，运行：

```bash
python server.py
```

你会看到类似输出：

```
 * Serving Flask app 'server'
 * Debug mode: on
 * Running on http://0.0.0.0:8000
```

### 第四步：用浏览器测试

打开浏览器，访问以下地址，观察结果：

**测试 1：访问根路径**
```
http://localhost:8000/
```
你会看到一段 HTML 文字。

**测试 2：获取全部任务（GET 请求）**
```
http://localhost:8000/api/tasks
```
你会看到浏览器显示一堆 JSON 数据：
```json
[
  {"id": 1, "title": "学 HTML", "done": false},
  {"id": 2, "title": "学 CSS", "done": false},
  {"id": 3, "title": "学 JavaScript", "done": false}
]
```
浏览器只是简单地发起了一个 GET 请求，Flask 收到后执行 `get_tasks()` 函数，把列表转成 JSON 返回。

**测试 3：获取单个任务**
```
http://localhost:8000/api/tasks/2
```
你会看到：
```json
{"id": 2, "title": "学 CSS", "done": false}
```

**测试 4：获取不存在的任务**
```
http://localhost:8000/api/tasks/99
```
你会看到：
```json
{"error": "任务不存在"}
```

**测试 5：用 Console 发一个 POST 请求**

按 F12 打开 DevTools -> Console 标签，输入以下代码然后回车：

```javascript
fetch('http://localhost:8000/api/tasks', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ title: '复习数据结构', done: false })
})
.then(function(res) { return res.json(); })
.then(function(data) { console.log('添加成功:', data); });
```

然后刷新 `http://localhost:8000/api/tasks`，你会看到多了一条记录。

### 代码逐段解释

如果你学过 C 语言的函数，这段 Python 代码的逻辑很好理解：

* `@app.route('/api/tasks')` 是一个"路由装饰器"。它的意思是：**当有人访问 `/api/tasks` 这个路径时，就调用下面这个函数**。这有点像 C 语言的函数指针表——浏览器"调用"了哪个路径，Flask 就去执行对应的函数。
* `jsonify(tasks)` 把 Python 的列表/字典转成 JSON 格式的字符串，并设置正确的 `Content-Type` 头。
* `<int:task_id>` 是 URL 参数——`/api/tasks/2` 中的 `2` 会被自动提取出来，赋给 `task_id` 参数。
* `methods=['POST']` 表示这个路由只接受 POST 请求。不写的话默认只接受 GET 请求。
* `app.run(debug=True)` 中的 `debug=True` 表示开启调试模式——每次你修改 `server.py` 并保存，服务器会自动重启。这对开发很方便。

## 四、Python 方案 vs Live Server 方案

你已经有了 Live Server（0.4 节安装的 VS Code 插件），为什么还要学 Python 启动服务器？

| 场景 | 用什么 |
|------|--------|
| 纯写 HTML/CSS/JS 静态文件，只为了在浏览器中预览 | Live Server（最方便） |
| 需要一个能返回动态数据的后端 API | Python Flask（或类似的框架） |
| 第 5 章之后，你学习了 fetch 请求，需要一个真正的 API 来练习从后端获取数据 | Python Flask |
| 你想理解"服务器端"到底是什么，而不是总用现成的工具 | 体验一次 `python -m http.server` |

一句话总结：**Live Server 适合纯前端学习阶段；当你开始接触"前后端交互"（fetch、AJAX）时，就需要一个能处理动态请求的后端服务**。

## 动手试试

1. 用 `python -m http.server 8000` 启动一个静态文件服务器，在浏览器中访问 `http://localhost:8000`，看看它是不是列出了你目录下的所有文件。
2. 安装 Flask（`pip install flask`），创建 `server.py`，运行它。
3. 用浏览器访问 `http://localhost:8000/api/tasks`，确认看到 JSON 数据。
4. 在 DevTools Console 中用 fetch 发一个 POST 请求，添加一条新任务，然后验证它是否被成功保存。
5. **挑战**：修改 `server.py`，添加一个新路由 `GET /api/tasks/done`，只返回 `done: true` 的任务。提示：用列表过滤。
6. 思考：这个小 Flask 服务和你生活中有没有用过的某个"后端 API"很像？（比如天气查询、快递查询）

## 本节小结

一行 `python -m http.server 8000` 就能启动一个静态文件 HTTP 服务器；进阶时用 Flask 可以快速构建能返回动态数据的 API。自此，你的电脑就是一个具备完整 HTTP 服务能力的开发环境。
