# 0.2 网页的三驾马车

## 本节你会学到什么

* HTML、CSS、JavaScript 分别负责网页的哪一部分工作
* 一个通俗的建筑比喻：骨架、装修、电路
* 三者在浏览器里是怎样协作的
* 一个完整但简单的网页示例，三种技术出现在同一个文件里
* 为什么要把三者分开而不是全揉在一起（关注点分离）

## 一、一个建筑比喻

想象你要盖一栋房子。盖房子的过程分三个阶段：

**第一阶段：搭骨架（HTML）**

你先把房子的骨架搭起来——几根柱子、房梁、楼板、隔墙。这时候房子还不好看，连墙都是毛坯水泥，但它已经有了明确的结构：这是客厅，那是卧室，那是厨房。

HTML 做的事情一模一样。它在网页上搭建"骨架"：这是一个标题、这是一个段落、这是一张图片、这是一个按钮。HTML 定义了**页面里有什么，以及这些内容的层次结构**。

**第二阶段：装修（CSS）**

骨架搭好后，装修队进场——墙刷成什么颜色、地板用什么材质、沙发摆哪里、窗帘什么花纹、灯挂多高。经过装修，房子变好看了，住着也舒服了。

CSS 做的就是这件事。它给 HTML 骨架做"装修"：标题用多大字、段落的行间距是多少、按钮是圆角还是直角、整个页面的背景色是什么。CSS 决定了**页面长什么样**。

**第三阶段：通电路和自动化（JavaScript）**

房子漂亮了，但还不够——灯不会亮、空调打不开、自动门不会动。于是电工进场，给房子布线、装开关、配遥控器。现在，按下开关灯亮了，遥控器能开空调，走到门口门能自动打开。

JavaScript 就是网页的"电路和神经"。它让网页有交互能力：点击按钮弹出一个对话框、滚动页面时导航栏变色、输入框中实时验证格式是否正确。JS 决定了**页面能做什么**。

> 把这三点总结成一句口诀：**HTML 管内容，CSS 管样式，JS 管行为。**

## 二、换个类比：写小说

如果你写过 struct 相关的 C 代码，换个角度再理解一遍：

* **HTML 像是 struct 的定义**：你声明这个结构体有哪些字段（标题、段落、图片）。它只定义"数据结构"，不管长什么样。
* **CSS 像是格式化输出的那部分代码**：你写 `printf("标题：%-20s\n", title)` 来决定怎么把内容展示出来，用多大字体、什么颜色。
* **JS 像是操作这个 struct 的业务逻辑**：如果用户点击了某个按钮，就修改某个字段的值，然后重新渲染。

## 三、一个完整示例：三驾马车出现在同一个文件里

以下是一个能直接保存为 `.html` 文件并在浏览器中打开的完整示例。注意看 HTML、CSS、JS 分别在哪、做了什么。

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>三驾马车的演示</title>

    <!-- ===== CSS 区域：管样式 ===== -->
    <style>
        /* 这些是 CSS：给 HTML 元素做"装修" */
        body {
            font-family: sans-serif;
            background-color: #f4f4f4;
            margin: 0;
            padding: 40px;
        }

        .card {
            background: white;
            border-radius: 12px;
            padding: 30px;
            max-width: 500px;
            margin: 0 auto;
            box-shadow: 0 2px 8px rgba(0,0,0,0.1);
        }

        h1 {
            color: #333;
            margin-top: 0;
        }

        p {
            line-height: 1.8;
            color: #555;
        }

        button {
            background-color: #007acc;
            color: white;
            border: none;
            padding: 10px 24px;
            border-radius: 6px;
            font-size: 16px;
            cursor: pointer;
        }

        button:hover {
            background-color: #005a9e;
        }

        #counter {
            font-weight: bold;
            color: #007acc;
        }
    </style>
</head>
<body>
    <!-- ===== HTML 区域：管内容 ===== -->
    <!-- 这些是 HTML 标签：定义网页的"骨架" -->
    <div class="card">
        <h1>你好，这是我的第一个网页</h1>

        <p>这是一个用<strong>三驾马车</strong>构建的演示页面。</p>

        <p>
            你点击了按钮 <span id="counter">0</span> 次。
        </p>

        <button id="clickBtn">点我!</button>
    </div>

    <!-- ===== JavaScript 区域：管行为 ===== -->
    <script>
        // 这些是 JavaScript：给网页加"电路"
        var count = 0;

        // 用 JS 找到 HTML 中的元素（通过 id）
        var button = document.getElementById('clickBtn');
        var counterSpan = document.getElementById('counter');

        // 给按钮绑定"点击"事件：每次点击，count 加 1，并更新页面
        button.addEventListener('click', function() {
            count = count + 1;
            counterSpan.textContent = count;
        });
    </script>
</body>
</html>
```

请把这个代码保存为一个文件（比如 `demo.html`），用浏览器打开，点击按钮观察效果。你看到：

* **HTML** 提供了按钮和文字（内容/骨架）
* **CSS** 让按钮是蓝色的、有圆角、鼠标悬停时变深（样式/装修）
* **JS** 让按钮在被点击时计数器加 1（行为/电路）

## 四、三者是怎么"协作"的

浏览器拿到 HTML 后，做这几件事：

1. 解析 HTML，构建一个树形结构（DOM 树，0.3 节会详细讲），每一个 HTML 标签变成树上的一个"节点"。
2. 遇到 `<style>` 标签，解析里面的 CSS 规则，记住"`.card` 的背景是白色"、"`button` 的背景是蓝色"等等。
3. 把 CSS 规则应用到 DOM 树的节点上，算出每个元素的最终样式。
4. 遇到 `<script>` 标签，执行里面的 JavaScript 代码。
5. JS 代码通过 `document.getElementById()` 这样的 API 找到 DOM 树上的节点，然后可以修改它们的内容、样式，或者给它们绑定事件。

关键理解：**HTML、CSS、JS 都在操作同一棵"DOM 树"**。HTML 创建了这棵树，CSS 给树上的节点穿衣服，JS 通过 API 操作这棵树上的节点（增、删、改、查）。

## 五、为什么要分开——关注点分离

初学者可能会想：能不能把颜色直接写在 HTML 标签上？能不能把点击行为也写在标签上？技术上可以，但不推荐。原因是一个设计原则叫"关注点分离（Separation of Concerns）"：

* 内容、样式、行为是三件**不同的事情**，把它们混在一起会让代码难以维护。
* 想象一下，如果你有 100 个页面，每个页面的按钮颜色都直接写在 HTML 标签上。有一天产品经理说"按钮颜色换成红色"，你得改 100 个文件。但如果颜色写在同一个 CSS 文件里，只需要改一行。
* 同理，如果你的项目有专门的设计师（负责 CSS）和专门的后端工程师（负责 JS 逻辑），分开的文件各自维护，互不干扰。

在实际项目中，HTML、CSS、JS 通常会放在**三个独立的文件**中：

```
my-project/
  index.html       -- HTML 结构
  style.css        -- CSS 样式
  main.js          -- JavaScript 逻辑
```

然后在 HTML 中通过 `<link>` 引入 CSS，通过 `<script src="...">` 引入 JS。本书前几章为了方便，会把它们写在同一个文件里，但请一定记住：**未来的正式项目要分开**。

## 动手试试

1. 把上面的完整示例代码保存为 `demo.html`，用浏览器打开，点几次按钮。
2. 然后尝试修改：
   * 把按钮的 `background-color` 改成红色（改 CSS）
   * 把按钮文字从"点我!"改成"使劲按!"（改 HTML）
   * 把每次点击的增量从 1 改成 2（改 JS 中的 `count = count + 1` 为 `count = count + 2`）
3. 每次修改后刷新浏览器，观察变化。体会"改样式不需要动 HTML"、"改行为不需要动样式"的感觉。

## 本节小结

HTML 是你盖房子的钢筋骨架，CSS 是装修和布置，JavaScript 是电路和自动化系统。三个技术各司其职，操作同一棵 DOM 树，共同构成你看到的每一个网页。
