# A.2 变量声明

## 本节你会学到什么

- `let` 声明可变变量，块作用域，最常用
- `const` 声明常量，不可重新赋值（但对象属性可以改）
- `var` 是老式声明，了解即可，不推荐使用
- `let`/`const` 的块作用域 vs `var` 的函数作用域
- JS 的命名规则和驼峰命名法

## 正文

### 一、let -- 你的主力声明方式

`let` 是 ES6 引入的变量声明方式，用来声明**可以改变**的变量。它和 C 语言中的普通变量最像。

```javascript
let age = 25;        // 声明并赋值
age = 26;            // 可以重新赋值
console.log(age);    // 26

let name;            // 可以先声明，稍后再赋值
name = "小明";
console.log(name);   // "小明"
```

`let` 有**块作用域**（block scope），也就是说，在 `{}` 内部声明的 `let` 变量，在 `{}` 外面访问不到。这和 C 语言的变量作用域规则是一样的。

```javascript
{
    let x = 10;
    console.log(x);  // 10, 正常
}
// console.log(x);   // 报错！x 在这里不可见
```

> 生活类比：`let` 声明的变量就像一个房间内部的东西，出了房间就看不到了。每个 `{}` 就是一道墙。

### 二、const -- 常量（但有个坑）

`const` 也是 ES6 引入的，用来声明**常量**。一旦赋值，就不能再重新赋值。

```javascript
const PI = 3.14159;
// PI = 3.14;       // 报错！不能重新赋值

const NAME = "小明";
// NAME = "小红";    // 报错！
```

**重要的坑：`const` 不能重新赋值，但如果它指向的是一个对象，对象内部的内容是可以改的。**

```javascript
const person = { name: "小明", age: 20 };
person.age = 21;          // 可以改！修改的是对象内部属性
console.log(person.age);  // 21

// person = { name: "小红" };  // 报错！不能把 person 指向新对象
```

换个类比：`const` 锁住的是"变量名指向谁"，而不是"指向的那个东西里面是什么"。就像你把家庭住址写在身份证上不能改，但家里可以重新装修。

> 最佳实践：**默认用 `const`**，当你确定这个变量的值需要改变时再用 `let`。这样代码意图更清晰。

### 三、var -- 老式的声明（知道就行，别用）

`var` 是 ES6 之前唯一的变量声明方式。它有两个"问题"：

1. **没有块作用域**，只有函数作用域
2. **变量提升（hoisting）** 会导致一些反直觉的行为

```javascript
// var 的块作用域"漏洞"
{
    var x = 10;
}
console.log(x);  // 10  居然能访问到！（用 let 的话会报错）

// var 的提升
console.log(y);  // undefined（没有报错！因为 var y 被"提升"了）
var y = 5;
```

这些行为在大型程序中容易导致 bug。所以 **ES6 之后，你应该只用 `let` 和 `const`，忘记 `var`**。如果你在别人的旧代码里看到 `var`，知道它是老式声明即可。

### 四、变量命名规则

JavaScript 的命名规则和 C 语言基本一致：

- 大小写敏感：`age`、`Age`、`AGE` 是三个不同的变量
- 首字符必须是字母、下划线 `_` 或美元符号 `$`
- 后续字符可以是字母、数字、下划线、美元符号
- **不能**用 JS 的保留字（如 `if`、`for`、`class`、`return` 等）

关于 `$`：JS 里可以用美元符号作为变量名的一部分，这在 C 语言中是不允许的。jQuery 库就用了 `$` 作为核心变量的名字。

### 五、命名惯例：驼峰命名法

JavaScript 世界的主流命名风格是**驼峰命名法**（camelCase）：

- 第一个单词小写，后面每个单词首字母大写
- 变量和函数：`myName`、`userAge`、`getUserInfo`
- 类名用大驼峰（PascalCase）：`Person`、`UserService`
- 常量（真正的常量，比如配置值）：`MAX_SIZE`、`API_URL`（全大写+下划线）

```javascript
// 推荐的命名
let userName = "张三";         // 驼峰命名
const MAX_LOGIN_ATTEMPTS = 5; // 全大写常量
let isLoggedIn = false;       // 布尔值常用 is/has 开头

// 不推荐的命名
let user_name = "张三";       // 下划线在 JS 中不主流（虽然能跑）
let UserName = "张三";        // 大驼峰通常给类用
```

## 与 C 语言的对比

C 语言的变量必须先声明类型（`int age = 25;`），JS 是**动态类型**语言，变量不需要写类型（`let age = 25;`）。JS 的 `let`/`const` 拥有和 C 类似的块作用域，但 `var` 没有。另外 C 中没有 `const` 对象属性的概念，JS 的 `const` 只锁住引用，这是一个需要适应的差异。

---

## 本节小结

- 日常用 `let` 声明可变变量，用 `const` 声明不可变变量
- `let`/`const` 有块作用域（`{}` 内有效），和 C 一样
- `var` 只有函数作用域，有提升问题，不推荐使用
- 命名用驼峰命名法：`myVariableName`
- JS 是动态类型语言，变量声明不需要写类型
