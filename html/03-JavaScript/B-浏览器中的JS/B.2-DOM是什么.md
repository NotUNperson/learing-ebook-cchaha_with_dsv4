# B.2 DOM 是什么

## 本节你会学到什么

- DOM 的全称与核心概念——浏览器把 HTML 变成了对象树
- `document` 对象——整个页面的"总入口"
- 三种节点类型：元素节点、文本节点、注释节点
- 用"家族树"类比理解 DOM 的层级结构
- 从 C 语言视角看 DOM——用链表造树 vs 浏览器内置树

## 正文

### 浏览器读 HTML，和你读菜单不一样

你打开一个 `.html` 文件时，看到的是标签和文字。但浏览器看到的不是字符串——它把整个 HTML **解析**成一棵**对象树**，然后存在内存里。这棵树就叫 **DOM**。

**DOM = Document Object Model**，中文叫"文档对象模型"。拆开理解：

- **Document（文档）**：整个 HTML 页面
- **Object（对象）**：每个标签、每段文字都被转换成 JS 对象
- **Model（模型）**：这些对象被组织成一棵树形结构

类比：你拿了一份纸质菜单——对你来说，它是印刷的文字和图片。但如果把它变成一份**电子表格**（Excel），每道菜就是一行记录，有"菜名""价格""辣度""分类"等属性，你可以排序、筛选、修改。DOM 就是浏览器把 HTML "录入"内存后形成的那个"电子表格"——只不过不是表格，是树。

### 从 HTML 到 DOM 树

假设有这么一段 HTML：

```html
<!DOCTYPE html>
<html>
  <head>
    <title>我的页面</title>
  </head>
  <body>
    <h1>欢迎</h1>
    <p>你好，<em>世界</em>！</p>
  </body>
</html>
```

浏览器解析后，会在内存中形成这样一棵树：

```
        document（文档根节点）
             |
           html（根元素）
          /     \
       head     body
        |       /  \
      title   h1    p
        |      |    | \
     "我的页面" "欢迎" "你好，" em "！"
                           |
                         "世界"
```

这棵树上的每一个方框，在 JS 中都是一个**节点（Node）**。JS 可以通过这些节点对象来读取、修改、删除、新增页面上的任何东西。

### document 对象——一棵树的入口

要操作 DOM 树，你首先要找到"入口"。这个入口就是 **`document`** 对象。

`document` 是浏览器自动创建的一个全局对象，代表整个 HTML 页面。你可以直接在任何 script 中使用它，无需声明。

打开任意网页，按 F12 进入控制台，输入：

```javascript
console.dir(document);
```

你会看到一个巨大的对象——里面包含了当前页面的所有信息。`document` 就像一棵族谱的封面：你想找张三，不能满世界喊"张三！"——你得先打开族谱（document），然后往下查。

### 三种节点类型

DOM 树上的节点主要有三种（实际上有 12 种，但这三种覆盖了 99% 的场景）：

| 节点类型 | 对应 HTML | nodeType 值 | 示例 |
|----------|-----------|-------------|------|
| 元素节点 | 任何 HTML 标签 | 1 | `<h1>`, `<p>`, `<div>` |
| 文本节点 | 标签里的文字 | 3 | "欢迎", "你好，" |
| 注释节点 | `<!-- ... -->` | 8 | `<!-- 这是一个注释 -->` |

- **元素节点**：你日常操作 99% 都是它。它有属性（class、id、src 等），可以嵌套子节点。
- **文本节点**：只包含文字，没有属性，没有子节点。它总是元素节点的孩子。
- **注释节点**：JS 也能读到注释！`nodeType === 8`。实际开发中极少操作它。

### 家族树类比——DOM 里的"爸爸""儿子""兄弟"

DOM 树的结构和家族谱一模一样。用家族称谓来理解 DOM 的遍历属性：

```html
<div id="parent">
  <p>老大</p>
  <p>老二</p>
  <span>老三</span>
</div>
```

- `<div id="parent">` 是 `<p>老大</p>`、`<p>老二</p>`、`<span>老三</span>` 的**父节点**（parentNode）
- 三个子元素互为**兄弟节点**（previousSibling / nextSibling）
- `<p>老大</p>` 里的 `"老大"` 文本是 p 的**第一个子节点**（firstChild / childNodes）
- `<div>` 是 `<body>` 的**子节点**，`<body>` 是 `<html>` 的子节点，`<html>` 是 `document` 的子节点

你在 B.3、B.6 等节中会反复用到这套"家族树"概念来在页面上查找和移动元素。

### 从 C 语言视角看 DOM

如果你学过 C 语言和数据结构，你可能会想：这棵树在底层是什么？

C 语言里如果要做一棵类似的树，你可能这样写：

```c
typedef struct Node {
    char* tagName;           // 标签名
    char* textContent;       // 文本内容
    struct Node* parent;     // 父节点指针
    struct Node* firstChild; // 第一个子节点
    struct Node* nextSibling;// 下一个兄弟
    // ... 还有很多属性
} Node;
```

DOM 本质上就是用这样的链表/指针结构在内存中表示整个页面。但区别是：

- **C 语言**：你需要自己 `malloc` 创建节点、手动管理内存、自己写 `free` 释放
- **浏览器**：所有这些都由浏览器引擎（如 Chrome 的 Blink、Firefox 的 Gecko）帮你管理——自动解析 HTML、构建 DOM 树、垃圾回收

你在 JS-A 篇中学过的"对象"和"属性访问"，现在终于要用来操作真实的页面结构了。

### 提示：JS-A 知识与 DOM 的衔接

- JS-A 篇中你学到的 `console.log`、对象属性访问（`.` 语法）、数组遍历（`for` 循环）——这些都是操作 DOM 的基本功
- DOM 节点本身是对象，有属性和方法，和你创建的普通对象操作方式一致
- HTML 中定义的 class、id 等也体现在 DOM 节点的属性中（B.5 详讲）

## 动手试试

1. 打开示例文件 `B.2-dom.html`，查看页面上显示的 DOM 树结构
2. 按 F12 打开控制台，输入 `console.dir(document)` 回车，展开看看 document 对象里面有什么
3. 在控制台继续输入 `document.body`，看看它是什么——这就是 `<body>` 元素节点
4. 试试 `document.body.childNodes`，观察返回的列表——里面混合了元素节点和文本节点（换行也是文本节点！）
5. 试着找到页面上 `<h1>` 元素：`document.querySelector('h1')`——这招会在 B.3 详细讲

## 本节小结

DOM 是浏览器把 HTML 解析成的一棵对象树。`document` 是树的根节点入口。节点分三种——元素（最多见）、文本、注释。DOM 树的层级关系和家族谱一致，用父/子/兄弟来理解最直观。相比 C 语言手动管理树结构，浏览器自动完成了 DOM 的创建、维护和销毁。
