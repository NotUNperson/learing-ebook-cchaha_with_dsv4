# B.1 JS 如何接入页面

## 本节你会学到什么

- script 标签的三种使用方式：内联、外部引用、行内事件
- script 标签放在 `<head>` 和 `<body>` 底部的区别
- `defer` 和 `async` 属性的作用与区别
- 现代项目推荐的 script 引入策略

## 正文

### 给房子装上"智能家电"

你已经学完了 HTML（房子的骨架）、CSS（房子的装修），现在终于来到了 JavaScript。JS 是什么？它是房子的"智能家电"——灯能自动亮、窗帘能遥控、门铃会发通知。HTML 定义"有什么"，CSS 定义"长什么样"，JS 定义"怎么动"。

但在这之前，有一个基础问题必须先解决：**JS 代码怎么接入 HTML 页面？**

类比：你要在房子里安装一个智能开关。你可以把开关直接装在墙上（内联），也可以从电箱拉一根线到开关（外部文件），还可以把开关贴在某个家具上——只有碰那个家具的时候它才起作用（行内事件）。

### 方式一：内联 script 块（最常用的学习方式）

直接在 HTML 文件的 `<script>` 标签里写 JS 代码：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>内联 script 示例</title>
</head>
<body>
  <h1>你好</h1>
  <script>
    // JS 代码直接写在这里
    console.log("Hello from inline script!");
  </script>
</body>
</html>
```

这就像把开关直接嵌在墙上——简单直接，非常适合学习和演示。本教程的绝大部分示例都会用这种方式。

### 方式二：外部 src 引用（生产环境推荐）

把 JS 代码写在一个独立的 `.js` 文件里，然后用 `<script src="...">` 引入：

```html
<script src="app.js"></script>
```

注意：**用了 `src` 属性的 script 标签，标签内部的代码会被忽略。** 你不能既写 `src` 又在标签内部写代码——二选一。

```html
<!-- 正确：外部文件 -->
<script src="app.js"></script>

<!-- 正确：内联代码 -->
<script>
  console.log("hello");
</script>

<!-- 错误：src 和内部代码同时存在，内部代码被忽略！ -->
<script src="app.js">
  console.log("这行不会执行！");
</script>
```

这就像从配电箱拉专线到开关——代码和页面分离，多个页面可以共用同一个 JS 文件，便于维护和缓存。

### 方式三：行内事件处理器（不推荐）

直接在 HTML 元素的属性里写 JS：

```html
<button onclick="alert('你点了我！')">点我</button>
```

这就像把开关用胶带贴在门把手上——只有碰门把手的时候它才工作。这种方式把 JS 和 HTML 混在一起，难以维护，现代项目几乎不再使用。了解它只是因为你看老代码时可能会遇到。

### script 放在哪——位置决定一切

浏览器解析 HTML 是**从上到下**的。当它遇到 `<script>` 标签时，会发生什么？这取决于你怎么放。

#### 默认行为：放在哪就立刻执行，阻塞 HTML 解析

```html
<head>
  <script src="heavy.js"></script>  <!-- 浏览器停下来下载+执行 JS，页面一片空白 -->
</head>
<body>
  <h1>用户要等 JS 执行完才能看到这行字</h1>
</body>
```

默认情况下，script 是"霸道"的：浏览器遇到它就停下来，下载（如果是外部文件）、执行，执行完了才继续解析后面的 HTML。如果你的 JS 文件很大或网络很慢，用户就会盯着白屏等待。

#### 放在 body 底部（传统方案）

```html
<body>
  <h1>用户先看到内容</h1>
  <p>页面已经渲染好了</p>
  <script src="app.js"></script>  <!-- 最后才加载 JS -->
</body>
```

把所有 script 放在 `</body>` 的前面。这样浏览器先把 HTML 内容渲染出来让用户看到，最后再去加载和执行 JS。这是用了很多年的经典方案，简单有效。

#### defer（现代方案，推荐）

```html
<head>
  <script src="app.js" defer></script>  <!-- 后台下载，HTML 解析完再执行 -->
</head>
<body>
  <h1>页面正常解析</h1>
</body>
```

`defer` 属性的行为：
1. 浏览器在后台下载 JS 文件，**不阻塞** HTML 解析
2. JS 文件下载完后**不会立即执行**，而是等到 HTML 全部解析完
3. 多个 defer 脚本会**严格按照书写顺序**执行（先写的先执行，后写的后执行）

#### async（完全独立）

```html
<head>
  <script src="ad.js" async></script>  <!-- 后台下载，下载完立即执行 -->
</head>
```

`async` 属性的行为：
1. 浏览器在后台下载 JS 文件，**不阻塞** HTML 解析
2. JS 文件**下载完就立即执行**，即使 HTML 还没解析完
3. 多个 async 脚本**不保证执行顺序**——谁先下载完谁先执行

#### 一张表对比三种模式

| 属性 | 下载是否阻塞 HTML | 执行时机 | 是否保证顺序 |
|------|-------------------|----------|-------------|
| 无（默认） | 阻塞 | 立即执行 | 保证 |
| defer | 不阻塞（后台下载） | HTML 解析完后 | 保证 |
| async | 不阻塞（后台下载） | 下载完立即 | 不保证 |

#### 什么情况用哪个？

- **defer**：你的 JS 需要操作 DOM（大多数情况），而且可能有多个 JS 文件之间有依赖关系。这是现代项目最推荐的写法。
- **async**：你的 JS 是完全独立的，不依赖 DOM 也不依赖其他 JS（比如统计代码、广告脚本）。
- **body 底部**：老方案，简单可靠，小项目没有 defer 也完全 OK。
- **默认 head**：几乎不要用，会严重影响页面加载速度。

### 关联 JS-A 篇知识

你在 A 篇里学过的所有 JS 语法（变量、函数、对象、数组、循环、条件判断等），现在终于可以真正在浏览器里运行了。之前你在 Node.js 环境或浏览器控制台里写的代码，现在要嵌在 HTML 页面里，和页面元素互动了。

## 动手试试

1. 打开示例文件 `B.1-script.html`，查看页面上的三句话分别由哪种方式输出，观察它们在页面上的显示顺序
2. 按 F12 打开控制台，看看 console.log 的输出顺序——和页面上的文字顺序一样吗？
3. 尝试把第一个 `<script>` 标签移到 `<head>` 里，刷新看效果（你会发现内联脚本中的 `document.getElementById` 返回 null——因为 DOM 还没加载！）
4. 思考：为什么行内事件处理器里的那行代码不需要放在 script 标签里？

## 本节小结

JS 接入页面有三种方式：内联 script 块（学习演示最方便）、外部 src 引用（生产环境推荐）、行内事件（不推荐）。script 标签的放置位置影响页面加载：默认阻塞解析，defer 后台下载+延迟执行+保证顺序，async 后台下载+立即执行+不保证顺序。初学者记住一句话：**head + defer，或 body 底部**。
