点击下面的按钮，会弹出一个 alert 对话框：

script 的位置与加载模式

打开浏览器控制台（F12），观察 console.log 的输出先后顺序：

  1. 默认 script：写在 body 里，浏览器解析到就立即执行（阻塞 HTML 解析）
  2. defer：写在 head 里但加了 defer，等 HTML 全部解析完才执行（控制台最后输出）
  3. 行内事件：不自动执行，只有用户点击按钮时才触发

一张表对比

属性 下载是否阻塞 HTML 执行时机 是否保证顺序 适用场景
无（默认） 阻塞 立即执行 保证 了解即可，避免用
defer 不阻塞（后台下载） HTML 解析完后 保证 需要操作 DOM 的脚本（推荐！）
async 不阻塞（后台下载） 下载完立即 不保证 独立脚本（如统计、广告）
body 底部 页面先渲染 HTML 解析完后 保证 简单可靠的小项目

推荐写法

<!-- 现代方案（推荐）：head + defer -->
<head>
  <script src="app.js" defer></script>
</head>

<!-- 传统方案（也 OK）：body 底部 -->
<body>
  ...页面内容...
  <script src="app.js"></script>
</body>

方式三：外部 src 引用

外部引用的写法是 <script src="app.js"></script>， 但由于本地文件跨域限制，这里不好做完整演示。你学过的知识已经够了：

  • 独立的 .js 文件只写 JS 代码，不用写 script 标签
  • HTML 通过 <script src="..."> 引入
  • 用了 src 属性后，标签内部的代码会被忽略
  • 这是生产环境最常用的方式
