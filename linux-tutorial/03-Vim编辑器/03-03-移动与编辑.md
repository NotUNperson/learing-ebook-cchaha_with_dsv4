# 03-03 移动与编辑：像弹钢琴一样打字

## 本节你会学到什么

- 使用 hjkl 代替方向键移动光标
- 使用 w、b、e 按单词跳跃
- 使用 0、$、^ 在行内快速定位
- 使用 gg、G、行号 G 在文件间跳转
- 使用 dd、yy、p 进行剪切复制粘贴
- 使用 u 和 Ctrl+r 撤销与重做
- 使用 . 重复上一次修改

## 正文

### 一、先讲一个钢琴指法的故事

想象一个钢琴初学者。他坐在钢琴前，想弹一个小节的旋律。每次弹下一个音符后，他都要低头看键盘，找到下一个音的位置，抬起手，移过去，再落下。弹一个小节能花十秒钟。

而一个训练有素的钢琴家呢？他的手放在琴键上就不再移动——手指天然地知道哪个键在哪里。弹完一个音，下一个音的手指已经准备好了。他的手从来不需要离开钢琴去"找"下一个位置，因为**每一根手指都有自己的地盘**。

**Vim 的移动命令就是帮你把双手固定在键盘主键区不动。** 在普通编辑器里，你的右手要不断离开主键区去按方向键、Home、End、PageUp、PageDown。每次手离开再回来，你都在浪费时间——就像钢琴初学者每次都要低头找琴键一样。

**生活类比**：普通编辑器的方向键就像你用食指一个一个戳手机屏幕打字。而 Vim 的 hjkl 移动就像你用十指盲打——手指不用看键盘，不用离开基准位置，速度极快。

### 二、hjkl：你的"方向键"

Vim 使用 `h`、`j`、`k`、`l` 四个键来代替方向键：

```
         k (上)
         |
h (左) --+-- l (右)
         |
         j (下)
```

| 按键 | 方向 | 记忆技巧 |
|------|------|---------|
| `h` | 向左 | h 是最左边的键 |
| `j` | 向下 | j 看起来像向下的箭头（下面有个弯钩） |
| `k` | 向上 | k 看起来像向上的箭头 |
| `l` | 向右 | l 在最右边 |

**为什么是这些键？** 因为这是你右手在键盘上的"基准位置"——食指在 j、中指在 k、无名指在 l、食指左移一格到 h。手指几乎不用移动，光标就已经动了。而方向键在键盘的最右下角，你得把整个右手移过去（这是很大的位移）。

**具体操作**（以下所有操作都在 Normal 模式下）：

打开 Vim，输入一些文字。不用方向键，试着用 hjkl 移动光标：

```
按 h → 光标向左移动一个字符（屏幕无变化，只是光标位置变了）
按 j → 光标向下移动一行
按 k → 光标向上移动一行
按 l → 光标向右移动一个字符
```

刚开始你可能会不习惯，手会不由自主地去摸方向键。没关系，这是正常的。坚持 2-3 天 hjkl 就会变成你的本能。

**一个小技巧**：如果你想强制自己不用方向键，可以在 .vimrc 里把方向键禁用：

```vim
" 禁用方向键，强迫自己用 hjkl
noremap <Up> <Nop>
noremap <Down> <Nop>
noremap <Left> <Nop>
noremap <Right> <Nop>
```

不过这不推荐给纯新手——先习惯 hjkl，等你想"戒掉"方向键的时候再加。

### 三、w、b、e：按"词"跳跃

hjkl 按单个字符移动，在一行文字里走太慢了。Vim 提供了按"单词"移动的命令：

| 按键 | 作用 | 记忆技巧 |
|------|------|---------|
| `w` | 跳到下一个单词的开头 | w = word（单词） |
| `b` | 跳到上一个单词的开头 | b = back（后退） |
| `e` | 跳到当前/下一个单词的结尾 | e = end（结尾） |

**演示文本**：

```
The quick brown fox jumps over the lazy dog.
```

假设光标在 `quick` 的 `q` 上：

- 按 `w` → 光标跳到 `brown` 的 `b`（下一个词的开头）
- 再按 `w` → 光标跳到 `fox` 的 `f`
- 按 `b` → 光标跳到 `brown` 的 `b`（上一个词的开头）
- 按 `e` → 光标跳到 `brown` 的 `n`（当前词的结尾）
- 再按 `e` → 光标跳到 `fox` 的 `x`（下一个词的结尾）

配合数字可以一次跳多个词：

- `3w` → 向前跳 3 个单词
- `5b` → 向后跳 5 个单词

**Vim 对"单词"的定义**：

Vim 区分两种"单词"：

1. **word**（小写 w、b、e）：由字母、数字、下划线组成。标点符号各算一个独立的 word。
2. **WORD**（大写 W、B、E）：由空格分隔的任何非空字符序列。

举例说明差异：

```
hello-world
```

- 对于小写 `w`：这是两个 word —— `hello` 和 `world`，因为 `-` 不是字母。按 `w` 会先跳到 `-`，再跳到 `w`。
- 对于大写 `W`：这是一个 WORD。按 `W` 会直接跳过去，到下一个空格后的第一个字符。

**日常建议**：大多数情况下用小写 `w`、`b`、`e` 就够了。当你发现标点符号在"碍事"的时候，改用大写 `W`、`B`、`E`。

### 四、行内快速跳转：0、$、^、f、t

在一行之内，你还会用到这些快速定位：

| 按键 | 作用 | 记忆技巧 |
|------|------|---------|
| `0` | 跳到行首（第一个字符） | 0 是开始，行首是开始 |
| `^` | 跳到本行第一个非空字符 | 正则里的 ^ 就是行首 |
| `$` | 跳到行尾 | 正则里的 $ 就是行尾 |
| `f` + 字符 | 向前查找字符，光标停在它上面 | f = find |
| `F` + 字符 | 向后查找字符 | |
| `t` + 字符 | 向前查找字符，光标停在它前面 | t = till（直到...之前） |
| `T` + 字符 | 向后查找字符，光标停在它后面 | |

**具体操作示例**：

```
文本:    Hello world! How are you?
              ^
光标在 world 的 w 上
```

- 按 `0` → 光标跳到 `H`（行首）
- 按 `$` → 光标跳到 `?`（行尾）
- 按 `^` → 光标跳到第一行的 `H`（第一个非空字符——这里 0 和 ^ 位置一样，但如果行首有空格就不一样了）
- 按 `f!` → 光标跳到 `!`（向前找感叹号）
- 按 `fw` → 光标跳到下一个 `w`（即 world 的 w）
- 按 `t?` → 光标跳到 `?` 前面的一个字符（即 `u`）

**`f` 和 `t` 的强大之处**：在一行文字里精确定位。比如你想删掉从光标到下一个逗号之间的所有内容：`dt,`（delete till comma）。想复制从光标到行尾的内容：`y$`。想修改从光标到下一个引号：`ct"`。

### 五、文件级移动：gg、G、Ctrl+f/b/d/u

| 按键 | 作用 | 记忆技巧 |
|------|------|---------|
| `gg` | 跳到文件开头 | g 就是 go，两个 g 就是 go to the very beginning |
| `G` | 跳到文件最后一行 | G 是大写的 go，大写的跳到最远 |
| `42G` | 跳到第 42 行 | 数字 + G |
| `:42` | 也是跳到第 42 行 | Command 模式方式 |
| `Ctrl+f` | 向下翻一页 | f = forward |
| `Ctrl+b` | 向上翻一页 | b = backward |
| `Ctrl+d` | 向下翻半页 | d = down |
| `Ctrl+u` | 向上翻半页 | u = up |

**一个生活场景**：你打开了一个 500 行的代码文件，报错信息说第 312 行有问题。在普通编辑器里，你要么滚鼠标中轮（滚半天），要么看编辑器有没有"跳转到行"的快捷键（很多编辑器 Ctrl+G）。在 Vim 里：`312G`，四个键搞定。或者 `:312` 回车。

### 六、编辑命令：dd、yy、p、x、u

到现在为止我们只讲了移动——但光移动不够，移动是为了"到达编辑地点"。下面是最基本的编辑命令：

#### `dd` — 删除（剪切）当前行

按 `dd` → 光标所在的整行消失。但这行不是真的"消失"了——它被存进了 Vim 的内部剪贴板（叫"寄存器"），你可以粘贴回来。

- `5dd` → 删除（剪切）5 行，包括当前行和下面 4 行

**生活类比**：`dd` 就像你用剪刀沿着行线把整行剪下来。剪下来的纸片你没有扔进垃圾桶——你暂时拿在了左手，准备贴到别处。

#### `yy` — 复制当前行

按 `yy` → 当前行被复制到剪贴板。屏幕上没有任何变化（因为复制不会改变文本），但这行已经在你的"左手"里了。

- `3yy` → 复制 3 行
- `y$` → 复制从光标到行尾
- `yw` → 复制一个单词

**注意**：`y` 在很多 Vim 教程里被叫做 "yank"。yank 是美式英语里"猛拉一下"的意思，在 Vim 语境下就是"把文本拉进剪贴板"。

#### `p` 和 `P` — 粘贴

- `p` → 在光标之后粘贴（below）
- `P` → 在光标之前粘贴（above）

**特别说明**：如果你 `dd` 删除了一整行，`p` 会把它粘贴到光标的**下一行**。如果你 `dw` 删除的是一个单词，`p` 会把它粘贴到光标的**后面**。

#### `x` — 删除光标下的字符

- `x` → 删除光标所在的单个字符（相当于退格键... 差不多）
- `5x` → 删除 5 个字符

#### `u` 和 `Ctrl+r` — 撤销和重做

- `u` → 撤销上一次修改
- `Ctrl+r` → 重做（把刚才撤销的操作恢复回来）

**Vim 的撤销非常强大**：它不只是"撤销一次按键"。在 Insert 模式下输入的一大段文字、在 Normal 模式下的一连串编辑——每次你从 Insert 模式回到 Normal 模式，这一整段编辑被当作"一步"来撤销。

举例：
1. 你按 `i` 进入 Insert 模式
2. 输入了三行文字
3. 按 `Esc` 回到 Normal 模式
4. 这整个"输入三行文字"在 Vim 看来是**一步操作**
5. 按 `u` 一次，三行全都撤销了

这个特性非常实用：如果你搞砸了一大段编辑，按 `u` 一次就够了，而不是像某些编辑器那样要按 Ctrl+Z 几十次。

### 七、`.` — 重复上一次修改

这是 Vim 最神奇但也最容易被忽视的命令之一。点号 `.` 会**重复你上一次做的修改操作**。

**什么算"修改操作"？** 任何改变了文本内容的操作：删除、插入、替换、缩进变更等等。移动不算（单纯的 hjkl、w、b 不会被重复）。

**场景示例**：

你有一个 Python 文件，里面很多行末尾都少了分号（Python 不需要分号，但假设你需要加）。

文本：
```
name = "Alice"
age = 30
city = "New York"
```

你想给每行后面加一个分号。

操作流程：
1. 光标在 name 那一行。按 `A`（跳到行尾并进入 Insert 模式），输入 `;`，按 `Esc`
2. 光标移到下一行（按 `j`）
3. 按 `.`（重复上一操作——跳到行尾并加分号）
4. 光标再移到下一行（`j`），再按 `.`
5. 如此重复，每行一个 `j` + `.` 就改完了

再一个场景：你需要在多处删除一个单词 "old"。你可以搜索到它，然后 `dw` 删除。找到下一个匹配按 `n`，然后按 `.`（重复 `dw`）。再 `n`，再 `.`。非常快速。

### 八、组合技：动词 + 名词

现在来看 Vim 语法最精妙的部分。你的编辑操作可以由"动词 + 动作范围"组成。

**动词（操作）**：
| 按键 | 操作 | 
|------|------|
| `d` | 删除（delete） |
| `y` | 复制（yank） |
| `c` | 修改（change）——删除并进入 Insert 模式 |
| `>` | 增加缩进 |
| `<` | 减少缩进 |
| `=` | 自动格式化缩进 |

**动作范围**（就是上面学的那些移动命令）：
| 按键 | 范围 |
|------|------|
| `w` | 到下一个词的开头 |
| `e` | 到当前/下一个词的结尾 |
| `$` | 到行尾 |
| `0` | 到行首 |
| `G` | 到文件尾 |
| `gg` | 到文件头 |

**组合出来**：

| 命令 | 含义 | 中文 |
|------|------|------|
| `dw` | delete word | 删除一个词 |
| `d$` | delete to end of line | 删到行尾 |
| `d0` | delete to beginning of line | 删到行首 |
| `dgg` | delete to beginning of file | 删到文件开头 |
| `dG` | delete to end of file | 删到文件末尾 |
| `yw` | yank word | 复制一个词 |
| `y$` | yank to end of line | 复制到行尾 |
| `cw` | change word | 修改一个词（删掉并等你输入新词） |
| `c$` | change to end of line | 修改到行尾 |
| `c2w` | change 2 words | 修改 2 个词 |
| `>>` | indent line | 当前行增加一级缩进 |
| `>3j` | indent current + next 3 lines | 4 行增加缩进（一个常用的格式化操作） |

**学习这些的关键**：不要背。你只需要记住动词和动作范围各自是一个"积木块"，然后像个孩子搭积木一样自由组合。你学会了 `dw`（删除一个词），自然就知道 `cw`（修改一个词）和 `yw`（复制一个词），因为它们只是换了动词。

### 九、更高级的"文本对象"

除了基于"移动范围"的操作，Vim 还有"文本对象"（text objects）这个概念。文本对象不依赖光标位置——它们直接选中**语义上的文本块**。

| 命令（动词 + 对象） | 含义 | 示例 |
|---------------------|------|------|
| `diw` | delete inner word | 删除光标所在词（无论光标在词中哪个位置） |
| `ciw` | change inner word | 修改光标所在词 |
| `yiw` | yank inner word | 复制光标所在词 |
| `di"` | delete inside quotes | 删除双引号内的内容 |
| `ci(` | change inside parentheses | 修改括号内的内容 |
| `da"` | delete a quoted string (including quotes) | 删除引号及其内容 |
| `dit` | delete inside HTML tag | 删除 HTML 标签内的内容 |
| `cip` | change inside paragraph | 修改当前段落 |

**`i` 和 `a` 的区别**：
- `i`（inner）：只操作**内部**内容，不包括包围符号
- `a`（a/an）：操作内部内容**加上**包围符号

举例：
```
文本: "Hello world"
       ^
       光标在双引号里面任何位置
```

- `ci"` → 引号内的 "Hello world" 被删除，光标在引号之间等待你输入新内容。引号**保留**。
- `ca"` → 整个 "Hello world"（包括引号）被删除。引号也**没了**。

**生活类比**：`i`（inner）就像吃橘子——你只吃里面的果肉，橘子皮留着。`a`（a/an）就像一口吞整个橘子——果肉和橘子皮都不剩。

### 十、初学阶段建议的学习路线

不要试图一天掌握所有命令。以下是一个建议的渐进学习计划：

**第 1 天**：专注 hjkl。不用方向键，只用 hjkl 移动光标。
**第 2 天**：加入 w、b、e。开始按"词"移动。
**第 3 天**：加入 0、$、gg、G。能在一行内和文件间快速跳转。
**第 4 天**：加入 dd、yy、p、u。能剪切、复制、粘贴、撤销。
**第 5 天**：加入 cw、ciw、c$。学会"修改"操作（change）。
**第 6 天**：加入 f、t。学会在一行内精确定位。
**第 7 天**：加入 .（重复）、/（搜索）。学会快速执行重复操作。

一周后，你对 Vim 的操作已经有了最基本的肌肉记忆。再往后，每天学 2-3 个新命令就行了。

## 动手试试

1. 用 Vim 打开一个新文件：`vim move_practice.txt`。
2. 进入 Insert 模式，输入 15 行文字。可以随便打字，或者复制粘贴一段文章。
3. 按 Esc 回到 Normal 模式。练习 hjkl 移动光标。试着只用这四个键在文字中游走 2 分钟。
4. 按 `gg` 跳到文件开头，按 `G` 跳到文件末尾。来回多试几次。
5. 尝试 `w` 和 `b` 在单词间跳跃。注意标点符号是怎么影响 `w` 和 `b` 的行为的。
6. 用 `dd` 删除一行，然后用 `p` 粘贴到别处。再用 `yy` 复制一行，粘贴。
7. 按 `u` 撤销几次操作，然后用 `Ctrl+r` 重做回来。
8. 找一个单词，用 `ciw` 把它改成别的单词。体会"不需要先选中再删除再输入"的效率。
9. 在一行文字中找到一个字符（比如逗号），用 `f,` 把光标跳过去。再用 `t,` 跳过去——注意这两种跳法的区别（一个是落在逗号上，一个是落在逗号前）。

## 本节小结

Vim 的移动和编辑命令遵循"动词+名词"的语法，你不是在背快捷键，而是在用一门编辑语言表达意图。hjkl 让你手指不离键位，w/b 让你按词跳跃，f/t 让你精准定位，dd/yy/p 让你操作文本，. 让你重复一切——这些组合在一起，让你的编辑速度达到手指思考的速度。

你能在单个文件内自由移动和编辑了。但在实际工作中，你经常需要在文件内做搜索替换，以及同时打开多个文件工作。下一节学习 Vim 的搜索、替换和多文件操作。
