# 03-02 四种模式：Vim 的"汽车档位"

## 本节你会学到什么

- 理解 Vim 四种模式各自的作用
- 掌握模式之间的切换方法（特别是如何退出 Insert 模式）
- 理解为什么 Normal 模式是"默认档位"
- 在 Insert 模式下进行基本的文字输入
- 使用 Visual 模式进行文本选择
- 使用 Command 模式执行保存、退出、设置等操作

## 正文

### 一、先讲一个汽车档位的故事

假设你第一次坐进一辆手动挡汽车。你面前有 P（驻车）、R（倒车）、N（空挡）、D（前进）四个档位。你的教练告诉你：

- 想往前开，挂 D 档，踩油门。
- 想倒车，挂 R 档，控制油门。
- 等红灯，挂 N 档，脚放刹车上。
- 停车熄火，必须挂 P 档。

刚开始你可能会手忙脚乱——有时挂着 P 档踩油门（车不动），有时挂着 R 档往前冲了一下（吓一跳）。但开了一个月之后，你换挡已经不用想了，完全是肌肉记忆。

**Vim 的模式就是你的"档位"。** Vim 有四种模式，每种模式下同样的按键会有不同的效果——就像同一个油门踏板，在 D 档是前进，在 R 档是倒车。

四种模式分别是：

| 模式 | 英文名 | 作用 | 类比 |
|------|--------|------|------|
| Normal（普通模式） | Normal mode | 浏览、导航、执行编辑命令 | P 档 + D 档的混合——默认停放状态，但随时准备行动 |
| Insert（插入模式） | Insert mode | 输入文字 | D 档——踩油门往前走，输入文字 |
| Visual（可视模式） | Visual mode | 选中文本块 | 用手在纸上圈出一段文字准备剪下来 |
| Command（命令模式） | Command-line mode | 执行保存、退出、搜索、替换等命令 | 中控台——调整空调、导航、音响 |

### 二、Normal 模式：你的"默认档位"

打开 Vim 后，你首先进入的就是 Normal 模式。屏幕左下角什么都没有（或显示文件名），不会像 Insert 模式那样出现 `-- INSERT --`。

**生活类比**：Normal 模式就像你的手机主屏幕。你不总是打开了某个 App——你大部分时间在主屏幕上，看到通知就点进去处理，处理完就返回主屏幕。Normal 模式就是 Vim 的"主屏幕"，大部分时间你都待在这里。

在 Normal 模式下，你在键盘上按的每一个字母键，都是一个**命令**，而不是输入文字。比如：

- 按 `h` —— 光标向左移动一格（不是输入字母 h）
- 按 `j` —— 光标向下移动一行
- 按 `x` —— 删除光标下的字符
- 按 `dd` —— 删除当前行
- 按 `u` —— 撤销上一步操作

**初学者最常见的困惑**：打开 Vim 就开始打字，但什么都没出现，或者出现奇怪的行为。原因很简单——你还在 Normal 模式（P 档），没有切换到 Insert 模式（D 档）。

**一个重要的习惯**：在 Vim 里，你**不应该长时间待在 Insert 模式**。正确的使用方式是——在 Normal 模式下移动到你想编辑的位置，然后按 `i` 进入 Insert 模式写文字，写完立即按 `Esc` 回到 Normal 模式。

```
大部分时间在 Normal 模式 → 需要输入了按 i 进 Insert → 输入完按 Esc 回 Normal → 继续移动...
```

这个节奏刚开始会觉得烦——"为什么写几个字就要按一次 Esc？"——但很快会形成肌肉记忆，就像打字时大拇指自动敲空格键一样自然。

### 三、Insert 模式：唯一的"输入文字"模式

Insert 模式是你真正输入文字的地方。进入 Insert 模式后，屏幕左下角会显示 `-- INSERT --`（或 `-- 插入 --`），这就像汽车仪表盘上显示"当前档位：D"。

从 Normal 模式进入 Insert 模式有多种方式，每种方式在"进入后光标落在哪里"这个细节上有区别：

| 按键 | 进入后光标位置 | 使用场景 |
|------|---------------|---------|
| `i` | 光标**之前**（当前位置不变） | 最常见的进入方式——想在光标当前位置插入文字 |
| `a` | 光标**之后**（光标向后移一位） | 想在光标后面追加文字——a 代表 append（追加） |
| `I` | 当前**行首**（第一个非空字符前） | 想在行首加东西——大写 I |
| `A` | 当前**行尾** | 想在行尾加东西——比如加个分号 |
| `o` | 当前行**下方另起一行** | 想写新的一行——o 代表 open a new line below |
| `O` | 当前行**上方另起一行** | 想在当前行上面插一行——大写 O |
| `s` | 删掉光标下的字符，然后进入 Insert | 想把一个字符替换成别的——s 代表 substitute |
| `S` | 删掉整行，然后进入 Insert | 想重写整行——大写 S |
| `cw` | 删掉从光标到当前词尾，然后进入 Insert | 想修改一个词——cw = change word |

我们来详细解释几个：

#### `i` vs `a` ——最常用的两个入口

假设你有一行文字，光标在字母 `l` 上（用 `|` 表示光标位置）：

```
hello world
    |
```

- 按 `i` → 进入 Insert 模式，光标位置不变。你输入的内容会出现在 `l` 之前。
- 按 `a` → 进入 Insert 模式，光标移到 `l` 之后。你输入的内容会追加在 `l` 之后。

**记忆技巧**：`i` 是 insert（插入），插入在光标前面。`a` 是 append（追加），追加在光标后面。

#### `o` 和 `O` ——换行插入

- 按 `o` → 在当前行下方新建一行，光标移到新行首，进入 Insert 模式。
- 按 `O` → 在当前行上方新建一行，光标移到新行首，进入 Insert 模式。

这两个键在写代码时极其常用——你写完一行，想写下一行的时候，按 `Esc` 回到 Normal，然后按 `o`（不用按 Enter 然后按 i），一气呵成。

#### 如何退出 Insert 模式

退出 Insert 模式回到 Normal 模式——这是每个初学者先要练熟的动作：

| 按键 | 效果 | 备注 |
|------|------|------|
| `Esc` | 回到 Normal 模式 | 最标准的方式 |
| `Ctrl+[` | 回到 Normal 模式 | 等效于 Esc，手指不用离开主键区去够 Esc 键 |
| `Ctrl+C` | 回到 Normal 模式（但不触发 InsertLeave 自动命令） | 大部分情况下和 Esc 一样，极少数情况有区别 |

**强烈建议**：如果你觉得 Esc 键太远（手指要离开主键区），可以试试 `Ctrl+[`。或者在你的终端/vim 配置里把 `jj` 映射为 Esc——快速连按两次 j 就退出 Insert 模式。很多 Vim 用户这么干。

### 四、Visual 模式：像用鼠标一样选中文本

Visual 模式让你**可视化地选择**一段文本，然后对这段文本执行操作。

**生活类比**：Visual 模式就像你在一张纸上用荧光笔划出一段文字——"这一段我要处理"。划好之后，你可以对这段文字做任何事：删除、复制、缩进、变大写、变小写。

有三种 Visual 模式：

| 按键 | 模式 | 选择方式 |
|------|------|---------|
| `v` | 字符 Visual | 按字符选择 |
| `V` | 行 Visual | 按整行选择 |
| `Ctrl+v` | 块 Visual | 按矩形块选择 |

#### `v` — 字符选择

```
按 v 进入 Visual 模式
用 h j k l 或方向键扩展选择范围
选好后，按 d 删除所选文本，或 y 复制，或 > 增加缩进，等等
```

具体步骤演示：

1. 你的文本是 "Hello beautiful world"
2. 光标在 "beautiful" 的 `b` 上
3. 按 `v`（屏幕左下角出现 `-- VISUAL --`）
4. 按 `e`（光标跳到词尾，整个 "beautiful" 被高亮选中）
5. 按 `d`（选中的文本被删除）
6. 文本变成 "Hello  world"
7. 你自动回到 Normal 模式

#### `V` — 行选择

```
按 V 进入行 Visual 模式
整行被选中
按 j/k 上下扩展，选中多行
```

这是最快的"选中整行然后操作"的方式。比如你想注释掉 5 行 Python 代码：`V` 进入行模式，按 `4j` 向下选 5 行，然后按 `>` 增加缩进。

#### `Ctrl+v` — 块选择

这是 Vim 的杀手级功能之一。它能选中一个**矩形区域**：

```
按 Ctrl+v 进入块 Visual 模式
用 h j k l 扩展矩形选区
按 I 进入插入（在块的前面统一插入）
输入内容
按 Esc 后，所有选中的行前面都出现了你输入的内容
```

**场景举例**：你有一段代码，想给 10 行每行前面都加上 `// ` 注释。用 VS Code 你得一行一行加，或者用多光标（还得按着 Alt）。在 Vim 里：`Ctrl+v` 选中 10 行的行首，`I// `（输入两个斜杠和一个空格），`Esc`。搞定。

### 五、Command 模式：下命令的地方

Command 模式（也叫命令行模式）是你对 Vim 本身发送指令的地方——保存、退出、设置选项、搜索、替换等等。

从 Normal 模式按 `:`（冒号）进入 Command 模式。屏幕左下角出现一个 `:`，光标跳到那等你输入命令。

| 命令 | 作用 | 说明 |
|------|------|------|
| `:w` | 保存（write） | 把当前内容写入磁盘 |
| `:q` | 退出（quit） | 关闭当前窗口/文件 |
| `:wq` | 保存并退出 | w + q 的组合 |
| `:q!` | 强制退出（不保存修改） | 当你改乱了想放弃所有修改时使用 |
| `:x` | 保存并退出 | 和 `:wq` 一样，但只在有修改时才写入 |
| `:e filename` | 打开另一个文件 | e = edit |
| `:help` | 打开帮助 | 在任何时候按 F1 也行 |
| `:set nu` | 显示行号 | nu = number |
| `:set nonu` | 隐藏行号 | |
| `:%s/old/new/g` | 全局替换 | % 是整个文件的意思 |
| `:!command` | 执行外部 shell 命令 | 比如 `:!ls` 列出当前目录文件 |

#### 搜索：`/` 和 `?`

严格来说，`/` 和 `?` 启动的搜索不算 Command 模式（它也有一个自己的小名字叫 search mode），但用法和 Command 模式很像——也是在屏幕底部出现输入提示：

- 按 `/` → 输入搜索词 → 按 Enter → 光标跳到下一个匹配处
- 按 `?` → 输入搜索词 → 按 Enter → 光标跳到上一个匹配处
- 搜索后按 `n` 跳到下一个匹配，`N` 跳到上一个匹配

#### 命令的简写

很多命令有更短的写法：

| 完整写法 | 简写 | 效果 |
|---------|------|------|
| `:write` | `:w` | 保存 |
| `:quit` | `:q` | 退出 |
| `:edit file` | `:e file` | 打开文件 |
| `:split` | `:sp` | 水平分屏 |
| `:vsplit` | `:vs` | 垂直分屏 |

#### 在 Normal 模式下快速保存

你可以用 `ZZ`（两个大写 Z）代替 `:wq` 来保存退出。按 `ZQ` 代替 `:q!` 强制退出。这些都是 Normal 模式下的快捷键，不用打冒号。

### 六、四种模式的流程图

```
                   打开 Vim
                      |
                      v
              +--> Normal 模式 <--+
              |       |           |
        按 i/I/a/A    |     按 v/V/Ctrl+v
         o/O/s/S      |           |
              |   按 Esc     |
              v       |           v
         Insert 模式 --+    Visual 模式
                           (选好后自动回 Normal)
              
              按 : 或 /
                      |
                      v
               Command 模式
               (执行完自动回 Normal)
```

**关键要点**：
- 所有模式最终都**回到 Normal 模式**
- Normal 模式是"家"——你从这里出发，也回到这里
- Insert 模式和 Visual 模式是临时状态，办完事立刻回 Normal
- Command 模式执行一条命令后自动回 Normal

### 七、常见新手错误与纠正

#### 错误一：在 Normal 模式下狂敲键盘

症状：打开 Vim 就打字，结果什么都写不上，各种奇怪的跳转和执行发生。

原因：你在 Normal 模式下——字母键是命令，不是输入。

解决：按 `i` 进入 Insert 模式再打字。如果已经搞乱了，按 `u` 多次撤销，或者 `:q!` 强制退出重新打开。

#### 错误二：忘带自己还在 Insert 模式里

症状：写了一些文字，然后用 hjkl 想移动光标，结果屏幕上出现了 hjkl 这四个字母。

原因：你在 Insert 模式下按了 hjkl，它们被当作文本输入了。

解决：先按 `Esc` 回到 Normal 模式，再按 hjkl 移动。删掉刚才输入的 hjkl。

**快速检查你当前在哪个模式**：看屏幕左下角。
- 什么都没有 → Normal 模式
- 显示 `-- INSERT --` → Insert 模式
- 显示 `-- VISUAL --` → Visual 模式
- 显示 `-- VISUAL LINE --` → 行 Visual 模式
- 显示 `-- VISUAL BLOCK --` → 块 Visual 模式
- 显示 `:` → Command 模式

#### 错误三：改变主意了但不知道怎么办

如果你在任何模式中不知道该怎么办，最安全的做法是：
1. 按几次 `Esc`（确保回到 Normal 模式）
2. 如果改乱了，按 `u` 撤销
3. 如果彻底不行了，`:q!` 退出

**Esc 是你的安全词**。无论发生什么，Esc 都能带你回到 Normal 模式这张安全网。

### 八、一个实用的节奏练习

下面是一段文本，你可以打开 Vim 跟着做，感受四种模式的切换节奏：

```
原始文本（你要创建这个文件）：
The quick brown fox jumps over the lazy dog.

任务：把 brown 改成 red，在 jumps 前面加 quickly，删掉 over。
```

**Vim 操作流程**：

1. 终端执行 `vim fox.txt`
2. 当前状态：**Normal 模式**（左下角无任何提示）
3. 按 `i` → 进入 **Insert 模式**（注意左下角出现 `-- INSERT --`）
4. 输入 `The quick brown fox jumps over the lazy dog.`
5. 按 `Esc` → 回到 **Normal 模式**
6. 按 `0`（跳到行首），按 `/brown` 然后回车（搜索 brown）→ **Normal 模式**中执行导航
7. 光标现在在 b 上。按 `cw` → 文本中 brown 被删除，同时进入 **Insert 模式**
8. 输入 `red`，按 `Esc` → 回到 **Normal 模式**
9. 按 `/jumps` 回车（搜索 jumps）→ **Normal 模式**
10. 按 `e`（跳到词尾），按 `a` → **Insert 模式**，输入一个空格和 `quickly`，按 `Esc` → **Normal 模式**
11. 按 `/over` 回车（搜索 over），按 `dw`（删除一个词）→ **Normal 模式**
12. 输入 `:wq` → **Command 模式**，保存并退出

完成后的文本：`The quick red fox jumps quickly the lazy dog.`

整个过程你切换了多次模式，但每次切换都是瞬间的。这就是 Vim 的正常节奏。

## 动手试试

1. 打开 Vim：`vim mode_practice.txt`。左下角没有 `-- INSERT --`，确认你在 Normal 模式。
2. 按 `i` 进入 Insert 模式。左下角出现 `-- INSERT --`。输入 5 行文字，每行随便写。按 `Esc` 回到 Normal 模式。左下角 `-- INSERT --` 消失。
3. 在 Normal 模式下，用 `hjkl` 移动光标。感受被解放的感觉——不用去按方向键了。
4. 按 `V` 进入行 Visual 模式。用 `j` 选中几行，然后按 `d` 删除。按 `u` 撤销。
5. 按 `Ctrl+v` 进入块 Visual 模式。用 `j` 和 `l` 选中一个矩形区域。按 `d` 删除它。按 `u` 撤销。
6. 按 `:` 进入 Command 模式，输入 `wq` 保存退出。

## 本节小结

Vim 的四种模式就像汽车的四个档位——Normal 是默认的"待命"状态，Insert 是"输入"状态，Visual 是"选择"状态，Command 是"下命令"状态。切换模式的熟练程度直接决定了你使用 Vim 的效率。牢记：Esc 永远是回家的路。
