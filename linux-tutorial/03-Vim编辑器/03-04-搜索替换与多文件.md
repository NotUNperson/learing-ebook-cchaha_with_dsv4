# 03-04 搜索替换与多文件：Vim 的生产力升级

## 本节你会学到什么

- 使用 `/` 和 `?` 在文件中正向和反向搜索
- 使用 `n` 和 `N` 在搜索结果间跳转
- 使用 `:%s/old/new/g` 做全局查找替换
- 理解替换命令的各种标志和范围限定
- 使用 `:vsp` 和 `:split` 分屏同时编辑多个文件
- 使用 `:e`、`:bn`、`:bp` 在多个文件之间切换

## 正文

### 一、搜索：在文字的海洋里找到坐标

#### 1.1 基本搜索：`/` 和 `?`

在 Normal 模式下，按 `/` 键，屏幕左下角会出现一个 `/` 符号，光标跳到那里。现在你可以输入搜索词了。

```
按 / → 屏幕左下角出现 /
输入 search_term → 屏幕左下角变为 /search_term
按 Enter → 光标跳到下一个匹配处
```

**正向搜索（`/`）**：从光标当前位置向下搜索。按 `n` 跳到下一个匹配，`N` 跳到上一个匹配。

**反向搜索（`?`）**：从光标当前位置向上搜索。按 `n` 跳到上一个匹配（因为方向是反的，n 也反了），`N` 跳到下一个匹配。

```bash
# 在 Vim 中实际操作：
按 /hello 回车  → 跳到下一个 "hello"
按 n           → 跳到再下一个 "hello"
按 N           → 回到上一个 "hello"
按 ?world 回车 → 向上搜索 "world"
按 n           → 跳到上一个 "world"
按 N           → 跳到下一个 "world"
```

**生活类比**：`/` 搜索就像在一本书里从当前页往下翻找关键词，`?` 就是从当前页往上翻找。`n` 就是"next"（下一个），`N` 就是"上一个"。

#### 1.2 搜索的高亮显示

默认情况下，搜索到的所有匹配都会被高亮显示。这个高亮有时很有用（一眼看到所有匹配在哪），有时很烦（到处都是黄色高亮，影响阅读）。

```vim
" 关闭搜索高亮（当前这次）
:nohlsearch    " 或者简写 :noh

" 永久关闭搜索高亮（加到 .vimrc 里）
set nohlsearch

" 或者保留高亮功能，但加一个快捷键一键关闭
nnoremap <leader><space> :nohlsearch<CR>
```

当你完成一次搜索后，高亮会一直保留，直到你进行下一次搜索或手动关闭。这其实是一个很有用的特性——它让你一目了然地看到你要找的东西在文件的哪些位置。

#### 1.3 搜索的大小写敏感性

```vim
" 在 .vimrc 中设置搜索时不区分大小写
set ignorecase

" 但如果在搜索词里包含大写字母，就自动区分大小写（智能大小写）
set smartcase
```

这两个设置搭配使用效果最好：
- `set ignorecase`：搜索 "hello" 时也匹配 "Hello"、"HELLO"
- `set smartcase`：但如果搜索词里有大写字母（比如 "Hello"），就只精确匹配 "Hello"

这样你大部分时候用小写搜索就行，偶尔需要精确匹配时输一个大写字母即可。

#### 1.4 搜索高亮词之间的快速跳转

除了 `n` 和 `N`，你还可以：

- 把光标放在一个词上，按 `*` → 自动搜索这个词（正向），相当于帮你打了 `/word` 然后回车
- 按 `#` → 自动搜索这个词（反向），相当于帮你打了 `?word` 然后回车

这两个按键是 Vim 用户的日常必备。你看到一个变量名，想知道它在哪被用到——光标放上去按 `*`，所有出现的地方都被高亮了。

### 二、查找替换：Vim 的"批量修改引擎"

#### 2.1 基本语法

Vim 的替换命令语法和 sed 很像：

```vim
:[range]s/old/new/[flags]
```

- `range`：范围（哪几行做替换）
- `s`：substitute（替换）
- `old`：要被替换的文本（支持正则）
- `new`：替换后的文本
- `flags`：标志（g=全局, c=确认, i=忽略大小写）

#### 2.2 替换范围

| 命令 | 范围 | 说明 |
|------|------|------|
| `:s/old/new/` | 当前行 | 只替换当前行的第一个匹配 |
| `:s/old/new/g` | 当前行 | 替换当前行的所有匹配 |
| `:5,10s/old/new/g` | 第 5 到第 10 行 | |
| `:.,$s/old/new/g` | 当前行到最后一行 | `.` 是当前行，`$` 是最后一行 |
| `:%s/old/new/g` | 整个文件 | `%` 是整个文件 |
| `:.,.+4s/old/new/g` | 当前行及下面 4 行 | |

**最常用的就是 `:%s/old/new/g`**——对整个文件做全局替换。这是你在 Vim 里最高频使用的 Command 模式命令之一。

#### 2.3 替换标志

| 标志 | 含义 | 示例 |
|------|------|------|
| `g` | 全局（替换一行中所有匹配） | `:%s/foo/bar/g` |
| `c` | 每次替换前确认 | `:%s/foo/bar/gc` |
| `i` | 忽略大小写 | `:%s/foo/bar/gi` |
| `I` | 区分大小写 | `:%s/foo/bar/gI` |
| `n` | 不实际替换，只报告匹配数量 | `:%s/foo/bar/gn` |

`c`（confirm）标志非常实用。当你不能 100% 确定每个匹配都要替换时，加 `c` 让 Vim 每次找到匹配都问你：

```
replace with bar (y/n/a/q/l/^E/^Y)?
```

选项含义：
- `y` → 替换这一个
- `n` → 跳过这一个
- `a` → 替换剩下的所有（不再问了）
- `q` → 退出替换
- `l` → 替换这一个后退出（last）
- `Ctrl+E` → 向下滚动一行
- `Ctrl+Y` → 向上滚动一行

**生活类比**：不加 `c` 的替换就像鼠标全选然后替换——快，但可能改错。加 `c` 就像一个人拿着两份清单，一个一个核对——慢一点，但稳妥。

#### 2.4 替换实战演示

```vim
" 整个文件: 把所有 foo 换成 bar
:%s/foo/bar/g

" 当前行: 把所有 foo 换成 bar
:s/foo/bar/g

" 第 10 到第 50 行: 把 foo 换成 bar
:10,50s/foo/bar/g

" 整个文件: 把 foo 换成 bar，但每次确认
:%s/foo/bar/gc

" 统计文件中出现了多少次 foo（不替换，只计数）
:%s/foo//gn

" 把行首的空格/制表符都删掉
:%s/^\s*//g

" 把行尾的空格都删掉
:%s/\s*$//g

" 在每行行首加上一个 # 注释
:%s/^/# /g

" 删除所有行尾的 # 注释
:%s/\s*#.*$//g

" 把所有连续的空格压缩成一个空格
:%s/\s\+/ /g
```

#### 2.5 替换中的正则引用

在替换的 `new` 部分，你可以引用匹配到的内容：

| 符号 | 含义 |
|------|------|
| `&` | 整个匹配文本 |
| `\0` | 整个匹配文本（同 &） |
| `\1` - `\9` | 第 N 个括号中匹配的内容 |
| `\r` | 换行符 |
| `\t` | 制表符 |

```vim
" 给所有数字加方括号
:%s/\d\+/[&]/g
" 123 → [123]

" 交换用逗号分隔的两列
:%s/\(.*\),\(.*\)/\2,\1/g
" hello,world → world,hello

" 给每行加双引号
:%s/.*/"&"/g
```

### 三、多文件操作：同时打开多个文件

在实际工作中，你很少只编辑一个文件。你可能需要同时查看一个 .h 头文件和一个 .c 源文件，或者同时修改配置文件和日志文件。

#### 3.1 在 Vim 中打开文件

```bash
# 启动时打开多个文件
vim file1.txt file2.txt file3.txt

# 启动后在 Vim 内部打开另一个文件
:e another_file.txt

# 重新加载当前文件（如果文件被其他程序修改了）
:e

# 打开文件但强制重新读取（放弃当前修改）
:e!
```

#### 3.2 在多个文件之间切换

启动时给了多个文件名，或者在 Vim 里用 `:e` 打开了多个文件后，Vim 把它们放在一个"缓冲区列表"（buffer list）里：

```vim
" 查看所有打开的缓冲区
:ls
" 输出示例：
"   1 %a   "file1.txt"           第 1 行
"   2 #    "file2.txt"           第 0 行
"   3      "file3.txt"           第 0 行

" 切换到下一个缓冲区
:bn          " buffer next

" 切换到上一个缓冲区
:bp          " buffer previous

" 切换到第 N 号缓冲区
:b3          " 切换到缓冲区 3

" 切换到最近编辑过的那个缓冲区（在两个文件间快速切换）
Ctrl+^       " 或者 :b#
```

`Ctrl+^`（按法是 Ctrl+Shift+6，因为 ^ 在 6 上）是非常常用的快捷键——在两个最近使用的文件之间来回切换，类似于 Alt+Tab 切换窗口。

#### 3.3 关闭缓冲区

```vim
" 关闭当前缓冲区（相当于关闭当前文件）
:bd          " buffer delete

" 关闭第 N 号缓冲区
:bd 3

" 关闭当前缓冲区但保留分屏窗口
:bp|bd #    " 先切换到上一个缓冲区，再关掉现在这个
```

### 四、分屏：同时看多个文件

分屏是 Vim 的另一大特色功能。你可以把 Vim 的窗口切成两块（或多块），同时查看不同的文件。

#### 4.1 水平分屏与垂直分屏

```vim
" 水平分屏（上下排列），打开新文件
:split newfile.txt        " 简写 :sp newfile.txt

" 水平分屏，打开同一个文件（看不同位置）
:split                    " 或 Ctrl+w s

" 垂直分屏（左右排列），打开新文件
:vsplit newfile.txt       " 简写 :vs newfile.txt

" 垂直分屏，打开同一个文件
:vsplit                   " 或 Ctrl+w v
```

#### 4.2 在分屏之间移动

所有分屏相关的操作都以 `Ctrl+w` 开头（w = window）：

| 快捷键 | 作用 |
|--------|------|
| `Ctrl+w h` | 移到左边的窗口 |
| `Ctrl+w j` | 移到下面的窗口 |
| `Ctrl+w k` | 移到上面的窗口 |
| `Ctrl+w l` | 移到右边的窗口 |
| `Ctrl+w w` | 循环切换到下一个窗口 |
| `Ctrl+w =` | 让所有窗口等宽/等高 |
| `Ctrl+w >` | 增加当前窗口宽度 |
| `Ctrl+w <` | 减少当前窗口宽度 |
| `Ctrl+w +` | 增加当前窗口高度 |
| `Ctrl+w -` | 减少当前窗口高度 |

**生活类比**：分屏就像你把两张纸并排放在桌面上。左边的纸写着需求文档，右边的纸是你正在写的代码。你不需要把左边的纸翻过来才能看右边的——两张都在视野里，目光左右扫就行。

#### 4.3 实用分屏场景

**场景一**：同时查看头文件和源文件

```vim
" 打开源文件
vim main.c

" 在右边垂直分屏打开头文件
:vsp main.h

" 现在左右两屏并排显示，可以在两边独立滚动
" Ctrl+w l 跳到右边
" Ctrl+w h 跳回左边
```

**场景二**：看同一个文件的不同部分

```vim
" 打开一个长文件
vim long_config.conf

" 水平分屏（自动是同一个文件）
:sp

" 跳到上面窗口的另一段配置
" 现在上下两个窗口可以看文件的不同部分
" 修改其中一个窗口，另一个窗口也会同步更新（因为是同一个文件）
```

**场景三**：启动时直接分屏打开两个文件

```bash
# -O 垂直分屏，-o 水平分屏
vim -O file1.txt file2.txt
vim -o file1.txt file2.txt
```

#### 4.4 标签页

除了分屏，Vim 也支持标签页（tabs），和浏览器的标签页差不多：

```vim
" 在新标签页中打开文件
:tabe filename        " tab edit

" 打开一个新的空白标签页
:tabnew

" 在标签页之间切换
gt                    " 下一个标签页
gT                    " 上一个标签页
:tabn                 " tab next
:tabp                 " tab previous

" 关闭当前标签页
:tabc                 " tab close
```

标签页 + 分屏可以组合使用——每个标签页里可以有多个分屏。

### 五、在文件间复制粘贴

当你有多个文件打开时，可以用 Vim 的寄存器来跨文件复制粘贴：

```vim
" 在文件 A 中：
" yy 复制一行（内容存入默认寄存器 ""）

" 切换到文件 B（Ctrl+^ 或 :bn）

" 在文件 B 中：
" p 粘贴

" 复制一个具名寄存器（如果需要保留多个剪贴内容）：
" "ayy   → 复制到寄存器 a
" "byy   → 复制到寄存器 b
" 切换到另一个文件
" "ap    → 粘贴寄存器 a 的内容
" "bp    → 粘贴寄存器 b 的内容
```

Vim 有 26 个具名寄存器（a-z），每个都可以独立保存内容，互不干扰。

### 六、打开目录浏览器

Vim 自带的文件浏览器叫 Netrw：

```vim
" 打开当前目录的文件浏览器
:Explore      " 简写 :E

" 在分屏中打开文件浏览器
:Sexplore     " 水平分屏，简写 :Sex
:Vexplore     " 垂直分屏，简写 :Vex

" 在文件浏览器中：
" j/k 上下移动
" Enter 打开文件/进入目录
" - 进入上级目录
" D 删除文件
" R 重命名文件
" % 创建新文件
```

这个内置的文件浏览器功能很基础，但应付简单场景足够了。如果你觉得不够用，可以在之后安装 NERDTree 插件（后面配置 Vim 的章节会讲）。

### 七、实战流程演练

最后，我们用一个完整的实战流程来串联本节的所有知识点。

**任务**：一个 Python 项目有三个文件——`main.py`、`utils.py`、`config.py`。你需要在三个文件之间来回编辑，并在 `config.py` 里做一次全局替换。

```
# 终端操作
vim main.py utils.py config.py
# Vim 打开，当前显示 main.py

# 搜索 "process" 看看这词在 main.py 里哪些地方出现了
/process 回车
# 按 n 浏览所有匹配
# 高亮显示了所有 "process"，一目了然

# 在 utils.py 里也有 "process" 需要看，切换到 utils.py
:bn
# 现在显示 utils.py
# 刚才搜索的 "process" 在这也高亮了

# 搜索 "TODO"，看看还有哪些待办事项
/TODO 回车

# 打开 config.py
:bn
# 现在显示 config.py

# 把所有的 "localhost" 替换成 "192.168.1.100"
:%s/localhost/192.168.1.100/g
# Vim 底部显示: 3 substitutions on 3 lines （替换了 3 处）

# 想同时看看 main.py 的代码——垂直分屏
:vsp main.py
# 现在左边是 config.py，右边是 main.py

# 在分屏间跳转
Ctrl+w l    # 跳到右边的 main.py
Ctrl+w h    # 跳回左边的 config.py

# 在 config.py 里有一段代码需要复制到 main.py
# 在 config.py 中，光标在你要复制的行上
"ayy         # 复制到寄存器 a
Ctrl+w l     # 跳到 main.py
"ap          # 粘贴寄存器 a 的内容

# 工作完成，保存所有文件
:wa          # write all —— 保存所有修改过的缓冲区
# 然后退出
:qa          # quit all —— 退出所有窗口
```

## 动手试试

1. 创建两个测试文件：`echo -e "line1\nline2\nline3\nhello\nhello world" > a.txt` 和 `echo -e "col1,col2\nval1,val2\nhello,world" > b.txt`。
2. 用 `vim a.txt b.txt` 打开两个文件。
3. 在 a.txt 中按 `/hello` 回车。观察匹配的高亮。按 `n` 几次。按 `N` 几次。
4. 执行 `:%s/hello/hi/g`。观察文件中的所有 hello 变成了 hi。按 `u` 撤销。
5. 执行 `:%s/hello/hi/gc`。Vim 会一个一个问你。试试 y（替换）和 n（跳过），体会确认模式。
6. 光标在 hello 这个词上，按 `*`。观察是否自动搜索了 hello。按 `n` 和 `N` 浏览。
7. 按 `:bn` 切换到 b.txt。输入 `:%s/,/ | /g` 把逗号替换成管道符。
8. 用 `:vsp a.txt` 垂直分屏打开 a.txt。用 `Ctrl+w l` 和 `Ctrl+w h` 在分屏之间跳转。
9. 在 b.txt 中用 `yy` 复制一行，`Ctrl+w h` 跳到 a.txt 窗口，`p` 粘贴。验证内容确实跨文件粘贴了。
10. 执行 `:wa` 保存所有，然后 `:qa` 退出。

## 本节小结

Vim 的搜索（`/`、`?`、`n`、`N`）让你在文件中快速定位，替换（`:%s/old/new/g`）让你批量修改内容，分屏（`:vsp`、`:sp`）和缓冲区管理（`:bn`、`:bp`）让你在多个文件间高效切换——这三项能力组合起来，让你在 Vim 里的工作流不亚于任何现代 IDE。
