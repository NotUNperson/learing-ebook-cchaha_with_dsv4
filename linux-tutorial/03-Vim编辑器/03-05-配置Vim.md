# 03-05 配置 Vim：让编辑器为你量身定做

## 本节你会学到什么

- 理解 .vimrc 配置文件的作用和位置
- 设置行号、缩进、语法高亮等常用选项
- 自定义快捷键映射
- 了解 vim-plug 插件管理器的安装和基本用法
- 写出一个适合初学者的 .vimrc 配置文件

## 正文

### 一、先讲一个搬新家的故事

你搬进一个新房子。开发商交付的房子有白墙、日光灯、标配的窗帘——能用，但总觉得缺了点什么。于是你开始装修：你换了暖色灯（看起来舒服），调了书桌的高度（用起来顺手），在墙上钉了几个挂钩挂常用工具（伸手就能拿到）。

**Vim 的默认配置就像那个刚交付的房子。它功能齐全，但一切都很"素"。** .vimrc 文件就是你的装修方案——你告诉 Vim：我想要行号、我想要自动缩进、我想要搜索时高亮、我想要按两个 j 就退出 Insert 模式。

默认情况下 Vim 没有 .vimrc——你得自己创建它。

### 二、.vimrc 文件的位置

Linux 下 .vimrc 的默认位置是：

```bash
~/.vimrc        # 用户的 Vim 配置文件
```

你可以用 Vim 创建或编辑它：

```bash
vim ~/.vimrc
```

**注意**：Vim 在某些系统上可能叫 `_vimrc`（Windows），或者配置文件在 `~/.vim/vimrc`。用以下命令检查 Vim 实际读取哪些文件：

```vim
:version        " 在 Vim 里执行，找 user vimrc file 的位置
:echo $MYVIMRC  " 直接显示当前加载的 vimrc 路径
```

#### 没有 .vimrc 时的行为

如果没有 .vimrc 文件，Vim 运行在"vi 兼容模式"，很多 Vim 的高级特性（比如语法高亮、多级撤销）会被禁用。所以创建 .vimrc 是使用 Vim 的第一步。

### 三、.vimrc 的基本语法

.vimrc 文件由一系列的 Vim 命令组成，和你在 Command 模式下输入的命令完全一样——只不过不用打冒号：

```vim
" 这是注释（双引号开头）

" 设置选项
set number          " 显示行号
set tabstop=4       " Tab 宽度为 4 个空格

" 自定义快捷键
nnoremap <C-s> :w<CR>    " Ctrl+s 保存文件

" 启用插件
call plug#begin('~/.vim/plugged')
Plug 'preservim/nerdtree'
call plug#end()
```

### 四、必知的 set 选项

以下是最常用的选项，按功能分类：

#### 4.1 外观

```vim
set number              " 显示左侧行号
set relativenumber      " 显示相对行号（当前行是 0，上下行显示相对距离）
set cursorline          " 高亮当前行（方便找到光标）
set showcmd             " 右下角显示正在输入的命令
set showmode            " 左下角显示当前模式
set laststatus=2        " 始终显示状态栏
set ruler               " 右下角显示光标位置（行号,列号）
```

关于 `number` 和 `relativenumber`：你可以同时设置两者——当前行显示绝对行号，其他行显示相对距离。这样你按 `5j` 向下跳 5 行时，能一眼看到第 5 行在哪。

#### 4.2 缩进

```vim
set autoindent          " 新行自动使用和上一行相同的缩进
set smartindent         " 智能缩进（根据代码语法自动调整）
set tabstop=4           " Tab 键显示为 4 个空格的宽度
set shiftwidth=4        " 自动缩进时使用 4 个空格
set expandtab           " 把 Tab 转换成空格（写 Python 时强烈建议开启）
set softtabstop=4       " 在插入模式下按 Tab 插入 4 个空格的混合
```

**"空格 vs Tab" 之争**：不同语言有不同的习惯。Python 官方推荐 4 个空格，Go 语言推荐 Tab，Makefile 必须用 Tab。你可以在 .vimrc 中设置全局默认，然后在特定文件类型中覆盖：

```vim
" 全局默认：4 个空格
set tabstop=4
set shiftwidth=4
set expandtab

" 对 Makefile 使用真正的 Tab
autocmd FileType make setlocal noexpandtab tabstop=8 shiftwidth=8

" 对 Go 使用真正的 Tab
autocmd FileType go setlocal noexpandtab tabstop=4 shiftwidth=4
```

#### 4.3 搜索

```vim
set hlsearch            " 搜索时高亮所有匹配
set incsearch           " 增量搜索——你还在打字 Vim 就在搜索了
set ignorecase          " 搜索默认忽略大小写
set smartcase           " 但搜索词含大写字母时自动区分大小写
```

`incsearch` 这个选项非常棒。当你按 `/` 然后开始打字时，不用按回车，光标已经跳到了第一个匹配处，随着你继续输入，匹配也在实时变化。

#### 4.4 编辑行为

```vim
set backspace=indent,eol,start    " 允许退格键跨越所有边界
set whichwrap+=<,>,[,],h,l        " 允许光标跨行（在行首按 h 跳到上一行尾）
set mouse=a                        " 允许使用鼠标（可以点击移动光标、拖动分屏边界）
set clipboard=unnamed              " 使用系统剪贴板（Vim 的复制能粘贴到其他程序）
set history=1000                   " 记住 1000 条命令历史和搜索历史
set undofile                       " 持久化撤销——关闭 Vim 后还能撤销
set autoread                       " 文件被外部修改时自动重新读取
set hidden                         " 允许切换缓冲区时不保存（不用每次 :bn 都问）
```

**`clipboard=unnamed` 这个设置特别重要**。默认情况下，Vim 的复制（yank）内容存在自己的寄存器里，和系统剪贴板是隔离的。你在 Vim 里 `yy` 复制了一行，切到浏览器 `Ctrl+V` 粘贴——没反应。开启 `clipboard=unnamed` 后，Vim 的默认寄存器就是系统剪贴板，跨程序粘贴畅通无阻。

不过这个功能需要 Vim 编译时开启 +clipboard 支持。检查方法：

```bash
vim --version | grep clipboard
# 如果看到 +clipboard，说明支持
# 如果看到 -clipboard，说明不支持，你可能需要安装 vim-gtk 或 vim-gnome 版本
```

#### 4.5 性能和兼容性

```vim
set nocompatible        " 关闭 vi 兼容模式（启用所有 Vim 特性）
set encoding=utf-8      " 使用 UTF-8 编码
set fileencoding=utf-8  " 文件默认以 UTF-8 保存
set ttyfast             " 告诉 Vim 你的终端很快（启用更多重绘优化）
set lazyredraw          " 执行宏时不要重绘屏幕（加快宏执行速度）
```

### 五、快捷键映射

你可以把常用操作映射到更顺手的快捷键上。

#### 5.1 映射的基本语法

```vim
" Normal 模式下的映射
nnoremap <快捷键> <要执行的操作>

" Insert 模式下的映射
inoremap <快捷键> <要执行的操作>

" Visual 模式下的映射
vnoremap <快捷键> <要执行的操作>
```

中间的 `nore` 表示 non-recursive（非递归），意思是映射不会触发其他映射。绝大部分情况下你应该用 `noremap` 而不是 `map`，避免意外的级联触发。

#### 5.2 常用的自定义映射

```vim
" 把 leader 键设为空格（默认是 \ ）
let mapleader = " "

" 用空格 + w 保存文件（不用打 :w 了）
nnoremap <leader>w :w<CR>

" 用空格 + q 退出
nnoremap <leader>q :q<CR>

" 用 Ctrl+j/k/h/l 在分屏窗口间移动（比 Ctrl+w j 方便很多）
nnoremap <C-j> <C-w>j
nnoremap <C-k> <C-w>k
nnoremap <C-h> <C-w>h
nnoremap <C-l> <C-w>l

" 用 jj 退出 Insert 模式（连按两次 j）
inoremap jj <Esc>

" 用 Ctrl+s 保存（覆盖终端暂停的默认行为——多数终端需要先 stty -ixon）
nnoremap <C-s> :w<CR>

" 在 Visual 模式下用 J/K 把选中的行向上/下移动
vnoremap J :m '>+1<CR>gv=gv
vnoremap K :m '<-2<CR>gv=gv

" 搜索后按空格取消高亮
nnoremap <leader>/ :nohlsearch<CR>
```

**什么是 leader 键？** leader 键是一个"前缀键"，你可以把它理解为"自定义命令的入口"。比如你把 leader 设为空格键后，`<leader>w` 就是"按空格再按 w"。这样你的自定义快捷键不会和 Vim 内置快捷键冲突，因为 Vim 本身没有以空格开头的快捷键组合。

**关于 jj 映射**：`inoremap jj <Esc>` 的意思是，在 Insert 模式下，连续快速按两个 j，就相当于按了 Esc。对于很多 Vim 用户来说，这个映射改变了他们的生活——不用伸手去够 Esc 了。

#### 5.3 `<CR>` 和 `<C-xxx>` 的含义

在 Vim 的映射命令中，尖括号有特殊含义：

| 符号 | 含义 |
|------|------|
| `<CR>` | Enter 键（Carriage Return） |
| `<Esc>` | Escape 键 |
| `<C-j>` | Ctrl + j |
| `<leader>` | leader 键 |
| `<Space>` | 空格键 |
| `<Tab>` | Tab 键 |

### 六、插件管理入门：vim-plug

Vim 有丰富的插件生态。管理插件的工具也有很多，推荐 **vim-plug**——它轻量、快速、配置简洁。

#### 6.1 安装 vim-plug

```bash
# Linux / macOS
curl -fLo ~/.vim/autoload/plug.vim --create-dirs \
    https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
```

安装成功后，`~/.vim/autoload/plug.vim` 文件就存在了。

#### 6.2 在 .vimrc 中声明插件

在 .vimrc 中添加插件声明块：

```vim
call plug#begin('~/.vim/plugged')

" 文件树浏览器（左侧显示目录树）
Plug 'preservim/nerdtree'

" 模糊文件查找（按文件名快速打开文件）
Plug 'junegunn/fzf'
Plug 'junegunn/fzf.vim'

" 状态栏美化
Plug 'vim-airline/vim-airline'

" 代码注释（gcc 注释/取消注释当前行）
Plug 'tpope/vim-commentary'

" 自动补全括号、引号
Plug 'jiangmiao/auto-pairs'

" 在代码行数旁边显示 git 修改状态
Plug 'airblade/vim-gitgutter'

call plug#end()
```

#### 6.3 安装和更新插件

在 .vimrc 中添加插件声明后，打开 Vim 执行：

```vim
:PlugInstall       " 安装所有列出的插件
:PlugUpdate        " 更新所有已安装的插件
:PlugClean         " 清理已从配置中删除的插件目录
:PlugStatus        " 查看插件状态
```

#### 6.4 推荐给初学者的插件

不建议一上来装几十个插件——那会让你迷失。以下是最实用的几个：

| 插件 | 作用 | 为什么推荐 |
|------|------|----------|
| nerdtree | 左侧文件树 | 方便浏览项目目录结构 |
| fzf.vim | 模糊查找文件 | 输入文件名的一部分就能快速打开 |
| vim-airline | 状态栏美化 | 显示当前模式、文件名、行列号、Git 分支等 |
| vim-commentary | 快速注释 | `gcc` 注释当前行，`gc` 在 Visual 模式注释选中行 |

**安装插件的心态**：先装最需要的 2-3 个，用熟再考虑加新的。很多初学者一看"Vim 必备插件 TOP 50"就全都装上，结果 Vim 启动变慢、快捷键冲突、遇到问题不知道是哪个插件引起的。

#### 6.5 插件的配置

很多插件需要在 .vimrc 中添加额外的配置：

```vim
" NERDTree 配置
nnoremap <leader>n :NERDTreeToggle<CR>   " 空格+n 切换文件树
let NERDTreeShowHidden=1                  " 显示隐藏文件

" fzf 配置
nnoremap <leader>f :Files<CR>             " 空格+f 模糊查找文件
nnoremap <leader>b :Buffers<CR>           " 空格+b 模糊切换缓冲区
```

### 七、颜色主题

Vim 内置了一些颜色主题：

```vim
" 查看所有可用的内置主题
:colorscheme <Tab>      " 按 Tab 自动补全看有哪些

" 设置主题
colorscheme desert      " 沙漠主题（黄底黑字，对眼睛友好）
colorscheme elflord     " 深色主题
colorscheme morning     " 浅色主题
```

你也可以下载第三方主题。比如著名的 gruvbox：

```vim
" 用 vim-plug 安装
Plug 'morhetz/gruvbox'

" 在 .vimrc 中启用
colorscheme gruvbox
set background=dark      " 暗色背景（也可以设为 light）
```

### 八、针对文件类型的自动配置

Vim 可以识别文件类型，并对不同文件类型应用不同的设置：

```vim
" 启用文件类型检测和插件
filetype plugin indent on

" 对特定文件类型的设置
augroup my_filetype_settings
    autocmd!
    " Python: 4 空格缩进
    autocmd FileType python setlocal tabstop=4 shiftwidth=4 expandtab

    " JavaScript: 2 空格缩进
    autocmd FileType javascript setlocal tabstop=2 shiftwidth=2 expandtab

    " Makefile: 必须用 Tab，不能用空格
    autocmd FileType make setlocal noexpandtab tabstop=8 shiftwidth=8

    " Markdown: 自动折行
    autocmd FileType markdown setlocal wrap linebreak

    " 在写纯文本时自动拼写检查
    autocmd FileType text,markdown setlocal spell
augroup END
```

`augroup` 和 `autocmd!` 的组合确保这些自动命令不会重复注册（每次重新加载 .vimrc 时会先清空同组的旧命令）。

### 九、完整的示例 .vimrc

配套的 `examples/vimrc-example` 文件包含了一个适合初学者的完整 .vimrc。你可以把它复制到 `~/.vimrc` 作为起点：

```bash
cp examples/vimrc-example ~/.vimrc
```

然后根据自己的习惯修改。

这个示例包含了：
- 基本的外观设置（行号、状态栏、语法高亮）
- 缩进设置
- 搜索设置
- 自定义快捷键
- 文件类型自动配置
- vim-plug 插件声明（按需取消注释安装）
- 每行都有中文或英文注释

### 十、调试 .vimrc

如果你的 .vimrc 出现了问题（比如设置 Vim 行为异常），可以这样排查：

```bash
# 不加载 .vimrc 启动 Vim（进入纯净模式）
vim -u NONE

# 不加载插件启动 Vim
vim --noplugin

# 在 Vim 中重新加载 .vimrc
:source ~/.vimrc
" 或简写
:so ~/.vimrc

# 查看某个选项的当前值
:set number?          " 查看 number 是否开启
:set tabstop?         " 查看 tabstop 的值

# 查看某个快捷键被映射到了什么
:map <leader>w        " 查看 <leader>w 映射
:nmap <C-s>           " 查看 Normal 模式下 Ctrl+s 的映射
```

**当 Vim 出问题时**，打开 Vim 然后 `:so ~/.vimrc` 看看有没有报错信息。如果有，根据报错信息去 .vimrc 里找对应的行修改。

## 动手试试

1. 检查你是否已有 .vimrc：`ls -la ~/.vimrc`。如果不存在，运行 `vim ~/.vimrc` 创建它。
2. 在 .vimrc 中添加 `set number`，保存退出。重新打开 Vim，确认左侧出现了行号。
3. 在 .vimrc 中添加 `set cursorline`，重新打开 Vim，确认当前行有高亮。
4. 在 .vimrc 中添加 `inoremap jj <Esc>`，重新打开 Vim。进入 Insert 模式（按 i），输入一些文字，然后快速连按两次 j。观察你是否回到了 Normal 模式。
5. 在 .vimrc 中添加 `set ignorecase` 和 `set smartcase`。在 Vim 中打开一个有大小写混合文字的文件，用 `/` 搜索一个小写词——然后搜索一个大写词——观察大小写行为的不同。
6. 把配套的 `vimrc-example` 复制到 `~/.vimrc`。打开 Vim，用 `:so ~/.vimrc` 重新加载。观察外观和行为的改变。
7. （可选）安装 vim-plug：运行安装命令。然后取消 .vimrc 中 `Plug 'preservim/nerdtree'` 的注释，在 Vim 中执行 `:PlugInstall`。安装完成后按 `<leader>n`（空格+n）打开 NERDTree 文件树。

## 本节小结

.vimrc 是 Vim 的配置中心——它让你定制外观（行号、高亮）、行为（缩进、搜索）、快捷键（映射）和插件（vim-plug）。从一个简单的配置开始，随着你使用 Vim 的深入逐步添加，让 Vim 越来越"属于你自己"。
