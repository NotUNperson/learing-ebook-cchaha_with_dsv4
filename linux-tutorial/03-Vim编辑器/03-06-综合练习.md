# 03-06 综合练习：用 Vim 完成一次完整编辑

## 本节你会学到什么

- 把 Vim 的模式切换、移动、编辑、搜索替换、保存退出串联成完整工作流
- 用 Vim 从头创建文件、编写内容、修改、格式化
- 在 Vim 内部使用外部命令
- 体会"全程不碰鼠标"的编辑体验
- 形成自己的 Vim 操作节奏

## 正文

### 一、任务描述

你要用 Vim 从头完成以下任务：

1. 创建一个 Markdown 文件 `weekly-report.md`
2. 写入本周的工作周报内容
3. 修改内容——调整顺序、删除废话、修改措辞
4. 用搜索替换统一某些术语
5. 在 Vim 内部运行 shell 命令检查字数
6. 保存退出，确认文件内容正确

**全程只用 Vim，不碰其他编辑器，不用鼠标。**

### 二、详细操作流程（每一步都告诉你按什么键、会发生什么）

#### 步骤 1：启动 Vim 并创建文件

打开终端，执行：

```bash
vim weekly-report.md
```

**屏幕变化**：终端窗口变成了 Vim 界面。  
**屏幕左上角**：一片空白（因为是新文件）。  
**屏幕左下角**：显示 `"weekly-report.md" [New File]`。  
**屏幕底部**：一行波浪线 `~` 表示空行（不是文件内容，只是占位符）。  
**当前模式**：Normal 模式（左下角没有 `-- INSERT --`）。

#### 步骤 2：进入 Insert 模式，开始写内容

**按下 `i` 键。** 注意，不是输入字母 i——i 键在 Normal 模式下是"进入 Insert 模式"的命令。

**屏幕变化**：左下角出现 `-- INSERT --`。这表示你现在在 Insert 模式了，按键会变成文字输入。

现在输入以下内容（逐字输入，包括空行）：

```
# 本周工作总结

## 完成的任务

1. 完成了用户登录模块的开发
2. 修复了首页加载速度慢的问题
3. 参与了代码评审会议

## 遇到的问题

数据库查询在高峰期偶尔超时。
怀疑是索引问题，下周需要排查。

## 下周计划

1. 优化数据库查询性能
2. 开始开发用户权限管理功能
3. 写单元测试
```

输入完后，**按下 `Esc` 键。**

**屏幕变化**：左下角的 `-- INSERT --` 消失了。你回到了 Normal 模式。

#### 步骤 3：浏览和检查内容

你现在在 Normal 模式。用以下按键检查你写的内容：

**按 `gg`**（连续按两次小写 g）→ 光标跳到文件开头（第一行第一个字符）。  
**屏幕变化**：光标在 `#` 上。

**按 `j` 几次** → 光标向下移动。  
**屏幕变化**：光标逐行向下移动。

**按 `G`**（单独一个大写 G，按住 Shift + g）→ 光标跳到文件最后一行。  
**屏幕变化**：光标在最后一行。

**按 `gg`** → 回到文件开头。

**按 `/数据库` 然后按 Enter** → 光标跳到了"数据库"这个词上。  
**屏幕变化**：所有出现的"数据库"都被高亮了。

**按 `n`** → 跳到下一个"数据库"。  
**屏幕变化**：光标跳到第二个"数据库"所在的位置。

**按 `N`** → 回到上一个"数据库"。  
**屏幕变化**：光标跳回前一个"数据库"。

**按 `:nohlsearch` 然后按 Enter** → 清除搜索高亮。  
**屏幕变化**：高亮全部消失，恢复正常显示。

#### 步骤 4：修改内容——删除一行

光标在"数据库查询在高峰期偶尔超时"这行上（用 `j` 或 `k` 移动到那里）。

**按 `dd`**（按两次小写 d）→ 这一整行被删除了。  
**屏幕变化**：该行消失，下面的行自动上移。

等一下——你觉得还是需要这行。**按 `u`**（撤销）。  
**屏幕变化**：被删的那行又回来了。

#### 步骤 5：修改内容——改变措辞

光标在"修复了首页加载速度慢的问题"这行。你想把"慢的"改成"缓慢的"。

**按 `f慢`** → 光标跳到"慢"字上。（f = find，向前查找字符）  
**屏幕变化**：光标定位在"慢"上。

**按 `cw`**（change word）→ "慢的"两个字符消失，你进入了 Insert 模式。  
**屏幕变化**：左下角显示 `-- INSERT --`，"慢的"两个字不见了。

输入 `缓慢的`。

**按 `Esc`** → 回到 Normal 模式。  
**屏幕变化**：该行变成了"修复了首页加载速度缓慢的问题"。

#### 步骤 6：修改内容——调整顺序

你觉得"代码评审会议"应该放在第一个任务的位置。也就是把第 3 条和第 1 条交换。

光标移到"1. 完成了用户登录模块的开发"这行。

**按 `dd`** → 这行被剪切。  
**屏幕变化**：第 1 条消失，原来的第 2 条变成了第 1 条。

光标移到"参与了代码评审会议"这行（现在是第 2 条，因为上面删了一行）。

**按 `p`** → 之前剪切的行被粘贴到这行的下面。  
**屏幕变化**："完成了用户登录模块的开发"出现在"参与了代码评审会议"下面。

但这样顺序还是不对——你希望"代码评审会议"在第一位。我们需要把当前行（被粘贴的内容在上面，实际想换到上面去）。

换个思路：在"代码评审会议"这行上面新建一行，把剪切的内容粘贴过来。

**按 `u` 两次**，撤销刚才的 dd 和 p 操作。回到原始状态。

用另一种方法：

光标在"1. 完成了用户登录模块的开发"这行。

**按 `V`**（大写 V，进入行 Visual 模式）→ 整行被选中（高亮）。  
**屏幕变化**：左下角显示 `-- VISUAL LINE --`，整行高亮。

**按 `j` 两次** → 下面两行也被选中（共选中 3 行——3 条任务）。  
**屏幕变化**：3 行一起高亮。

现在你想把第 3 行移到第 1 行位置。先按 `Esc` 退出 Visual 模式，用更直接的方法：

光标移到第 3 行（"参与了代码评审会议"）。

**按 `dd`** → 剪切第 3 行。

光标移到第 1 行（"完成了用户登录模块的开发"）。

**在 Normal 模式下按 `P`（大写 P）** → 在光标之前（上方）粘贴。  
**屏幕变化**："参与了代码评审会议"出现在第一行，原来的第 1 和第 2 行被推到下面。

任务列表现在变成了：
```
## 完成的任务

1. 参与了代码评审会议
2. 完成了用户登录模块的开发
3. 修复了首页加载速度缓慢的问题
```

#### 步骤 7：用搜索替换来统一术语

你觉得全文中"开发"这个词不够正式，想改成"研发"。

**按 `gg`** → 回到文件开头。

输入 **`:%s/开发/研发/g`** 然后按 Enter。

这一行的含义是：
- `:` 进入 Command 模式
- `%` 整个文件范围
- `s/开发/研发/g` 把所有"开发"替换成"研发"
- 按 Enter 执行

**屏幕变化**：Vim 底部显示类似 `3 substitutions on 3 lines` 的消息。说明替换了 3 处。

浏览文件确认替换结果。所有"开发"都变成了"研发"。

#### 步骤 8：在 Vim 内部运行 shell 命令

你想确认一下这个文件有多少字。

在 Normal 模式下输入 **`:!wc -w %`** 然后按 Enter。

这行命令的含义：
- `:!` 在 Vim 中执行外部 shell 命令
- `wc -w` 统计单词数
- `%` Vim 里代表当前文件名

**屏幕变化**：Vim 暂时切换到终端界面，显示了 `wc` 的输出（一个数字，代表单词数），底部显示 `Press ENTER or type command to continue`。按 Enter 回到 Vim。

再试一个操作——把当前日期插入到文件中：

**按 `G`** → 跳到文件末尾。  
**按 `o`** → 在文件末尾下面新建一行，进入 Insert 模式。  
**屏幕变化**：新的一行出现了，光标在新行首，左下角显示 `-- INSERT --`。

输入 `---` 然后回车。输入 `编写日期：`。按 `Esc` 回到 Normal 模式。

现在在光标处插入当前日期。输入 **`:r !date`** 然后按 Enter。

`:r` 是 "read" 的简写，`!date` 表示执行 date 命令并把输出读入。  
**屏幕变化**：date 命令的输出（如 `Thu May 15 14:30:00 CST 2026`）被插入到当前光标位置。

#### 步骤 9：给内容做最后美化

**按 `gg`** → 回到文件开头。

你想把"遇到的问题"改成全部大写，显得更正式。光标移到"遇到的问题"这行（用 `j` 移动）。

**按 `0`** → 跳到行首。

**按 `f遇`** → 光标跳到"遇"字上。

**按 `v`**（进入字符 Visual 模式）。**按 `e`**（跳到词尾）。"遇到的"这三个字被选中高亮。

**按 `U`**（大写 U）→ 选中的文本变成大写。  
**屏幕变化**："遇到的问题"变成了"遇到的问题"——等等，中文没有大小写区别，这个操作对中文无效。按 `u` 撤销。

换个实际有用的美化：把每行的缩进调整到一致。

检查一下每个二级标题（## 开头的行）前后是否有空行。用 `/##` 搜索所有二级标题，按 `n` 逐个检查。如果没有空行，在 Normal 模式下用 `o` 和 `O` 添加。

#### 步骤 10：保存并退出

内容检查无误后，保存退出。

输入 **`:wq`** 然后按 Enter。

- `:w` 保存（write）
- `:q` 退出（quit）
- 合在一起就是保存并退出

**屏幕变化**：Vim 关闭，回到了终端。Vim 底部似乎短暂闪过 `"weekly-report.md" 22L, 450C written`（行数和字符数）。

#### 步骤 11：验证结果

在终端执行：

```bash
cat weekly-report.md
```

你应该能看到你刚才创建、编辑、保存的全部内容。

### 三、操作总结

回顾一下你在这次练习中用到的所有 Vim 操作：

| 操作 | 按键 | 用途 |
|------|------|------|
| 打开文件 | `vim filename` | 启动 Vim |
| 插入文字 | `i` | 进入 Insert 模式 |
| 退出编辑 | `Esc` | 回到 Normal 模式 |
| 文件开头 | `gg` | 跳到第一行 |
| 文件末尾 | `G` | 跳到最后一行 |
| 下移一行 | `j` | 光标向下 |
| 搜索 | `/keyword` Enter | 查找关键字 |
| 下一个匹配 | `n` | 跳到下一处 |
| 清除高亮 | `:nohlsearch` | 取消搜索高亮 |
| 删除整行 | `dd` | 剪切当前行 |
| 撤销 | `u` | 撤销上一步 |
| 查找字符 | `f字` | 行内精准跳转 |
| 修改单词 | `cw` | 删除当前词并进入编辑 |
| 粘贴 | `p` / `P` | 粘贴到光标后/前 |
| 选中多行 | `V` + `j/k` | 行 Visual 模式 |
| 全局替换 | `:%s/old/new/g` | 整个文件查找替换 |
| 执行命令 | `:!cmd` | 在 Vim 内运行 shell |
| 插入命令输出 | `:r !cmd` | 把命令结果插入文件 |
| 保存退出 | `:wq` | 保存并退出 |

这些操作涵盖了日常 Vim 使用 80% 的场景。熟练使用它们，你就能在服务器上高效地编辑任何文本文件。

### 四、进阶挑战

如果你觉得上面的练习还不够过瘾，试试这些附加任务（同样只用 Vim，不碰鼠标）：

**挑战 1：从零创建一个 Markdown 表格**

在 Vim 中新建一个文件，用学到的技巧快速创建以下表格：

```
| 名称 | 数量 | 价格 |
|------|------|------|
| 苹果 | 3    | 5.00 |
| 香蕉 | 5    | 3.50 |
| 橙子 | 2    | 6.00 |
```

提示：创建第一行后，用 `yy` 复制、`p` 粘贴快速生成多行。用 `ciw` 修改单元格内容。

**挑战 2：用 Vim 的"列块编辑"添加行号**

创建一个包含 10 行文字的简单文件。用 `Ctrl+v` 进入块 Visual 模式，选中 10 行的行首，按 `I` 输入 `1. `，然后按 `Esc`。你能预测会发生什么吗？

提示：所有选中的行前面都会出现 `1. `。Vim 不会自动递增数字——你需要用其他技巧（如 `g<C-a>` 递增），但这需要先了解。

**挑战 3：查找替换 + 确认模式**

创建一个包含多个 "hello" 和 "Hello" 的文件。执行 `:%s/hello/hi/gc`。Vim 会在每个匹配处停下来问你。试试回答 y（替换）、n（跳过）、a（全部替换不问了）、q（退出）。

### 五、养成 Vim 习惯的几点建议

1. **前两周强制自己只用 Vim 写东西**——即使效率变低了也要坚持。笔记、日记、临时的文本片段，全用 Vim。
2. **把鼠标放远一点**——放到你够不着的地方。切断"摸鼠标"的肌肉记忆回路。
3. **每天学 2 个新命令**——早上学，当天就用。不要再"先收藏以后再看"。
4. **在普通编辑器里想念 Vim 的命令**——当你按 j 想向下移动但在浏览器里打出了 j 时，说明 Vim 已经渗入你的肌肉记忆了。这时候可以考虑在浏览器/IDE 里装 Vim 模式插件（如 VSCode 的 Vim 插件、浏览器的 Vimium）。
