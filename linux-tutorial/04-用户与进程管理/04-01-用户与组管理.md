# 04-01 用户与组管理

## 本节你会学到什么

- 理解用户（UID）、组（GID）和权限的基本概念，以及 Linux 如何使用它们控制访问
- 读懂 `/etc/passwd`、`/etc/shadow`、`/etc/group` 三个文件的格式和含义
- 使用 `useradd`、`usermod`、`userdel` 创建、修改和删除用户
- 使用 `groupadd`、`groupmod`、`groupdel` 管理用户组
- 使用 `id`、`groups`、`passwd` 查看身份和修改密码

## 公司门禁卡系统：理解 Linux 用户与组

想象你在一栋大型办公楼里上班。大楼入口有门禁闸机，每张员工卡里记录着你的工号和所属部门。刷一下卡，系统就知道你是谁、属于哪个部门、有没有权限进入特定楼层。

Linux 的用户与组系统，本质上就是这套"门禁卡系统"的电子版。

在这套系统中，有四个核心概念：

1. **用户（User）**：就像大楼里的每一个员工。每个人有一张独一无二的门禁卡，卡上记录着你的工号。在 Linux 里，这个"工号"就叫 **UID（User ID，用户标识号）**。系统真正识别的是 UID 这个数字，而不是你看到的用户名 `zhangsan`。
2. **组（Group）**：就像公司里的部门。财务部的人能进财务部的办公室，技术部的人能进服务器机房。一个用户可以同时属于多个组（比如你既属于技术部，又属于篮球队兴趣小组）。
3. **root 用户**：就像公司的超级管理员，他有所有房间的万能钥匙。root 的 UID 永远是 0，拥有系统的最高权限。Linux 常说"用 root 时要万分小心"，就像你不会让一个实习生拿着所有服务器的 root 密码到处跑。
4. **普通用户**：普通员工的权限有限，只能进自己的工位和公共区域。在 Linux 里，普通用户的 UID 通常从 1000 开始。

### 为什么需要多用户系统？

再想一个场景：你和室友共用一台台式电脑。如果你只有一个账户，那么你的工作文档、浏览器收藏夹、微信聊天记录，会和室友的完全混在一起。他随时可以打开你的文件，修改甚至删除它们。

有了多用户系统，你们各用各的账户，互不干扰：
- 你的文件放在 `/home/你的用户名/` 下面，室友默认看不到
- 你可以装一个只在你自己账户下可用的软件，不会影响室友的环境
- 系统日志会记录是谁在什么时间做了什么操作，出了问题能追溯

这就是 Linux 多用户系统的意义：**隔离**与**安全**。

## 三个密码世界的档案：/etc/passwd、/etc/shadow、/etc/group

想象公司的 HR 系统里有三本重要的花名册：

### 第一本：《员工花名册》 -- /etc/passwd

这本册子记录了每个员工的基本信息。Linux 的 `/etc/passwd` 文件每行就是一个用户，格式如下：

```
用户名:密码占位符:UID:GID:描述:家目录:登录Shell
```

来看一个真实例子：

```
zhangsan:x:1000:1000:张三:/home/zhangsan:/bin/bash
```

拆解一下这段"暗号"：

| 字段 | 示例值 | 含义 |
|------|--------|------|
| 用户名 | `zhangsan` | 登录时输入的名字 |
| 密码占位符 | `x` | 密码不存这里，"x"表示密码已转移到 `/etc/shadow` |
| UID | `1000` | 用户ID，系统真正用的是这个数字 |
| GID | `1000` | 主组ID，代表你的"默认部门" |
| 描述（GECOS） | `张三` | 用户全名或备注，可以留空 |
| 家目录 | `/home/zhangsan` | 你登录后默认进入的文件夹 |
| 登录Shell | `/bin/bash` | 你用的命令行解释器 |

注意密码字段写的是 `x`，这说明真正的密码存在隔壁的 `/etc/shadow` 里。这是出于安全考虑 -- `/etc/passwd` 所有人都能读（不然 `ls` 命令都不知道文件是谁的），但密码的密文绝对不能让普通人看到。

你可以用 `grep` 快速查找某个用户：

```bash
# 查看 root 用户信息
grep "^root:" /etc/passwd

# 查看自己的信息
grep "^$(whoami):" /etc/passwd
```

### 第二本：《机密密码簿》 -- /etc/shadow

这本册子只对 root 开放阅读权限。它存储的是加密后的密码哈希值，以及密码策略信息。

格式如下：

```
用户名:加密密码:上次修改天数:最小修改间隔:最大有效期:警告天数:宽限天数:过期天数:保留字段
```

例子：

```
zhangsan:$y$j9T$abcd1234...:19500:0:99999:7:::
```

各字段含义：

| 字段 | 含义 |
|------|------|
| 加密密码 | 密码的哈希值。如果以 `!` 或 `*` 开头，说明账户被锁定 |
| 上次修改天数 | 从 1970-01-01 算起的天数 |
| 最小修改间隔 | 两次修改密码的最短间隔（天），0 表示随时可改 |
| 最大有效期 | 密码最多能用多少天，之后必须修改 |
| 警告天数 | 密码到期前多少天开始提醒用户 |
| 宽限天数 | 密码过期后还有多少天可以登录 |
| 过期天数 | 从 1970-01-01 算起，账户在哪天之后彻底失效 |

类比一下：公司的门禁卡电子锁会记录"这张卡最后一次换密码是什么时候"、"密码还有多久作废"、以及"如果三个月没来上班，卡自动失效"。`/etc/shadow` 做的就是这个工作。

**安全提示**：`/etc/shadow` 的权限通常是 `640`（root 可读写，shadow 组可读），普通用户无法读取。这就像 HR 把薪资信息锁在保险柜里，普通员工看不到别人的工资。

### 第三本：《部门花名册》 -- /etc/group

这本册子记录了每个部门有哪些成员。

格式：

```
组名:密码占位符:GID:成员列表
```

例子：

```
wheel:x:10:zhangsan,lisi
```

解读：
- 组名叫 `wheel`（在 CentOS/RHEL 里这是管理员组）
- 密码占位符 `x`（组密码极少使用）
- GID 为 10
- 成员有 `zhangsan` 和 `lisi`，用逗号分隔

在 Debian/Ubuntu 系统里，管理员组的名字叫 `sudo` 而不是 `wheel`。不管叫什么，只要用户属于这个组，就能通过 `sudo` 命令获得临时的管理员权限。

## 用户管理的日常操作

### 创建新员工：useradd

新同事报到，HR 需要在系统里登记。在 Linux 里，对应的命令是 `useradd`。

```bash
# 最基本的创建用户（不创建家目录，不指定 Shell）
sudo useradd newuser

# 推荐用法：创建家目录 + 指定 bash 为默认 Shell
sudo useradd -m -s /bin/bash newuser

# 同时指定 UID、主组、附加组和备注
sudo useradd -m -s /bin/bash -u 1500 -g staff -G wheel,docker -c "新员工小王" newuser
```

参数解释：
- `-m`：自动创建家目录（`/home/newuser`），并从 `/etc/skel` 复制模板文件（`.bashrc`、`.profile` 等）
- `-s /bin/bash`：指定登录 Shell
- `-u 1500`：手动指定 UID
- `-g staff`：指定主组（这个组必须先存在）
- `-G wheel,docker`：加入附加组（逗号分隔，不加空格）
- `-c`：填写备注（GECOS 字段）

`/etc/skel` 是个好东西。它就像公司给新员工统一配备的"入职礼包" -- 标准化的鼠标垫、笔记本、工牌套。你可以在 `/etc/skel` 里放一些默认的配置文件，这样每个新用户创建时都会自动拿到一份。

创建用户后，别忘了设密码：

```bash
sudo passwd newuser
# 然后交互式输入密码两次
```

### 修改员工信息：usermod

同事调岗了，或者加了新的公司门禁权限，需要更新档案。`usermod` 就是干这个的。

```bash
# 修改备注（描述信息）
sudo usermod -c "资深工程师小李" zhangsan

# 把用户加入附加组（APPEND 模式，一定要加 -a，否则会覆盖原有组）
sudo usermod -aG docker zhangsan
sudo usermod -aG wheel zhangsan

# 改家目录
sudo usermod -d /home/newhome -m zhangsan

# 锁定用户（禁止登录）
sudo usermod -L zhangsan

# 解锁用户
sudo usermod -U zhangsan

# 改登录 Shell
sudo usermod -s /usr/bin/zsh zhangsan
```

**特别提醒**：`-G` 如果不加 `-a`（append），会把你之前加入的附加组全部替换掉。这相当于你本来有财务部和工会的门禁权限，结果 HR 更新时忘了加 `-a`，只留了财务部的权限，工会的就丢了。所以请记住：**改附加组要用 `-aG`，永远带着那个 `a`**。

还有一个找不同：`usermod -L` 锁定用户，实际上是在 `/etc/shadow` 的密码字段前加了一个 `!` 号，让密码验证永远失败。这不是删账号，只是让它暂时不能登录，就像你暂时冻结了同事的门禁卡而不是把卡销毁。

### 删除员工：userdel

同事离职了，需要注销档案：

```bash
# 只删用户，保留家目录和文件
sudo userdel zhangsan

# 删用户 + 同时删除家目录和邮件池
sudo userdel -r zhangsan
```

`-r` 参数就像离职手续中的"清理工位" -- 把个人物品打包带走，工位清空。不加 `-r` 的话，相当于人走了但工位上的东西还留着。

## 组管理的日常操作

### 创建部门：groupadd

```bash
# 创建新组
sudo groupadd developers

# 创建组并指定 GID
sudo groupadd -g 2000 developers
```

### 修改部门信息：groupmod

```bash
# 改名
sudo groupmod -n devs developers

# 改 GID
sudo groupmod -g 2500 devs
```

### 删除部门：groupdel

```bash
# 删除组（前提是这个组不是任何用户的主组）
sudo groupdel devs
```

注意：如果一个组还是某些用户的主组（在 `/etc/passwd` 里被引用），`groupdel` 会拒绝删除。就像你没法解散一个还有人的部门一样。

## 常用查询命令

```bash
# 查看我是谁（UID、GID、所属组）
id

# 查看指定用户的信息
id zhangsan

# 只看所属组
groups
groups zhangsan

# 查看当前登录的用户名
whoami

# 查看当前有哪些用户登录了系统
who
w
```

## 动手试试

**练习：创建一个完整的用户并验证**

请打开终端，按以下步骤操作（需要 sudo 权限）：

1. 创建一个名为 `testuser` 的新用户，要求：
   - 创建家目录
   - 设置 Shell 为 `/bin/bash`
   ```bash
   sudo useradd -m -s /bin/bash testuser
   ```

2. 给这个用户设置密码：
   ```bash
   sudo passwd testuser
   ```

3. 查看 `/etc/passwd` 中新用户的信息：
   ```bash
   grep "^testuser:" /etc/passwd
   ```

4. 用 `id` 命令查看该用户的 UID 和所属组：
   ```bash
   id testuser
   ```

5. 创建一个新组 `testgroup`，并把 `testuser` 加入这个组：
   ```bash
   sudo groupadd testgroup
   sudo usermod -aG testgroup testuser
   ```

6. 再次用 `id testuser` 验证，确认 `testgroup` 出现在组列表中。

7. 清理（可选）：
   ```bash
   sudo userdel -r testuser
   sudo groupdel testgroup
   ```

## 本节小结

Linux 的用户与组系统就像公司的门禁卡管理，用户（UID）是员工身份，组（GID）是部门归属，三个关键文件（`/etc/passwd`、`/etc/shadow`、`/etc/group`）分别是花名册、密码簿和部门表，通过 `useradd`、`usermod`、`userdel`、`groupadd` 等命令即可完成日常的身份管理操作。
