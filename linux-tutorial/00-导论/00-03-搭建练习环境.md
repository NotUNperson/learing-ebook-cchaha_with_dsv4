# 00-03 搭建练习环境

## 本节你会学到什么

- 理解 WSL2 的本质和优势，并学会安装配置
- 知道如何使用 VirtualBox 创建 Linux 虚拟机
- 了解云服务器作为练习环境的利弊和基本操作
- 对比三种方案，选择最适合自己的练习方式

---

你还没有 Linux 系统，但别担心。你不需要把现在的 Windows 删掉重装，也不需要再买一台新电脑。这一节我们介绍三种在 Windows 上跑 Linux 的方式。它们各有优劣，你只需要选一种。

## 方案一：WSL2（推荐给所有 Windows 10/11 用户）

### WSL2 是什么

WSL 全称是 Windows Subsystem for Linux（Windows 的 Linux 子系统）。直白地说，就是微软在 Windows 里内置了一个"翻译官"，让 Linux 程序可以直接在 Windows 上运行，不需要虚拟机。

第一代 WSL（WSL1）是一个翻译层：Linux 的系统调用被翻译成 Windows 的系统调用。性能还行，但有些操作（比如 Docker）支持不好。

WSL2 则完全换了一个思路：它实际上是一个**轻量级的虚拟机**，里面跑着一个真正的 Linux 内核。但这个虚拟机被微软深度集成到 Windows 里，你几乎感觉不到它的存在。WSL2 的启动速度极快（一秒左右），文件读写速度比 WSL1 快很多，而且完全兼容 Docker 等工具。

**类比**：WSL1 像是在 Windows 这个说中文的人旁边配了一个同声传译，听到英文（Linux 系统调用）就立刻翻译成中文（Windows 系统调用）。WSL2 则直接在办公室里隔了一个小房间（轻量虚拟机），里面的人说的是纯英文（真正的 Linux 内核），但通过一扇门（网络和文件共享）和外面的中文世界无缝交流。

### 安装 WSL2（Windows 10 版本 2004+ 或 Windows 11）

打开你的 PowerShell（右键点击开始菜单 -> Windows PowerShell 或 Windows 终端），**以管理员身份运行**，然后一行命令搞定：

```powershell
wsl --install
```

这个命令会自动做以下事情：
1. 启用"虚拟机平台"和"Linux 子系统"这两个 Windows 功能
2. 下载并安装 WSL2 的最新 Linux 内核
3. 将 WSL2 设置为默认版本
4. 下载并安装 Ubuntu（默认发行版）

命令执行完毕后，**重启电脑**。重启后系统会自动弹出一个 Ubuntu 的终端窗口，提示你创建一个用户名和密码。这个用户名和密码是你在这个 Linux 环境里的账号，跟 Windows 的账号没有任何关系。设置好之后，你就有了一个可以使用的 Linux 终端。

```bash
# 重启后首次进入，会看到类似这样的提示：
Installing, this may take a few minutes...
Please create a new UNIX username: zhangsan
New password: ********
Retype new password: ********
zhangsan@YOUR-PC:~$
```

看到那个以 `$` 结尾的提示符了吗？恭喜你，你已经进入了 Linux 的世界。

### 如果想安装不同的发行版

```powershell
# 在 PowerShell 中查看可用的发行版
wsl --list --online

# 安装特定的发行版，比如 Debian
wsl --install -d Debian

# 查看你已经安装了哪些
wsl --list --verbose
```

### WSL2 的使用体验

安装好 WSL2 之后，你在 Windows 上就有了一个 Linux 终端。你可以：

- 在 Linux 终端里用 `cd /mnt/c/Users/你的用户名/Desktop` 访问 Windows 桌面上的文件
- 在 Windows 文件资源管理器的地址栏输入 `\\wsl$\Ubuntu\home\你的用户名`，用图形界面浏览 Linux 里的文件
- 在 VS Code 里安装 "Remote - WSL" 插件，直接在 Windows 的 VS Code 里编辑 Linux 里的代码

**类比**：WSL2 就像在你家（Windows）里开了一扇任意门，门通向另一间公寓（Linux）。你可以随时走过去（打开终端），走过去之后你就完全在 Linux 的房间里了。但如果你需要拿 Windows 房间里的东西（文件），你也可以通过走廊（/mnt/c/）走过去拿。

### WSL2 的优缺点

优点：
- 安装最简单，一行命令搞定
- 启动极快，敲一个命令就进去了
- 和 Windows 深度集成，文件互通、剪贴板互通
- 占用的内存和硬盘都比较少
- 免费

缺点：
- 没有图形界面，纯命令行（不过对学 Linux 来说这反而是优点，逼你学命令）
- 不适合测试网卡配置、防火墙等底层网络功能（网络是共享 Windows 的）
- 需要 Windows 10 2004 以上或 Windows 11

## 方案二：VirtualBox 虚拟机

### 虚拟机是什么

虚拟机（Virtual Machine）就是在一台物理电脑上用软件模拟出另一台电脑。你可以把它想象成一个"电脑模拟器"：你的 Windows 上运行着一个软件，这个软件里有着跟真实电脑一模一样的东西--虚拟的 CPU、虚拟的内存、虚拟的硬盘、虚拟的网卡。你在里面装一个操作系统，它还以为自己跑在真实的硬件上。

**类比**：虚拟机就像在大房子里用隔板隔出一个小房间。小房间有自己完整的四面墙、一个门、独立的空调和电表。小房间里的人（Linux）不知道自己在别人家（Windows）里，他觉得这就是一个完整的房间。而房子的主人（你）可以随时推门进去看 Linux 在干什么，不想用了就关上门让它睡觉。

### 安装 VirtualBox

1. 访问 https://www.virtualbox.org/ 下载 VirtualBox（这个软件是免费的）
2. 安装 VirtualBox（一路"下一步"即可）
3. 去你选择的发行版官网下载 .iso 镜像文件（比如 Ubuntu 的 https://ubuntu.com/download）
4. 打开 VirtualBox，点击"新建"，给虚拟机起个名字，选择类型为 Linux
5. 分配内存（建议至少 2048 MB，也就是 2GB）
6. 创建虚拟硬盘（建议至少 20GB，选择"动态分配"以节省实际硬盘空间）
7. 创建完成后，点击"设置" -> "存储" -> 在光驱那里加载你下载的 .iso 文件
8. 启动虚拟机，进入安装界面

安装过程和在一台真实电脑上装系统差不多：选择语言、选择时区、设置用户名和密码、等待安装完成。

### 安装后要做的一件事：安装增强功能

系统装好后，你会发现鼠标在虚拟机和 Windows 之间切换很麻烦（要点一下才能切换），而且分辨率可能不对。这时候需要安装"Guest Additions"（增强功能）：

在虚拟机窗口的菜单栏里，点击"设备" -> "安装增强功能"，然后按照提示操作。装完之后：
- 鼠标可以在虚拟机和 Windows 之间自由移动，不需要按切换键
- 分辨率自适应窗口大小
- 可以直接拖拽文件从 Windows 到虚拟机里
- 剪贴板可以互通

### VirtualBox 的优缺点

优点：
- 有完整的图形桌面，跟一台真实电脑一模一样
- 可以做任何实验，即使把系统搞坏了，删掉虚拟机重建即可（快照功能更是可以随时"时光倒流"）
- 完全不影响 Windows
- 适合需要图形界面的操作（比如用浏览器测试、用 IDE 写代码）

缺点：
- 比较吃内存（虚拟机本身要占用你分配给它的内存，加上 Windows 自己的开销，你的电脑最好有 8GB 以上的内存）
- 启动慢（相当于开一台新电脑）
- 文件在虚拟机和 Windows 之间传来传去不太方便（虽然有共享文件夹功能）

## 方案三：云服务器

### 云服务器是什么

云服务器就是一台放在别人机房里、通过互联网连接的真实电脑。你不需要买硬件，只需要在云厂商（阿里云、腾讯云、华为云、AWS 等）的网站上注册一个账号，几分钟就能创建出一台 Linux 服务器，然后通过 SSH 远程连接操作它。

**类比**：云服务器就像是长租公寓。你不需要买一套房子（买服务器硬件），你只需要在 APP 上签约租一年（在云厂商网站上创建实例）。公寓里的基础设施（水电、宽带、物业）都由平台提供，你只需要拎包入住。你可以随时退房换公寓，今天住上海，明天搬到纽约。

### 创建云服务器（以阿里云为例）

1. 注册阿里云账号（https://aliyun.com），需要实名认证
2. 搜索"云服务器 ECS"
3. 点击"创建实例"
4. 选择配置：
   - 地域：选离你最近的（比如华东 1 杭州）
   - 镜像：选择 Ubuntu 22.04 或 Rocky Linux 9
   - 实例规格：选最便宜的（1 核 1GB 内存，学习足够用）
   - 网络：使用默认的 VPC 和安全组
   - 登录凭证：选择"密钥对"或"密码"
5. 确认订单并支付
6. 创建完成后，你会获得一个**公网 IP 地址**（比如 47.96.123.456）

然后你需要在 Windows 上装一个 SSH 客户端来连接。Windows 10/11 自带了 SSH 客户端（在 PowerShell 或 CMD 里直接用 `ssh` 命令），你也可以用更友好的图形化客户端如 MobaXterm 或 Tabby。

```powershell
# 在 PowerShell 或 CMD 中连接你的云服务器
ssh root@你的公网IP
# 输入你设置的密码
```

### 云服务器的优缺点

优点：
- 不需要在自己电脑上装任何东西（除了 SSH 客户端）
- 它是一台"真实的 Linux 服务器"，跟你面试工作时用的环境一模一样
- 随时随地可以从任何电脑连上去（在教室、图书馆、网吧都能连）
- 你可以把做好的个人项目（网站、博客、API 服务）部署在上面，让全世界访问
- 很多云厂商有学生优惠或新用户免费试用

缺点：
- 需要花钱（虽然很便宜，最低配一个月几十块钱）
- 需要实名认证
- 依赖网络连接，断网就没法操作
- 需要一些基本的网络安全意识（改默认端口、设置防火墙规则）

## 三种方案对比总结

| 对比维度 | WSL2 | VirtualBox | 云服务器 |
|---------|------|------------|---------|
| 安装难度 | 最简单 | 中等 | 中等 |
| 启动速度 | 极快（1 秒） | 慢（30 秒+） | 无需启动 |
| 图形界面 | 无 | 有完整桌面 | 无（除非装 VNC） |
| 内存占用 | 较低 | 高 | 不占本地内存 |
| 费用 | 免费 | 免费 | 需要付费 |
| 和 Windows 互通 | 极好 | 一般 | 靠网络传输 |
| 学习效果 | 好 | 好 | 最好（真实环境） |
| 对电脑要求 | Win10 2004+ | 8GB+ 内存 | 无要求 |

**我的建议**：如果你的 Windows 版本满足条件，首选 **WSL2**。它安装最快、启动最快、和 Windows 集成最好。当你学完本教程的 90% 内容后，如果发现自己对 Linux 很感兴趣，想往运维或后端方向发展，再去申请一台云服务器来"玩真的"。

## 动手试试

1. 如果你的 Windows 10 版本在 2004 以上，现在就打开 PowerShell（管理员），运行 `wsl --install`
2. 重启后设置你的第一个 Linux 用户名和密码
3. 在 Linux 终端里输入 `whoami`，看看显示的是不是刚才设置的用户名
4. 输入 `uname -a`，看看系统信息
5. 输入 `exit` 退出 Linux，回到 PowerShell

（如果你的 Windows 版本太旧或不方便装 WSL，那就去 virtualbox.org 下载 VirtualBox 并开始安装）

## 本节小结

在 Windows 上练习 Linux 有三种方式：WSL2（最简单，推荐）、VirtualBox（有完整桌面）、云服务器（最接近真实工作场景）。新手建议从 WSL2 开始。
