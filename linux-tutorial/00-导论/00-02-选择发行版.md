## 发行版家族：两大门派

如果你打开 DistroWatch.com（一个专门追踪各发行版动态的网站），你会发现 Linux 发行版有好几百个。但别慌，大部分发行版都是从少数几个"老祖宗"派生出来的。对于新手来说，只需要了解两大流派：

**第一派：Debian 系（也叫 apt 系）**

这个名字来自 Debian 这个发行版。Debian 是 1993 年由 Ian Murdock 创立的，名字是他和他女朋友 Debra 的名字组合（Deb + Ian = Debian，Linux 圈最浪漫的命名了）。Debian 的最大特点是**极度稳定**和**完全由社区维护**，没有任何商业公司控制。

从 Debian 派生出来的最著名发行版是 **Ubuntu**。Ubuntu 由南非企业家 Mark Shuttleworth 创立的 Canonical 公司维护，每两年出一个长期支持版（LTS，Long Term Support），提供 5 年以上的安全更新。Ubuntu 的目标是"让 Linux 对人类友好"，它在 Debian 的稳定基础上加了很多易用性的改进。

Debian 系发行版使用 **APT**（Advanced Package Tool）来管理软件包，安装软件的命令是 `apt install`。如果你看到一个教程里写 `apt-get install something`，那肯定是在 Debian 系上。

Debian 系的"族谱"大致如下：

```
Debian (1993)
├── Ubuntu (2004) ─── Canonical 公司维护，最流行
│   ├── Linux Mint ─── 更接近 Windows 界面的体验
│   ├── Pop!_OS ─── System76 公司出品，针对开发者和游戏优化
│   └── elementary OS ─── 模仿 macOS 的颜值
├── Kali Linux ─── 渗透测试专用（黑客工具集）
└── Raspberry Pi OS ─── 树莓派官方系统
```

**第二派：Red Hat 系（也叫 rpm 系 / dnf 系）**

Red Hat 公司成立于 1993 年，是 Linux 商业化的先驱。他们的商业模式很简单：软件免费，服务收费。一套 Red Hat Enterprise Linux（RHEL）本身不要钱，但要获得官方技术支持和安全更新认证，你得付费订阅。

这个模式对个人用户不太友好（谁想为一台个人电脑买企业订阅呢？），但对银行、政府、保险公司的机房来说正合适。

Red Hat 系使用 **RPM**（Red Hat Package Manager）作为包格式，用 **DNF**（Dandified Yum）或旧版 **YUM** 作为包管理器。安装软件的命令是 `dnf install` 或 `yum install`。

Red Hat 系的"族谱"如下：

```
Red Hat Enterprise Linux (RHEL) ─── 商业版，需要订阅
├── Fedora ─── Red Hat 赞助的社区版，新技术试验田
├── CentOS (已消亡) ─── 曾经是 RHEL 的免费克隆版
│   └── CentOS Stream ─── RHEL 的"预览版"，滚动更新
└── Rocky Linux / AlmaLinux ─── CentOS 的继任者，RHEL 的免费克隆版
```

**CentOS 的故事**值得单拎出来讲。CentOS 曾经是服务器领域最受欢迎的免费发行版，因为它就是 RHEL 去掉商标后的完全克隆版。但 2020 年 Red Hat 宣布终止 CentOS 项目，改为 CentOS Stream（一个"预览版"而非稳定克隆版）。这件事在 Linux 社区引起了轩然大波，很多人感觉被"背叛"了。

于是 CentOS 的原联合创始人 Gregory Kurtzer 创建了 **Rocky Linux**（以另一位已故的 CentOS 联合创始人 Rocky McGough 命名）。Rocky Linux 的目标和原来的 CentOS 一模一样：做 RHEL 的免费、稳定、1:1 克隆版。同时，CloudLinux 公司也推出了 **AlmaLinux** 作为另一个替代品。

**类比**：这就像你去买牛仔裤。Levi's 就是 RHEL（正品，价格高）。突然有一家工厂用完全相同的布料和工艺生产了一模一样的牛仔裤，换了个牌子叫 CentOS，价格只有十分之一。大家纷纷转买这个牌子的牛仔裤。结果有一天，Levi's 收购了这家工厂，把 CentOS 这个牌子改成了一条实验性产品线（水洗一次就掉色的那种）。于是工厂里的老员工愤而离职，新开了一家叫 Rocky 的工厂，继续生产以前那种高品质的牛仔裤。

## 四大发行版详细对比

| 特性 | Ubuntu | Debian | Rocky Linux | Fedora |
|------|--------|--------|-------------|--------|
| 所属派系 | Debian 系 | Debian 系（祖宗） | Red Hat 系 | Red Hat 系 |
| 维护方 | Canonical 公司 | 社区志愿者 | Rocky 社区 | Red Hat 赞助 |
| 发布节奏 | 半年/小版，两年/LTS | 约两年一次 | 跟随 RHEL | 约半年一次 |
| 稳定性 | LTS 版很稳 | 极稳 | 极稳 | 较新，偶尔翻车 |
| 软件新旧 | LTS 偏旧但安全 | 偏旧但极其稳定 | 偏旧但极其稳定 | 很新 |
| 桌面体验 | 最好，开箱即用 | 需要自己配置 | 不推荐做桌面 | 不错的桌面体验 |
| 默认桌面 | GNOME（已修改） | 自选 | GNOME | GNOME（原版） |
| 社区支持 | 极为丰富 | 非常丰富 | 快速增长中 | 丰富 |
| 企业使用 | 大量中小企业 | 追求极致稳定的场景 | 替代 CentOS | 较少 |

## 我该选哪一个？

这取决于你的目标。大多数读者学习 Linux 是为了以下某个原因，我们逐一分析：

**情况一：纯新手，想有一台"像 Windows 一样好用的 Linux 桌面"**

选 **Ubuntu LTS**（当前最新 LTS 是 24.04）。理由：
- 安装简单，全程图形化向导
- 中文支持好，输入法装好就能用
- 网上 90% 的 Linux 教程以 Ubuntu 为例，你能直接复制粘贴命令
- 应用商店（Ubuntu Software）里面常用的软件都有
- 官方论坛（Ask Ubuntu）和中文社区都非常活跃，遇到问题一搜就有答案

如果你觉得 Ubuntu 的界面不太习惯，可以试试 **Linux Mint**。它默认的桌面布局（底部任务栏、左下角开始菜单、右下角系统托盘）和 Windows 几乎一模一样，从 Windows 迁移过来几乎零学习成本。

**类比**：Ubuntu 像是丰田卡罗拉--不是最快的，不是最酷的，但是最靠谱、最省心、修车铺最多的。新手第一辆车买卡罗拉永远不会错。

**情况二：想学服务器运维，以后做后端开发或运维工程师**

选 **Rocky Linux** 或 **AlmaLinux**。理由：
- 中国的互联网公司服务器大多用 CentOS（现在正在向 Rocky/Alma 迁移），你在自己电脑上学的内容和公司的生产环境完全一致
- RHEL 体系是国内企业级 Linux 的事实标准，RHCE（Red Hat 认证工程师）是最受认可的 Linux 证书
- 你用 `dnf`、`systemctl`、`firewalld`、SELinux 这些 Red Hat 系的工具，跟面试官聊起来完全对得上

如果你暂时不确定走哪条路，**Ubuntu LTS 仍然是最好的起点**。Ubuntu 的服务器版（Ubuntu Server）也很流行，而且 Amazon AWS、Google Cloud 上的 Ubuntu Server 使用量非常大。两条路在技术上是相通的：掌握了 Ubuntu 上的命令行操作，切换到 Rocky Linux 只需要适应一下包管理器的命令差异（`apt` vs `dnf`），别的几乎一样。

**情况三：想在老旧电脑上运行 Linux，让它起死回生**

选带轻量桌面的发行版。**Lubuntu**（Ubuntu 的轻量变体）或 **Linux Mint Xfce 版** 都可以在 1GB 内存的老电脑上流畅运行。你把家里那个开机要五分钟的老笔记本装上 Lubuntu，它会像新电脑一样快。

**类比**：Windows 10 就像一个 200 斤的大汉，需要大房子（8GB 内存、SSD 硬盘）才能住得舒服。Lubuntu 像一个 40 斤的小孩，一个单人间（1GB 内存）就能活蹦乱跳。

**情况四：只是想试试，不想改动现在的 Windows 系统**

先别急着装系统。下一节我们会讲三种不破坏现有系统的练习方式：
- WSL2：在 Windows 里直接跑一个 Linux，无缝切换
- VirtualBox：装一个虚拟机，Linux 跑在一个窗口里
- 云服务器：在云厂商那里租一台，通过 SSH 远程连接

建议你读完下一节再决定用哪种方式开始。

## 发行版选择速查表

```
我想要...
├── ...最像 Windows 的体验         → Linux Mint
├── ...最好看的桌面                → Ubuntu 或 elementary OS
├── ...最稳定的服务器              → Debian 或 Rocky Linux
├── ...最新的软件                  → Fedora
├── ...学习企业级技能              → Rocky Linux
├── ...在老电脑上跑               → Lubuntu 或 Linux Mint Xfce
├── ...渗透测试/网络安全          → Kali Linux
└── ...终极定制自由               → Arch Linux（不推荐新手）
```

## 动手试试

1. 打开浏览器，访问 https://distrowatch.com/，看看右侧的"Page Hit Ranking"，了解当前哪些发行版最热门
2. 访问 https://ubuntu.com/download，看看 Ubuntu 的下载页面长什么样。注意区分 LTS 版本和普通版本
3. 搜索"Ubuntu vs Linux Mint screenshot"，看看两种桌面长什么样，选择你更喜欢的那一个

## 本节小结

Linux 发行版分为 Debian 系（apt）和 Red Hat 系（dnf）两大派别。新手第一台 Linux 推荐 Ubuntu LTS，想做服务器运维推荐 Rocky Linux，想拯救老电脑推荐 Lubuntu。
