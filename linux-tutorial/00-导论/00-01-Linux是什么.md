# 00-01 Linux 是什么

- 明白 Linux 为什么在今天如此重要
- 知道 Linux 和 Windows/macOS 的根本不同之处
- 建立"一切皆文件"的 Linux 核心世界观

---

如果你每天打开电脑，点一下"开始菜单"，找到浏览器图标，然后上网看视频--那你用的很可能是 Windows。Windows 就像一辆自动挡汽车：你只需要踩油门和刹车，引擎盖下面发生了什么，你不用管。而 Linux 就像一辆手动挡赛车：你可以看到引擎的每一个零件，可以亲手调整每一个参数。刚开始学开手动挡确实有点手忙脚乱，但一旦掌握了，你会发现自己对汽车的掌控力提升了不止一个档次。

这就是 Linux 最大的魅力：它把计算机的控制权完完整整地交到你手里。

## 操作系统内核：计算机的管家

每一台计算机都需要一个"管家"来管理硬件资源--CPU、内存、硬盘、网卡等等。这个管家叫做**操作系统内核（Kernel）**。内核的工作非常好理解：

- 当微信要保存一张图片到硬盘，内核负责跟硬盘说"喂，把这个数据写到某个位置"
- 当浏览器要上网，内核负责指挥网卡收发网络数据
- 当你同时开了二十个程序，内核负责分配 CPU 时间，让每个程序都能分到计算资源

你可以把内核想象成一家大公司里的**前台接待员加保安队长**：所有外来人员（程序）想进入公司（使用硬件），都必须先经过这个前台。前台核实你身份（权限检查），告诉你哪个会议室可以用（内存分配），如果有人捣乱就把你轰出去（进程管理）。

Windows 有 Windows 的内核，macOS 有 macOS 的内核（叫 XNU），而 Linux 就是另一种内核。问题的关键在于：你在电脑城买到的"Windows 系统 U 盘"，里面包含的不只是内核，还有桌面、文件管理器、计算器、写字板、浏览器等等一大堆软件。而 "Linux" 这个词，严格来说只指那个内核本身。

**类比**：好比一辆车的发动机（内核）和整车（操作系统）的区别。丰田凯美瑞和雷克萨斯 ES 用的是同一款发动机，但外观、内饰、座椅、音响完全不同。Linux 就是那款发动机，不同的**发行版（Distribution）**就是不同的车型。

## 发行版：一辆完整的车

光有一个内核，你什么也干不了。你不会想直接跟内核打交道--那要写 C 语言代码才行。你需要一个完整的操作系统，包括：

| 组件 | 作用 | 类比 |
|------|------|------|
| 内核（Kernel） | 管理硬件 | 发动机 |
| Shell（命令行） | 接收你的指令 | 方向盘和油门 |
| 核心工具（GNU 工具集） | 文件操作、文本处理 | 底盘和刹车系统 |
| 桌面环境（GUI） | 图形化界面 | 车身外壳和座椅 |
| 包管理器 | 安装/卸载软件 | 4S 店的配件仓库 |

把这些组件打包在一起，经过配置和测试，形成一个可以直接安装使用的完整系统，这就是**Linux 发行版**。常见的发行版有 Ubuntu、Debian、CentOS、Rocky Linux、Fedora、Arch Linux 等。你在网上下载的那个 ".iso" 文件，就是某个发行版的安装镜像。

**类比**：你家装修房子。内核是地基和承重墙（不能动，也不能少），发行版是精装修方案。开发商给了你几种装修套餐：经济型（Debian 稳定版）、舒适型（Ubuntu 带全套桌面）、极简型（Arch 连墙纸都不贴，你自己来）。你选一个套餐，拎包入住。

## Linux 的历史：一个"业余爱好"如何改变了世界

故事要从 1991 年说起。芬兰赫尔辛基大学有个 21 岁的大学生叫林纳斯-托瓦兹（Linus Torvalds）。他买了一台新电脑，想在上面运行 Unix 系统（当时大学里主流的操作系统），但商业 Unix 太贵了，于是他决定自己写一个。

1991 年 8 月 25 日，林纳斯在 Usenet 新闻组发了一封著名的帖子：

> "我在做一个（免费的）操作系统（只是爱好，不会像 GNU 那么专业）……"

他绝对没想到，这个"业余爱好"三十多年后会运行在全球 96% 以上的服务器、100% 的超级计算机、以及超过 30 亿台 Android 手机上。

**类比**：这就像一个小伙子在自家车库里开始鼓捣一个木工活，结果三十年后全世界最大的商场、机场、火车站都用了他的家具设计。不是因为他一开始就野心勃勃，而是因为他做的东西**开放、透明、允许任何人来改进**。

Linux 之所以能够迅速成长，还因为它踩在了巨人的肩膀上。早在 1983 年，理查德-斯托曼（Richard Stallman）就发起了 **GNU 项目**，目标是创造一套完全自由的操作系统。GNU 项目写好了编译器（GCC）、编辑器（Emacs）、Shell（Bash）、以及大量核心工具，但独缺一个内核。Linux 内核的出现正好填补了这个空缺。所以严谨地说，我们日常用的是 **GNU/Linux 系统**，不过大家习惯上就叫 Linux。

## 为什么 Linux 在今天如此重要

如果 Linux 只是一个"极客的玩具"，它不会存活到今天。真正让它无处不在的原因有四个：

**1. 免费开源。** 你不会收到微软的律师函说你侵权，也不需要为每台服务器购买授权。对于互联网公司来说，这意味着可以部署成千上万台服务器而不用支付操作系统费用。

**2. 服务器领域的统治地位。** 你每天用的微信、淘宝、抖音、百度，后端服务器几乎全部运行 Linux。AWS、Azure、Google Cloud 上最常见的操作系统也是 Linux。如果你将来想从事后端开发、运维、网络安全、大数据、人工智能中的任何一个方向，Linux 是绕不过去的。

**3. Android 底层是 Linux。** 全球 30 多亿部 Android 手机的内核就是 Linux。你其实已经在用 Linux 了，只是被一层"手机壳"包裹着，你没意识到而已。你世界的每个角落都有 Linux 在运转--路由器、智能电视、汽车中控、甚至你的微波炉。

**4. 学习计算机原理的最佳工具。** Windows 把底层的复杂性全部隐藏了起来，你想看也看不到。Linux 把所有东西都摊开给你看：进程长什么样（/proc 目录）、设备怎么通信（/dev 目录）、系统怎么配置（/etc 下的文本文件）。对于学计算机的人来说，Linux 是最好的教科书。

**类比**：Windows 像是去餐厅吃饭，你看到的是精美的菜品（图形界面），后厨发生了什么你不知道。Linux 像是开放式厨房，你可以走到灶台前看厨师怎么做的，甚至可以自己动手炒两个菜。对于想学做菜（学计算机）的人来说，开放式厨房当然是最好的选择。

## Linux 和 Windows 的核心思维差异

在 Windows 里，你习惯了这样的世界：

- C 盘、D 盘、E 盘，每个盘是一个独立的"容器"
- 程序放在 `C:\Program Files\`，自己的文件放在 `D:\我的文档\`
- 想改个系统设置，去"控制面板"里点点点
- 安装软件，下载 .exe 然后下一步下一步下一步

在 Linux 里，世界观完全不同：

- **只有一个根目录 `/`**。没有 C 盘 D 盘的概念。所有硬盘分区、U 盘、光盘、网络存储全都"挂载"到 `/` 下面的某个目录（比如 U 盘挂到 `/media/usb`）
- **一切皆文件。** 普通文件是文件，目录是文件，键盘是文件（`/dev/input/`），硬盘是文件（`/dev/sda`），进程信息也是文件（`/proc/`）。甚至你当前运行程序的"内存脑图"都可以通过读取文件来查看
- **配置是文本文件。** 没有注册表这种东西。所有系统配置、软件配置都是纯文本文件，用任何编辑器都能打开修改
- **软件来自统一的"仓库"。** 不用去网上搜 `.exe` 下载。一个命令 `apt install` 或 `dnf install`，系统会自动从官方仓库下载安装

**类比**：Windows 是一个"封闭式小区"，物业规定了统一的装修风格，你看不到管道怎么走的。Linux 是一个"开放式街区"，水电煤气管道全部在地上明铺，你可以自己改管道、自己装水龙头，但前提是你要知道在改什么--不然你可能把整栋楼的供水搞炸。

## 动手试试

虽然你还没有安装 Linux，但你可以在本节先在脑海中建立正确概念。请做以下练习：

1. 打开你 Windows 电脑的"此电脑"（或"我的电脑"），看看有几个盘符（C 盘、D 盘等）
2. 在心里想象一下：如果把所有的盘都合并成一棵树，根节点叫 `/`，C 盘变成 `/c`，D 盘变成 `/d`——这就是 Linux 的目录结构雏形
3. 打开你的浏览器，搜索"Top 500 supercomputer Linux"，看看目前全球最快的 500 台超级计算机中有多少台运行 Linux（你会发现是 100%）

## 本节小结

Linux 是一个开源的操作系统内核，搭配 GNU 工具集和各种软件组件，形成了现在我们使用的 Linux 发行版。它的设计哲学是"一切皆文件"，把对计算机的完整控制权交给用户。
