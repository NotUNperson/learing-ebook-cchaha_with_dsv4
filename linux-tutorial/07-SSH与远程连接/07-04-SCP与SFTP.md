# 07-04 SCP 与 SFTP

## 本节你会学到什么

- 使用 `scp` 在本地和远程服务器之间复制文件和目录
- 使用 `sftp` 进入交互式文件管理界面
- 使用 `rsync` 实现高效的增量目录同步
- 理解三种工具的适用场景和选择策略
- 用"快递寄送"的类比记住文件传输的核心概念

---

你已经会登录远程服务器了。接下来最常做的事是什么？传文件。

你的电脑上有新版本的代码，需要传到服务器上。服务器的日志需要下载到本地分析。几十台服务器要同步同一个配置目录。这些都需要安全、高效的文件传输工具。

SSH 生态提供了三个核心工具：`scp`（一次性复制）、`sftp`（交互式浏览加复制）、`rsync`（增量同步）。它们都跑在 SSH 的加密通道上，保证传输安全。

---

## 类比：快递服务的三种模式

**scp** 就像 **普通快递**。你把包裹交给快递员，告诉他地址，他就送过去了。一次性动作，简单直接。适合"就寄这一件"的场景。

**sftp** 就像 **自助仓储超市**。你自己推购物车进去，在货架上挑挑拣拣，选好的东西打包结账带走。适合"我还不确定要传哪些文件，要先看看"的场景。

**rsync** 就像 **搬家公司的分批托运**。第一次把一车东西送过去，之后每次只送"新增加或改动过"的东西。省时省力。

---

## SCP：安全复制

SCP（Secure Copy）是最简单的文件传输工具，语法非常直观：

```bash
scp [选项] 源 目标
```

路径中如果涉及远程，格式是 `user@host:路径`。

### 上传（本地到远程）

```bash
# 上传单个文件
scp myapp.tar.gz user@server:/opt/deploy/

# 上传整个目录（-r 递归）
scp -r ./dist/ user@server:/var/www/

# 上传多个文件
scp file1.txt file2.txt file3.txt user@server:/home/user/

# 指定端口（注意是大写 -P！）
scp -P 2222 backup.tar.gz user@server:/backup/
```

注意：SCP 的端口选项是 `-P`（大写），而 SSH 的端口选项是 `-p`（小写）。这种不一致是历史原因造成的，很多人都会搞混。记住：**SCP = 大写 P**。

### 下载（远程到本地）

```bash
# 下载单个文件
scp user@server:/var/log/syslog ./local_logs/

# 下载整个目录
scp -r user@server:/etc/nginx/ ./nginx_config_backup/

# 下载多个文件
scp user@server:'/var/log/{syslog,auth.log}' ./logs/
```

### 常用选项

| 选项 | 作用 | 场景 |
|------|------|------|
| `-r` | 递归复制目录 | 传输整个目录 |
| `-p` | 保留文件属性（时间戳、权限） | 备份场景 |
| `-P` | 指定端口（大写） | 非标准端口 |
| `-C` | 传输时压缩 | 慢速网络 |
| `-q` | 安静模式 | 脚本中使用 |
| `-v` | 详细模式 | 排查问题 |
| `-l N` | 限速 N Kbit/s | 避免占满带宽 |

### SCP 的局限性

SCP 简单好用，但它有几个局限：
- 不支持断点续传（传一半断了得重头来）
- 不支持增量传输（每次都传完整文件）
- 不支持排除特定文件（.log、.tmp 文件也一并传）
- 没有进度条（大文件传输时不知道进展）

这些局限在传输大量文件时就很明显了。这时就需要 `rsync`。

## SFTP：交互式文件管理

SFTP 启动后会进入一个交互式界面，类似于一个简化的 FTP 客户端，但一切都在 SSH 加密通道内。

### 连接

```bash
sftp user@host
sftp -P 2222 user@host    # 指定端口
```

连接成功后，你会看到 `sftp>` 提示符，等待你输入命令。

### 交互式命令速查

**导航类（和 Linux 命令很像，加上 `l` 前缀是本地的意思）**

| 命令 | 作用 |
|------|------|
| `pwd` | 显示远程当前目录 |
| `lpwd` | 显示本地当前目录 |
| `cd /path` | 切换远程目录 |
| `lcd /path` | 切换本地目录 |
| `ls` | 列出远程目录内容 |
| `lls` | 列出本地目录内容 |
| `mkdir dir` | 创建远程目录 |
| `lmkdir dir` | 创建本地目录 |

**传输类**

| 命令 | 作用 |
|------|------|
| `get file` | 下载文件到本地当前目录 |
| `get -r dir` | 递归下载目录 |
| `put file` | 上传文件到远程当前目录 |
| `put -r dir` | 递归上传目录 |
| `mget *.log` | 批量下载（按模式匹配） |
| `mput *.txt` | 批量上传 |
| `reget file` | 断点续传下载 |
| `reput file` | 断点续传上传 |

**其他**

| 命令 | 作用 |
|------|------|
| `!command` | 在本地 Shell 执行命令 |
| `help` | 显示帮助 |
| `bye` / `exit` | 退出 |

### 在脚本中使用 SFTP

SFTP 可以通过重定向实现非交互式操作：

```bash
sftp user@host << EOF
put /local/file1.txt /remote/path/
put /local/file2.txt /remote/path/
get /remote/log.txt /local/
bye
EOF
```

**类比**：交互式 SFTP 就像你走进一家店，先看看有什么，再决定买什么。非交互式 SFTP 则是你在网上下了单，快递员按清单拣货送到你手里。

## rsync：增量同步之神

`rsync` 不是 SSH 家族的一员，但它可以通过 SSH 传输，是生产环境中最重要的文件同步工具。如果你只能记住这一节的一个工具，记住 `rsync`。

### rsync 的核心优势

1. **增量传输**：只传有变化的部分，不是整个文件重新发
2. **断点续传**：传一半断了可以接着传
3. **选择性同步**：可以用 `--exclude` 跳过不需要的文件
4. **一致性保证**：加 `--delete` 让目标端和源端完全一致
5. **丰富选项**：限速、压缩、保留属性、干跑等等

### 基本语法

```bash
# 本地同步
rsync [选项] 源目录/ 目标目录/

# 远程同步（走 SSH）
rsync [选项] /local/dir/ user@host:/remote/dir/
rsync [选项] user@host:/remote/dir/ /local/dir/
```

注意源目录末尾的 `/`：
- `src/`（有斜杠）：同步目录**内容**，不包括 `src` 目录本身
- `src`（无斜杠）：同步目录**本身**（在目标创建 `src` 目录）

```bash
# 有斜杠：把 src 里面的东西放到 backup 里
rsync -av src/ backup/
# 结果：backup/file1, backup/file2...

# 无斜杠：src 整个目录放到 backup 里
rsync -av src backup/
# 结果：backup/src/file1, backup/src/file2...
```

这个细微差别让无数人踩过坑，请你一定记住。

### 核心选项

```bash
rsync -avzP --delete --exclude='*.log' /source/ user@host:/dest/
```

| 选项 | 含义 |
|------|------|
| `-a` | 归档模式：保留权限、时间戳、符号链接等（`-rlptgoD` 的组合） |
| `-v` | 详细输出，看到传了哪些文件 |
| `-z` | 传输时压缩 |
| `-P` | 显示进度 + 支持断点续传 |
| `-n` | dry-run（干跑）：只显示会做什么，不真执行 |
| `--delete` | 删除目标端多余文件（让两端完全一致） |
| `--exclude='*.log'` | 排除特定模式文件 |
| `--bwlimit=5000` | 限速 5000 KB/s |
| `-e 'ssh -p 2222'` | 指定 SSH 选项 |

### 生产常用组合

```bash
# 同步网站文件（最常用）
rsync -avzP --delete /var/www/ user@server:/var/www/

# 执行前先干跑检查（重要习惯！）
rsync -avzn --delete /var/www/ user@server:/var/www/

# 限速同步，避免占满带宽
rsync -avzP --bwlimit=5000 /bigdata/ user@host:/backup/
```

### rsync 的增量备份原理

```bash
# 第一次：完整备份
rsync -av /data/ /backups/full/

# 第二次：增量备份，利用 --link-dest 做硬链接去重
rsync -av --link-dest=/backups/full/ /data/ /backups/incr_1/
```

`--link-dest` 让 rsync 对没变化的文件创建硬链接而不是复制。这样每个"增量备份"看起来是完整目录，但其实不占用额外磁盘空间（没变的文件都硬链接到之前的备份）。

**类比**：就像你每天拍一张办公桌的照片。第一天是完整的照片。第二天只有桌面上多了一杯咖啡，你只需要标注"昨天照片基础上，这里多了一杯咖啡"。rsync 就是这个逻辑的具体实现。

## 三种工具的选择策略

| 场景 | 推荐工具 | 原因 |
|------|----------|------|
| 传一个配置文件到服务器 | `scp` | 最简单，一行搞定 |
| 从服务器下载日志，不确定日志文件名 | `sftp` | 交互式浏览后再下载 |
| 每天同步整个网站目录 | `rsync` | 增量传输，省时省流量 |
| 批量部署代码到 10 台服务器 | `rsync` + 循环 | 增量 + 批量 |
| 传输 10GB 的数据库备份 | `rsync -P` | 支持断点续传 |
| 浏览远程服务器的文件结构 | `sftp` | 就像在本地浏览文件 |

## 动手试试

1. 创建几个测试文件，用 `scp` 传到远程服务器（或本机的 `/tmp/` 目录）
2. 用 `sftp` 连接远程服务器，在交互式界面里执行 `ls`、`pwd`、`get` 等命令
3. 创建一个包含多个文件的测试目录，用 `rsync -avzn` 先干跑查看效果，再用 `rsync -avz` 真正同步
4. 修改源目录中的一个文件，再次运行 `rsync`，观察它只传输了变化的文件
5. 用 `rsync --exclude='*.log'` 排除日志文件，或 `--exclude='node_modules/'` 排除依赖目录
6. 在 `rsync -n`（干跑）和真正执行之间，养成"先看再做"的好习惯

---

## 本节小结

SCP 适合快速复制单个文件，语法简单直观。SFTP 提供交互式文件管理界面，适合浏览和选择性传输。rsync 是文件同步的终极工具，增量传输、断点续传、选择性排除，是生产环境文件同步的标准方案。三剑客配合使用，覆盖所有远端文件传输需求。
