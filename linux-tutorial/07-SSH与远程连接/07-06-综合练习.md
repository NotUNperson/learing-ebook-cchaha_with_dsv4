# 07-06 综合练习：tmux 会话保持

## 本节你会学到什么

- 理解 tmux 的"会话保持"能力及其与普通 SSH 的本质区别
- 掌握 tmux 的三层结构：会话 (Session)、窗口 (Window)、窗格 (Pane)
- 熟练使用 tmux 的常用快捷键进行分屏、切换、脱离
- 编写一个自动化脚本，SSH 登录后自动连接或创建 tmux 会话
- 用"云端桌面"的类比理解 tmux 的核心价值

---

假设你正在远程服务器上跑一个数据导入脚本，预计要跑 2 小时。你已经等了 1 小时 45 分钟，马上要完成了。这时候你的笔记本突然弹出一个提示："电量不足，即将休眠。"

没有 tmux 的话，SSH 连接会断开，那个跑了 105 分钟的脚本会被无情地杀死。你得从头开始。这种痛苦，用过 SSH 的都懂。

tmux 就是来解决这个问题的。它让你的远程工作状态"云存储"在服务器上，即使你的本地连接断了，工作仍在继续 -- 就像你在服务器上租了一个"永不关机的小桌面"。

---

## 类比：云端桌面 vs 临时借用的电脑

**普通 SSH** = 你去网吧上机。你一走（断开连接），电脑上的东西就没了，别人可能重启了机器。

**tmux** = 你在云服务商那里租了一台虚拟桌面。你在自己笔记本上远程操控它，即使笔记本关机了，虚拟桌面还在云上继续运行。你换一台电脑重新连接，桌面原封不动。

这就是 tmux 的本质：**终端会话和网络连接的解耦**。你的工作状态（tmux 会话）活在服务器上，网络连接只是一个"窗口"，窗口关了可以再开。

---

## tmux 的三层结构

tmux 有三个层级，从大到小：

```
Session（会话）
  └── Window（窗口）
        └── Pane（窗格）
```

**类比**：一栋写字楼。

- **Session（会话）** = 你在写字楼里租的一个套间。可以有多个房间。
- **Window（窗口）** = 套间里的一个房间。可以有一个或多个窗格。
- **Pane（窗格）** = 房间里的一个工作区。你可以在一个屏幕上同时做好几件事。

你可以同时拥有多个 Session（不同的项目），每个 Session 里有多个 Window（不同的任务），每个 Window 里可以分多个 Pane（同时监控多个东西）。

## 安装和基本使用

### 安装

```bash
# Ubuntu/Debian
sudo apt install tmux

# CentOS/RHEL
sudo yum install tmux

# 验证安装
tmux -V
```

### 第一个 tmux 会话

```bash
# 创建并进入一个命名会话
tmux new -s mywork

# 你会进入一个新的 Shell 界面，底部出现状态栏
# 在这个 Shell 里运行的任何东西，断开后都不会丢失
```

在 tmux 里面，随便跑几个命令，然后按 `Ctrl+B` 然后按 `d`（脱离 detach）。你会回到原来的终端，tmux 会话在后台继续运行。

```bash
# 查看所有活动会话
tmux ls
# 输出类似：mywork: 1 windows (created Fri May 15 14:30:00 2026)

# 重新连接（reattach）
tmux attach -t mywork

# 或者简写
tmux a -t mywork
```

你会发现之前跑的命令、输出的内容，全都还在 -- 就像你从来没离开过一样。

---

## 前缀键：Ctrl+B 是 tmux 的"翻译官"

在 tmux 内部，你不能直接使用常规快捷键（比如 `Ctrl+C` 会被 tmux 拦截或处理）。tmux 规定了一个"前缀键" -- 你需要先按 `Ctrl+B`，松开，然后再按下一个键，tmux 才会识别为"快捷键"。

**类比**：`Ctrl+B` 就像是按下对讲机的通话键。你不按通话键的时候，说的话是给身边人听的（给当前的 Shell）。按下通话键之后，说的话是给对讲机另一头的人听的（给 tmux 的指令）。

如果你觉得 `Ctrl+B` 按起来不顺手，可以在 `~/.tmux.conf` 中改成 `Ctrl+A`（很多 screen 老用户喜欢这个）：

```bash
set -g prefix C-a
unbind C-b
bind C-a send-prefix
```

## 窗格管理：一块屏幕当多块用

这是 tmux 最常用的功能之一。不用开多个 SSH 窗口，一个连接里就能同时看到多个终端。

```bash
# 在 tmux 会话中：

Ctrl+B  %         # 在右边创建一个新窗格（垂直分屏，左右排列）
Ctrl+B  "         # 在下面创建一个新窗格（水平分屏，上下排列）

Ctrl+B  方向键     # 切换到相邻的窗格
Ctrl+B  o         # 切换到下一个窗格
Ctrl+B  ;         # 切换到上一个活动窗格

Ctrl+B  z         # 当前窗格全屏 / 恢复（zoom）
Ctrl+B  x         # 关闭当前窗格（需要确认）
```

**实用场景**：

```
+---------------------------+---------------------------+
|  htop（系统监控）          |  ssh user@other-server     |
|                           |  管理另一台服务器           |
|                           |                           |
|                           +---------------------------+
|                           |  tail -f /var/log/syslog  |
|                           |  实时查看日志              |
+---------------------------+---------------------------+
```

一个 SSH 连接，四个窗格，同时监控系统状态、管理远程服务、查看日志 -- 像不像一个运维指挥中心？

---

## 窗口管理：像浏览器标签页

如果窗格还不够（屏幕太挤），你可以创建多个窗口，就像浏览器打开多个标签页：

```bash
Ctrl+B  c         # 创建新窗口
Ctrl+B  n         # 下一个窗口（next）
Ctrl+B  p         # 上一个窗口（previous）
Ctrl+B  0-9       # 直接跳到第 N 号窗口
Ctrl+B  ,         # 重命名当前窗口
Ctrl+B  &         # 关闭当前窗口
Ctrl+B  w         # 列出所有窗口
```

**实用场景**：

| 窗口名 | 用途 |
|--------|------|
| `monitor` | htop + tail 日志，监控用 |
| `deploy` | 部署代码，git pull + build |
| `db` | 数据库操作，mysql 客户端 |
| `edit` | Vim 编辑配置文件 |
| `misc` | 杂项，临时测试命令 |

用 `Ctrl+B ,` 给每个窗口重命名，底部状态栏会显示窗口列表，工作和切换非常流畅。

---

## 复制模式：回滚查看历史输出

普通终端的历史输出是有限制的（由终端模拟器的滚动缓冲决定），而且 tmux 默认情况下鼠标滚轮滚动的是 tmux 的历史缓冲区，不是 Shell 的历史。

```bash
Ctrl+B  [         # 进入复制模式
```

在这个模式下：
- 用方向键或 `hjkl`（vi 风格）移动光标
- `PageUp` / `PageDown` 翻页
- `Space` 开始选择文本
- `Enter` 复制选中的文本并退出
- `q` 退出复制模式
- `/关键词` 向下搜索
- `?关键词` 向上搜索
- `g` 跳到最开头
- `G` 跳到最末尾

如果你在 `~/.tmux.conf` 中启用了 `set -g mouse on`，甚至可以直接用鼠标滚轮滚动和点选文本，不需要进入复制模式。

## 会话共享：结对编程的神器

tmux 的一个高级功能是**两个终端同时 attach 到同一个会话**。两个人看到完全一样的屏幕，任何一方的操作双方都能看到：

```bash
# 先创建一个共享会话
tmux new -s pair-programming

# 另一个人在另一个终端（或另一台机器通过 SSH）也 attach
tmux attach -t pair-programming

# 或者用 ssh 一行完成
ssh user@server -t tmux attach -t pair-programming
```

这就是远程结对编程的基础设施。不需要任何第三方工具，SSH + tmux 就够了。

## 自动化连接脚本

每次 SSH 到服务器后手动输入 `tmux new` 或 `tmux attach` 也挺烦的。写一个自动检测的脚本：

```bash
#!/bin/bash
# auto-tmux.sh -- SSH 登录后自动进入 tmux
REMOTE=$1
SESSION=${2:-main}

ssh -t "$REMOTE" "
    if tmux has-session -t '$SESSION' 2>/dev/null; then
        tmux attach -t '$SESSION'
    else
        tmux new -s '$SESSION'
    fi
"
```

`ssh -t` 的 `-t` 很重要 -- 它强制分配伪终端，没有它 tmux 无法启动。

## 推荐配置：~/.tmux.conf

```bash
# ========================================
# ~/.tmux.conf
# ========================================

# 启用鼠标（强烈推荐！滚轮、点击都能用）
set -g mouse on

# 窗口和窗格编号从 1 开始（直觉更自然）
set -g base-index 1
setw -g pane-base-index 1

# 减少 Escape 等待时间
set -sg escape-time 10

# 增大历史回滚行数
set -g history-limit 50000

# 启用 256 色
set -g default-terminal "screen-256color"

# 用友好的快捷键分屏
bind | split-window -h      # Ctrl+B | 左右分
bind - split-window -v      # Ctrl+B - 上下分

# vi 风格移动窗格
bind h select-pane -L
bind j select-pane -D
bind k select-pane -U
bind l select-pane -R

# 状态栏颜色
set -g status-bg colour236
set -g status-fg white
```

创建后重载配置：

```bash
tmux source-file ~/.tmux.conf
```

## 在没有 tmux 时的备选方案：nohup

如果你登录后发现服务器没装 tmux（又没有 sudo 权限安装），可以用传统的 `nohup` 命令应急：

```bash
nohup your_command > output.log 2>&1 &
```

`nohup` 让命令忽略 SIGHUP 信号（终端关闭时发出的"挂断"信号），`&` 让命令在后台运行。但它的体验远不如 tmux -- 你无法"重新接回去"看中间输出，只能等它跑完看日志。

这就是为什么学 tmux 是值得的。

## tmux vs screen 简要对比

GNU Screen 是 tmux 的前辈，功能类似。目前 tmux 是主流选择。

| 特性 | tmux | GNU Screen |
|------|------|------------|
| 开发活跃度 | 活跃维护 | 更新较少 |
| 配置文件 | `~/.tmux.conf` | `~/.screenrc` |
| 窗格分屏 | 原生支持 | 需要补丁 |
| 状态栏 | 更灵活美观 | 较基础 |
| 会话共享 | 原生支持 | 支持 |
| 学习曲线 | 中等 | 中等 |

除非你在维护老旧系统只能用 screen，否则选择 tmux。

## 常见问题

**Q: tmux ls 显示 "no server running on ..."**
A: 这是正常的，说明当前没有 tmux 会话在运行。启动一个新会话即可：`tmux new -s mysession`

**Q: 快捷键 Ctrl+B 没反应？**
A: 确认按法正确：按住 `Ctrl`，点一下 `B`，松开所有键，再按后面的键。不是 Ctrl+B 和其他键同时按住。

**Q: tmux 中鼠标滚轮不能滚动？**
A: 在 `~/.tmux.conf` 中加 `set -g mouse on`。或在不改配置的情况下按 `Ctrl+B [` 进入复制模式再滚动。

**Q: tmux 里的颜色不好看？**
A: 在 `~/.tmux.conf` 中加 `set -g default-terminal "screen-256color"`，同时确保你的终端模拟器支持 256 色。

## 动手试试

1. 安装 tmux：`sudo apt install tmux`
2. 创建第一个会话：`tmux new -s test`
3. 在会话中分屏：`Ctrl+B %`（左右分），`Ctrl+B "`（上下分）
4. 在各个窗格中运行不同命令（如 `top`、`htop`、`watch date`）
5. `Ctrl+B d` 脱离会话，然后 `tmux a -t test` 重新连接
6. 写一个测试脚本在 tmux 中运行：
   ```bash
   for i in {1..60}; do
       echo "计数：$i -- $(date)"
       sleep 5
   done
   ```
   然后 `Ctrl+B d` 脱离，过一会再连接回来，看到计数器仍然在跑
7. 创建 `~/.tmux.conf`，至少启用鼠标支持
8. 创建新窗口 `Ctrl+B c`，在两个窗口间切换 `Ctrl+B n/p`
9. SSH 到远程服务器并重复以上步骤，体验真正的"断线重连"
10. 在两个终端 attach 同一个会话，观察同步效果

## 本节小结

tmux 是远程工作的"会话保持神器"，它让你的终端会话不受网络连接中断的影响，重新连接后一切如初。掌握了三层结构（Session/Window/Pane）、前缀键操作（Ctrl+B）、窗口和窗格管理，你就能在远程服务器上搭建一个"永不关机的云端桌面"，大幅提升工作效率。
