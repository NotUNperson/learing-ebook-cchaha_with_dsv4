# 01-08 综合练习

## 本节你会学到什么

- 运用本章全部知识完成一个完整的项目目录搭建
- 从零规划目录结构，设置合理的权限和所有权
- 实际体会在一个接近真实场景中如何使用所学命令
- 练习管道和重定向的组合使用
- 建立"文件操作即做事"的 Linux 思维习惯

---

这一节是整个第一章的大综合练习。你已经学了十多个命令，每个单独看好像都不太难。但真正工作的时候，你不是在孤立地使用某个命令，而是把一串命令像搭积木一样组合起来，完成一个完整的任务。这一节我们就来模拟一个真实场景。

## 场景设定

假设你刚加入一家叫"极客科技"的创业公司。公司准备用 Linux 服务器跑一个 Web 应用。你的任务是：**从零开始搭建这个项目的目录结构和权限体系。**

项目需求：
- 项目根目录叫 `/home/你的用户名/projects/geekapp`（实际练习中我们用家目录下的路径模拟）
- 项目需要 `src`（源代码）、`docs`（文档）、`logs`（日志）、`config`（配置）、`scripts`（脚本）、`backups`（备份）六个子目录
- 不同目录需要不同的权限：源代码目录所有人都能读但不能改；日志目录需要可写；配置目录只有特定组能访问；脚本需要可执行
- 项目有一个 README.md 和几个占位文件
- 需要一些软链接来方便日常操作（比如在项目根目录放一个指向 logs 最新日志的快捷方式）

## 第一步：规划目录树

在动工之前，先用纸笔（或者脑海）画出你要创建的目录结构：

```
geekapp/                        (755, 所有者:你, 所属组:geekdev)
├── README.md                   (644)
├── src/                        (755)
│   ├── main.py                 (644)
│   ├── utils.py                (644)
│   └── config.example.py       (644)
├── docs/                       (755)
│   ├── api.md                  (644)
│   └── guide.md                (644)
├── logs/                       (775, 所属组需要写权限)
│   └── app.log                 (664)
├── config/                     (750, 只有所有者和组能进)
│   ├── app.conf                (640, 组可读但不可写)
│   └── database.conf           (600, 只有所有者能读写)
├── scripts/                    (755)
│   ├── start.sh                (755, 要可执行！)
│   ├── stop.sh                 (755)
│   └── deploy.sh               (755)
├── backups/                    (750)
│   └── README.md               (644)
└── latest.log -> logs/app.log  (软链接，方便快速查看最新日志)
```

## 第二步：从零开始搭建

现在我们一步步来。你可以跟着下面的命令在你的 Linux 终端里敲。

### 2.1 创建用户组（模拟团队）

```bash
# 在实际练习中，这一步需要 sudo 权限
# 如果是在你的个人练习环境（WSL2/虚拟机），可以跳过或模拟
# 我们先看看有没有现成的组可以用
groups
# 如果没有，我们后续用自己名字的组来模拟
```

在实际工作环境（有 root 权限）中，管理员会这么做：
```bash
# sudo groupadd geekdev          # 创建开发组
# sudo usermod -aG geekdev zhangsan  # 把你加入开发组
```

但在个人练习中，我们直接用自己的默认组来模拟。

### 2.2 创建项目根目录

```bash
# 进入家目录
cd ~

# 创建项目根目录
mkdir -p ~/projects/geekapp
cd ~/projects/geekapp

# 确认位置
pwd
# 应该显示 /home/你的用户名/projects/geekapp
```

### 2.3 创建所有子目录

```bash
# 一次性创建所有需要的子目录
mkdir -p src docs logs config scripts backups

# 验证
ls -la
```

### 2.4 创建文件

```bash
# 快速创建多个空文件
touch README.md
touch src/main.py src/utils.py src/config.example.py
touch docs/api.md docs/guide.md
touch logs/app.log
touch config/app.conf config/database.conf
touch scripts/start.sh scripts/stop.sh scripts/deploy.sh
touch backups/README.md

# 给文件添加一些初始内容
echo "# GeekApp" > README.md
echo "print('Hello from GeekApp')" > src/main.py
echo "# API 文档" > docs/api.md
echo "# 部署指南" > docs/guide.md
echo "# 备份说明" > backups/README.md

# 给脚本添加 shebang（让它们成为可识别的脚本）
for script in scripts/*.sh; do
    echo "#!/bin/bash" > "$script"
    echo "# GeekApp 脚本" >> "$script"
done

# 验证目录结构
find . -type f -o -type d | sort
```

### 2.5 设置正确的权限

这是最关键的一步。不同文件需要不同权限：

```bash
# 目录：755 (rwxr-xr-x) -- 所有者完全控制，其他人可进入可查看
# 普通源代码/文档文件：644 (rw-r--r--) -- 所有者可改，其他人只读
# 日志文件：664 (rw-rw-r--) -- 所有者+组可写，其他人只读
# 配置文件：640 (rw-r-----) -- 组可读但不可写，其他人不可见
# 私密文件：600 (rw-------) -- 只有所有者能读写
# 可执行脚本：755 (rwxr-xr-x)

# 第一步：所有目录设为 755
find . -type d -exec chmod 755 {} \;

# 第二步：所有普通文件设为 644
find . -type f -exec chmod 644 {} \;

# 第三步：特殊目录和文件的精细调整

# 日志目录需要组写权限
chmod 775 logs
chmod 664 logs/app.log

# 配置目录更私密
chmod 750 config
chmod 640 config/app.conf
chmod 600 config/database.conf

# 备份目录
chmod 750 backups

# 脚本需要可执行
chmod 755 scripts/*.sh

# 验证所有权限
find . -exec ls -ld {} \;
```

### 2.6 设置所有权（如果有条件）

```bash
# 如果你有 sudo 权限且创建了 geekdev 组：
# sudo chown -R zhangsan:geekdev ~/projects/geekapp

# 如果没有，查看当前所有权：
ls -la ~/projects/geekapp
```

### 2.7 创建软链接

```bash
# 在项目根目录创建一个指向最新日志的软链接
ln -s logs/app.log latest.log

# 验证
ls -l latest.log
# lrwxrwxrwx 1 ... latest.log -> logs/app.log

# 通过软链接查看日志内容
cat latest.log
# (目前应该是空的，因为我们只创建了文件没写内容)
```

### 2.8 模拟日常工作流程

```bash
# 1. 修改源代码
echo "" >> src/main.py
echo "def greet(name):" >> src/main.py
echo "    return f'Hello, {name}!'" >> src/main.py

# 2. 查看修改后的代码行数
wc -l src/main.py

# 3. 把日志写点内容
echo "[INFO] Application started at $(date)" >> logs/app.log
echo "[INFO] User zhangsan logged in" >> logs/app.log

# 4. 通过软链接查看最新日志
cat latest.log

# 5. 给代码做个备份
cp src/main.py backups/main.py.bak.$(date +%Y%m%d)

# 6. 查看备份文件
ls -l backups/

# 7. 统计整个项目有多少个文件
find . -type f | wc -l

# 8. 统计所有代码文件的总行数
find . -name "*.py" -exec cat {} \; | wc -l
```

## 第四步：挑战任务

如果你完成了基础练习，试试下面的挑战任务（需要用到本章全部知识）：

**挑战 1：权限审计报告**

用你学会的命令生成一个"权限审计报告"：

```bash
# 列出所有权限过于宽松的文件（其他人可写的）
echo "=== 权限审计报告 ===" > ~/projects/geekapp/audit.txt
echo "扫描时间: $(date)" >> ~/projects/geekapp/audit.txt
echo "" >> ~/projects/geekapp/audit.txt

echo "--- 其他人可写的文件（安全风险）---" >> ~/projects/geekapp/audit.txt
find ~/projects/geekapp -type f -perm -o+w >> ~/projects/geekapp/audit.txt

echo "" >> ~/projects/geekapp/audit.txt
echo "--- 所有人都可执行的文件 ---" >> ~/projects/geekapp/audit.txt
find ~/projects/geekapp -type f -perm -a+x >> ~/projects/geekapp/audit.txt

echo "" >> ~/projects/geekapp/audit.txt
echo "--- 文件统计 ---" >> ~/projects/geekapp/audit.txt
echo "总文件数: $(find ~/projects/geekapp -type f | wc -l)" >> ~/projects/geekapp/audit.txt
echo "总目录数: $(find ~/projects/geekapp -type d | wc -l)" >> ~/projects/geekapp/audit.txt
echo "总代码行数: $(find ~/projects/geekapp -name '*.py' -exec cat {} \; | wc -l)" >> ~/projects/geekapp/audit.txt

cat ~/projects/geekapp/audit.txt
```

**挑战 2：数据库配置文件的硬链接保护**

```bash
# database.conf 是重要文件，创建一个硬链接作为"保险"
# 防止有人不小心 rm 了原文件
cd ~/projects/geekapp
ln config/database.conf config/database.conf.protected

# 验证它们共享同一个 inode
ls -li config/database.conf config/database.conf.protected

# 即使有人删除了 database.conf，数据还在
# （至少还有一个 database.conf.protected 指向这个 inode）
```

**挑战 3：软链接打造"快捷操作面板"**

```bash
# 在家里目录创建几个指向项目的快捷方式
ln -s ~/projects/geekapp ~/geekapp
ln -s ~/projects/geekapp/logs ~/geekapp-logs
ln -s ~/projects/geekapp/config ~/geekapp-config

# 现在你可以这样快速操作：
# cd ~/geekapp
# cd ~/geekapp-logs
# less ~/geekapp-config/app.conf
```

## 本节小结

本章学习了 Linux 文件系统的完整知识链：目录树结构（小区规划）、导航查看（pwd/cd/ls）、文件操作（touch/mkdir/cp/mv/rm）、查看内容（cat/less/head/tail/wc）、权限系统（chmod）、所有权（chown/chgrp）、链接机制（ln/ln -s）。综合练习用"搭建项目"的场景把这些命令串联起来，让你看到它们在实际工作中是如何配合的。
