# 01-01 Linux 目录树

## 本节你会学到什么

- 理解 Linux 单一根目录 `/` 的设计哲学
- 掌握 `/home`、`/etc`、`/var`、`/usr`、`/tmp` 等关键目录的用途
- 能够用"小区规划"的类比来记忆目录结构
- 知道哪些目录的改动需要管理员权限，哪些不需要
- 能说出 FHS（文件系统层次标准）是什么

---

在 Windows 里，打开"此电脑"，你看到的是这样的世界：

```
C:\
  程序文件\
  用户\
  Windows\
D:\
  游戏\
  电影\
E:\  (U盘)
  资料\
```

每个盘符（C、D、E）都是一个独立的"容器"。你可以把 C 盘想象成一栋房子的地下室，D 盘是车库，E 盘是临时放门口的快递柜。它们彼此独立，各管各的。

Linux 完全不是这样。Linux 只有一个根，叫 `/`（读音是"根"或"slash"）。所有东西--硬盘分区、U 盘、光驱、网络共享--全部挂在这一个根下面。没有 C 盘、D 盘之分，全世界都从 `/` 出发。

## 一棵倒着长的树

Linux 的目录结构是一棵**倒着长的树**。根（`/`）在最上面，所有目录和文件都是根的分支，一层一层向下延展：

```
/                          ← 根（整棵树的起点）
├── bin/                   ← 基本命令（ls, cp, mv 就住在这里）
├── boot/                  ← 启动系统需要的文件（内核镜像等）
├── dev/                   ← 设备文件（硬盘、键盘、鼠标都表现为文件）
├── etc/                   ← 配置文件（系统设置、软件设置都在这里）
├── home/                  ← 用户的家目录
│   ├── zhangsan/          ← 张三的家
│   └── lisi/              ← 李四的家
├── lib/                   ← 系统库文件（相当于 Windows 的 .dll）
├── media/                 ← 自动挂载的 U 盘、光盘
├── mnt/                   ← 手动挂载的临时存储
├── opt/                   ← 第三方商业软件（可选）
├── proc/                  ← 进程和系统信息（虚拟文件系统，在内存里）
├── root/                  ← root 用户的家目录
├── run/                   ← 运行时临时文件（重启后清空）
├── sbin/                  ← 系统管理命令
├── srv/                   ← 服务数据（网站、FTP 等）
├── sys/                   ← 内核和驱动信息（虚拟文件系统）
├── tmp/                   ← 临时文件（重启后可能清空）
├── usr/                   ← 用户程序和数据（你装的软件大部分在这里）
│   ├── bin/               ← 用户命令
│   ├── lib/               ← 用户库文件
│   ├── local/             ← 手动编译安装的软件
│   └── share/             ← 共享数据（文档、图标、字体）
└── var/                   ← 经常变化的数据（日志、缓存、数据库）
    ├── log/               ← 系统日志
    ├── cache/             ← 缓存
    └── spool/             ← 打印队列、邮件队列
```

这棵树里的每一个节点要么是目录（相当于 Windows 里的文件夹），要么是文件。实际上在 Linux 里，目录也是一种特殊的文件。

**类比**：Linux 目录树就像一棵真实的树。树根（`/`）扎在土里，树干（`/usr`、`/etc` 等一级目录）向上生长，树枝（子目录）不断分叉，树叶就是最终的文件。你在 Windows 里是坐着车从 C 盘 "车库里" 开车去 D 盘 "二楼卧室"，中间要"跨一个容器"（切换盘符）。在 Linux 里，你从树根出发，顺着树枝往下走，永远在一棵树上攀爬，不会"换一棵树"。

## 用"小区规划"来记忆关键目录

假设你是一个小区规划师，你要设计一个很大的居民小区。你如何划分区域？Linux 的目录设计思路和小区规划几乎一模一样：

### `/home` -- 居民楼

每一个住户（用户）都有自己的房间（家目录）。张三住在 `/home/zhangsan`，李四住在 `/home/lisi`。张三可以在自己房间里随意折腾--扔东西（创建文件）、搬家（移动文件）、搞装修（修改文件），但他没有钥匙进入李四的房间（除非李四给他开门，或者物业管理员 root 来强制开门）。

你登录系统后，默认就在自己的 `/home/你的用户名/` 目录下。提示符里的 `~` 就是这个目录的简写。`~` 读作"tilde"，但在 Linux 里它就是"我家"的意思。`cd ~` 就是"回家"，`cd ~/Documents` 就是"回我家然后去书房"。

```bash
# 以下两条命令是等价的
cd /home/zhangsan
cd ~
```

### `/etc` -- 物业管理中心

小区里的物业办公室。这里保存着整个小区的规章制度和配置记录册：什么时候收垃圾、访客怎么登记、游泳池几点开放。`/etc` 里全是纯文本的配置文件，你可以用任何文本编辑器打开查看和修改。

`/etc/passwd` 记录着小区有哪些住户，`/etc/fstab` 记录着哪些储物间（硬盘分区）该挂在小区的哪里，`/etc/ssh/sshd_config` 记录着远程门禁系统的设置。

**类比**：`/etc` 就像是物业管理处墙上挂的那个文件夹，里面夹着各种表格和规定。你想改门禁密码？去 `/etc` 里翻翻。想改垃圾回收时间？去 `/etc` 改个配置。在 Windows 里，这些配置分散在注册表、组策略、控制面板各处；在 Linux 里，全都在 `/etc` 下面，透明公开。

### `/var` -- 监控中心 + 仓库

小区的监控录像、快递收发明细、垃圾站的存货量--这些每天都在变的东西就放在 `/var`。它名字里的 "var" 是 "variable"（可变的）的缩写。

`/var/log/` 是监控录像：所有系统日志都在这里。哪天出了问题（比如有人半夜翻墙进小区），保安来这里查录像。

`/var/cache/` 是临时存放区：比如小区门口的快递柜。你网购的东西先放快递柜，等你来取，长期没人取的会被清理。

`/var/spool/` 是待处理任务队列：打印店等着打印的文件、邮局等着寄出的信件。

**类比**：Windows 里没有这么集中的"变化数据"目录。日志在事件查看器里，缓存藏在 `C:\Users\用户名\AppData\` 的各个角落。Linux 把一切"会变的东西"集中在 `/var`，方便监控和维护。

### `/usr` -- 商业街 + 公共设施

小区里的商铺、超市、健身房、图书馆。所有居民共享的软件和资源都在 `/usr`。

`/usr/bin/` 就是商业街上开的各种店铺（日常使用的命令）。你逛街时候需要的东西这里都有。

`/usr/share/` 就是小区的公告栏和公共图书馆（文档、字体、图标等共享数据）。

`/usr/local/` 比较特殊。它是留给小区居民自己开的"私房菜馆"--你手动编译安装的软件放这里，不影响小区统一的商业街管理。

**注意**：`/usr` 不是 "user"（用户）的缩写！它最初的设计意图是 "Unix System Resources"（Unix 系统资源）。这是一个历史遗留的命名问题，你只要记住它放的是"共享的软件和资源"就好。

### `/tmp` -- 小区门口的临时垃圾堆放点

所有人都可以往这里扔东西（所有用户都能读写），但清洁工（系统重启）会把这里清空。不适合存放重要物品。

**重要警告**：不要把重要文件放在 `/tmp`！有些系统配置成每次重启就清空 `/tmp`，有系统会定期清理。`/tmp` 只适合放"就算丢了也无所谓"的临时文件。

### `/root` -- 物业经理的私人办公室

root 用户（超级管理员）的家目录。普通居民不要随便进去翻东西，弄坏了全小区的秩序受影响。

注意区别：`/` 是"根目录"（整个小区的大门），`/root` 是 root 用户的家（物业经理的办公室）。它们是完全不同的两个地方。

新手常犯的错误：把 `/` 和 `/root` 搞混。`/` 是最高层级，`/root` 只是 `/` 下面的一个子目录。

### 其他重要区域

| 目录 | 用途 | 小区类比 |
|------|------|---------|
| `/boot` | 系统启动文件 | 小区发电站（系统启动必不可少的核心设备） |
| `/dev` | 设备文件 | 小区房产证档案柜（每个硬件对应一个文件） |
| `/proc` | 进程和系统信息（虚拟） | 小区实时监控仪表盘（不是实体，是运行时的状态显示） |
| `/mnt` 和 `/media` | 挂载点 | 外来车辆的临时停车场（U 盘、光盘、网络存储） |
| `/srv` | 服务数据 | 物业自营的食堂和洗衣房（网络服务提供的数据） |
| `/opt` | 第三方商业软件 | 小区门口的连锁加盟店（非社区标配的商业软件） |

## 你可以在哪里"随便玩"

作为普通用户，以下目录你是可以自由创建、修改、删除文件的：

- `/home/你的用户名/`（你自己的家，随便折腾）
- `/tmp/`（临时垃圾堆，大家都能用，但不保证永久保存）

以下目录你**只能看，不能改**（需要 root 权限才能改）：

- `/etc/`（物业管理中心，普通居民只能看规章制度，不能改）
- `/usr/`（公共设施，只有物业经理能改建）
- `/boot/`（发电站，谁敢乱碰？）
- `/bin/` 和 `/sbin/`（核心商业街，店铺不能随便拆）

这个权限设计是 Linux 安全模型的基石。即使你胡乱操作，最多毁掉自己家（`/home/你的用户名/`），不会影响整个小区。在 Windows 里，很多恶意软件利用"你是管理员"的身份来干坏事；在 Linux 里，你平时就是普通居民，想干坏事得先让 root 给你开门。

**类比**：Windows 默认给用户的权限就像给了每个住户一把万能钥匙，能开任何房间包括配电房的门（虽然现在 UAC 在改善这种情况）。Linux 默认只给你自己房间的钥匙，你要进配电房？找管理员借钥匙，而且是"用一次还一次"（sudo）。

## FHS -- 小区的"规划法"

这么多目录的用途不是某人拍脑袋决定的，而是遵循一个叫 **FHS**（Filesystem Hierarchy Standard，文件系统层次标准）的规范。这个规范由 Linux 基金会维护，绝大多数 Linux 发行版都遵循它。它确保了：

- 你在 Ubuntu 上学到的目录结构，在 Rocky Linux 上一样适用
- 安装软件时，软件知道把文件放到哪里（二进制放 `/usr/bin`，配置文件放 `/etc`，日志放 `/var/log`）
- 管理员可以快速定位问题（日志肯定在 `/var/log`，网卡配置肯定在 `/etc` 下面）

了解这套规范，你就像是一个了解"小区规划法"的建筑师。不管去到哪个 Linux 系统，你都能快速摸清它的结构。

## 动手试试

在你的 Linux 终端中，执行以下操作：

1. 输入 `cd /` 跳转到根目录，然后输入 `ls` 看看根目录下有哪些子目录。你能认出 `/home`、`/etc`、`/var`、`/tmp` 吗？
2. 输入 `cd ~` 回到你的家目录
3. 输入 `ls /etc` 看看物业管理中心有多少配置文件（你会发现有很多）
4. 输入 `ls /var/log` 看看监控录像目录
5. 输入 `cat /etc/os-release` 看看这个配置文件里写了什么（显示当前系统版本信息）
6. 输入 `df -h` 查看磁盘使用情况，观察所有分区是如何"挂"在目录树上的

## 本节小结

Linux 只有一棵从 `/` 开始的目录树，所有硬件和分区都挂载在树下。关键目录记住：`/home` 住人、`/etc` 管配置、`/var` 记日志、`/usr` 放软件、`/tmp` 存临时文件。
