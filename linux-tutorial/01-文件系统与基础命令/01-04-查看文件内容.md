# 01-04 查看文件内容

## 本节你会学到什么

- 用 `cat` 快速查看小文件的全部内容
- 用 `less` 翻页浏览大文件，掌握翻页操作
- 用 `head` 和 `tail` 只看文件的开头和结尾
- 用 `nl` 给文件加行号阅读
- 用 `wc` 统计文件的行数、单词数和字符数

---

文件建好了，但你站在门口，还不知道里面写了什么。在 Windows 里，你双击一个 .txt 文件，记事本自动打开，内容一览无余。在 Linux 的命令行里，你有五种不同的方式查看文件内容，各有各的适用场景。

## cat -- 一口气从头读到尾

`cat` 是 "concatenate"（连接）的缩写，它的原始设计目的是把多个文件的内容拼接起来输出。但因为它的默认行为就是把文件内容输出到屏幕上，所以被大家当作最常用的文件查看工具。

```bash
# 查看一个文件
cat /etc/hostname

# 查看多个文件（会按顺序依次输出，等同于把它们"拼接"起来）
cat file1.txt file2.txt file3.txt

# 在输出中显示行号
cat -n file.txt

# 把多个文件内容合并成一个新文件
cat part1.txt part2.txt > full.txt
```

**类比**：`cat` 就像你把一部电影从头到尾一次性播完，中间不能暂停、倒带、快进。对于短视频（小文件）这完全没问题，但如果是一部三小时的电影（几万行的大日志文件），画面会飞速闪过，你什么也看不清。

```bash
# 对于 /var/log/syslog 这种可能有几十万行的日志文件
cat /var/log/syslog
# 屏幕会疯狂滚动，你几乎什么都看不到
```

所以 `cat` 最适合查看**内容很短**的文件（几十行以内），比如配置文件。对于大文件，你需要换一个工具。

**Windows 对比**：`cat` 相当于你在 CMD 里用的 `type` 命令。`type file.txt` 在 Windows 上做的事情和 `cat file.txt` 在 Linux 上完全一样。

## less -- 翻页浏览，进退自如

`less` 是 Linux 世界里最重要的文件查看工具。它的名字是一个幽默的 self-referencing 梗：早先有个类似工具叫 `more`（只能向下翻页），`less` 的意思是"比 more 更多"--因为 less 既可以向下翻也可以向上翻（less is more）。

```bash
# 用 less 打开一个文件
less /var/log/syslog

# 用 less 查看一个命令的输出（通过管道）
ls -la /etc | less
```

进入 `less` 之后，你的键盘就变成了一套"视频播放遥控器"：

| 按键 | 功能 | 类比 |
|------|------|------|
| `空格键` | 向下翻一页 | 翻到下一页 |
| `b` | 向上翻一页（back） | 翻回上一页 |
| `j` 或 `下箭头` | 向下滚动一行 | 慢放 |
| `k` 或 `上箭头` | 向上滚动一行 | 倒放 |
| `g` | 跳到文件开头（go） | 跳到片头 |
| `G`（大写） | 跳到文件末尾 | 跳到片尾 |
| `/关键词` | 向下搜索关键词 | Ctrl+F |
| `?关键词` | 向上搜索关键词 | 反向查找 |
| `n` | 跳到下一个搜索结果 | 查找下一个 |
| `N`（大写） | 跳到上一个搜索结果 | 查找上一个 |
| `q` | 退出 less | 关掉播放器 |

**类比**：`less` 就像你用一个带全部操控功能的视频播放器看一部电影。你可以快进（空格键）、倒带（b）、逐帧播放（j/k）、搜索特定场景（/关键词）、直接跳到开头或结尾（g/G）。最重要的是，这个播放器不会一次性把整部电影加载到内存--它只加载你当前在看的那一页，所以即使电影有 100GB（超大文件），播放器也不会卡。

这就是 `less` 比直接在编辑器里打开大文件强得多的地方。当你用 VSCode 打开一个 500MB 的日志文件时，它会尝试把整个文件读进内存，你的电脑可能直接卡死。`less` 只会加载你当前屏幕能显示的那几十行，内存占用几乎为零。

```bash
# less 的日常使用场景
# 场景1：查看大型日志文件
less /var/log/syslog

# 场景2：查看上一个命令的长输出
dmesg | less

# 场景3：查看并搜索
less /etc/nginx/nginx.conf
# 在 less 中按 /server 搜索 "server"
```

退出 `less` 的方法是按 `q`（quit）。

## head 和 tail -- 只看头尾

很多情况下你不需要看一个文件的全部内容，只需要看头几行或尾几行。

### head -- 看开头

```bash
# 看文件的前 10 行（默认值）
head file.txt

# 看文件的前 20 行
head -n 20 file.txt

# 看文件的前 3 行
head -3 file.txt

# 查看 /etc/passwd 的前几行（看看系统有哪些用户）
head -5 /etc/passwd
```

**类比**：`head` 就像你拿起一本书，只看目录和前几页，快速判断这本书讲什么。不需要把整本书看完就知道大意。

### tail -- 看结尾

```bash
# 看文件的最后 10 行（默认值）
tail file.txt

# 看文件的最后 20 行
tail -n 20 /var/log/syslog

# 最实用的功能：实时追踪文件更新（-f）
tail -f /var/log/syslog
```

`tail -f` 是 Linux 管理员最常用的命令之一。它不会在显示最后几行后就退出，而会持续"监听"文件，每当有新内容被追加到文件末尾，就立刻显示在屏幕上。

**类比**：`tail -f` 就像你站在一台打印机的出纸口旁边，每打印完一张纸（日志文件新增一行），你立刻就能看到。这在排查线上问题时非常有用：你在一个终端窗口运行 `tail -f /var/log/nginx/error.log`，在另一个窗口访问你的网站，如果网站报错了，`tail -f` 的窗口会立刻显示错误信息，不用你手动去刷新。

```bash
# 实战场景：同时追踪多个日志文件
tail -f /var/log/syslog /var/log/auth.log
# 每个文件的更新都会实时显示，并标注来自哪个文件

# 显示文件的第 50 行到第 100 行（head + tail 组合技）
head -100 file.txt | tail -51
# head 先取前 100 行，tail 再从这 100 行里取最后 51 行（= 第 50 到第 100 行）
```

**head 和 tail 的组合技**：

```bash
# 假设一个文件有 1000 行，你想看中间的第 450-460 行
head -460 huge_file.txt | tail -11

# 或者你只想要第 100 行
head -100 huge_file.txt | tail -1
```

这里的竖线 `|` 叫做**管道**（pipe），它是 Linux 最强大的特性之一。管道的本质是把左边命令的输出"传送"给右边命令作为输入。我们后面会有专门的一节讲管道和重定向，你现在只需要记住这个组合的用法。

## nl -- 给文件加行号

`nl` 是 "number lines" 的缩写。有些文件你想看内容，还想看行号。`cat -n` 也能做到，但 `nl` 更灵活：

```bash
# 给文件加行号
nl file.txt

# 只给非空行加行号
nl -b t file.txt
# -b t 表示 "body numbering: t (non-empty lines only)"

# 给所有行加行号（包括空行）
nl -b a file.txt
```

```bash
# 举例
cat > demo.txt << 'EOF'
第一行
第二行

第四行（第三行是空行）
EOF

nl -b t demo.txt
# 输出:
#      1  第一行
#      2  第二行
#
#      3  第四行（第三行是空行）
# 注意空行没有编号

nl -b a demo.txt
# 输出:
#      1  第一行
#      2  第二行
#      3
#      4  第四行（第三行是空行）
# 空行也有了编号
```

**类比**：`nl` 就像是给文件的每一行贴上编号标签。在编程和调试时，报错信息经常说"第 XX 行有错误"，你打开文件后用 `nl` 或 `cat -n` 就能快速定位到对应的行。

## wc -- 统计文件的"体格"

`wc` 是 "word count" 的缩写，但它不止能统计单词数。

```bash
# 统计文件的行数、单词数、字节数
wc file.txt

# 只统计行数
wc -l file.txt

# 只统计单词数
wc -w file.txt

# 只统计字符数
wc -m file.txt

# 只统计字节数
wc -c file.txt
```

```bash
# 实例
wc /etc/passwd
# 输出: 45  78  2380 /etc/passwd
# 解释: 45 行，78 个单词，2380 字节

# 看看你的系统有多少用户
wc -l /etc/passwd
# /etc/passwd 每行代表一个用户，行数就是用户数
```

**类比**：`wc` 就像你去体检时做的身体测量--身高、体重、体脂率。`wc -l` 量"行数"（文件的身高），`wc -c` 量"字节数"（文件的体重），`wc -w` 量"单词数"（某种维度上的宽度）。

### wc 的实用场景

```bash
# 场景1：看看某个目录下有多少文件
ls -1 /etc | wc -l

# 场景2：看看一段代码有多少行
wc -l *.py

# 场景3：统计多个文件的总行数
cat *.txt | wc -l

# 场景4：看看 Apache 日志有多少条请求记录
wc -l /var/log/apache2/access.log
```

## 动手试试

在你的 Linux 终端中完成以下操作：

1. 查看你的主机名：`cat /etc/hostname`
2. 查看你的系统用户列表（只看头 5 个）：`head -5 /etc/passwd`
3. 用 `less` 打开一个大文件来练习翻页和搜索：`less /etc/services`（一个记录了各种网络服务端口的文件）。在 less 中试试：
   - 按空格翻页
   - 按 `/http` 搜索 "http"
   - 按 `n` 跳到下一个匹配项
   - 按 `g` 跳到开头，按 `G` 跳到结尾
   - 按 `q` 退出
4. 用 `tail -f` 实时观察系统日志（如果权限不够会报错，没关系）：`tail -f /var/log/syslog`（按 Ctrl+C 退出）
5. 统计 `/etc/passwd` 的行数，看看系统有多少个用户账户：`wc -l /etc/passwd`
6. 用 `cat -n /etc/hosts` 带行号查看 hosts 文件

## 本节小结

小文件用 `cat`，大文件用 `less`（按 q 退出），只看头尾用 `head`/`tail`（`tail -f` 实时追踪），统计行数用 `wc -l`。
