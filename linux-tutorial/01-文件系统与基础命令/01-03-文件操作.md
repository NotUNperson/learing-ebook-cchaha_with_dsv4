# 01-03 文件操作

## 本节你会学到什么

- 用 `touch` 创建空文件
- 用 `mkdir` 创建目录（包括递归创建多级目录）
- 用 `cp` 复制文件和目录
- 用 `mv` 移动和重命名文件
- 用 `rm` 删除文件 -- 以及为什么这是 Linux 里最"危险"的命令

---

在 Windows 里，你每天都在对文件做四件事：新建（右键 -> 新建 -> 文本文档）、复制（Ctrl+C -> Ctrl+V）、移动（拖拽到另一个文件夹）、删除（右键 -> 删除，或者 Shift+Delete 永久删除）。Linux 的命令行当然也能做这些事，而且更快、更灵活。但有一个命令的使用方式和 Windows 完全不同，没搞清楚的话可能会酿成大错--那就是删除命令。

## touch -- 创建文件和更新时间戳

`touch` 命令最初的设计目的是更新文件的时间戳（把文件的"最后修改时间"和"最后访问时间"设为当前时间），就像你"摸"了一下文件。但实际上它被用得最多的功能是：**创建空文件**。

```bash
# 创建一个空文件
touch readme.md

# 同时创建多个文件
touch file1.txt file2.txt file3.txt

# 使用花括号展开（Bash 特有），批量创建带编号的文件
touch report_{1..5}.txt
# 等价于 touch report_1.txt report_2.txt ... report_5.txt
```

当目标文件不存在时，`touch` 会创建一个大小为 0 字节的空文件。当目标文件已经存在时，`touch` 会把它的时间戳更新为当前时间（文件内容不变）。

**类比**：`touch` 就像你走过去摸了一下墙上的签到表。如果这张表（文件）之前就在墙上，你只是更新了"最近来访时间"；如果墙上没有这张表，你就在墙上贴了一张空白的签到表（新建空文件）。

```bash
# 验证：查看文件的详细信息，注意时间戳
touch hello.txt
ls -l hello.txt
# 输出: -rw-r--r-- 1 zhangsan zhangsan 0 May 15 14:30 hello.txt

# 等几秒钟再用 touch 碰一下同一个文件
sleep 5
touch hello.txt
ls -l hello.txt
# 注意时间戳变了！从 14:30 变成了 14:30:05（或其他当前时间）
```

## mkdir -- 创建目录

`mkdir` 是 "make directory" 的缩写，用来创建文件夹。在 Windows 里你右键 -> 新建 -> 文件夹，在 Linux 里就是 `mkdir`。

```bash
# 创建一个目录
mkdir my_project

# 一次创建多个目录
mkdir docs images scripts

# 创建多级目录（如果父目录不存在，自动创建）
mkdir -p project/src/components
# 这会自动创建 project/、project/src/、project/src/components/ 三层目录
# 不加 -p 的话，如果 project/ 不存在，会报错
```

`-p` 选项（parents，父目录）非常重要。在编写脚本时，如果你不确定父目录是否存在，用 `mkdir -p` 是最安全的做法。

```bash
# 不加 -p：父目录不存在时报错
mkdir a/b/c
# mkdir: cannot create directory 'a/b/c': No such file or directory

# 加了 -p：自动创建所有需要的父目录，如果目录已存在也不报错
mkdir -p a/b/c
# 成功，无报错
```

**类比**：`mkdir` 就像在小区里盖房子。`mkdir my_project` 是直接在地面上盖一层平房。`mkdir -p project/src/components` 是盖一栋三层小楼，地基（project）和一楼（src）和二楼（components）一次性全部盖好。如果不加 `-p`，你需要先打地基、再盖一楼、再盖二楼，分三步走。

## cp -- 复制文件

`cp` 是 "copy" 的缩写，就是复制粘贴。基本语法是 `cp 源文件 目标位置`。

```bash
# 复制一个文件
cp source.txt target.txt

# 复制到另一个目录，保留原名
cp source.txt /tmp/

# 复制并重命名
cp source.txt /tmp/backup.txt

# 复制整个目录（必须加 -r，递归复制）
cp -r my_project/ my_project_backup/

# 复制时保留文件属性（权限、时间戳等）
cp -a my_project/ my_project_backup/
```

几个重要选项：

| 选项 | 含义 | 什么时候用 |
|------|------|-----------|
| `-r` | 递归复制（复制目录必须加） | 复制整个文件夹时 |
| `-i` | 覆盖前询问确认 | 想避免误覆盖时 |
| `-v` | 显示复制过程 | 想知道复制了哪些文件时 |
| `-a` | 归档模式（保留所有属性） | 做完整备份时 |
| `-u` | 只复制新文件（目标没有或源更新才复制） | 增量备份时 |

**常见坑：cp 默认会覆盖同名文件而不提示。** 如果你执行 `cp file.txt /backup/`，而 `/backup/file.txt` 已经存在，它会静默覆盖，之前的文件就没了。如果想避免这种情况，可以使用 `cp -i`（交互模式，覆盖前询问）或者设置别名 `alias cp='cp -i'`。

**类比**：`cp` 就像复印机。你放一份原件进去，它吐出一份复印件。原件没有任何变化。`cp -r` 是把一整本书（目录）逐页复印，装订成一本新书。

## mv -- 移动和重命名

`mv` 是 "move" 的缩写。在 Linux 里，移动文件和重命名文件用的是同一个命令，因为从文件系统的角度看，它们本质上都是"改变文件在目录树中的路径"。

```bash
# 重命名文件（在同一目录下"移动"）
mv old_name.txt new_name.txt

# 移动文件到另一个目录
mv file.txt /tmp/

# 移动并重命名
mv file.txt /tmp/renamed.txt

# 移动整个目录（不需要额外选项，和移动文件一样）
mv my_project/ /tmp/

# 移动多个文件到一个目录
mv file1.txt file2.txt file3.txt target_dir/
```

**为什么移动和重命名是同一个命令？** 因为在 Linux 的文件系统中，文件名只是文件在目录树上的一个"标签"。把文件从 `/home/zhangsan/file.txt` 改名为 `/home/zhangsan/report.txt`，和把它移动到 `/tmp/file.txt`，本质上都是在修改这个"标签"的内容。Windows 也差不多（在同一个分区内移动文件很快，跨分区移动才需要复制数据），只是 Windows 把它们分成了两个不同的操作。

**类比**：文件在硬盘上的实际数据就像你家房子的物理位置（不会变）。文件名和路径就像你的快递地址。`mv` 做的事就是修改你的快递地址标签，但房子本身没动。重命名相当于"把门牌号 101 改成 102"，移动文件相当于"把这栋房子的地址从 A 街道改到 B 街道"。当然，如果跨文件系统（跨硬盘分区）移动，那就相当于真的搬家了（数据要复制到新位置，然后删除旧的）。

## rm -- 删除文件，Linux 里最需要小心的命令

终于聊到 `rm`（remove，删除）了。这是我必须花最多篇幅讲的命令，因为它是 Linux 新手最容易犯下不可挽回错误的命令。

### 基本用法

```bash
# 删除一个文件
rm file.txt

# 删除多个文件
rm file1.txt file2.txt file3.txt

# 删除前询问确认（推荐新手使用）
rm -i file.txt
# 会提示: rm: remove regular empty file 'file.txt'? (y/n)

# 删除整个目录及其所有内容
rm -r my_directory/

# 强制删除，不询问（危险！）
rm -f file.txt

# 强制递归删除整个目录（极其危险！）
rm -rf my_directory/
```

### Windows 和 Linux 删除的根本区别

这是从 Windows 转过来的新手最需要理解的一点。

在 Windows 里，你删除一个文件：
1. 右键 -> 删除 -> 文件进了回收站
2. 后悔了？去回收站还原，文件就回来了
3. 即使 Shift+Delete 永久删除，也可以用第三方恢复软件（如 Recuva）找回来

在 Linux 里，`rm` 删除的文件：
1. **不进回收站**，直接标记为可覆盖
2. **没有确认提示**（除非你加了 `-i` 选项）
3. **几乎没有恢复的可能**（除非你用了特殊的文件系统快照或备份）

**类比**：Windows 的删除像你把废纸扔进垃圾桶--垃圾还在垃圾桶里，你可以翻垃圾桶找回来。Linux 的 `rm` 像你把废纸扔进碎纸机--瞬间变成纸屑，再也拼不回来了。

### 为什么 Linux 要这样设计

这不是设计缺陷，而是设计哲学。Linux 的设计者认为：

1. 命令行是为"知道自己要干什么"的人设计的。如果你输入了 `rm` 命令，你就应该知道后果
2. 服务器上没有"回收站"这种奢侈的东西。一台服务器可能同时在跑几百个服务，产生大量临时文件，如果每个文件删除前都要确认，管理员会被烦死
3. 简洁即安全。每加一个确认步骤就可能导致"确认疲劳"（用户机械性地按 y，反而失去了确认的意义）

### 几个"恐怖故事"级别的 rm 误操作

这些不是传说，是真实发生过的生产事故：

```bash
# 事故1：本想删除当前目录下的所有 .tmp 文件，多打了一个空格
rm -rf / tmp/     # 注意 / 和 tmp/ 之间有个空格！
# 这等价于：删除根目录 /（整个系统），然后删除 tmp/
# 系统瞬间被摧毁

# 事故2：变量为空时的灾难
rm -rf $UNSET_VARIABLE/   # 如果变量 UNSET_VARIABLE 是空的
# 这等价于 rm -rf /
# 整个系统被清空

# 事故3：本想在 /tmp 下操作，不小心切到了根目录
cd /tmp
# ... 一顿操作 ...
cd /          # 不小心切回来了
rm -rf *      # 从根目录删除一切！
```

这些事故看起来像是"那么蠢的操作谁干得出来"，但疲劳的凌晨三点、着急上线的时候、心里想着别的事手上还在敲命令的时候，这些事故真的会发生。

### 安全使用 rm 的五个习惯

**习惯一：永远先 `ls` 后 `rm`。**

在删除之前，用 `ls` 看看你要删除什么：

```bash
# 不要直接 rm /tmp/project/*
# 先 ls 看看会删除哪些文件
ls /tmp/project/*
# 确认无误后，再按上箭头把 ls 改成 rm
rm /tmp/project/*
```

**习惯二：使用 `rm -i` 或设置别名。**

```bash
# 临时使用交互模式
rm -i *.tmp

# 永久设置：在 ~/.bashrc 中添加
alias rm='rm -i'
# 这样每次 rm 都会询问确认（但如果已经养成机械按 y 的习惯也没用）
```

**习惯三：使用 `trash-cli` 工具。**

```bash
# 安装 trash-cli（Ubuntu/Debian）
sudo apt install trash-cli

# 用 trash 替代 rm，文件会进入回收站
trash file.txt

# 查看回收站
trash-list

# 还原文件
trash-restore
```

这相当于给 Linux 装了 Windows 风格的回收站。

**习惯四：用 `find` 配合 `-exec` 代替通配符。**

```bash
# 删除所有 .log 文件
find . -name "*.log" -exec rm {} \;

# 带确认的版本
find . -name "*.log" -ok rm {} \;
# -ok 和 -exec 一样，但每个操作前会询问
```

**习惯五：备份！备份！备份！**

重要文件操作前先备份。可以是简单的 `cp`，也可以使用 `git` 做版本控制。

```bash
# 操作重要文件前的保命操作
cp important_config.conf important_config.conf.bak
# 再修改 important_config.conf
# 改了发现不对？用 .bak 还原
```

## 动手试试

在你的 Linux 终端中完成以下操作：

1. 在家目录下创建一个练习目录：`mkdir -p ~/linux-practice/file-ops`
2. 进入练习目录：`cd ~/linux-practice/file-ops`
3. 用 `touch` 创建三个文件：`touch a.txt b.txt c.txt`
4. 用 `ls -l` 查看刚才创建的文件（注意文件大小是 0）
5. 用 `mkdir -p level1/level2/level3` 创建三层嵌套目录
6. 用 `cp a.txt b_copy.txt` 复制一个文件
7. 用 `mv c.txt level1/` 把 c.txt 移动到 level1 目录
8. 用 `mv b_copy.txt b_new_name.txt` 重命名一个文件
9. 用 `ls -R` 递归查看整个目录结构，确认所有操作成功
10. 用 `rm -i *.txt` 交互式删除所有 txt 文件（你会被逐条询问）
11. 最后用 `rm -rf ~/linux-practice/file-ops` 清理练习目录

## 本节小结

`touch` 创建文件，`mkdir -p` 创建多级目录，`cp` 复制，`mv` 移动和重命名，`rm` 删除。记住：`rm` 在 Linux 里没有回收站，删了就没了，操作前先用 `ls` 确认你想删什么。
