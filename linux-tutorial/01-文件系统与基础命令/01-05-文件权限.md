# 01-05 文件权限

## 本节你会学到什么

- 理解 Linux 权限模型：r（读）、w（写）、x（执行）
- 知道为什么权限要分三组：所有者、所属组、其他人
- 掌握 chmod 数字法（例如 755、644）的含义和计算方式
- 掌握 chmod 符号法（例如 u+x、g-w、o=r）的灵活用法
- 理解"可执行"权限对文件和目录的不同含义

---

在 Windows 里，文件权限对大多数普通用户来说基本上是透明的。你双击一个文件，能打开就是能打开，不能打开就弹个警告框。你不太会去想"这个文件我有读取权限吗？有写入权限吗？"

在 Linux 里，权限是写在每个文件的"脸上"的。你每执行一次 `ls -l`，都在看每个文件的权限信息。而且，权限不仅仅关乎安全，还直接影响你日常的每一个操作：为什么我改不了这个配置文件？为什么我运行这个脚本提示"Permission denied"？为什么我进不去那个目录？

Linux 的权限模型非常简洁：只有三种操作（读、写、执行），分给三类人（所有者、同组、其他人）。但这个简单的模型却构成了整个 Linux 安全体系的基础。

## 一个生活中的类比：你的公寓

想象你住在公寓楼里的一间房。你对这间房有三个层次的"权限"：

- **读（r）**：你可以看到房间里的东西。能看到桌上有什么文件，柜子里有什么物品。
- **写（w）**：你可以改变房间里的东西。可以移动家具，可以在墙上贴海报，可以在桌上写东西。
- **执行（x）**：你可以进入这间房。如果你连房门都打不开，读和写都无从谈起。

现在，你的公寓有三把钥匙，分给三类人：

1. **你自己（所有者 / Owner）**：你当然有全部三把钥匙。你可以进房间、看东西、改东西。
2. **你的家人（所属组 / Group）**：你老婆/老公/孩子住在同一个公寓的其他房间。你给家人配了部分钥匙--他们可以进你的房间（x），看看你在干什么（r），但不能乱翻你的抽屉（没有 w）。
3. **外人（其他人 / Others）**：邻居、外卖小哥、来收物业费的。你最多让他们在门口看一眼（r），或者根本不让他们靠近你的房门（没有任何权限）。

**类比**：Linux 的文件权限就是这套"房门钥匙"系统。每个文件都有三组权限（所有者、所属组、其他人），每组有三种权限（读、写、执行）。`ls -l` 显示的那一串 `-rwxr-xr--` 就是这九把钥匙的分布图。

## 看懂 ls -l 的权限字符串

当你执行 `ls -l` 时，每一行开头的 10 个字符就是权限信息：

```bash
ls -l
# -rwxr-xr--  1 zhangsan  developers  4096 May 15 10:30 script.sh
```

让我们把这 10 个字符拆开来看：

```
 -   rwx   r-x   r--
(1)  (2)   (3)   (4)
```

| 位置 | 字符 | 含义 |
|------|------|------|
| (1) 第 1 位 | `-` | 文件类型。`-` = 普通文件，`d` = 目录，`l` = 符号链接 |
| (2) 第 2-4 位 | `rwx` | **所有者（Owner）**的权限：读、写、执行 |
| (3) 第 5-7 位 | `r-x` | **所属组（Group）**的权限：读、执行（没有写权限） |
| (4) 第 8-10 位 | `r--` | **其他人（Others）**的权限：只读 |

所以 `-rwxr-xr--` 翻译成人话就是：

- 这是一个**普通文件**（不是目录不是链接）
- **所有者**可以读、写、执行（rwx，完全控制）
- **所属组**成员可以读和执行，但不能修改（r-x）
- **其他人**只能看，不能改也不能执行（r--）

再看几个常见例子：

```
-rw-r--r--  普通文件，所有者可读写，其他人只能读（最常见的文件权限）
-rwx------  普通文件，只有所有者能读写执行，其他人没有任何权限（私密脚本）
drwxr-xr-x  目录，所有者完全控制，其他人可以进入和查看（最常见的目录权限）
-r--------  普通文件，只有所有者能读，连写都不行（只读文件）
```

## 三个权限的精确含义

### r（Read，读取，数字代号 4）

**对文件**：可以查看文件的内容。能用 `cat`、`less` 打开阅读。
**对目录**：可以列出目录中有哪些文件（能用 `ls`）。但如果只有 r 没有 x，你只能看到文件名，看不到文件的详细信息（`ls -l` 会显示问号）。

**类比**：对文件有 r 权限就像你能透过玻璃窗看到房间里有什么。对目录有 r 权限就像你能看到走廊里的房间号清单--你知道 301 房间存在，但你没钥匙（没有 x）的话进不去。

### w（Write，写入，数字代号 2）

**对文件**：可以修改文件的内容。能用编辑器修改、能用 `>` 覆盖。
**对目录**：可以在目录中创建、删除、重命名文件。注意：删除一个文件需要的是文件**所在目录**的 w 权限，而不是文件本身的 w 权限。

```bash
# 有趣的事实：你可以删除一个只读文件！
touch test.txt
chmod 444 test.txt   # 设为只读（r--r--r--）
rm test.txt
# rm 会问: rm: remove write-protected regular empty file 'test.txt'?
# 按 y 就能删除！因为删除文件需要的是目录的写权限，不是文件的写权限
```

**类比**：文件的 w 权限就像白板笔--你可以在白板上写字或擦掉重写。目录的 w 权限就像是房间的钥匙--你能进房间后往里面放新家具（创建文件），也能把家具扔掉（删除文件）。

### x（Execute，执行，数字代号 1）

**对文件**：可以把这个文件当作程序来运行。对于编译好的二进制程序（如 `/bin/ls`）和脚本文件（如 `script.sh`），必须有 x 权限才能执行。

**对目录**：可以"穿越"这个目录（cd 进去），可以访问目录里的文件。目录的 x 权限也叫"搜索权限"。

```bash
# 没有 x 权限的目录：你进不去
mkdir test_dir
chmod 666 test_dir    # rw-rw-rw-，有读写但没有执行
cd test_dir
# bash: cd: test_dir: Permission denied
# 即使有读写权限，没有 x 也进不去目录

chmod 766 test_dir    # rwxrw-rw-，所有者有执行权限
cd test_dir           # 成功进入
```

**类比**：目录的 x 权限就是走廊门的钥匙。r 让你知道走廊里有哪些房间，x 让你能走进走廊。没有 x，你只能在走廊口张望（最多 `ls` 一下），但没法走进去，更没法进入走廊里的任何房间。

## chmod -- 修改权限

`chmod` 是 "change mode"（更改模式）的缩写。有两种方法使用它：数字法和符号法。

### 数字法（八进制表示法）

这是最常用的方法。每种权限对应一个数字：

| 权限 | 字母 | 数字值 | 记忆方法 |
|------|------|--------|---------|
| 读 | r | 4 | 4 个字母的 "read"…好吧其实用二进制记更好 |
| 写 | w | 2 | |
| 执行 | x | 1 | |
| 无权限 | - | 0 | |

这三个数字是**八进制位的权重**。要计算一组权限的数字，把它们加起来：

```
r + w + x = 4 + 2 + 1 = 7    （读写执行）
r + w     = 4 + 2     = 6    （读写）
r + x     = 4 + 1     = 5    （读执行）
r         = 4               （只读）
w         = 2               （只写，很少见）
x         = 1               （只执行，也很少见）
-         = 0               （无权限）
```

三个数字分别代表**所有者**、**所属组**、**其他人**的权限：

```bash
chmod 755 script.sh
# 7 (rwx) = 所有者：读写执行
# 5 (r-x) = 所属组：读 + 执行
# 5 (r-x) = 其他人：读 + 执行

chmod 644 document.txt
# 6 (rw-) = 所有者：读写
# 4 (r--) = 所属组：只读
# 4 (r--) = 其他人：只读

chmod 600 secret.key
# 6 (rw-) = 所有者：读写
# 0 (---) = 所属组：无权限
# 0 (---) = 其他人：无权限
```

**最常见的权限组合速查**：

| 数字 | 权限 | 适用场景 |
|------|------|---------|
| `777` | rwxrwxrwx | 所有人完全控制（安全风险高，不推荐） |
| `755` | rwxr-xr-x | 可执行程序/脚本、目录（最常用） |
| `700` | rwx------ | 私密脚本或目录，只有所有者能用 |
| `644` | rw-r--r-- | 普通文件，所有者可修改，其他人只读（最常用） |
| `600` | rw------- | 私密文件，只有所有者能读写（如 SSH 私钥） |
| `400` | r-------- | 只读文件，连所有者都不能改 |

**类比**：数字法就像给三种人分别发"权限积分卡"。你（所有者）获得 7 分（满分，什么都能干），你家人（同组）获得 5 分（能看能用不能改），陌生人获得 5 分（同样待遇）。`chmod 755` 就是直接说出这三个数字，干净利落。

### 符号法（字母表示法）

数字法快速但不直观。符号法可以精确地增加或移除某个权限，不需要重新计算全部数字。

```bash
# 语法: chmod [who][operator][permission] file

# who: u (user/所有者), g (group/所属组), o (others/其他人), a (all/所有人)
# operator: + (添加权限), - (移除权限), = (设置为精确权限)
# permission: r (读), w (写), x (执行)
```

```bash
# 给所有者添加执行权限
chmod u+x script.sh
# 不管之前是什么权限，所有者现在多了 x

# 给所有人添加读权限
chmod a+r document.txt

# 移除所属组的写权限
chmod g-w document.txt

# 精确设置其他人的权限为只读
chmod o=r document.txt
# 不管 o 之前有什么权限，现在只有 r

# 给所有者和所属组添加执行权限（一次操作影响两个对象）
chmod ug+x script.sh

# 移除所有人的执行权限
chmod a-x script.sh
```

**符号法的优势**：你不需要知道当前的权限是什么。你只想"让这个文件可以执行"，`chmod +x` 就搞定了，不用去算 755 还是 644。这在自动化脚本里特别有用。

**类比**：数字法像是"我重新给你分配一套完整的权限卡"（不管之前有什么，全部换成新的）。符号法像是"在你的现有权限卡上增加/减少一项权限"（在原有基础上微调）。

### 递归修改权限

```bash
# 把整个目录及其所有子文件/子目录的权限都改掉
chmod -R 755 my_project/
```

警告：`chmod -R` 会把目录和文件都设成同样的权限。但文件和目录的理想权限通常是不一样的--文件通常不需要 x（执行），目录需要 x。所以 `chmod -R 755` 可能会给不该有执行权限的文件加上 x。更精细的做法是用 `find` 配合 `chmod`：

```bash
# 先把所有目录设为 755
find my_project -type d -exec chmod 755 {} \;
# 再把所有文件设为 644
find my_project -type f -exec chmod 644 {} \;
```

## 动手试试

在你的 Linux 终端中完成以下操作：

1. 创建练习目录和文件：
```bash
mkdir ~/linux-practice/permissions
cd ~/linux-practice/permissions
touch myfile.txt
echo "echo 'Hello World'" > myscript.sh
```

2. 查看默认权限：`ls -l`
3. 尝试运行脚本：`./myscript.sh`（会提示 Permission denied，因为没有 x 权限）
4. 添加执行权限：`chmod +x myscript.sh`
5. 再次运行：`./myscript.sh`（现在能执行了）
6. 用数字法改变权限：`chmod 600 myfile.txt`，然后用 `ls -l` 确认
7. 尝试读取 myfile.txt：`cat myfile.txt`（所有者能读，没问题）
8. 把 myfile.txt 设为只读：`chmod 400 myfile.txt`，尝试写入：`echo "test" > myfile.txt`（会提示 Permission denied）
9. 创建一个目录：`mkdir test_dir`，查看权限：`ls -ld test_dir`（注意目录默认权限通常是 755）
10. 移除目录的执行权限：`chmod 666 test_dir`，试试 `cd test_dir`（进不去！这证明了目录的 x 权限的重要性）
11. 清理：`cd ~ && rm -rf ~/linux-practice/permissions`

## 本节小结

Linux 权限用 r(4)、w(2)、x(1) 三种操作，分给所有者、所属组、其他人三类用户。`chmod 755` 让所有人能读能执行但只有所有者能修改，是目录和可执行文件的标准权限。`chmod 644` 让所有者能写、其他人只读，是普通文件的标准权限。
