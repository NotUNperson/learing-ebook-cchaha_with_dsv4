# 02-04 sed：流水线上的质检员

## 本节你会学到什么

- 理解 sed 的"流编辑"模式与传统编辑器的区别
- 使用 `s/old/new/g` 进行查找替换
- 使用 `d` 命令删除指定行
- 使用 `i` 和 `a` 在指定位置前后插入文本
- 使用 `p` 和 `-n` 精确打印选中的行
- 在管道中把 sed 当作文本转换器来使用

## 正文

### 一、先讲一个流水线质检员的故事

想象一条汽水生产流水线。一瓶瓶汽水在传送带上前进，经过一个质检工位。质检员的工作很简单：

- 如果看到标签歪了，把它扶正（替换）。
- 如果看到一瓶有裂缝的，直接从传送带上拿掉（删除）。
- 如果看到瓶子之间有空位，塞一瓶新的进去（插入）。

质检员不会把整条传送带停下来——他就在流水线上，瓶子来一瓶处理一瓶，处理完就放回去继续往前走。

**sed** 就是这条传送带上的质检员。它的全名叫 **S**tream **Ed**itor（流编辑器），"stream"就是流水线。sed 从输入流中逐行读取文本，按你指定的规则处理每一行，然后把结果输出。它**不会修改原始文件**（除非你用 `-i` 选项明确要求），它只是把改过的版本输出出来。

**生活类比**：sed 就像你拿一支荧光笔在复印机旁工作。你把原稿放进去，在复印出来的副本上用荧光笔做标记、划掉某些行、替换某些词。原稿始终完好，你改动的是副本。

### 二、sed 的基本语法

```bash
sed [选项] '命令' 文件名
sed [选项] -e '命令1' -e '命令2' 文件名   # 多个命令
```

命令由两部分组成：一个**地址**（address，指定对哪些行操作）和一个**动作**（command，做什么操作）。

如果没有指定地址，sed 默认对**所有行**执行动作。

```bash
# 语法结构
sed '地址 动作' 文件

# 地址可以是：
#   3        → 第 3 行
#   3,5      → 第 3 到第 5 行
#   /pattern/ → 匹配某模式的行
#   $        → 最后一行

# 动作可以是：
#   s/old/new/ → 替换
#   d          → 删除
#   i\         → 在匹配行之前插入
#   a\         → 在匹配行之后追加
#   p          → 打印
```

### 三、sed 的四大核心操作

#### 3.1 替换（substitute）：`s` 命令

这是 sed 最常用的操作，占据了你 90% 的 sed 使用场景。

```bash
# 基本格式
sed 's/旧内容/新内容/' 文件名

# 示例：把 hello 替换成 hi
echo "hello world" | sed 's/hello/hi/'
# 输出: hi world

# 示例：把 apple 替换成 orange
echo "I have an apple" | sed 's/apple/orange/'
# 输出: I have an orange
```

**关键：`g` 标志**

默认情况下，`s` 命令只替换每行中**第一个**匹配项。如果你想把一行里的所有旧内容都替换掉，要加 `g`（global，全局）：

```bash
echo "apple pie and apple juice" | sed 's/apple/orange/'
# 输出: orange pie and apple juice  （只替换了第一个 apple）

echo "apple pie and apple juice" | sed 's/apple/orange/g'
# 输出: orange pie and orange juice  （全部替换了）
```

**其它标志**

| 标志 | 含义 |
|------|------|
| `g` | 全局替换（一行中所有匹配） |
| `i` | 忽略大小写（GNU sed 中可以用 `I`） |
| `数字` | 只替换第 N 个匹配 |
| `p` | 打印替换过的行 |

```bash
# 只替换第 2 个出现的 apple
echo "apple apple apple" | sed 's/apple/orange/2'
# 输出: apple orange apple

# 替换并打印被修改的行
sed -n 's/error/ERROR/p' app.log
```

**分隔符不一定是 `/`**

如果你的搜索内容本身就包含 `/`（比如文件路径），用 `/` 做分隔符就会变成一堆转义地狱：

```bash
# 难读：替换路径
sed 's/\/home\/user\/old/\/opt\/new/g' config.txt

# 好读：用 # 做分隔符
sed 's#/home/user/old#/opt/new#g' config.txt

# 也可以用 | 或 : 或 , 
sed 's|/var/log|/opt/log|g' config.txt
sed 's:/usr/bin:/usr/local/bin:g' config.txt
```

**生活类比**：分隔符就像你写信封时选什么笔——圆珠笔、钢笔、铅笔都行，只要你别用和信封本身颜色太接近的笔。内容里有 `/` 你就换个分隔符，就像白纸上不要用白笔写字。

#### 3.2 删除（delete）：`d` 命令

`d` 命令从输出中删除指定行。记住——它只是不输出这些行了，原始文件不受影响。

```bash
# 删除第 3 行
seq 1 5 | sed '3d'
# 输出: 1 2 4 5 （第 3 行不见了）

# 删除第 2 到第 4 行
seq 1 5 | sed '2,4d'
# 输出: 1 5

# 删除所有空行
sed '/^$/d' config.txt

# 删除所有以 # 开头的注释行
sed '/^#/d' config.txt

# 删除最后一行
sed '$d' file.txt
```

**实用案例**：快速清理一个配置文件，只留下有效配置：

```bash
# 删除注释行和空行
sed '/^#/d; /^$/d' /etc/ssh/sshd_config
```

#### 3.3 插入（insert）和追加（append）：`i` 和 `a`

- `i`：在当前行**之前**插入新行
- `a`：在当前行**之后**追加新行

```bash
# 在第 2 行之前插入一行
seq 1 5 | sed '2i\---我是插入的行---'
# 输出:
# 1
# ---我是插入的行---
# 2
# 3
# 4
# 5

# 在第 2 行之后追加一行
seq 1 5 | sed '2a\---我是追加的行---'
# 输出:
# 1
# 2
# ---我是追加的行---
# 3
# 4
# 5

# 在文件的最后一行之后追加
sed '$a\这是新增的最后一行' file.txt

# 在文件的第一行之前插入
sed '1i\这是新增的第一行' file.txt
```

#### 3.4 打印（print）：`p` 命令

`p` 命令单独使用时会打印匹配的行——但要注意，sed 默认就会输出所有行，所以单独 `p` 会导致匹配行被打印**两次**。

```bash
# 错误用法：匹配行会被打印两次
seq 1 10 | sed '/5/p'
# 第 5 行出现两次！

# 正确用法：配合 -n 抑制默认输出，只打印匹配行
seq 1 10 | sed -n '/5/p'
# 只输出 5

# 打印第 3 到第 6 行（相当于 head + tail）
seq 1 10 | sed -n '3,6p'

# 打印包含 error 的行
sed -n '/error/p' log.txt
```

### 四、sed 的地址（Address）详解

地址决定了 sed 命令作用于哪些行。掌握地址，你就能精确控制 sed 的行为。

#### 4.1 数字地址

```bash
sed '3s/hello/hi/' file.txt      # 只替换第 3 行的 hello
sed '3,5d' file.txt               # 删除第 3 到第 5 行
sed '3,$d' file.txt               # 删除第 3 行到最后一行（$ 表示最后一行）
sed '1,3!d' file.txt              # ! 表示取反，即保留第 1 到 3 行，删除其他
```

#### 4.2 正则地址

```bash
sed '/error/s/ERROR/FIXED/' log.txt    # 只对包含 error 的行做替换
sed '/start/,/end/d' file.txt          # 删除从包含 start 的行到包含 end 的行
sed '/^#/d' config.txt                 # 删除所有注释行
```

#### 4.3 多命令组合

```bash
# 用分号分隔多个命令
sed 's/apple/orange/g; s/banana/grape/g' fruits.txt

# 或使用多个 -e
sed -e 's/apple/orange/g' -e 's/banana/grape/g' fruits.txt

# 按顺序执行：先删除注释行，再做替换
sed -e '/^#/d' -e 's/localhost/127.0.0.1/g' config.txt
```

**生活类比**：sed 的地趾逻辑就像 Excel 的筛选功能——你可以"对 A 列包含 error 的所有行，把 B 列的值替换成 FIXED"。先筛选（地址），再操作（命令）。

### 五、sed 在管道中的威力

sed 配合管道使用时，就成了一条自动化文本处理流水线上的关键工位：

```bash
# 把 ls 输出中的长路径简化显示
ls -la | sed 's/home\/user/~/g'

# 从 ps 输出中提取 PID 和进程名
ps aux | sed -n '2,$p' | sed 's/  */ /g'

# 读取日志，找出错误，格式化输出
cat app.log | grep ERROR | sed 's/ERROR/***ERROR***/g' | less

# 生成一个 markdown 表格的骨架
seq 1 5 | sed 's/.*/| & |  |  |/' 
```

### 六、sed 的实战场景

#### 场景一：批量修改配置文件

```bash
# 把某个配置文件里的所有 8080 端口改成 9090
sed 's/8080/9090/g' server.conf > server.conf.new

# 或者（如果你确认没问题）直接改原文件
sed -i 's/8080/9090/g' server.conf
```

`-i` 选项表示 "in-place"（原地修改），会直接修改文件。强烈建议第一次使用时先**不加 `-i`**，预览效果后再加上。
`-i.bak` 会在修改前自动创建备份文件：

```bash
# 安全做法：修改原文件，同时创建 .bak 备份
sed -i.bak 's/8080/9090/g' server.conf
# 会生成 server.conf.bak 作为原始文件的备份
```

#### 场景二：提取日志中的特定信息

```bash
# 从一个 Web 访问日志中提取所有 HTTP 状态码
sed -n 's/.*" [0-9]\{3\} \([0-9]*\).*/\1/p' access.log | sort | uniq -c
```

#### 场景三：代码重构

```bash
# 把项目中所有文件里的 oldFunctionName 重命名为 newFunctionName
sed -i 's/oldFunctionName/newFunctionName/g' src/*.py

# 给所有 Python 文件添加一行 import
sed -i '1i\import sys' src/*.py
```

#### 场景四：清理数据

```bash
# 删除 CSV 文件的所有空行
sed -i '/^$/d' data.csv

# 去掉每行末尾多余的空格（这在代码 lint 中很常见）
sed -i 's/[ \t]*$//' src/*.py
```

### 七、sed 的高级技巧速览

这里列举几个常用但稍高级的技巧，你可以先感受一下，不一定立刻掌握：

```bash
# 引用匹配的内容：& 表示整个匹配文本
echo "hello world" | sed 's/world/(&)/'
# 输出: hello (world)   —— & 被替换为 world

# 分组引用：\1 \2 引用第 N 个括号中的内容
echo "John Smith" | sed 's/\([A-Za-z]*\) \([A-Za-z]*\)/\2, \1/'
# 输出: Smith, John   —— 交换了姓和名

# 只处理某个范围的行
sed '10,20s/foo/bar/g' file.txt   # 只替换第 10 到 20 行的 foo

# 处理多行（在处理 XML/HTML 时有时有用）
sed 'N; s/\n/, /'  # 把两行合并成一行，用逗号分隔
```

### 八、sed 常见错误与排错

**错误一**：忘了 `g` 标志，只替换了每行第一个匹配。

```bash
echo "the cat and the hat" | sed 's/the/a/'
# 输出: a cat and the hat    ← 第二个 the 没被替换

echo "the cat and the hat" | sed 's/the/a/g'
# 输出: a cat and a hat      ← 全都替换了
```

**错误二**：在 sed 命令中使用单引号内的变量。

```bash
# 在单引号内，$var 不会被 shell 展开
name="world"
echo "hello world" | sed 's/world/$name/'    # 输出: hello $name （字面值）

# 解决方法：改用双引号
echo "hello world" | sed "s/world/$name/"    # 输出: hello world
```

**错误三**：忘记 `-i` 不会改原文件，但第一次就用 `-i` 改错了后悔。

```bash
# 安全流程：
# 1. 先不加 -i 预览
sed 's/old/new/g' file.txt | head -10
# 2. 确认无误后加 -i（最好加 .bak）
sed -i.bak 's/old/new/g' file.txt
```

**错误四**：把 `sed` 和文件名写反。

```bash
sed file.txt 's/old/new/'    # 错误！sed 不认识这个顺序
sed 's/old/new/' file.txt    # 正确
```

### 九、sed vs grep vs awk：三兄弟的分工

| 工具 | 擅长什么 | 类比 |
|------|---------|------|
| `grep` | 查找/过滤行 | 图书馆查书员 |
| `sed` | 查找并替换/删除/修改 | 流水线质检员 |
| `awk` | 结构化文本的提取和计算 | 会计在报表上做统计 |

在一行命令里，它们经常搭配使用：

```bash
# grep 找出相关行 → sed 修改内容 → awk 做统计
grep "ERROR" app.log | sed 's/.*ERROR //' | awk '{sum+=$1} END {print sum}'
```

## 动手试试

1. 创建测试文件：`echo -e "apple\nbanana\napple pie\norange" > fruits.txt`。执行 `sed 's/apple/mango/' fruits.txt`，观察输出。再执行 `sed 's/apple/mango/g' fruits.txt`，比较差异。
2. 执行 `seq 1 10 | sed '3,7d'`。观察哪些数字被删除了。
3. 执行 `seq 1 5 | sed '2i\***插入行***'`，看插入发生在哪一行之前。
4. 执行 `sed -n '3,5p' /etc/passwd`，验证只打印了第 3 到第 5 行。
5. 创建文件 `echo -e "# 注释1\n有效配置=真\n# 注释2\n\n空行上面这行" > test.conf`。执行 `sed '/^#/d; /^$/d' test.conf`，看是否只剩有效配置行。
6. 用管道组合：`ps aux | sed -n '1p; /bash/p'`，看结果（打印表头 + 所有 bash 进程行）。
7. 试试 `sed -i.bak`：`echo "old text" > test.txt && sed -i.bak 's/old/new/' test.txt`，然后 `cat test.txt` 和 `cat test.txt.bak`。

## 本节小结

`sed` 是一个流编辑器，它像流水线上的质检员一样逐行处理文本，用 `s` 做替换、`d` 做删除、`i`/`a` 做插入追加、`p` 配合 `-n` 做精确打印。默认不修改原文件（安全预览先用），加 `-i` 才原地修改。它在管道中配合 grep 和 awk，构成了 Linux 文本处理的三件套。
