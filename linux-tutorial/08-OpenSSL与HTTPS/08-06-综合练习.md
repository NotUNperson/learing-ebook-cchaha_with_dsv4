## 步骤 1：安装 Nginx 和基础配置

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Nginx
sudo apt install nginx -y

# 启动并设置开机自启
sudo systemctl enable --now nginx

# 确认运行
sudo systemctl status nginx
```

创建你的站点根目录和测试页面：

```bash
# 创建站点目录
sudo mkdir -p /var/www/example.com

# 写一个测试页面
sudo tee /var/www/example.com/index.html << 'EOF'
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>我的 HTTPS 站点</title>
    <style>
        body {
            font-family: system-ui, sans-serif;
            max-width: 600px;
            margin: 100px auto;
            text-align: center;
        }
        .lock { font-size: 4rem; }
        .green { color: #22c55e; }
    </style>
</head>
<body>
    <div class="lock green">&#x1f512;</div>
    <h1>恭喜！HTTPS 配置成功！</h1>
    <p>这个页面通过加密连接传输</p>
</body>
</html>
EOF
```

创建 HTTP 站点配置：

```nginx
sudo tee /etc/nginx/sites-available/example.com << 'EOF'
server {
    listen 80;
    server_name example.com www.example.com;

    root /var/www/example.com;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}
EOF

# 启用站点
sudo ln -s /etc/nginx/sites-available/example.com /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default

# 测试配置
sudo nginx -t

# 重载
sudo systemctl reload nginx
```

## 步骤 2：配置 DNS

去你的域名注册商（或 DNS 服务商）的管理后台，添加 A 记录：

```
类型: A
主机记录: @       （表示 example.com）
记录值:  你的服务器公网 IP
TTL:     600

类型: A
主机记录: www     （表示 www.example.com）
记录值:  你的服务器公网 IP
TTL:     600
```

验证 DNS 生效：

```bash
# 等待 DNS 传播（通常几分钟到几小时）
nslookup example.com
# 或
dig example.com +short
# 或在 Windows 上：
# nslookup example.com

# 测试 HTTP 访问
curl -I http://example.com
# 应该返回 HTTP 200
```

## 步骤 3：用 certbot 申请证书

```bash
# 安装 certbot
sudo apt install certbot python3-certbot-nginx -y

# 申请证书（certbot 自动修改 Nginx 配置）
sudo certbot --nginx -d example.com -d www.example.com

# 交互式选择：
# 1. 输入你的邮箱（用于到期提醒和紧急通知）
# 2. 同意服务条款（A）
# 3. 是否订阅邮件（可选，Y/N）
# 4. 选择是否自动重定向 HTTP 到 HTTPS（推荐选 2：自动重定向）
```

certbot 会自动修改你的 Nginx 配置，加入 SSL 证书路径和 HTTPS 设置。

查看修改后的配置：

```bash
sudo cat /etc/nginx/sites-available/example.com
```

你会看到 certbot 添加了 `listen 443 ssl`、`ssl_certificate`、`ssl_certificate_key` 等指令，以及一个 HTTP 到 HTTPS 的重定向 server 块。

## 步骤 4：加固 Nginx HTTPS 配置

certbot 的自动配置偏保守。在 `/etc/nginx/sites-available/example.com` 的 HTTPS server 块中添加优化项：

```nginx
server {
    # ... certbot 生成的配置 ...

    # ====== 新增以下优化 ======

    # HTTP/2
    listen 443 ssl;       # certbot 默认的
    # 把上面改成：
    listen 443 ssl http2; # 启用 HTTP/2

    # 协议版本
    ssl_protocols TLSv1.2 TLSv1.3;

    # 加密套件（放在 ssl_certificate 之后）
    ssl_ciphers ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384:ECDHE-ECDSA-CHACHA20-POLY1305:ECDHE-RSA-CHACHA20-POLY1305;
    ssl_prefer_server_ciphers off;

    # ECDH 曲线
    ssl_ecdh_curve X25519:prime256v1:secp384r1;

    # 性能优化
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;
    ssl_session_tickets off;

    # DH 参数
    ssl_dhparam /etc/nginx/dhparam.pem;

    # OCSP Stapling
    ssl_stapling on;
    ssl_stapling_verify on;
    ssl_trusted_certificate /etc/letsencrypt/live/example.com/chain.pem;
    resolver 8.8.8.8 8.8.4.4 valid=300s;
    resolver_timeout 5s;

    # 安全头
    add_header Strict-Transport-Security "max-age=63072000; includeSubDomains; preload" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    # ... 其余 certbot 生成的配置保持不变 ...
}
```

生成 DH 参数文件：

```bash
# 可能需要 2-5 分钟
sudo openssl dhparam -out /etc/nginx/dhparam.pem 2048
```

测试并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 步骤 5：验证站点安全性

### 5.1 浏览器检查

访问 `https://example.com`，检查：
- 地址栏出现小锁图标
- 点击小锁 -> 查看证书 -> 确认是 Let's Encrypt 签发的
- 证书 SAN 包含 `example.com` 和 `www.example.com`

### 5.2 curl 检查

```bash
# 查看 TLS 握手信息
curl -vI https://example.com 2>&1 | grep -E "SSL|subject|issuer|HTTP/"

# 期望输出：
# * SSL connection using TLSv1.3
# * subject: CN=example.com
# * issuer: C=US; O=Let's Encrypt; CN=R3
# < HTTP/2 200
```

### 5.3 OpenSSL 检查

```bash
# 确认 TLS 1.3 可用
echo | openssl s_client -connect example.com:443 -tls1_3 2>/dev/null | grep "Protocol"
# 输出：Protocol  : TLSv1.3

# 查看证书链
openssl s_client -connect example.com:443 -showcerts </dev/null 2>/dev/null | grep -E "subject=|issuer="
```

### 5.4 SSL Labs 检测

打开 `https://www.ssllabs.com/ssltest/`，输入 `example.com`，等待 2-3 分钟。目标评级：**A+**。

如果你的评级不到 A+，检查：
- 是否启用了 TLS 1.0/1.1？（应该禁用）
- HSTS 头是否设置了？
- 加密套件中是否包含弱算法？
- OCSP Stapling 是否生效？

### 5.5 安全头检查

打开 `https://securityheaders.com/`，输入你的域名，查看安全头评分。

## 步骤 6：自动续期验证

```bash
# 测试自动续期（干跑，不会真续）
sudo certbot renew --dry-run

# 检查 systemd 定时器
sudo systemctl status certbot.timer
sudo systemctl list-timers | grep certbot

# 手动续期一次确认流程正常
sudo certbot renew

# 重载 Nginx
sudo systemctl reload nginx
```

## 步骤 7：证书过期监控

即使有自动续期，也要有人盯着。创建一个简单的监控脚本：

```bash
sudo tee /usr/local/bin/cert-check.sh << 'SCRIPT'
#!/bin/bash
# 证书过期检查脚本，放到 crontab 里每天跑一次

DOMAIN="example.com"
THRESHOLD_DAYS=14    # 少于这个天数就告警
CERT_FILE="/etc/letsencrypt/live/$DOMAIN/cert.pem"
ALERT_EMAIL="your-email@example.com"

if [ ! -f "$CERT_FILE" ]; then
    echo "证书文件不存在: $CERT_FILE"
    exit 1
fi

# 获取证书过期日期
END_DATE=$(openssl x509 -enddate -noout -in "$CERT_FILE" | cut -d= -f2)
END_EPOCH=$(date -d "$END_DATE" +%s)
NOW_EPOCH=$(date +%s)

DAYS_LEFT=$(( ($END_EPOCH - $NOW_EPOCH) / 86400 ))

echo "证书剩余天数: $DAYS_LEFT"

if [ $DAYS_LEFT -lt $THRESHOLD_DAYS ]; then
    echo "WARNING: 证书即将过期！剩余 $DAYS_LEFT 天"
    # 发送告警邮件（需要 mail 命令）
    echo "域名 $DOMAIN 的 SSL 证书将在 $DAYS_LEFT 天后过期" | \
        mail -s "SSL 证书过期告警: $DOMAIN" $ALERT_EMAIL
fi
SCRIPT

sudo chmod +x /usr/local/bin/cert-check.sh

# 每天下午 2 点检查
(crontab -l 2>/dev/null; echo "0 14 * * * /usr/local/bin/cert-check.sh >> /var/log/cert-check.log 2>&1") | crontab -
```

## 故障排查清单

| 症状 | 可能原因 | 检查命令 |
|------|----------|---------|
| 浏览器显示"不安全" | 证书过期或域名不匹配 | `openssl x509 -in cert.pem -dates -subject` |
| Nginx 无法启动 | 证书和私钥不匹配 | 对比 modulus MD5 |
| SSL Labs 评分 B | 启用了弱加密 | 检查 `ssl_ciphers` 和 `ssl_protocols` |
| 部分用户访问慢 | OCSP Stapling 失败 | `openssl s_client -status -connect domain:443` |
| certbot 续期失败 | 端口 80 不可达 | `curl http://domain/.well-known/acme-challenge/` |
| HSTS 误配置后无法回退 | HSTS 被浏览器记住 | 清除浏览器 HSTS 缓存或修改 `max-age=0` |
