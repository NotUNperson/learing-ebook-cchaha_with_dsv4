# 06-02 变量与环境变量

## 本节你会学到什么

- 在 Shell 脚本中定义、引用、修改和删除变量
- 理解普通变量和环境变量的区别
- 使用 `export` 命令将变量传递给子进程
- 掌握 `$0`、`$1`、`$#`、`$?`、`$$` 等特殊变量的用法
- 用便利贴 vs 公告栏的类比记住变量的作用范围

---

上一节我们让脚本"说了话"（echo 输出），但脚本还不会"记东西"。一个不会记东西的脚本就像每次做饭都要重新出门买菜 -- 明明昨天买过的，今天又得跑一趟。

变量就是脚本的"记忆纸条"。它让你把信息存下来，后面随时取用。

---

## 便利贴 vs 公告栏：理解变量的作用范围

先讲一个非常重要的概念。想象你在办公室工作：

- **便利贴（自定义变量）**：你写在便利贴上的东西，只有你自己看得到。旁边同事看不到你的便利贴内容。等你下班走人（脚本结束），便利贴就扔进垃圾桶了。

- **公告栏（环境变量）**：办公室公告栏上贴的通知，所有人（进程）都看得到。新来的同事（新进程）也能看到。公告栏上的东西通常在"装修"的时候（系统启动）就贴上了，一直挂在那里。

这就是普通变量和环境变量的核心区别 -- **作用范围**。

```bash
my_note="这是便利贴"      # 只有当前 Shell 能看到
echo $HOME              # 环境变量，所有程序都能看到
```

## 定义变量：给盒子贴标签

在 Shell 中定义变量非常简单，但你一定要记住一条铁律：

**等号两边不能有空格！**

```bash
name="小明"        # 正确
age=25            # 正确
name = "小明"     # 错误！bash 会把 name 当成命令执行
```

为什么不能有空格？因为 bash 解释命令时，用空格来分隔"命令"和"参数"。如果你写 `name = "小明"`，bash 会以为你要执行一个叫 `name` 的命令，后面跟着 `=` 和 `"小明"` 两个参数。

**类比**：这就像填表格。表格上的"姓名：____"位置是固定的，你不能在"姓名"后面加一个空格再写内容。变量赋值也是"紧密绑定"的。

变量命名规则：
- 只能使用英文字母、数字和下划线
- 不能以数字开头
- 区分大小写（`NAME` 和 `name` 是两个不同的变量）

```bash
user_name="正确"
_user="正确"
file2="正确"
2file="错误"        # 不能以数字开头
my-name="错误"      # 连字符不允许
```

## 引用变量：把记下的东西拿出来用

定义好变量后，用 `$` 符号来"取"它的值。加上 `$` 就是"取这个变量的内容"，不加 `$` 就是"这个变量名本身"。

```bash
name="小明"
echo $name       # 输出：小明
echo ${name}     # 输出：小明（花括号版本）
echo name        # 输出：name（没有 $，原样输出）
```

花括号 `${}` 什么时候必须用？当你需要把变量和后面的文字"粘"在一起的时候：

```bash
prefix="log"
echo "${prefix}_2026.txt"   # 输出：log_2026.txt
echo "$prefix_2026.txt"     # 输出：空（bash 去找变量 prefix_2026 了）
```

花括号就像划出一道清晰的分界线，告诉 bash："变量到这里为止，后面的是别的东西"。

---

## 变量类型：Shell 变量很"随和"

和 Python、Java 等编程语言不同，Shell 变量不区分类型。你存进去的东西默认就是字符串：

```bash
a="hello"
b=123
c=3.14
```

这里 `b=123` 看起来是数字，但它其实是字符串 `"123"`。当你做算术运算时（后面会讲到），bash 会自动转换。这种"宽松"的设计有时候方便，有时候会埋坑 -- 我们在条件判断那节会深入讨论。

**类比**：Shell 变量就像一个大口袋，什么都能往里面装 -- 硬币（数字）、纸条（字符串）、小玩意儿（文件路径）。口袋不关心你装什么，只管装进去和掏出来。

---

## 只读变量：覆上保鲜膜

有时候你希望某个变量的值永远不变（比如圆周率 PI、配置文件路径）。用 `readonly` 把它"锁死"：

```bash
readonly PI=3.14159
PI=3.14    # 报错！不能修改只读变量
```

`declare -r` 效果相同：

```bash
declare -r CONFIG_PATH="/etc/myapp"
echo $CONFIG_PATH
# CONFIG_PATH="/opt/myapp"   # 也会报错
```

**类比**：就像博物馆里的展品，加了玻璃罩 -- 你可以看，但不能碰。只读变量给你的脚本增加了一层保护，防止误改重要的值。

---

## 删除变量：撕掉便利贴

不需要的变量可以删掉，释放"记忆"：

```bash
temp="马上要删掉"
echo $temp       # 输出：马上要删掉
unset temp
echo $temp       # 输出：空（变量已经不存在了）
```

注意：只读变量不能用 `unset` 删除。就像你没法把博物馆展品上的玻璃罩撕下来。

## 环境变量：系统的"公告栏"

环境变量是系统启动时就设置好的一些全局变量。你可以在终端敲 `env` 或者 `printenv` 查看所有环境变量。

最常见的有：

| 变量 | 含义 | 示例值 |
|------|------|--------|
| `$HOME` | 当前用户的家目录 | `/home/xiaoming` |
| `$USER` | 当前用户名 | `xiaoming` |
| `$PWD` | 当前工作目录 | `/home/xiaoming/docs` |
| `$PATH` | 命令搜索路径 | `/usr/bin:/bin:/usr/sbin` |
| `$SHELL` | 当前 Shell | `/bin/bash` |
| `$LANG` | 系统语言 | `zh_CN.UTF-8` |
| `$HOSTNAME` | 主机名 | `myserver` |

`$PATH` 需要特别讲一下。当你在终端敲 `ls` 的时候，bash 怎么知道 `ls` 这个程序在哪里？它就是去 `$PATH` 里列出的目录中挨个找。如果你的可执行文件放在不在 `$PATH` 里的目录中，你就必须写完整路径（比如 `/home/xiaoming/bin/mytool`）才能执行。

`$PATH` 里的目录用冒号 `:` 分隔：

```bash
echo $PATH
# 输出：/usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin
```

这就像手机的通讯录 -- 你打电话的时候不用记号码，通讯录帮你找到对应的号码。`$PATH` 就是命令的通讯录。

## export：把你的便利贴贴到公告栏上

默认情况下，你在脚本里定义的变量只在这个脚本里有效。如果脚本里面调用了另一个脚本（子进程），那个子脚本看不到你定义的变量。就像你写在便利贴上的东西，同事看不到。

`export` 命令就是"把便利贴贴到公告栏上"：

```bash
my_var="hello"           # 普通变量，只有自己能看到
export my_var            # 现在子进程也能看到它了

# 或者一行搞定：
export APP_ROOT="/opt/myapp"
```

**经典场景**：你写了一个部署脚本，里面要调用 `./install.sh`。`install.sh` 需要知道软件装到哪里。你可以在主脚本里 `export INSTALL_DIR="/opt/myapp"`，子脚本用 `echo $INSTALL_DIR` 就能拿到了。

注意：子进程对变量的修改不会影响父进程。就像同事在公告栏旁边贴的便签纸不是公告栏的一部分。如果子进程需要"回传"数据给父进程，一般用文件、管道或退出码来实现。

## 特殊变量：脚本的"身份证"和"通讯录"

bash 提供了一些特殊的"内置变量"，它们不用你定义，脚本运行时就自动有了。

| 变量 | 含义 | 类比 |
|------|------|------|
| `$0` | 脚本本身的名字 | 你的身份证上的姓名 |
| `$1` ~ `$9` | 第 1 到第 9 个命令行参数 | 快递包裹上的收件人信息 |
| `$#` | 命令行参数的个数 | 包里有几件东西 |
| `$@` | 所有参数（每个独立） | 列出每件东西的清单 |
| `$*` | 所有参数（合并为一个）| 把所有东西揉在一起 |
| `$?` | 上一条命令的退出码 | 考试成绩：0=及格，非0=不及格 |
| `$$` | 当前脚本的进程 ID (PID) | 你的工号 |
| `$!` | 最后一个后台进程的 PID | 最后入职的同事的工号 |

`$@` 和 `$*` 的区别需要特别说明：

```bash
# 假设执行 ./script.sh a b c

# $@ = "a" "b" "c" （三个独立的参数）
for arg in "$@"; do echo $arg; done
# 输出三行

# $* = "a b c" （一个合并的大参数）
for arg in "$*"; do echo $arg; done
# 输出一行：a b c
```

大部分时候你应该用 `"$@"`，因为它保留了每个参数的独立性。

`$?` 是调试时的好朋友：

```bash
ls /some/dir
if [ $? -eq 0 ]; then
    echo "命令执行成功"
else
    echo "命令执行失败"
fi
```

每条命令执行完都会设置 `$?`，0 代表成功，非 0 代表某种错误。这是一个非常重要的机制，后面讲错误处理时会反复用到。

## 变量与引号的"三角关系"

变量配合引号有一些微妙的行为，这里总结一下：

```bash
name="xiao ming"    # 变量值里有空格

echo $name          # 输出：xiao ming（bash 做了单词拆分）
echo "$name"        # 输出：xiao ming（双引号保护了空格）
echo '$name'        # 输出：$name（单引号不解析变量）
```

最佳实践：**始终用双引号包裹变量引用**。除非你确定变量不包含空格或通配符，否则不加引号可能导致意外的单词拆分。

```bash
file="my document.txt"
rm $file        # 危险！bash 会尝试删除 "my" 和 "document.txt" 两个文件
rm "$file"      # 正确！删除 "my document.txt" 这一个文件
```

## 本节小结

变量是脚本的"记忆体"，用 `$` 引用、`=` 赋值（不能有空格）、`unset` 删除。环境变量是系统的"公告栏"，用 `export` 可以把你的变量贴上去让子进程看到。特殊变量如 `$1`、`$?`、`$$` 是脚本与外部世界交互的关键桥梁。
