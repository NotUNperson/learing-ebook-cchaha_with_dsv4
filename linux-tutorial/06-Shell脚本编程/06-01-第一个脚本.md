# 06-01 第一个脚本

## 本节你会学到什么

- 编写你的第一个 Shell 脚本并让它运行起来
- 理解 shebang（`#!/bin/bash`）的作用和原理
- 掌握 `echo` 命令输出文本和变量的方法
- 学会使用 `chmod +x` 给脚本添加可执行权限
- 理解 Shell 脚本与 Windows 批处理文件（.bat）的异同

---

你有没有遇到过这种情况：每天登录服务器都要敲一串相同的命令 -- 查看磁盘空间、检查进程状态、备份几个重要文件。每次手动敲一遍，不仅枯燥，还容易漏掉某个步骤。要是能把这串命令"录下来"，下次一键执行就好了。

Shell 脚本就是干这个的。它本质上是一个文本文件，里面写了一行一行的命令，系统会按顺序帮你执行。你可以把它想象成一个"录音带"：你把要说的话（命令）录进去，下次直接播放就行。

如果你用过 Windows，可能见过 `.bat` 批处理文件 -- 双击就能执行一串 DOS 命令。Shell 脚本就是 Linux 世界的批处理，但功能强大得多：它有变量、条件判断、循环、函数，本质上是一门完整的编程语言。

---

## 从一个简单的脚本开始

打开你的终端（或者 Vim），创建一个新文件，起名叫 `hello.sh`：

```bash
vim hello.sh
```

在文件里写入以下内容：

```bash
#!/bin/bash

# 这是我的第一个 Shell 脚本
# 井号开头的是注释，不会被执行

echo "你好，世界！"
echo "这是我写的第一个脚本，感觉不错！"

# 显示当前时间
echo "现在是：$(date)"
```

保存退出后，用下面的命令执行它：

```bash
bash hello.sh
```

你应该能在屏幕上看到：

```
你好，世界！
这是我写的第一个脚本，感觉不错！
现在是：2026年 05月 15日 星期五 14:30:00 CST
```

恭喜！你已经写了人生第一个 Shell 脚本。

---

## shebang：告诉系统该找谁

脚本第一行的 `#!/bin/bash` 有个很酷的名字叫 **shebang**（发音类似"谁帮"）。它由 `#!` 组成，后面跟着解释器的路径。

它的作用就像快递单上的收件人地址。你把脚本交给操作系统，操作系统看到 shebang，就知道应该把脚本送给 `/bin/bash` 去执行。如果你写的是 `#!/usr/bin/python3`，系统就会送给 Python 解释器去执行。

你可以用 `which` 命令确认 bash 的位置：

```bash
which bash
# 输出通常是 /bin/bash 或 /usr/bin/bash
```

常见的 shebang 写法：

```bash
#!/bin/bash          # Bash 脚本
#!/bin/sh            # 通用 Shell 脚本（兼容性更好，但功能少）
#!/usr/bin/python3   # Python 脚本
#!/usr/bin/env bash  # 更灵活的写法（从 PATH 中找 bash）
```

`#!/usr/bin/env bash` 这种写法更"宽容" -- 它不硬编码 bash 的路径，而是从系统的环境变量 `PATH` 中去寻找 bash。在 bash 安装在不同位置的系统上（比如 macOS 和 Linux），这种写法更通用。

不过对于学习来说，直接写 `#!/bin/bash` 就可以了，这也是 99% 的情况下的标准写法。

## chmod +x：给脚本"通行证"

默认情况下，你新建的文件是没有可执行权限的。你可以用 `ls -l` 看看：

```bash
ls -l hello.sh
# 输出类似：-rw-r--r-- 1 user user 150  5月 15 14:30 hello.sh
```

最前面是 `-rw-r--r--`，看不到 `x`（executable，可执行）。这意味着你不能直接 `./hello.sh` 运行它。

你需要给它加上执行权限：

```bash
chmod +x hello.sh
```

然后再看：

```bash
ls -l hello.sh
# 输出类似：-rwxr-xr-x 1 user user 150  5月 15 14:30 hello.sh
```

多了三个 `x`，现在可以直接运行了：

```bash
./hello.sh
```

注意前面的 `./` -- 它表示"当前目录"。Linux 不像 Windows 会自动搜索当前目录下的可执行文件，你必须明确告诉系统：我要执行的是当前目录下的 `hello.sh`。

这个机制其实是一种安全保护。想象有人在你的目录里放了一个叫 `ls` 的恶意脚本，如果系统自动执行当前目录的文件，你敲 `ls` 的时候就会中招。加上 `./` 相当于你在说："我知道我在干什么，我就是要执行当前目录的这个文件。"

**类比**：`chmod +x` 就像是给文件盖了一个"可运行"的章。没有这个章，系统不会执行它。这就像你把一首歌下载到电脑里，首先得确认它不是病毒，然后才能双击播放。

## echo：脚本的"嘴巴"

`echo` 是 Shell 脚本中最常用的命令之一，它的作用就是"说" -- 把文字输出到屏幕上。

基本用法：

```bash
echo "Hello World"
echo '单引号也可以'
echo 不加引号也行但容易出问题
```

三种引号的区别（这是一个重要知识点）：

```bash
name="小明"

echo "你好，$name"      # 双引号会解析变量 → 你好，小明
echo '你好，$name'      # 单引号不解析变量 → 你好，$name
echo 你好，$name        # 无引号也会解析，但空格可能导致问题
```

记住一个口诀：**双引号里面能"看见"变量，单引号里变量被"关起来了"**。

`echo` 的常用选项：

```bash
echo -n "不换行"       # -n 表示不在末尾加换行符
echo -e "第一行\n第二行"  # -e 启用转义字符（\n=换行，\t=制表符）
```

转义字符（escape characters）这个名词听起来吓人，其实很简单。因为键盘上没法直接敲出"换行"这个动作，所以用 `\n` 来代替。就像你用 `:-)` 代替笑脸一样，是一种约定好的符号。

## 注释：写给未来的自己

以 `#` 开头的内容是注释，不会被脚本执行：

```bash
# 这是一个注释
echo "这行会执行"  # 行尾注释也可以
```

注释就像是代码里的便利贴。你现在写的脚本，三个月后回头看可能完全忘记为什么要那样写。好的注释能帮你快速回忆。不要吝啬写注释，"未来的你"会感谢"现在的你"。

给脚本加一个标准的文件头注释是个好习惯：

```bash
#!/bin/bash
# ==========================================
# 脚本名称：backup.sh
# 功能描述：备份 /var/www 目录到 /backup
# 作者：小明
# 创建日期：2026-05-15
# ==========================================
```

## 类比：批处理文件（.bat）和脚本的关系

如果你用过 Windows 的批处理文件 `.bat`，理解 Shell 脚本会容易很多。它们的目标是一致的：把命令打包，批量执行。

| 特性 | Windows 批处理 (.bat) | Linux Shell 脚本 (.sh) |
|------|----------------------|------------------------|
| 解释器 | cmd.exe | bash / sh |
| 注释 | `REM` 或 `::` | `#` |
| 输出 | `echo` | `echo` |
| 变量 | `%VAR%` | `$VAR` 或 `${VAR}` |
| 条件判断 | `IF ... ELSE` | `if ... then ... else ... fi` |
| 循环 | `FOR ... DO` | `for ... do ... done` |

本质上，它们都是"命令的清单"。但 Shell 脚本的能力远超批处理 -- 你可以把它当一门真正的编程语言来用。

## 为什么要学脚本？

这个问题就像是问"为什么要学做饭"。你当然可以顿顿叫外卖（每次手动敲命令），但学会做饭（写脚本）以后：

1. **省时间**：重复性的工作一键完成。每天花 5 分钟手动做的事，写成脚本只要 1 秒。
2. **不出错**：手动操作容易漏步骤，脚本每次执行的结果都是一致的。
3. **能分享**：把脚本给队友，他们也能用你的方法做同样的事。知识沉淀下来了。
4. **可组合**：小脚本可以组合成大的自动化流程。就像乐高积木，一块块拼出复杂的东西。

Linux 系统的日常维护、部署、监控，几乎都能用脚本来完成。运维工程师通常把脚本当"瑞士军刀"用。

## 动手试试

这是一个 5 分钟能完成的小练习：

1. 创建一个新文件 `myinfo.sh`，加入 shebang `#!/bin/bash`
2. 在脚本里用 `echo` 输出你的名字、今天的日期和当前工作目录
3. 给脚本添加至少 3 条注释
4. 用 `chmod +x myinfo.sh` 添加执行权限
5. 执行脚本，确认输出正确
6. 执行 `echo $?` 查看退出码（应该是 0）

提示：日期用 `date`，当前目录用 `pwd`，主机名用 `hostname`。

---

## 本节小结

Shell 脚本就是把命令写入文件，让系统批量执行。`#!/bin/bash` 告诉系统用什么解释器，`chmod +x` 赋予脚本执行权限，`echo` 负责输出文字。它就像 Windows 的 `.bat` 批处理文件，但功能强大得多。
