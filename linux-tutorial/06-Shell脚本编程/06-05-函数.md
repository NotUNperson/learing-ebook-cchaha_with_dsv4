# 06-05 函数

## 本节你会学到什么

- 在 Shell 中定义和调用函数（两种写法）
- 向函数传递参数并在函数内部使用 `$1`、`$@`
- 区分函数的两种"返回"机制：`return`（退出码）和 `echo`（数据）
- 使用 `local` 关键字保护函数内的局部变量
- 理解函数就像"微波炉预设程序"的类比

---

到目前为止，你写的脚本代码都是"铺开"的 -- 所有命令按照顺序从上到下排开。这在短脚本里没问题，但如果脚本有几百行，里面有相似的逻辑重复出现，维护起来就头疼了。

函数就是把这些重复的逻辑打成一个"包"，起个名字，以后直接叫名字就能用。就像微波炉上的"热牛奶"按钮 -- 你不需要每次都设 500W 中火 2 分钟，按一个键就行。

---

## 类比：微波炉预设程序

微波炉操作面板上有一排快捷键：热牛奶、解冻、烤土豆、热菜。每个按钮背后都是一组预设好的指令（火力多大、烤多久、是否翻面）。

写脚本也是一样：

```bash
# 没有函数，每次都要写一遍
echo "开始备份 /etc ..."
tar -czf /backup/etc-$(date +%Y%m%d).tar.gz /etc
echo "备份完成"

echo "开始备份 /var/www ..."
tar -czf /backup/www-$(date +%Y%m%d).tar.gz /var/www
echo "备份完成"

# 有了函数，一键搞定
backup_dir() {
    local dir="$1"
    echo "开始备份 $dir ..."
    tar -czf "/backup/$(basename $dir)-$(date +%Y%m%d).tar.gz" "$dir"
    echo "备份完成"
}

backup_dir /etc
backup_dir /var/www
```

函数的本质就是**给一组命令起名字**。定义了之后，你可以像用普通命令一样用它。

## 定义函数：两种写法

Shell 中定义函数有两种写法，功能完全一样，选择哪种看个人喜好：

```bash
# 写法一：function 关键字
function 函数名 {
    命令
}

# 写法二：函数名+括号（我个人更推荐这个）
函数名() {
    命令
}
```

**注意**：定义函数时，花括号 `{}` 里的命令可以另起一行或者和函数名同行，但函数体必须和括号分开。调用函数时不用加括号，直接写函数名就行：

```bash
# 定义
say_hi() {
    echo "你好"
}

# 调用（注意没有括号！）
say_hi
```

如果你是 Python 程序员，可能会不自觉地写成 `say_hi()` -- 这在 Shell 里是错误语法。记住：**定义时加括号，调用时不加括号**。

## 函数参数：函数内部的"局部通道"

函数内部也可以使用 `$1`、`$2`、`$@` 等特殊变量，但它们代表的是**传给函数的参数**，而不是传给脚本的参数。这个区别很重要！

```bash
#!/bin/bash

show_args() {
    echo "函数内部："
    echo "  函数名：${FUNCNAME[0]}"   # 当前函数名
    echo "  参数1=\$1"
    echo "  参数2=\$2"
    echo "  参数总数：$#"
}

echo "脚本参数1=$1"
echo "脚本参数2=$2"
echo "---"
show_args "函数参数A" "函数参数B"
```

如果脚本执行 `./script.sh hello world`，输出是：

```
脚本参数1=hello
脚本参数2=world
---
函数内部：
  函数名：show_args
  参数1=函数参数A
  参数2=函数参数B
  参数总数：2
```

看到了吗？函数内部的 `$1` 和脚本的 `$1` 是两套独立的"频道"，互不干扰。`$0` 永远是脚本名，即使你在函数里访问它。

**类比**：脚本参数就像楼下的信箱 -- 给整栋楼的。函数参数就像办公室的桌面收纳盒 -- 只给你一个人的。不同的人（函数）有不同的收纳盒，但都用同一个信封（`$1` 这个"称呼"），只是内容不同。

## 返回值：return 和 echo 的分工

Shell 函数的"返回"有两种完全不同的机制，一定要分清楚：

### return：返回退出码

```bash
is_root() {
    if [ "$EUID" -eq 0 ]; then
        return 0   # 是 root
    else
        return 1   # 不是 root
    fi
}

if is_root; then
    echo "以 root 身份运行"
fi
```

- `return` 只能返回 **0 到 255** 之间的整数
- `return 0` 表示"成功/是/真"
- `return 非0` 表示"失败/否/假"
- 函数调用直接放在 `if` 后面，无需用 `[ ]` 包裹

### echo：返回数据

```bash
get_sum() {
    local result=$(( $1 + $2 ))
    echo "$result"
}

total=$(get_sum 10 20)
echo "10 + 20 = $total"
```

- `echo` 返回的是**字符串数据**，可以是任意内容
- 调用函数时用 `$()` 或反引号把输出捕获到变量里
- 函数内部的其他 `echo` 也会被捕获到！所以调试信息要用 `>&2` 输出到 stderr

**踩坑提醒**：很多初学者在函数里用 `echo` 输出调试信息，然后调用者用 `$()` 捕获输出时把调试信息也抓进去了。解决办法：调试信息发到 stderr：

```bash
calculate() {
    echo "正在计算..." >&2    # 发到 stderr，不会被 $() 捕获
    echo $(( $1 * $1 ))       # 这是真正的"返回值"
}

result=$(calculate 5)    # result=25，不会包含"正在计算..."
```

## local：给变量设置"门禁"

默认情况下，函数里定义的变量是**全局的** -- 这会带来严重后果！

```bash
#!/bin/bash

name="张三"

change_name() {
    name="李四"    # 没有 local！直接修改了全局变量
}

echo "修改前：$name"   # 张三
change_name
echo "修改后：$name"   # 李四 -- 被改了！
```

加上 `local` 关键字后，变量就只在函数内部有效：

```bash
name="张三"

change_name() {
    local name="李四"   # 加了 local，不影响外面的 name
    echo "函数内：$name"  # 李四
}

echo "修改前：$name"   # 张三
change_name
echo "修改后：$name"   # 张三 -- 没被改变！
```

**类比**：`local` 就像是办公室的隔间墙壁。你在自己的隔间里贴便利贴（定义局部变量），外面的人看不到，也不会被影响。没有 `local` 则是开放式办公区，你贴的任何东西大家都看得到，也可能被不小心撕掉。

**最佳实践**：函数里定义的所有变量都应该用 `local`，除非你**明确需要**修改全局变量。

```bash
# 好习惯
my_func() {
    local temp_file="/tmp/temp_$$"
    local result=""
    local i
    # ...
}
```

## 函数库：把常用函数打包成"工具箱"

你可以把常用的函数放在一个独立文件里，成为一个"函数库"。需要时导入它。

比如创建 `lib-utils.sh`：

```bash
#!/bin/bash
# lib-utils.sh — 通用工具函数库

log_info() { echo "[INFO] $(date '+%H:%M:%S') $1"; }
log_warn() { echo "[WARN] $(date '+%H:%M:%S') $1" >&2; }
log_error() { echo "[ERROR] $(date '+%H:%M:%S') $1" >&2; }

check_root() {
    if [ "$EUID" -ne 0 ]; then
        log_error "此脚本需要 root 权限"
        exit 1
    fi
}

ensure_dir() {
    local dir="$1"
    if [ ! -d "$dir" ]; then
        mkdir -p "$dir" && log_info "创建目录：$dir"
    fi
}
```

然后在你的主脚本里用 `source`（或缩写 `.`）加载它：

```bash
#!/bin/bash
# 导入函数库
source "$(dirname "$0")/lib-utils.sh"

# 或者用 .（点命令），效果一样
# . "$(dirname "$0")/lib-utils.sh"

check_root
ensure_dir "/opt/myapp/logs"
log_info "部署完成"
```

`source` 和 `.` 的效果完全一样：它们把指定文件的内容"搬"到当前位置执行，就像是把那个文件的代码复制粘贴过来一样。

## 函数与递归：函数调用自己

Shell 函数也支持递归（自己调用自己）。最经典的例子是计算阶乘：

```bash
factorial() {
    local n=$1
    if [ $n -le 1 ]; then
        echo 1
    else
        local prev=$(factorial $((n - 1)))
        echo $((n * prev))
    fi
}

echo "5 的阶乘 = $(factorial 5)"
# 输出：5 的阶乘 = 120
```

不过说实话，Shell 里递归用得不多。Shell 的设计目标不是做复杂的数学或算法计算，而是做**系统管理任务的自动化**。如果你发现自己在写很复杂的递归逻辑，也许该考虑换 Python 了。

**类比**：Shell 脚本就像螺丝刀，特别好拧螺丝（文件操作、命令编排）。你能用它撬钉子，但不如直接拿锤子（Python）。

## 本节小结

函数就是把一组命令打包起名，让脚本更模块化。定义时加括号，调用时不加括号。函数内部用 `$1` 接参数，用 `local` 隔离变量，用 `return` 回退出码、`echo` 回数据。`source` 可以把函数库导入当前脚本。每一个好的函数就像一个微波炉预设程序 -- 一句命令完成一组预设动作。
