# 06-04 循环

## 本节你会学到什么

- 使用 `for...in` 遍历列表、数组和命令输出
- 使用 `for((;;))` 写出 C 语言风格的计数循环
- 使用 `while` 在条件满足时反复执行
- 使用 `until` 在条件不满足时反复执行
- 使用 `break` 和 `continue` 控制循环流程

---

有一天你的老板说："小明，把公司 100 台服务器上都创建一个 `/data/logs` 目录。" 你总不能 SSH 登录 100 次，手敲 100 遍 `mkdir`。这时候你需要的是"循环"。

循环就是"把一件事重复做好多次"，每次的数据不同，但操作相同。它就像是工厂里的流水线 -- 机器臂（循环体）重复做同一个动作，但每来一个零件（数据），就会针对这个零件做一次处理。

---

## 生活类比：给全班发卷子

老师在课堂上要发考卷。她有两种方式：

**方式一（for...in 循环）**：老师说："张三、李四、王五、赵六，上来拿卷子。" -- 她知道名单，按名单一个一个叫。

```bash
for student in 张三 李四 王五 赵六; do
    echo "请 $student 来拿卷子"
done
```

**方式二（while 循环）**：老师说："只要还有人没拿到卷子，我就继续发。" -- 她不知道具体有多少人，只知道"没发完就继续"。

```bash
while [ 还有人没拿到 ]; do
    echo "发一份卷子"
done
```

## for...in：逐个点名

`for...in` 是最常用的循环类型。它遍历一个列表，每次取出一个元素放入变量，然后执行循环体。

```bash
for 变量 in 列表; do
    操作的命令
done
```

### 遍历固定列表

```bash
for color in 红色 蓝色 绿色 黄色; do
    echo "我喜欢 $color"
done
```

### 遍历数字范围

```bash
# 花括号展开
for i in {1..10}; do
    echo "第 $i 次循环"
done

# seq 命令
for i in $(seq 1 10); do
    echo "第 $i 次循环"
done
```

### 遍历文件

```bash
# 遍历当前目录下所有 .txt 文件
for file in *.txt; do
    echo "处理文件：$file"
    wc -l "$file"
done
```

这里 `*.txt` 是通配符（globbing），bash 会自动把它展开成匹配的文件列表。

**类比**：`for...in` 就像点名册。你有一个名单（列表），然后一个一个念（迭代），念到谁，谁就做对应的事。

### 遍历命令输出

```bash
# 遍历所有用户
for user in $(awk -F: '{print $1}' /etc/passwd); do
    home_dir=$(eval echo ~$user)
    echo "用户 $user 的家目录：$home_dir"
done
```

但这里有个坑：命令输出里如果包含空格或特殊字符，可能会被错误拆分。更安全的方式是用 `while read`（后面会讲）。

## for((;;))：C 语言风格计数

如果你学过 C 语言或 Java，你会对 `for((;;))` 感到非常亲切：

```bash
for ((初始化; 条件; 更新)); do
    命令
done
```

```bash
# 从 1 数到 10
for ((i = 1; i <= 10; i++)); do
    echo "计数器：$i"
done

# 倒计时
for ((i = 10; i >= 1; i--)); do
    echo "$i..."
done
echo "发射！"
```

`for((;;))` 的好处是可以在循环体内做数值计算，比如：

```bash
# 计算 1 到 100 的和
sum=0
for ((i = 1; i <= 100; i++)); do
    ((sum += i))
done
echo "1 到 100 的和是：$sum"
```

注意 `((sum += i))` 是算术运算的写法（用双括号包起来），这个在讲函数时会详细展开。

## while：条件为真一直干

`while` 循环在你不知道要循环多少次的时候特别有用。它的逻辑是："只要条件成立，就继续。"

```bash
while [ 条件 ]; do
    命令
done
```

### 经典用法：逐行读取文件

```bash
# 逐行读取日志文件，找出包含 ERROR 的行
while IFS= read -r line; do
    if [[ "$line" == *ERROR* ]]; then
        echo "发现错误：$line"
    fi
done < /var/log/syslog
```

这个例子值得展开讲讲：
- `IFS=` 临时取消默认的空格/制表符拆分
- `read -r` 的 `-r` 表示不解释反斜杠
- `< /var/log/syslog` 把文件内容重定向到 while 循环的输入

**类比**：`while` 就像是自动饮水机 -- 只要杯子没满（条件为真），水就一直流。

### 死循环和退出条件

```bash
# 故意写一个死循环
while true; do
    echo "按 Ctrl+C 停止我"
    sleep 1
done
```

`Ctrl+C` 是通用的"中断当前程序"的快捷键。如果你不小心写了个死循环，不用惊慌，按 `Ctrl+C` 就能停下来。

```bash
# 带退出条件的循环
count=0
while true; do
    echo "检查中... 第 $((++count)) 次"
    if [ $count -ge 10 ]; then
        echo "检查完毕"
        break
    fi
    sleep 1
done
```

## until：条件为假一直干

`until` 和 `while` 是双胞胎，只不过逻辑反过来：

| while | until |
|-------|-------|
| 条件为真时执行 | 条件为假时执行 |
| 条件为假时停止 | 条件为真时停止 |

```bash
# 等待服务端口就绪
# 当端口还没开放时，一直等
until nc -z localhost 3306; do
    echo "等待 MySQL 启动..."
    sleep 2
done
echo "MySQL 已启动！"
```

`nc -z` 是检查网络端口是否通畅的命令，`-z` 表示只检查不发送数据（类似"ping 端口"）。

**类比**：`until` 就是"等到...为止"。就像你等朋友："**直到**他出现为止，我一直等。" 翻译过来就是 `until 他出现了; do 等待; done`。

大多数 `while` 循环都可以改写为 `until` 循环（反过来写条件就行），选择哪个取决于哪种表达更自然。

## break 和 continue：循环的控制杆

### break：紧急刹车

`break` 会跳出整个循环，即使条件还没满足：

```bash
for i in {1..100}; do
    if [ $i -gt 5 ]; then
        echo "已经够了，停止！"
        break
    fi
    echo "循环第 $i 次"
done
# 输出 1 到 5，然后停止
```

在嵌套循环中，`break N` 可以跳出多层循环：

```bash
for outer in {1..3}; do
    for inner in {1..5}; do
        if [ $inner -eq 3 ]; then
            break 2   # 跳出两层循环
        fi
        echo "  outer=$outer, inner=$inner"
    done
done
```

### continue：跳过当前

`continue` 跳过一次循环中后面的代码，直接进入下一次：

```bash
for i in {1..10}; do
    if [ $i -eq 3 ] || [ $i -eq 7 ]; then
        continue   # 跳过 3 和 7
    fi
    echo "处理第 $i 项"
done
```

**类比**：`break` 是直接关掉水龙头，不洗了。`continue` 是"这件衣服跳过不洗，拿下一件"。一个是终止整个洗衣过程，一个是跳过某一件衣服。

## 实战模式：常见循环用法

### 模式一：批量文件处理

```bash
# 将所有 .jpg 文件重命名为 .png
for file in *.jpg; do
    [ -e "$file" ] || continue   # 没有 .jpg 文件就跳过
    mv "$file" "${file%.jpg}.png"
    echo "重命名：$file -> ${file%.jpg}.png"
done
```

`${file%.jpg}` 是从变量尾部去掉 `.jpg`（字符串截取，下一节会详细讲）。

### 模式二：等待服务启动

```bash
# 等待网页服务启动（最多等 30 秒）
max_wait=30
elapsed=0
while ! curl -s http://localhost:8080/health > /dev/null 2>&1; do
    if [ $elapsed -ge $max_wait ]; then
        echo "超时：服务未在 ${max_wait} 秒内启动"
        exit 1
    fi
    echo "等待服务启动... ${elapsed}s"
    sleep 2
    elapsed=$((elapsed + 2))
done
echo "服务已就绪！"
```

### 模式三：遍历数组

```bash
servers=("web01" "web02" "db01" "cache01")
for server in "${servers[@]}"; do
    echo "部署到 $server..."
    # ssh $server "sudo systemctl restart nginx"
done
```

## 循环和管道的注意事项

在管道中的循环有一个重要的坑：**管道会创建子进程**。

```bash
# 这段代码有 bug！
count=0
cat /etc/passwd | while read line; do
    ((count++))   # 在子 Shell 里修改 count
done
echo "共 $count 行"   # 输出 0！因为子 Shell 的修改不回传
```

解决方案是使用**进程替换**或**here-string**：

```bash
# 方法一：重定向（不创建子进程）
count=0
while read line; do
    ((count++))
done < /etc/passwd
echo "共 $count 行"

# 方法二：进程替换
count=0
while read line; do
    ((count++))
done < <(cat /etc/passwd)
```

## 本节小结

循环让脚本拥有了"重复执行"的能力。`for...in` 适合遍历已知列表，`for((;;))` 适合计数循环，`while` 在条件为真时循环，`until` 在条件为假时循环。`break` 退出循环，`continue` 跳过本次。注意管道中的子进程陷阱。
