# 06-03 条件判断

## 本节你会学到什么

- 使用 `if/then/elif/else/fi` 写出条件分支结构
- 理解 `test`、`[ ]`、`[[ ]]` 的区别和适用场景
- 用文件测试操作符检查文件是否存在、是否为目录、是否可执行
- 用字符串和数值比较操作符做逻辑判断
- 用 `&&` 和 `||` 写出紧凑的条件控制逻辑

---

上一节我们让脚本学会了"记东西"（变量），但光有记忆还不够。生活中你需要做判断：下雨就带伞，晴天就不带。脚本也一样，需要根据不同的情况走不同的路。这就是条件判断的作用。

---

## 生活类比：交通路口的红绿灯

想象你开车到一个十字路口：

- **如果**是绿灯，你就直行
- **否则如果**是黄灯，你减速准备停车
- **否则**（红灯），你停车等待

翻译成 Shell 脚本就是：

```bash
if [ "$light" = "green" ]; then
    echo "通行"
elif [ "$light" = "yellow" ]; then
    echo "减速"
else
    echo "停车"
fi
```

`if` 开头，`fi` 结尾（就是 if 倒过来写）。这个对称设计不是开玩笑 -- 它让你一眼就能看到"条件块"的边界，类似于括号的对称性。

---

## if 的基本结构

Shell 中 if 语句的几种形态：

### 形态一：最简单的 if

```bash
if [ 条件 ]; then
    满足条件时执行的命令
fi
```

`then` 可以写在单独一行，也可以和 `if` 同行但用分号分隔：

```bash
if [ 条件 ]
then
    命令
fi
```

### 形态二：if-else（二选一）

```bash
if [ 条件 ]; then
    命令A
else
    命令B
fi
```

### 形态三：if-elif-else（多选一）

```bash
if [ 条件1 ]; then
    命令A
elif [ 条件2 ]; then
    命令B
elif [ 条件3 ]; then
    命令C
else
    命令D    # 以上都不满足时执行
fi
```

**类比**：if 结构就像流水线上的分拣机。物品（数据）沿着传送带过来，每经过一个检查站，符合条件就被分流到对应的篮子。最后不满足任何条件的，就掉进 `else`（默认篮子）。

## 条件测试的三兄弟：test、[ ]、[[ ]]

Shell 中做条件判断有三种写法，它们是"三兄弟"，功能类似但各有千秋。

### 老大：test 命令

最原始的写法，直接用 `test` 命令：

```bash
if test -f /etc/passwd; then
    echo "文件存在"
fi
```

### 老二：[ ] （方括号）

`[ ]` 实际上就是 `test` 命令的一个别名。当你写 `[ -f /etc/passwd ]` 时，bash 实际上执行的是 `test -f /etc/passwd`。

```bash
if [ -f /etc/passwd ]; then
    echo "文件存在"
fi
```

用 `[ ]` 时有几个"坑"必须注意：
- `[` 后面必须有空格，`]` 前面必须有空格
- 变量引用要用双引号包起来，防止变量为空时语法错误

```bash
# 正确写法
if [ "$name" = "小明" ]; then ...

# 错误写法（缺少空格）
if ["$name" = "小明"]; then ...

# 危险写法（如果 $name 为空，就会变成 [ = "小明" ]，语法错误）
if [ $name = "小明" ]; then ...

# 安全写法
if [ "$name" = "小明" ]; then ...
```

**类比**：`[ ]` 就像是一个微妙的仪器 -- 它的运行需要"呼吸空间"（空格），如果太"挤"就会出故障。记住 `[` 和 `]` 都是独立的命令，不是语法糖，所以空格是必须的。

### 老三：[[ ]] （双方括号）

`[[ ]]` 是 bash 的扩展，功能比 `[ ]` 更强，也更安全：

```bash
if [[ "$name" == "小明" ]]; then ...
```

`[[ ]]` 的优势：
1. **不要求给变量加双引号**：即使在变量为空的情况下也不会报语法错误
2. **支持 `&&` 和 `||`**：条件之间可以用 `&&`（与）和 `||`（或）连接
3. **支持 `=~` 正则表达式匹配**：这是 `[ ]` 做不到的
4. **支持 `==` 模式匹配**：可以使用通配符

```bash
# 正则匹配（[[ ]] 独有功能）
phone="13812345678"
if [[ "$phone" =~ ^[0-9]{11}$ ]]; then
    echo "这是有效的手机号"
fi

# 模式匹配
filename="backup_2026.tar.gz"
if [[ "$filename" == backup_* ]]; then
    echo "这是一个备份文件"
fi
```

**使用建议**：如果你确定脚本只在 bash 下运行，推荐使用 `[[ ]]`；如果你希望脚本在任何 Shell 下都能运行（跨平台），用 `[ ]`。学习阶段两种都要掌握。

## 文件测试：检查文件的状态

脚本里最常见的操作就是操作文件 -- 创建、读取、删除。在操作之前，最好先"探路"：这个文件真的存在吗？我有权限吗？它是目录还是文件？

bash 提供了一组文件测试操作符：

| 操作符 | 含义 | 用法示例 |
|--------|------|----------|
| `-f` | 是否是普通文件 | `[ -f /etc/passwd ]` |
| `-d` | 是否是目录 | `[ -d /tmp ]` |
| `-e` | 路径是否存在（不关心类型） | `[ -e /some/path ]` |
| `-r` | 是否可读 | `[ -r file.txt ]` |
| `-w` | 是否可写 | `[ -w file.txt ]` |
| `-x` | 是否可执行 | `[ -x script.sh ]` |
| `-s` | 文件是否存在且大小大于 0 | `[ -s file.txt ]` |
| `-L` | 是否是符号链接 | `[ -L link ]` |
| `-nt` | file1 是否比 file2 新 (newer than) | `[ file1 -nt file2 ]` |
| `-ot` | file1 是否比 file2 旧 (older than) | `[ file1 -ot file2 ]` |

实际应用场景：

```bash
# 备份前检查目录是否存在
backup_dir="/backup/data"
if [ ! -d "$backup_dir" ]; then
    mkdir -p "$backup_dir"
    echo "创建备份目录：$backup_dir"
fi

# 检查配置文件是否为空
config="/etc/myapp/config.ini"
if [ ! -s "$config" ]; then
    echo "警告：配置文件为空，使用默认配置"
    cp /etc/myapp/default.ini "$config"
fi
```

**类比**：文件测试就是在"敲门"。你到一个朋友家门口，先看看门牌号对不对（`-f`），再试探一下门有没有锁（`-r`），最后决定要不要进去。盲目操作文件而不先检查，就像直接破门而入 -- 可能造成灾难。

## 字符串比较：判断文字内容

| 操作符 | 含义 | 用法示例 |
|--------|------|----------|
| `=` 或 `==` | 相等 | `[ "$a" = "$b" ]` |
| `!=` | 不等 | `[ "$a" != "$b" ]` |
| `-z` | 字符串为空（zero length） | `[ -z "$a" ]` |
| `-n` | 字符串不为空 | `[ -n "$a" ]` |
| `<` / `>` | 字典序比较（需要 `[[ ]]`） | `[[ "$a" < "$b" ]]` |

```bash
username=""

if [ -z "$username" ]; then
    echo "用户名为空，请先登录"
    exit 1
fi

if [ "$answer" = "yes" ]; then
    echo "你选择了是"
elif [ "$answer" = "no" ]; then
    echo "你选择了否"
else
    echo "请输入 yes 或 no"
fi
```

小技巧：比较时可以把常量放在左边，防止变量为空时出现问题：

```bash
# 更安全的写法
if [ "yes" = "$answer" ]; then ...
```

这看起来很别扭，但确实更安全 -- 如果 `$answer` 为空，`[ "" = "yes" ]` 是合法的语法。

## 数值比较：用字母而不是符号

在 `[ ]` 中做数值比较，不能用 `>` `<` `>=`，而是用字母操作符。这是因为 `>` 在 Shell 中是重定向符号。

| 操作符 | 含义 | 记忆技巧 |
|--------|------|----------|
| `-eq` | 等于 (equal) | eq = equal |
| `-ne` | 不等于 (not equal) | ne = not equal |
| `-lt` | 小于 (less than) | lt = less than |
| `-le` | 小于等于 (less or equal) | le = less or equal |
| `-gt` | 大于 (greater than) | gt = greater than |
| `-ge` | 大于等于 (greater or equal) | ge = greater or equal |

```bash
age=20

if [ $age -ge 18 ]; then
    echo "已成年"
fi

if [ $age -lt 18 ]; then
    echo "未成年"
fi
```

如果用的是 `[[ ]]`，则可以直接用 `>` `<`：

```bash
if [[ $age -ge 18 ]]; then ...   # 也行
if [[ $age > 17 ]]; then ...     # 也行（但要注意是字典序不是数值序）
```

不过在 `[[ ]]` 中，`>` `<` 是字典序比较（字符串比较），不是数值比较。所以数值操作时 `-ge`、`-lt` 等 still 是最可靠的选择。

## 组合条件：与、或、非

生活中我们经常组合条件："如果明天下雨**而且**气温低于 10 度，我就在家看书。" Shell 中也有对应的逻辑运算：

```bash
# 与运算：两个条件都满足
if [ $age -ge 18 ] && [ "$has_id" = "yes" ]; then
    echo "可以入场"
fi

# 在 [[ ]] 中甚至可以更简洁
if [[ $age -ge 18 && $has_id == "yes" ]]; then
    echo "可以入场"
fi

# 在 [ ] 中可以用 -a（与）和 -o（或）
if [ $age -ge 18 -a "$has_id" = "yes" ]; then
    echo "可以入场"
fi

# 或运算：任一条件满足
if [ "$answer" = "y" ] || [ "$answer" = "yes" ]; then
    echo "确认操作"
fi

# 非运算：取反
if [ ! -f "/tmp/lock" ]; then
    echo "没有锁文件，可以安全执行"
fi
```

**使用建议**：用 `&&` 和 `||` 连接多个 `[ ]`，比用 `-a` 和 `-o` 更清晰，也更容易维护。

## 单行条件：命令行中的快捷判断

在命令行里，你经常看到这种写法：

```bash
# 命令成功则执行下一个（&&）
cd /var/log && ls -la
# 意思是：先 cd 到 /var/log，如果成功了（退出码 0），然后 ls

# 命令失败则执行另一个（||）
mkdir /backup 2>/dev/null || echo "目录创建失败，可能已存在"
# 意思是：尝试 mkdir，如果失败了，输出提示

# 三目运算的 Shell 版本
[ -d /backup ] && echo "备份目录已存在" || echo "需要创建备份目录"
```

**类比**：`&&` 就像多米诺骨牌 -- 前面倒下了，后面才会倒。`||` 就像备用方案 -- 主方案不行，Plan B 顶上。

## 本节小结

条件判断让脚本拥有了"选择"的能力。`if/then/elif/else/fi` 是条件分支的基本结构，`[ ]`（test 命令）和 `[[ ]]`（bash 增强版）用于表达条件，文件测试、字符串比较、数值比较是最常用的三种判断类型。`&&` 和 `||` 让条件控制更紧凑高效。
