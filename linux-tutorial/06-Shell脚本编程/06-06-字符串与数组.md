Shell 脚本处理最多的东西是什么？不是数字，不是图片 -- 是**文本**。文件名、输出信息、配置项、命令行参数... 全都是文本（字符串）。所以 Shell 内置了一套非常强大的"字符串操作"机制，让你不用调用外部命令就能完成常见的文本处理。

本节还会讲到数组。你不会想用 50 个变量 `server01`、`server02`... `server50` 来管理服务器列表吧？一个数组搞定。

## 字符串操作：Shell 的"瑞士军刀"

字符串操作的核心语法是**参数扩展**（Parameter Expansion），它的格式是 `${变量名 操作符 参数}`。这个语法全部在 bash 内部完成，不需要调用 `sed`、`awk` 等外部命令，所以速度很快。

---

## 获取字符串长度

```bash
name="Hello World"
echo ${#name}       # 输出：11
```

`#` 在这里不是注释，而是在 `${}` 里面表示"求长度"。这就像是拿了一把尺子量一下字符串有多少个字符。

---

## 截取子串：切蛋糕的艺术

```bash
url="https://www.example.com/index.html"

echo ${url:0:5}        # 输出：https （从位置0开始，取5个字符）
echo ${url:8:11}       # 输出：www.example （从位置8开始，取11个字符）
echo ${url:8}          # 输出：www.example.com/index.html （从位置8开始到末尾）
echo ${url: -10}       # 输出：index.html （倒数10个字符，注意冒号后面有空格）
```

**类比**：字符串就像一根法棍面包。`${url:8:11}` 就是说"从第 8 厘米处切一刀，再切 11 厘米长的一段"。`${url: -10}` 是"从尾巴倒数 10 厘米切一段"。

## 模式匹配截取：最实用但也最难记

这是 Shell 字符串操作中最强大的部分，用 `#`、`##`、`%`、`%%` 来"抠掉"不需要的部分。

记住一个口诀：
- `#` 是**左边**（键盘上 `#` 在 `$` 的左边）
- `%` 是**右边**（百分号往往写在数字右边）
- 一个符号是**最短匹配**
- 两个符号是**最长匹配**（贪婪）

```bash
url="https://www.example.com/docs/index.html"

# #  从左边去掉最短匹配
echo ${url#*//}         # www.example.com/docs/index.html（去掉 https://）

# ## 从左边去掉最长匹配
echo ${url##*/}         # index.html（去掉所有路径，只留文件名）

# %  从右边去掉最短匹配
echo ${url%/*}          # https://www.example.com/docs（去掉文件名）

# %% 从右边去掉最长匹配
echo ${url%%/*}         # https:（去掉所有路径）
```

实际应用场景：

```bash
# 获取文件扩展名
filename="backup_2026.tar.gz"
echo ${filename##*.}            # gz（取最后的扩展名）
echo ${filename%.*}             # backup_2026.tar（去掉最后的扩展名）
echo ${filename%%.*}            # backup_2026（去掉所有扩展名）

# 获取目录路径
filepath="/home/user/docs/readme.txt"
echo ${filepath%/*}             # /home/user/docs（目录部分）
echo ${filepath##*/}            # readme.txt（文件名部分）
```

**类比**：`##` 就像吃冰棍 -- 从左边开始咬掉最长的一段。`%%` 则像是从冰棍棒的反方向咬 -- 从右边开始咬掉最长的一段。

## 字符串替换：查找并替换

```bash
text="apple orange apple banana apple"

echo ${text/apple/PEAR}         # PEAR orange apple banana apple（只替换第一个）
echo ${text//apple/PEAR}        # PEAR orange PEAR banana PEAR（替换所有）
echo ${text/#apple/PEAR}        # PEAR orange apple banana apple（只替换开头的）
echo ${text/%apple/PEAR}        # apple orange apple banana PEAR（只替换结尾的）
```

记忆规则：
- 一个 `/` = 替换第一个
- 两个 `//` = 替换全部
- `/` 前面加 `#` = 只匹配开头
- `/` 前面加 `%` = 只匹配结尾

如果"新字符串"为空，就等于删除：

```bash
echo ${text//apple/}             #  orange  banana （删除了所有 "apple"）
```

## 大小写转换

```bash
name="linux"

echo ${name}      # linux（原始）
echo ${name^}     # Linux（首字母大写，^ 是向上箭头）
echo ${name^^}    # LINUX（全大写，两个^^）
echo ${name,}     # linux（首字母小写，逗号向下）
echo ${name,,}    # linux（全小写，两个逗号）
```

记忆：`^` 箭头向上 = 变大写，`,` 逗号向下 = 变小写。一个符号改第一个字母，两个符号改全部。

## 默认值和空值判断

这是一组非常实用的语法，用于处理"变量可能为空"的情况：

| 语法 | 含义 | 示例 |
|------|------|------|
| `${var:-默认值}` | var 为空时使用默认值，但不修改 var | `${port:-8080}` |
| `${var:=默认值}` | var 为空时赋值默认值，并返回结果 | `${port:=8080}` |
| `${var:+替换值}` | var 不为空时才使用替换值 | `${debug:+--verbose}` |
| `${var:?错误信息}` | var 为空时打印错误并退出脚本 | `${config:?配置文件未设置}` |

```bash
# 场景：给变量提供默认值
port=${PORT:-8080}
echo "服务将运行在端口 $port"

# 场景：只有开启调试模式时才加 -v 选项
debug_mode="yes"
./script.sh ${debug_mode:+-v}    # 等价于 ./script.sh -v

# 场景：必须设置的关键变量
data_dir=${DATA_DIR:?请设置 DATA_DIR 环境变量再运行此脚本}
```

**类比**：`${var:-默认值}` 就像是自动售货机的备选方案 -- "如果百事可乐卖完了，给我可口可乐。" 你不会改变售货机里的存货（不修改原变量），只是拿到了一罐替代品。

## 数组：从一个个变量到一个"集装箱"

### 普通数组（索引数组）

数组让你把一组相关的数据放在一个"集装箱"里：

```bash
# 定义数组（小括号，空格分隔元素）
servers=("web01" "web02" "db01" "cache01")

# 访问元素（索引从 0 开始）
echo ${servers[0]}         # web01
echo ${servers[2]}         # db01

# 访问所有元素
echo ${servers[@]}         # web01 web02 db01 cache01

# 数组长度
echo ${#servers[@]}        # 4

# 修改元素
servers[1]="web02-new"
echo ${servers[1]}         # web02-new

# 追加元素
servers+=("monitor01")
echo ${servers[@]}         # web01 web02-new db01 cache01 monitor01

# 删除元素
unset servers[3]           # 删除 cache01
echo ${servers[@]}         # web01 web02-new db01 monitor01

# 删除整个数组
unset servers
```

### 遍历数组的三种方式

```bash
fruits=("苹果" "香蕉" "橘子")

# 方式一：直接遍历值（最常用）
for fruit in "${fruits[@]}"; do
    echo "$fruit"
done

# 方式二：遍历索引
for i in "${!fruits[@]}"; do
    echo "索引 $i = ${fruits[$i]}"
done

# 方式三：C 风格循环
for ((i = 0; i < ${#fruits[@]}; i++)); do
    echo "[$i] ${fruits[$i]}"
done
```

注意遍历时 `"${fruits[@]}"` 要加双引号，否则水果名字里如果有空格，会被错误拆分。

## 关联数组：字符串当索引的数组

Python 里叫 dict（字典），JavaScript 里叫 object，Shell 里叫**关联数组**（associative array）。它是一个"键 -> 值"的映射表。

**声明**是必须的一步（和普通数组不同）：

```bash
# 第一步：声明（必须！）
declare -A student_scores

# 第二步：赋值
student_scores["张三"]=95
student_scores["李四"]=87
student_scores["王五"]=92

# 访问
echo ${student_scores["张三"]}     # 95

# 获取所有键
echo ${!student_scores[@]}         # 张三 李四 王五

# 获取所有值
echo ${student_scores[@]}           # 95 87 92

# 遍历
for name in "${!student_scores[@]}"; do
    echo "$name: ${student_scores[$name]} 分"
done

# 检查键是否存在
if [ -n "${student_scores["赵六"]+exists}" ]; then
    echo "赵六有成绩"
else
    echo "赵六没有成绩"
fi

# 删除
unset "student_scores[李四]"
```

**类比**：关联数组就像是真正的字典。你想查"apple"的意思，翻到"a"区找到"apple"这个条目，它的释义就是值。你不能用数字编号来翻字典（第 100 个词是什么？），只能用"单词"来查。

关联数组的典型应用场景：
- 配置文件解析（`key=value`）
- 统计数据（"每个 IP 访问了多少次"）
- 命令行参数解析（`--host` -> `localhost`）

## 本节小结

Shell 的参数扩展语法 `${}` 是处理字符串的利器：用 `#`/`%` 做模式截取，用 `/` 做替换，用 `^^`/`,,` 做大小写转换。数组分为索引数组（数字下标）和关联数组（`declare -A`，字符串下标），分别适合有序列表和键值映射场景。掌握这些技巧，处理文本和批量数据会快 10 倍。
