# 04-02 自签名证书

## 本节你会学到什么

- 理解证书的生成流程：私钥 -> CSR -> 证书
- 用 openssl 命令亲手生成一张自签名证书
- 搞懂自签名证书和 CA 签名证书的本质区别
- 明白为什么浏览器会提示"不安全"

## 证书是怎么产生的：三步流程

在上一节我们讲了证书是 CA 盖章的"身份证明"。那这张证明具体是怎么造出来的？整个过程分为三步：

**第一步：生成私钥。** 私钥是你自己保管的绝密文件，谁都不能给。相当于你自己的签名印章——只有你知道什么样，别人仿不出来。

**第二步：生成 CSR（Certificate Signing Request，证书签名请求）。** CSR 里面包含了你的身份信息（公司名、域名等）和你的公钥。相当于你填好了一张申请表，上面写着"我是某某网站，这是我的公开资料"。

**第三步：签名生成证书。** 把 CSR 交给 CA，CA 核实后用自己的私钥给 CSR 盖章，生成证书。如果是自签名，那就是你自己给自己盖章——用你自己的私钥给 CSR 签名。

生成流程图解：

```
私钥 (server.key)           CSR (server.csr)
    |                            |
    +---------- CSR <------------+
    |
    +---------- 签名 ---------> 证书 (server.crt)
```

## 自签名 vs CA 签名：工作证明的类比

**自签名证书**就像你自己给自己开的工作证明。你是"张三公司"，你自己在证明上签个字说"我，张三，确认我在张三公司工作"。听起来很荒谬对吧？除了你自己，没有任何一个第三方认可这张证明。

**CA 签名证书**则像是你公司 HR 给你开的工作证明，上面有公司公章。办事单位（浏览器）一看，这家公司的公章我认识，我信。证明有效。

浏览器之所以对自签名证书报"不安全"，不是因为你的加密不够强——事实上自签名证书的加密强度和 CA 证书一模一样。浏览器报警是因为**无法验证你的身份**。它不认识你的"公章"。

用快递签收来类比：快递员让你签收快递时，他认识顺丰的工服和工牌，知道你是正规快递公司的人。如果来一个穿便服的人说"我是快递员，把包裹给我"，你敢给吗？他的"快递身份"是自签名的——除了他自己，没人能证明他真的是快递员。

## 什么时候用自签名证书

自签名证书不是完全没用。以下场景用自签名完全合理：

- 本地开发调试
- 公司内网系统
- 测试环境
- 个人学习实验

在这些场景下，你是唯一访问的人，你知道这个网站的身份是真的，所以不需要第三方 CA 来证明。但如果是面向公网的服务，一定要用 CA 签名的证书（Let's Encrypt 可以免费申请）。

## 完整操作步骤

下面是在命令行中一步步生成自签名证书：

```bash
# 第 1 步：生成 2048 位 RSA 私钥
openssl genrsa -out server.key 2048

# 第 2 步：生成证书签名请求 (CSR)
# -subj 里的参数：
#   /CN= 是 Common Name，填你要用 HTTPS 的域名
#   /C=  国家代码
#   /ST= 省份
#   /L=  城市
#   /O=  组织名
openssl req -new \
    -key server.key \
    -out server.csr \
    -subj "/C=CN/ST=Beijing/L=Beijing/O=MyCompany/OU=Dev/CN=example.test"

# 第 3 步：用私钥自签名，生成证书（有效期 365 天）
openssl x509 -req \
    -in server.csr \
    -signkey server.key \
    -out server.crt \
    -days 365

# 第 4 步（可选）：查看证书内容
openssl x509 -in server.crt -text -noout | head -20
```

你也可以直接用配套的脚本一键生成，脚本在 `examples/04-02/generate-cert.sh`。把脚本中的 `DOMAIN` 变量改成你的实际域名，然后运行：

```bash
chmod +x generate-cert.sh
./generate-cert.sh
```

生成后你会得到三个文件：
- `server.key` —— 私钥，绝对不能泄露。权限应设为 600（只有文件所有者可读写）
- `server.csr` —— 签名请求。自签名场景下可以删除，申请 CA 签名时需要提交这个
- `server.crt` —— 证书。这个文件是公开的，配给 Nginx 用

## 动手试试

1. 打开终端，进入 `examples/04-02/` 目录
2. 运行 `bash generate-cert.sh`
3. 观察每一步的输出，理解生成了什么
4. 用 `openssl x509 -in server.crt -text -noout` 仔细查看证书内容，找到以下字段：
   - `Issuer`（颁发者）是谁？注意到它和 `Subject`（主体）是一样的——这说明是自签名
   - `Validity` 显示了什么有效期？
   - `Subject Public Key Info` 显示了什么加密算法？

## 本节小结

证书生成分三步：先生成私钥，再生成 CSR（申请表），最后用私钥签名得到证书；自签名证书加密强度和 CA 证书一样，只是缺少第三方身份担保。
