## 正文

### 什么是虚拟主机

"虚拟主机"这个名字听起来有点玄乎，但概念其实很简单：**一台物理服务器上跑多个网站，每个网站有自己独立的域名**。

这对于成本和效率来说意义重大。一台云服务器每月 50 块钱，上面可以跑十个甚至更多的网站。你不需要为每个小网站单独买一台服务器，Nginx 会根据访问的域名"分拣"请求，把它们送到对应的网站目录。

用**一栋写字楼里的多家公司**来类比：

美团、字节、阿里都在同一栋写字楼里租了办公室，大楼的地址（IP 地址）是同一个——比如"北京朝阳区 XX 路 88 号"。但快递员来了不会搞混，因为包裹上写着不同的公司名字（域名）。大楼前台（Nginx）看到"美团"的快递就通知美团的人下来取，看到"字节"的快递就通知字节的人。

这里的关键点是：**多个域名可以解析到同一个 IP 地址**。DNS 就是那个"写公司名字"的环节。

### server_name 的三种匹配方式

`server_name` 指令用来指定这个 server 块负责哪个域名。当请求到达 Nginx 时，Nginx 会检查请求头中的 `Host` 字段，和所有 server 块的 `server_name` 做匹配。

Nginx 支持三种匹配方式：

**1. 精确匹配**

```nginx
server {
    listen 80;
    server_name www.example.com;
}
```

只有当访问者敲的是 `www.example.com` 才进入这个 server 块。多一个字符、少一个点都不行。

**2. 通配符前缀匹配**

```nginx
server {
    listen 80;
    server_name *.example.com;
}
```

`*` 可以匹配任意字符串。这个配置会匹配 `a.example.com`、`b.example.com`、`blog.example.com` 等任何以 `.example.com` 结尾的域名。

```nginx
server {
    listen 80;
    server_name www.example.*;
}
```

这个配置匹配 `www.example.com`、`www.example.cn`、`www.example.org` 等。

**3. 正则表达式匹配**

```nginx
server {
    listen 80;
    server_name ~^(www\.)?(?<domain>.+)\.com$;
}
```

以 `~` 开头表示这是一个正则表达式。这个例子匹配所有 `.com` 域名，前面可选 `www.`。`(?<domain>.+)` 是一个命名捕获组，后续可以在配置中用 `$domain` 引用这个值。

### 匹配优先级

当多个 server_name 都能匹配同一个请求时，Nginx 按以下顺序选择：

1. 精确名称（如 `www.example.com`）
2. 以 `*` 开头的最长通配符（如 `*.example.com`）
3. 以 `*` 结尾的最长通配符（如 `mail.*`）
4. 第一个匹配的正则表达式（按配置文件中出现的顺序）
5. 都没有匹配时，使用 `default_server`

用**快递分拣中心**来类比：分拣员拿到一个包裹，先看地址是否精确匹配某个分拣口（精确匹配），如果没有，再看是否符合"XX 省 XX 市"的通配规则（前缀通配），接着看"XX 路"的规则（后缀通配），最后查正则规则，实在分不出来的就扔到"默认处理区"（default_server）。

### default_server —— 兜底处理

当访问者的域名和所有 server_name 都不匹配时，Nginx 默认使用第一个定义的 server 块（按配置文件加载顺序）。但你可以显式指定哪个 server 是"默认的"：

```nginx
server {
    listen 80 default_server;
    server_name _;
    return 444;  # Nginx 特有的状态码，直接关闭连接，不返回任何内容
}
```

这个配置就像一个"垃圾邮件过滤器"——任何没有明确收件人（域名不匹配）的请求，直接丢弃。

`server_name _;` 中的下划线不是什么特殊语法，只是一个占位符，表示"名字无所谓"。关键是 `default_server` 标记。

`default_server` 有几个实用场景：
- 屏蔽直接通过 IP 访问你服务器的请求
- 拦截恶意扫描（有些扫描器会随机构造 Host 头）
- 提供一个"这个域名没配置"的提示页面

### 完整示例：一台服务器跑两个网站

假设你要在一台服务器上同时托管一个公司官网和一个人博客：

```
同一台服务器（IP: 192.168.1.100）
├── www.mycompany.com  -> /var/www/company/
└── blog.myblog.com    -> /var/www/blog/
```

对应配置：

```nginx
# 网站 A：公司官网
server {
    listen 80;
    server_name www.mycompany.com;

    root /var/www/company;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}

# 网站 B：个人博客
server {
    listen 80;
    server_name blog.myblog.com;

    root /var/www/blog;
    index index.html;

    location / {
        try_files $uri $uri/ =404;
    }
}

# 默认 server：拦截其他域名的请求
server {
    listen 80 default_server;
    server_name _;
    return 444;
}
```

当有人访问 `http://www.mycompany.com` 时，Nginx 去 `/var/www/company/` 找文件。当访问 `http://blog.myblog.com` 时，去 `/var/www/blog/` 找文件。两个网站完全独立，互不影响。

### 本地测试：用 hosts 文件模拟域名

在上线之前，你需要本地测试虚拟主机配置。但你还没有买域名，DNS 更没有指向你的电脑。这时候修改本机的 hosts 文件就能模拟：

**Windows**：`C:\Windows\System32\drivers\etc\hosts`
**Linux/Mac**：`/etc/hosts`

在文件末尾添加：

```
127.0.0.1  www.mycompany.com
127.0.0.1  blog.myblog.com
```

保存后，你在浏览器输入 `www.mycompany.com`，操作系统会直接从 hosts 文件中读取 IP 地址（127.0.0.1），不再去公网 DNS 查询。这就让你可以在本地用自定义域名测试 Nginx 的虚拟主机功能。

hosts 文件就像你手机通讯录里的"快捷拨号"——你给张三打电话不用背他的号码，通讯录已经帮你做好了名字到号码的映射。

## 动手试试

1. 在你的 Nginx 配置中创建两个 server 块，分别配置不同的域名（比如 `site-a.local` 和 `site-b.local`），指向不同的 `root` 目录。
2. 在两个 root 目录下各放一个 `index.html`，内容要能区分是哪个网站（比如一个写"这是 A 站"，另一个写"这是 B 站"）。
3. 修改 hosts 文件，把这两个域名都指向 `127.0.0.1`。
4. 执行 `nginx -t` 和 `nginx -s reload`，然后在浏览器分别访问这两个域名，确认看到不同的页面。
5. 额外挑战：添加一个 `default_server`，用 `return 444` 直接关闭连接，然后直接用 IP（`http://127.0.0.1`）访问，看看是什么效果。

## 本节小结

虚拟主机让一台服务器通过不同域名区分多个网站。`server_name` 支持精确、通配符和正则三种匹配方式，`default_server` 作为兜底处理无法匹配的请求。
