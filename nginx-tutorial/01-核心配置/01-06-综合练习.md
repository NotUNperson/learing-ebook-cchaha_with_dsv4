## 正文

### 练习目标

这次我们要搭建一个完整的"多站点静态服务器"。场景设定如下：

你是一个自由职业者，同时维护着两个网站：
- **个人作品集网站**（域名：`portfolio.local`）——展示你的项目和技能，有一个首页、一个项目展示页
- **技术文档站**（域名：`docs.local`）——记录你的学习笔记和技术文档

你需要用一台服务器（也就是你现在的 Nginx）同时托管这两个网站，每个有独立的域名、独立的文件目录、独立的错误页面。

先确认一下你的目录结构：

```
nginx/
├── conf/
│   └── nginx.conf          (主配置，include 我们的站点配置)
├── html/
│   ├── site-a/             (作品集网站)
│   │   ├── index.html
│   │   └── projects.html
│   └── site-b/             (文档站)
│       └── index.html
│   └── 404.html            (共享的 404 页面)
```

### 步骤二：编写 Nginx 配置

```nginx
# 作品集网站
server {
    listen 80;
    server_name portfolio.local;

    root html/site-a;
    index index.html;

    # 首页
    location / {
        try_files $uri $uri/ =404;
    }

    # 项目页：精确匹配 /projects.html，直接返回文件
    location = /projects.html {
        # root 继承父级
        # 不需要额外配置
    }

    # 为作品集图片单独设置缓存（即使现在还没有图片）
    location /images/ {
        expires 30d;
        try_files $uri =404;
    }

    # 404 错误页面
    error_page 404 /404.html;
    location = /404.html {
        root html;
    }
}

# 技术文档站
server {
    listen 80;
    server_name docs.local;

    root html/site-b;
    index index.html;

    # 文档站首页和子页面
    location / {
        try_files $uri $uri/ =404;
    }

    # 共享 404 页面
    error_page 404 /404.html;
    location = /404.html {
        root html;
    }
}

# 默认 server —— 通过 IP 访问时拒绝连接
server {
    listen 80 default_server;
    server_name _;
    return 444;
}
```

### 步骤三：配置 hosts 文件

在你的 hosts 文件中添加：

```
127.0.0.1  portfolio.local
127.0.0.1  docs.local
```

- Windows：`C:\Windows\System32\drivers\etc\hosts`（用管理员权限编辑）
- Linux/Mac：`sudo nano /etc/hosts`

### 步骤四：测试与验证

```bash
# 1. 测试配置语法
nginx -t

# 2. 重载配置
nginx -s reload

# 3. 在浏览器或 curl 中测试
curl http://portfolio.local/
curl http://docs.local/
curl http://portfolio.local/projects.html
curl http://portfolio.local/不存在的页面  # 应该返回你的 404 页面

# 4. 查看日志，验证请求被正确记录
tail -f logs/access.log
```

### 步骤五：验证要点

完成搭建后，逐一确认以下事项：

1. 访问 `http://portfolio.local/` 看到作品集首页
2. 访问 `http://docs.local/` 看到文档站首页
3. 访问 `http://portfolio.local/projects.html` 看到项目列表
4. 访问 `http://portfolio.local/不存在的页面` 看到自定义 404 页面（而非浏览器默认 404）
5. 通过 IP `http://127.0.0.1/` 访问时，浏览器显示"无法访问"（因为 default_server 返回 444 直接关闭了连接）
6. 打开 `logs/access.log`，能看到每一条请求的日志记录，包括成功（状态码 200）和失败（状态码 404）

## 动手试试

这份"动手试试"本身就是整个练习。按照上面五个步骤，从头到尾搭建出这个多站点静态服务器。如果中途遇到任何错误：

1. 首先看 `error.log` 里的错误信息
2. 用 `nginx -t` 检查配置语法
3. 确认 hosts 文件配置正确（可以用 `ping portfolio.local` 看是否解析到 127.0.0.1）
4. 确认 HTML 文件放在了正确的目录

整个练习应该在 10-15 分钟内完成。完成后你就拥有了一台真正能跑多网站的 Web 服务器了。

## 本节小结

本章全部核心知识的综合实践——虚拟主机实现多站点点隔离，location 匹配控制路由规则，error_page 自定义错误页面，访问日志记录验证一切是否正常。
