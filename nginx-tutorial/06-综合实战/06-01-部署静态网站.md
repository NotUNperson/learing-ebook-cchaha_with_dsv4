# 06-01 部署静态网站

## 本节你会学到什么

- 从零部署一个静态网站的完整流程
- 理解部署的三个阶段：准备 -> 配置 -> 上线
- 整合 Gzip、缓存、HTTPS、自定义错误页到实际项目中
- 学会上线前的基本检查和验证方法

## 开一家实体店：部署类比

部署一个网站就像开一家实体店。整个流程可以分成三个阶段：

**第一阶段：装修（准备文件 + 写配置）**

你要开店，首先要有店面（服务器）和商品（网站文件）。然后你规划店铺的布局：正门在哪里（80 端口/443 端口）、货架怎么摆（目录结构）、要不要装防盗门（HTTPS）、要不要装监控（日志）、是不是要在门口设安检（访问控制）。在 Nginx 里，"规划"就是写配置文件。

**第二阶段：摆货（上传文件）**

装修好了，你把商品摆上货架。网站文件（HTML、CSS、JS、图片）上传到服务器的指定目录。这个目录就是 Nginx 配置中的 `root` 指令指向的路径。

**第三阶段：开门营业（启动 Nginx）**

一切就绪，你打开卷帘门——`nginx -s reload` 或者 `systemctl start nginx`。顾客（浏览器）开始进来浏览。你通过日志（"收银记录"）观察今天的客流情况。

## 完整流程：从零到上线

### 第一步：准备服务器环境

首先确认 Nginx 已经安装：

```bash
# 检查 Nginx 是否已安装
nginx -v

# 如果未安装
# Ubuntu/Debian:
apt-get update && apt-get install nginx
# CentOS/RHEL:
yum install nginx
```

### 第二步：创建网站目录结构

```bash
# 创建网站根目录
mkdir -p /var/www/mysite
mkdir -p /var/www/mysite/css
mkdir -p /var/www/mysite/js
mkdir -p /var/www/mysite/images
mkdir -p /var/www/mysite/errors
```

好的目录结构就像超市合理的货架布局。假设你是顾客，进入超市后你希望一眼就能找到"零食区"、"饮料区"、"生鲜区"。网站的目录结构也是一样的道理——清晰的分类让别人（包括几个月后的你自己）一眼就能看懂。

推荐的网站目录结构：

```
/var/www/mysite/
    index.html          <- 首页
    404.html            <- 自定义错误页
    css/
        style.css       <- 样式
    js/
        app.js          <- 脚本
    images/
        logo.png        <- 图片
    errors/
        404.html        <- 404 错误页
        50x.html        <- 50x 错误页
```

### 第三步：准备网站文件

最简单的方式是先创建一个 `index.html`。本节配了一个示例页面在 `examples/06-01/site/index.html`，你可以直接复制使用。

```bash
# 把网站文件复制到根目录
cp -r examples/06-01/site/* /var/www/mysite/
```

### 第四步：生成 HTTPS 证书（可选但推荐）

```bash
# 用之前学的方法生成自签名证书（或使用 Let's Encrypt）
mkdir -p /etc/nginx/certs
cd /etc/nginx/certs
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
    -keyout server.key -out server.crt \
    -subj "/CN=mysite.test"
```

### 第五步：编写 Nginx 配置

创建一个完整的配置文件 `examples/06-01/nginx.conf`，包含以下要点：

- HTTP 80 端口自动跳转 HTTPS
- HTTPS 443 端口配置 SSL 证书
- Gzip 压缩全局开启
- 静态资源分类缓存（CSS 7 天、图片 30 天、字体 1 年）
- JSON 格式日志
- 自定义 404 和 50x 错误页面
- HSTS 安全头

### 第六步：启用配置并测试

```bash
# 1. 测试配置语法
nginx -t

# 2. 如果测试通过，重载配置
nginx -s reload

# 或者直接启动（如果还未启动）
systemctl start nginx
```

### 第七步：验证部署

```bash
# 验证 HTTP -> HTTPS 跳转
curl -I http://mysite.test

# 验证 HTTPS 正常响应
curl -k https://mysite.test

# 验证 Gzip
curl -k -H "Accept-Encoding: gzip" -I https://mysite.test | grep Content-Encoding

# 验证缓存头
curl -k -I https://mysite.test/css/style.css 2>/dev/null | grep -E "Cache-Control|Expires"

# 验证 404 页面
curl -k -I https://mysite.test/nonexistent-page
```

## 常见部署问题

**问题：访问网站看到 403 Forbidden**

通常是 Nginx 没有读取网站文件的权限。检查：
- 网站目录是否有正确的权限（`chmod -R 755 /var/www/mysite`）
- Nginx 运行用户（通常是 `www-data` 或 `nginx`）是否是目录的所有者或有读取权限
- SELinux 是否在拦截（CentOS 常见）：`setenforce 0` 临时关闭测试

**问题：修改了文件但浏览器看到的是旧版本**

这是缓存问题。在验证阶段可以打开浏览器开发者工具，勾选 "Disable cache"，或者在地址后加一个随机参数 `?v=123` 强制绕过缓存。

**问题：nginx -t 报错 "unknown directive"**

某个指令在当前 Nginx 版本中不支持。先用 `nginx -V` 确认编译时包含了哪些模块。比如 `gzip` 模块通常默认包含，但 `ssl` 模块在某些精简编译版本中可能缺失。

## 部署检查清单

上线前，对照这份清单逐项检查：

- [ ] 配置文件语法检查通过（`nginx -t`）
- [ ] 80 端口正确跳转到 HTTPS（`curl -I http://...`）
- [ ] HTTPS 正常访问（浏览器地址栏有小锁）
- [ ] Gzip 生效（`Content-Encoding: gzip` 响应头）
- [ ] 静态资源缓存策略正确（Cache-Control 头）
- [ ] 404 页面显示正常（访问一个不存在的路径）
- [ ] 50x 页面存在（`/50x.html` 可访问）
- [ ] 日志文件正常生成
- [ ] 网站根目录和文件的权限正确

## 动手试试

1. 在你的服务器上创建 `/var/www/mysite/` 目录结构
2. 把 `examples/06-01/site/index.html` 复制到 `/var/www/mysite/index.html`
3. 用 `examples/06-01/nginx.conf` 作为配置文件（修改域名和证书路径）
4. 生成自签名证书（或跳过 SSL，先只用 80 端口测试）
5. 启动 Nginx，用浏览器和 curl 验证网站可以正常访问
6. 故意访问一个不存在的页面（比如 `/abc`），检查 404 页面是否显示
7. 对照上面的部署检查清单，逐项确认

## 本节小结

部署静态网站 = 准备文件（摆货）+ 写配置（装修规划）+ 启动 Nginx（开门营业），上线前用检查清单逐项验证 HTTPS、Gzip、缓存和错误页面。
